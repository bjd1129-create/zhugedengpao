# 小花团队自我进化机制

> 这是小花团队的自我进化操作系统。每个成员（Agent）都必须遵循这套机制。

---

## 核心架构

```
老庄（用户）
    ↓
小花（主协调者）
    ├── 洞察者（市场研究）
    ├── 代码侠（工程实现）
    ├── 配色师（视觉设计）
    ├── 文案君（内容创作）
    └── 其他Agent
    ↓
.learnings/（进化记忆）
content/（研究产出）
memory/（每日日志）
MEMORY.md（长期记忆）
```

---

## 进化基础设施

### 1. .learnings/ 目录（每个Agent都有）

每个Agent工作目录下都有 `.learnings/`：

```
.learnings/
├── LEARNINGS.md        # 重要经验教训
├── ERRORS.md           # 犯过的错误（根因+解决方案）
└── FEATURE_REQUESTS.md # 想要但还没有的功能
```

**规则：错误发生 → 立即写入 ERRORS.md，不要依赖记忆。**

### 2. memory/ 目录

```
memory/
└── YYYY-MM-DD.md       # 每日工作日志
```

- 记录每天完成的任务、发现的问题、下次待办
- 格式简洁，用checklist

### 3. MEMORY.md

位于 agents/[name]/MEMORY.md

包含：
- 用户（老庄）偏好和特征
- 关键教训（不重复犯错）
- 待办追踪
- 进化记录

## 进化循环

### 定时进化（Cron）

```
openclaw cron create "小花团队进化" --interval=5h --message "执行团队进化任务"
```

Cron触发 → 主协调者分配任务 → 各Agent执行 → 结果写入content/ → 更新memory/

### 事件驱动进化

当发生以下情况时立即触发进化：
- 发现重大错误 → 写入.learnings/ERRORS.md
- 用户反馈负面 → 分析根因，更新MEMORY.md
- 任务连续失败 → 记录并调整策略

## 进化报告

每次完整进化后生成报告：`content/[Agent名]-[任务名]-YYYY-MM-DD.md`

报告结构：
```
# [任务名] | YYYY-MM-DD

## 执行情况
## 关键发现
## 发现的问题
## 下次待办
```

## 自我进化技能（self-evolve）

安装：`openclaw skills install be1human/self-evolve`

能力：
- 实时改写模糊的prompt
- 发现能力缺口时自动创建新skill
- 上下文丢失后自动更新记忆结构

**⚠️ 权限极大，仅限主协调者和核心角色使用。**

## 进化原则

1. **行动 > 反思**：问题发现立即改，不写多版报告
2. **错误不过夜**：每次犯错当天写入.learnings/
3. **量化追踪**：任务完成要记录产出（文件、token消耗、时间）
4. **拒绝重复**：ERRORS.md里的错误不再犯第二次
5. **进化报告循环终结**：发现问题直接改，不写进化报告

## 常见问题

**Q：什么时候更新MEMORY.md？**
A：每次完成重要任务后、发现关键教训时、用户偏好变化时。

**Q：进化报告和MEMORY.md的区别？**
A：进化报告是产出文档，MEMORY.md是记忆文件。前者记录工作结果，后者记录需要长期记住的信息。

**Q：.learnings/和MEMORY.md的错误记录有什么区别？**
A：.learnings/ERRORS.md是通用错误库（任何Agent都可查），MEMORY.md是个人记忆（只有该Agent需要知道）。
