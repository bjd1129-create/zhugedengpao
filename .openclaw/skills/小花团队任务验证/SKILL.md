# 任务验证：完成度系统检查

## 核心原则

> **"不是确认它能工作，而是尝试破坏它"**
> "代码看起来正确" ≠ 完成。运行它。
> "大概没问题" ≠ 完成。验证它。

## 验证检查清单

### 1. 需求验证
- [ ] 原始任务要求是否清晰？
- [ ] 所有要求都有对应产出吗？
- [ ] 边缘情况是否考虑？

### 2. 产出验证
- [ ] 文件是否真的存在于指定路径？
- [ ] 文件内容是否有实质内容（非空、非占位）？
- [ ] 格式是否符合预期（Markdown结构、代码语法）？

### 3. 质量验证
- [ ] 内容是否避免了"AI味"？
- [ ] 数据是否有来源标注？
- [ ] 逻辑是否自洽？

### 4. 可用性验证
- [ ] 其他Agent能直接使用这个产出吗？
- [ ] 需要额外转换才能用吗？
- [ ] 有没有明显的遗漏？

## 验证流程

1. **获取上下文**：理解任务描述和实际产出
2. **执行检查**：逐项验证上述清单
3. **呈现报告**：清晰区分通过/警告/阻塞

## 输出格式

```
## 任务验证报告：[任务名称]

### ✅ 通过
- [验证通过的项目]

### ⚠️ 警告
- [潜在问题，建议关注]

### ❌ 阻塞
- [必须修复才能认为完成的问题]

### 总结
[总体评价 + 下一步建议]
```

## 使用示例

```
/任务验证
/任务验证 T-009竞品分析
验证一下今天的产出
```
