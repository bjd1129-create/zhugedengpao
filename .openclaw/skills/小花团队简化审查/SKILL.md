# 简化审查：内容质量三维度并行审查

## Phase 1: 识别变更

检查 `content/` 目录下的最新产出文件，或用户指定的工作成果。

## Phase 2: 三个并行审查Agent

同时启动三个审查维度：

### Agent 1: 复用性审查

1. **查找现有资源**：是否有可复用的模板、框架、结论？
2. **标记重复内容**：是否在重复已有研究？
3. **标记可复用部分**：哪些内容可以抽象为通用模板？

### Agent 2: 质量审查

1. **逻辑完整性**：论点是否有充分证据支撑？
2. **信息来源**：数据是否来自可靠的一级来源？
3. **表达清晰度**：是否避免了"AI味"（套话、过度分点）？
4. **受众匹配度**：目标读者是否能读懂？

### Agent 3: 效率审查

1. **是否有冗余**：是否存在重复段落或啰嗦表达？
2. **结构是否高效**：读者能快速找到重点吗？
3. **是否有遗漏**：关键信息是否缺失？

## Phase 3: 汇总修复

等待三个Agent完成，聚合发现，直接修复可改进的问题。
对于有争议的发现，说明理由后请用户决策。

## 输出格式

```
## 简化审查报告

### ✅ 通过
- [检查通过的项目]

### ⚠️ 需改进
- [建议改进的项目 + 具体修改方案]

### ❌ 阻塞问题
- [必须修复才能使用的问题]

### 已修复
- [已直接修复的问题]
```

## 使用示例

```
/简化审查
/简化审查 最近的研究报告
帮我审查一下这个内容
```
