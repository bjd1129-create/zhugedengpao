# 记忆审查：三层记忆整理

## 目标

审查团队的记忆体系，生成变更建议报告。**不自动应用**——呈现给用户审批后再执行。

## 三层记忆架构

| 层级 | 文件 | 用途 | 可见范围 |
|------|------|------|----------|
| 项目级 | `MEMORY.md` | 团队长期记忆、决策、教训 | 所有Agent |
| 个人级 | `memory/YYYY-MM-DD.md` | 每日工作日志 | 单个Agent |
| 学习库 | `.learnings/` | 错误记录、经验教训 | 所有Agent可查 |

## 审查步骤

### 1. 收集所有记忆层

- 读取 `MEMORY.md`
- 读取今日和昨日的 `memory/YYYY-MM-DD.md`
- 读取 `.learnings/LEARNINGS.md` 和 `.learnings/ERRORS.md`

### 2. 分类整理

对每条记忆，判断最佳存放位置：

| 目标位置 | 什么放这里 | 示例 |
|----------|------------|------|
| **MEMORY.md** | 跨会话重要信息、决策、教训 | "老庄偏好简洁回复"、"不要编造数据" |
| **memory/当日.md** | 当日工作日志、任务记录 | "完成了T-009竞品分析" |
| **.learnings/ERRORS.md** | 犯过的错误及解决方案 | "web_fetch失败 → 改用browser" |
| **.learnings/LEARNINGS.md** | 重要经验 | "v2026.3.22有13个Breaking Changes" |
| **留在原处** | 临时上下文、不确定的信息 | 本次会话的中间思考 |

### 3. 识别问题

- **重复**：同一信息出现在多个地方
- **过时**：被新信息取代的旧内容
- **冲突**：不同记忆之间的矛盾
- **缺失**：应该记录但没记录的重要信息

### 4. 呈现报告

```
## 记忆审查报告

### 📈 晋升建议（→ MEMORY.md）
- [条目] → 晋升理由

### 🗑️ 清理建议
- [重复/过时/冲突] → 清理理由

### ❓ 需确认（模糊条目）
- [条目] → 为什么不确定

### ✅ 无需行动
- [已妥善存放的条目]
```

## 重要规则

- **先报告，后修改**：不经过用户审批不修改任何文件
- **逐条审批**：用户可以批准或拒绝每条建议
- **不要猜测**：不确定时提问，不要假设

## 使用示例

```
/记忆审查
/记忆审查 今天的收获
帮我整理一下记忆
```
