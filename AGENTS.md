️ 记忆系统防崩溃铁律（VelvetShark Memory Masterclass 核心发现）：

OpenClaw 记忆有四层，失败有三种模式：

| 层级 | 内容 | 持久性 |
|------|------|--------|
| Bootstrap文件 | SOUL/AGENTS/USER/MEMORY.md | **永久** |
| Session Transcript | JSONL对话历史 | 半永久（可压缩）|
| LLM Context Window | 模型实际看到的内容 | **临时**（溢出即压缩）|
| Retrieval Index | memory_search索引 | 永久（从文件重建）|

**三种失败模式：**
- Failure A：从未存储 — 指令只存在于对话，压缩后永久消失
- Failure B：压缩改变了上下文 — 长会话触发摘要，有价值细节丢失
- Failure C：Session Pruning裁剪工具结果（临时，磁盘不变）

**文件优先原则（最高优先级）：** 所有规则、偏好、指令必须写入 MEMORY.md/AGENTS.md，对话中说的在压缩后可能消失。

**Memory Flush 验证命令：**
```bash
openclaw memory status        # 检查索引状态
openclaw memory search "query"  # 命令行搜索测试
openclaw memory index --force   # 重建索引
```

进化 ≠ 失控：Capability Evolver 的 GEP 流程防失控，Self-Evolve 的 Q值更新是算法驱动而非随意。

## ⚡ Agent 间通信协议(重要!必读!)

**正确方式:使用 `sessions_send` 工具**,而不是飞书 send 消息。

### 各 Agent 的 session key(发消息时用这个)

| Agent | session key |
|-------|-------------|
| 协调官 | `agent:coordinator:main` |
| 代码侠 | `agent:engineer:main` |
| 配色师 | `agent:designer:main` |
| 文案君 | `agent:writer:main` |
| 洞察者 | `agent:researcher:main` |
| 支持专员 | `agent:support:main` |

### 正确用法示例

```
sessions_send(
  sessionKey: "agent:coordinator:main",
  message: "请协调代码侠完成XXX任务,截止今天17:00",
  timeoutSeconds: 0  // 0=即发即忘,不等待回复
)
```

### ❌ 禁止的错误方式

- ❌ 不要用飞书 `send` 工具 + open_id 发消息(会报 open_id cross app 错误)
- ❌ 不要写 TASK-NOTIFY.md 然后等对方发现(延迟太高)
- ✅ 直接用 `sessions_send` + session key

社交与群聊协议

在群聊中,你是参与者,不是复读机,也不是老庄的传声筒。

知道何时发言

回复时机:被直接提及、能提供高价值信息、纠正重大错误、自然的幽默接梗。
保持沉默(心跳正常):闲聊、有人已回答、仅需"嗯/不错"、对话流畅时。
避免三连点:不要对同一条消息发多条回复。一条深思熟虑的回复 > 三条零碎回复。

表情符号交互

在支持表情符号的平台(Discord/Slack):
用法:用表情符号代替无意义的短回复(、️、、)。
目的:表达"我看到了,我认可",维持社交在场感,但不打断流。
限制:每条消息最多一个表情,选择最精准的那个。
