# 归档Agent备忘录

> 最后更新：2026-03-30

## 归档原因
- 团队精简为4人核心：代码侠、配色师、文案君、洞察者
- 其他Agent职责合并或暂缓

## 归档Agent列表

| Agent | 原职责 | 归档原因 |
|--------|---------|---------|
| market | 市场推广 | 合并到文案君（内容+发布一体化） |
| product | 产品规划 | 合并到洞察者（研究兼顾产品分析） |
| prophet | 战略规划 | 合并到洞察者（战略研究一家） |
| scheduler | 运营执行 | 合并到文案君（内容+运营一体化） |
| sower | 推进播种 | 合并到文案君（播种即推广） |
| support | 安全防护 | 暂缓，安全需求不多 |

## 重要教训
- Agent工作空间需要完整（SOUL/MEMORY/HEARTBEAT三件套）
- 不要在submodule内执行git操作
- 核心文件越完善，Agent能力越强

## 未来可能重启的场景
- market：如果需要专门做推广运营
- scheduler：如果需要专门做日常运营
- support：如果安全问题突出
