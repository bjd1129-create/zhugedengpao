## 🔴 P0：漫画第一话渲染Bug（id=0时标题和格数全错）

**文件：** `comic.html` 第170行 `showStory` 函数

**Bug 1 - undefined 标题：**
```javascript
container.innerHTML = '...<div class="episode-title">'+stories[id-1].emoji+' '+stories[id-1].title+'</div>...';
```
当 `id=0`（漫画第一话）时，`stories[0-1]` = `stories[-1]` = **undefined**，导致标题显示"undefined undefined"。

**Bug 2 - 奇怪的单格渲染：**
代码逻辑上看 `storyPanels.forEach` 应该渲染8格，但截图只显示1格且标题是"第1格"。可能是 `story` 字段被错误赋值（见下条），导致 `storyPanels` 数组为空或只有1个元素。

**Bug 3 - 重复的 story 键：**
```javascript
legacyPanels.push({
  story: s,  // ← 重复key，JavaScript取最后一个
  src: '...comic-lobster-story'+s+'-p'+p+'.jpg',
  story: s,  // ← 同样是 s，但写法冗余
  label: '第'+p+'格'
});
```
虽然两个 `story` 值相同（都是 `s`），但这个冗余写法暴露了代码质量问题。另外，漫画第一话（id=0）的8个面板在 `allPanels.filter(p=>p.story===0)` 时理论上应该返回8个——但截图只显示1格，原因未明，可能是浏览器缓存了旧版JS。

**修复方案：**
- 修正 `stories[id-1]` → 当 `id=0` 时特殊处理
- 删除重复的 `story: s` 键
- 清理冗余代码，重新部署

### 🥈 P2：右下角浮蛙加"呼吸灯"效果

**现状：** 右下角的小花浮蛙是静态的，像个贴纸，不够"活"。

**具体做法：**
给浮动气泡加一个 CSS 呼吸动画：
```css
@keyframes breathe {
  0%, 100% { transform: scale(1) translateY(0); }
  50% { transform: scale(1.04) translateY(-2px); }
}
.floating-lobster { animation: breathe 3s ease-in-out infinite; }
```
- 动画周期：3秒（足够慢，不干扰阅读）
- 悬浮高度：-2px（微小起伏，不抢注意力）
- 配合微微的阴影变化，效果像在呼吸

**为什么重要：** 呼吸感是"活"和"静态"的分界线。访客看到会下意识觉得"这只龙虾在动"，哪怕他们没有意识到。

**执行者：** 代码侠（1行CSS即可实现）
