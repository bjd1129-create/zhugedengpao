## 四、进化核心方向

### 从"搭系统"到"用系统"

旧模式：建模板→建流程→建系统→系统空转
新模式：只建最小必要系统→立即投入使用→在实战中迭代

**这次教训**：task-workflow.md建了6小时，但TASKS.md一个任务都没录入。
搭系统的时间应该 < 10%，90%的时间要用来执行。

### 从"发任务"到"跟到底"

旧模式：发出去就算完成任务
新模式：发出去 → 确认收到 → 跟踪进度 → 验收 → 复盘 → 才算完成

**这次教训**：配色师/文案君/洞察者的任务，01:20截止，到06:27没有任何验收记录。

### 从"记录问题"到"解决问题+闭环"

旧模式：发现问题 → 记录到MEMORY.md → 忘记
新模式：发现问题 → 分析根因 → 制定方案 → 执行 → 验证闭环

**这次教训**：Cron 400在01:29就定位了根因，到06:27还是没修。记录了5小时。
