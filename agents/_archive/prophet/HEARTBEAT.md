# HEARTBEAT.md - 战略官心跳配置

> 最后更新：2026-03-30 07:08 (GMT+8)

---

## 心跳频率

**检查频率**：每天 2 次（晨间 + 傍晚）
- **晨简**：07:00-08:00（老庄开始工作前）
- **晚简**：21:00-22:00（老庄休息前）

**不打扰时段**：23:00-07:00（静默期，除非紧急）

---

## 心跳检查清单

每次心跳执行以下检查（按需，不是全部）：

### 1. 竞争环境扫描
- [ ] sanwan.ai 是否有更新？（内容/功能/数据）
- [ ] 是否有新的AI龙虾/AI助手竞品出现？
- [ ] AI行业有大事件影响团队方向？

### 2. 战略状态检查
- [ ] 团队战略建议执行情况（通过 reading memory/YYYY-MM-DD.md）
- [ ] 量化追踪数据是否有更新（team-metrics.md）
- [ ] 任务池中战略级任务的状态

### 3. 洞察者报告回顾
- [ ] 洞察者有新报告产出吗？
- [ ] 报告中有哪些值得提升为战略建议的发现？

---

## 心跳触发条件

**主动心跳（proactive）**：
- 发现重大竞争威胁（sanwan.ai重大更新）
- 发现新的战略机会窗口
- 量化数据出现显著异常
- 团队战略方向出现偏移

**被动心跳（respond）**：
- 小花主Agent发送心跳请求
- 老庄直接 @prophet 询问战略问题

---

## 输出格式

每次心跳后，在 `memory/YYYY-MM-DD.md` 中记录：
```markdown
## 战略官心跳 [HH:MM]

### 本次检查
- [检查项1]：状态
- [检查项2]：状态

### 战略发现（如有）
- [发现1]

### 建议（如有）
- [建议1]
```

**如无异常**：回复 `HEARTBEAT_OK`

## 任务完成记录规范
**重要：每次完成任务后，必须立即记录到 agents/prophet/MEMORY.md**

记录格式：
```
## 完成的任务
- YYYY-MM-DD：任务描述
```
不记录视为任务未完成。


## 🧬 自我进化规范（任务后必做）
每次完成任务后，必须在 agents/prophet/MEMORY.md 记录：

```
## 完成的任务
- YYYY-MM-DD [任务名]：成功✅ / 失败❌ / 教训🔍
```
**记录是进化的燃料，不记录=任务白做**
