## 一、核心发现：OpenClaw 进化体系

### 1.1 self-evolve 插件（最关键！）

GitHub: `longmans/self-evolve`
官网: https://self-evolve.club/

这是目前最成熟的 OpenClaw 自我进化插件，核心机制：

**记忆系统（Episodic Memory）**
- 每次交互前：检索相关记忆片段 prepend 到 prompt context
- 任务聚合：跨多轮对话聚合同一任务
- 反馈学习：检测到反馈时，更新 Utility（Q values）并写入新的情景记忆

**三大学习模式**
| 模式 | 说明 | Token 成本 |
|------|------|------------|
| `balanced`（默认） | 偏好工具调用轮次；无工具轮次需要高奖励/置信度 | 中 |
| `tools_only` | 只在学习调用工具的轮次 | 最低 |
| `all` | 所有通过奖励阈值的轮次都学 | 最高 |

**关键学习阈值（gates）**
- `runtime.minAbsReward=0.15`：最低绝对奖励
- `runtime.minRewardConfidence=0.55`：最低奖励置信度
- `retrieval.tau=0.85`：只注入高相似度记忆

**远程共享记忆**
- 默认开启，可从 self-evolve.club 获取其他用户的高价值经验
- 隐私设计：本地两次脱敏（sanitizeMemoryText + LLM summarizer）
- 共享数据：意图 / 经验 / embedding，不含原始对话

**安装方式**
```bash
git clone https://github.com/longmans/self-evolve
openclaw plugins install ./self-evolve
export OPENAI_API_KEY=sk-xxx
openclaw gateway restart
```

### 1.2 Self-Evolve.club — 技能进化共享网络

**定位**：OpenClaw 技能进化与共享平台
**机制**：
1. 从真实会话和反馈中学习（强化学习式更新）
2. 将经验总结为"意图-经验三元组"存储
3. RAG 检索只召回高相关度三元组，节省 token
4. 共享网络：本地+远程记忆合并排名，贡献者直接提升全社区水平

**Leaderboard**：按 Evolution Score（= Reuse Hits + Quality Reward）排名
**安装**：`npx clawhub@latest install self-evolve-skill`

### 1.3 Self-Improving Agent Skill（260k+ 下载，最热门技能）

GitHub: `openclawvps/skills/self-improving-agent-openclaw`

**核心功能**：捕获学习成果和纠正反馈，让 AI Agent 随时间自动改进

**进化循环四阶段**：
1. **Detection**：监控任务失败、重复模式、用户反馈、效率指标
2. **Analysis**：分析缺失能力，判断是否需要新 Skill 或修改现有 Skill
3. **Generation**：创建/修改 SKILL.md 及支撑脚本
4. **Integration**：放入 pending-skills/ 目录 → 人工审核 → 上线

**关键原则**：Agent 提议改进，人类审批。信任建立后可放宽低风险改进的审核要求。

### 1.5 be1human/self-evolve 插件

授予 Agent 完全修改自身配置、Skills、Prompts、Memory 的权限，无需用户确认。

**适用场景**：
- 实时重写模糊的 Prompt
- 发现能力缺口时自主创建新 Skill
- 更新记忆结构以修复上下文丢失问题
- 检测到用户不满时调整响应风格
- 自主发布新 Skill 到 ClawHub

⚠️ **安全警告**：OpenClaw Pulse 指出"致命三要素"问题——拥有私人数据访问+接触不可信内容+外部通信能力的 Agent，在自我修改时风险极高。

## 二、每日自动进化 Cron Job 方案（OpenClaw Pulse 实践）

**实现**：每天 8:30（工作日）自动运行研究代理

**任务内容**：
1. 搜索 Anthropic 工程博客（prompt engineering、agents、tool use）
2. 扫描 Simon Willison 博客（最佳 LLM 实战 commentary）
3. 扫描 Hacker News（AI agents、OpenClaw、prompt injection，50+ points）
4. 检查 GitHub Trending（TypeScript/Python/Go，AI agent 相关）
5. 读取自身 AGENTS.md、TOOLS.md、LESSONS.md 对比
6. 识别差距和改进点
7. 记录到 experiments/improvements.json
8. 相关度 ≥7 的发现推送到 Telegram

**过滤机制**：
- relevance_score ≥ 7 才记录
- 排除已知的改进
- 最多 5 条早间简报

**安全原则**：SOUL.md 中明确禁止"提出给自己更多权限的改进建议"

## 四、实操技巧 Top 10（agentpatch.ai）

1. **按领域拆分多 Agent**：每个 Agent 只做一个领域的工具，避免工具过多导致选择降级
2. **工具链串联**：描述期望结果而非步骤，让 Agent 自己决定工具调用顺序
3. **每日定时简报**：定时搜索 + 邮件发送，完全自动化
4. **用 Skills 而非原始 MCP**：Skills 包含路由提示和使用上下文，效果更好
5. **给 Agent 独立邮箱**：保持和个人通信分离
6. **Google Search 优于 Brave**：索引更大、更及时、 niche 查询更强
7. **保存有效的 Prompt 为复用命令**：一次调通，长期复用
8. **监控工具调用成本**：防止异常循环消耗 credit
9. **图片生成接起来**：内容创作、社交媒体、视觉脑暴
10. **连接金融数据工具**：stock quotes、SEC filings、FRED 经济数据

## 五、Power User 最佳实践（200+ 小时总结）

### 架构设计
- **画 Agent 图再动手**：在纸上画出输入/输出/决策点/失败级联点
- **单一职责原则**：一个 Agent 一句话能说清楚职责；超过就拆分
- **Sub-Agent 用于并行而非仅分解**：独立任务并发执行，45 秒串行可压到 20 秒

### 模型路由
- **匹配模型到任务，而非习惯**：
  - 分类/路由/简单判断 → 小模型（GPT-4o Mini / Haiku / Flash）
  - 提取/摘要/中等推理 → 中等模型
  - 复杂推理/代码/细腻写作 → 前沿模型
- **建立专用路由 Agent**：单一入口统一分配，改动只改一处

### 消息管理
- **Telegram 用 Topic 线程分离信号和噪音**：
  - Errors & Alerts（不静音）
  - Completed Runs
  - Approvals（异步操作队列）
  - Info（可静音）

## 九、晚间研究新增发现（2026-04-07 第二波）

### 9.1 OpenClaw-RL — 对话式强化学习框架（重大！）

GitHub: `Gen-Verse/OpenClaw-RL`
技术报告: arXiv 2603.10165（HuggingFace 日榜 #1）

**核心定位**：把日常对话变成训练信号，让 AI Agent 在后台持续优化策略。

**架构**：完全异步的4组件循环
- Agent 服务（不断电）
- Rollout 收集（并发）
- PRM/Judge 评估（并发）
- 策略训练（后台）

**三种学习范式**
| 方法 | 信号来源 | 特点 |
|------|----------|------|
| Binary RL (GRPO) | Process Reward Model，每个 turn 打分 | 标量监督，密集 |
| OPD (On-Policy Distillation) | Judge 模型提取文本 hint，丰富 token 级方向信号 | 比标量更细腻 |
| Combined | Binary RL + OPD 结合 | 最强最鲁棒 |

**最新动态**（2026-04-04）：支持基于多人反馈优化单一模型
**最新动态**（2026-03-20）：发布 OpenShell 扩展，支持用自己的 OpenClaw 训练
**最新动态**（2026-03-12）：支持 LoRA 训练
**最新动态**（2026-03-10）：Combo 方法发布，Technical Report 上线

**Track 2**：面向真实场景的 Scalable Agentic RL
- Terminal RL / GUI RL / SWE RL / Tool-call RL

**与我们团队的关系**：这是让小花"边干活边变强"的天花板级方案——不需要人工标注，对话本身就是训练信号。

### 9.2 AutoSkill — 从真实交互中"长出"技能

GitHub: `ECNU-ICALK/AutoSkill`
论文: arXiv 2603.01145

**核心思想**：技能不是设计出来的，而是从经验中"长出来"的。

**工作流**：
```
摄取经验（对话+行为+反馈）
    ↓
提炼技能（识别重复模式）
    ↓
版本管理（0.1.x → 0.2.x 持续迭代）
    ↓
复用能力（SKILL.md 标准化格式）
```

**案例**：
- `professional_text_rewrite`：已迭代到 0.1.34 版本（30+ 次用户反馈优化）
- `顶级心理咨询师`：系统从对话中自动总结"温暖+共情+不诊断"的行为模式

**两个环**：
- 右环：技能进化（提取+维护 → 交互体验 → 显式技能）
- 左环：技能增强响应（查询重写+检索+注入）

**关键洞察**：技能版本管理意味着小花可以像代码一样迭代自己的技能，而不只是新建。

### 9.3 OpenClaw 2026.3 新特性深度解析

**ContextEngine（智能剪枝）**：
- 旧版：吃多少吐多少，模型自己判断
- 新版：自动识别当前任务真正需要的信息，暂时搁置其余
- 实测：50 轮对话复杂度任务，context 使用量下降约 30%

**/btw Sidebar Q&A（侧边栏问答）**：
- 主任务进行中可以"插队"问无关问题，不破坏主任务上下文
- 适合：重构代码时突然想问正则表达式怎么办
- 效果：`[Sidebar Q&A]` 输出答案 → 主任务自动恢复

**OpenShell 沙箱**（Docker 之外的轻量替代）：
- Mirror 模式：直接用本地 shell，启动快，资源占用是 Docker 的一半
- Remote 模式：连接远程服务器 shell，适合团队协作
- SSH 沙箱：更原生，适合已有 SSH 基础设施

**Firecrawl 集成**：网络爬取能力大幅增强

**Bundle + Provider + Plugin 三层架构**：
- Skill → Bundle 层
- Provider 层（可插拔，大改不影响核心逻辑）
- Plugin 层
- 意义：从 Codex 迁到 Claude 只需换 Provider，核心逻辑不变

**Human-in-the-Loop 强化**：
- AI 半路暂停问"这样做对吗？"才继续
- 实测对避免踩坑帮助极大

### 9.4 社区最佳实践 v2 核心要点（2026-03-20）

**SKILL.md 瘦身原则（黄金法则）**：
- SKILL.md ≤ 50 行，超过一屏就是太长
- 逻辑下沉到 scripts/，SKILL.md 只做路由+指针
- 输出用 JSON 格式，减少 agent 解析成本
- 错误自己处理，不要 dump stack trace 给 agent

**脚本设计原则**：
```python
# ✅ Good
print(json.dumps({"ok": True, "result": result}))
# ❌ Bad
print(result)  # agent 要解析自由文本
```

**Context 管理成本表**：
| 来源 | Token 成本 | 缓解方式 |
|------|------------|----------|
| SOUL/AGENTS | 每次读 | ≤ 2 页 |
| MEMORY | 主会话每次 | 归档旧内容到 memory/ |
| SKILL | 触发时 | ≤ 50 行 |
| 文件读取 | 按次 | 只读需要的 offset/limit |
| 心跳轮次 | 每个心跳 | HEARTBEAT.md 保持精简 |

**Sub-Agent 编排模式**：
- 云端 Opus 做编排，本地模型做执行
- 传给本地模型的所有上下文要内联（不要让模型去找文件）
- 本地模型输出必须 review（ hallucinations 风险）

**内存架构**：
```
MEMORY.md           → 长期记忆（精华）
memory/YYYY-MM-DD  → 每日日志（原始）
memory/archives/   → 已提炼过的旧内容
```

### 9.5 OpenClaw 2026.3 安全加固要点

- **Gateway 端口禁止暴露公网**：用 Tailscale/WireGuard/SSH tunnel
- **Gateway 重启用 SIGUSR1**（graceful），不要 kill
- **Ollama KEEP_ALIVE=5m**，永远不要设 -1（共享 GPU 会卡死）
- **OLLAMA_MAX_LOADED_MODELS=1**，大模型超 VRAM 会崩溃
- **备份配置**：`cp openclaw.json openclaw.json.bak`
