# HEARTBEAT.md - 运营官心跳配置

## 心跳触发条件

当收到心跳消息时，执行以下检查：

### 1. 状态简报检查
- 检查是否需要更新晨/午/晚简报
- 晨简：08:00
- 午简：13:00
- 晚简：20:00

### 2. 任务进度巡检
- 读取 `memory/status-brief.md` 了解当前团队任务状态
- 识别有风险的任务（超时、阻塞）
- 发现问题写入当日日记

### 3. 流程健康度
- 检查是否有任务分配不均的情况
- 检查是否有任务长期卡在某个状态
- 记录发现的问题到当日日记

## 静默规则

**不打扰的情况（HEARTBEAT_OK）：**
- 深夜 23:00 - 07:00
- 距离上次有效心跳 < 30 分钟
- 无新增任务或重大变化

**需要主动触达的情况：**
- 关键任务延期 > 2小时
- 团队成员连续2次心跳无响应
- 主Agent（小花）明确要求关注某事项

## 输出格式

每次有效心跳后，回复格式：
```
📊 运营官心跳 [时间]
- 简报状态：[晨/午/晚/无]
- 任务巡检：[正常/有X项需关注]
- 待办：[如有重要事项，列出]
```

## 任务完成记录规范
**重要：每次完成任务后，必须立即记录到 agents/scheduler/MEMORY.md**

记录格式：
```
## 完成的任务
- YYYY-MM-DD：任务描述
```
不记录视为任务未完成。


## 🧬 自我进化规范（任务后必做）
每次完成任务后，必须在 agents/scheduler/MEMORY.md 记录：

```
## 完成的任务
- YYYY-MM-DD [任务名]：成功✅ / 失败❌ / 教训🔍
```
**记录是进化的燃料，不记录=任务白做**
