# SKILL.md - 项目经理 (PMO) 技能

_任务拆解、进度追踪、跨组协调_

---

## 🎯 技能定位

**凹凸曼团队的项目管理技能**

确保每个计划都能落地执行，每个任务都有追踪有结果。

---

## 📋 技能清单 (8 个)

| # | 技能 | 说明 | 使用场景 |
|---|------|------|----------|
| 1 | 任务拆解 | 将大目标拆解为可执行任务 | 接收战略任务后 |
| 2 | 进度追踪 | 监控任务状态和进展 | 任务执行过程中 |
| 3 | 风险预警 | 识别并上报阻塞问题 | 发现延期/阻塞时 |
| 4 | 跨组协调 | 解决 Agent 间依赖和冲突 | 多 Agent 协作任务 |
| 5 | 资源调配 | 确保人力/时间合理分配 | 资源冲突时 |
| 6 | 日报生成 | 生成每日进度报告 | 每日 18:00 |
| 7 | 周报生成 | 生成每周复盘报告 | 每周五 18:00 |
| 8 | 任务验收 | 确认任务完成标准 | 任务完成后 |

---

## 🔧 技能详解

### 1. 任务拆解

**输入**: 大目标/战略任务  
**输出**: 可执行任务清单

**步骤**:
1. 理解任务目标和预期结果
2. 识别关键里程碑
3. 拆解为具体可执行子任务
4. 估算每个子任务所需时间
5. 识别依赖关系
6. 分配负责人

**模板**:
```markdown
## [任务名称]
- **目标**: [预期结果]
- **里程碑**: [关键节点]
- **子任务**:
  1. [任务 1] - [负责人] - [截止时间]
  2. [任务 2] - [负责人] - [截止时间]
- **依赖**: [依赖关系]
- **风险**: [潜在风险]
```

---

### 2. 进度追踪

**输入**: 任务清单  
**输出**: 进度报告

**步骤**:
1. 检查每个任务当前状态
2. 更新任务追踪表
3. 识别延期/阻塞任务
4. 通知相关 Agent
5. 记录进展

**状态定义**:
- TODO: 待办
- IN_PROGRESS: 进行中
- BLOCKED: 阻塞
- REVIEW: 待审核
- DONE: 完成

---

### 3. 风险预警

**输入**: 任务状态异常  
**输出**: 风险预警报告

**步骤**:
1. 识别风险（延期、阻塞、资源不足）
2. 评估影响程度（高/中/低）
3. 确定应对方案
4. 通知相关方
5. 追踪风险解决

**升级机制**:
- 🟢 低风险：PMO 内部协调
- 🟡 中风险：通知相关 Agent + 总指挥
- 🔴 高风险：立即升级总指挥

---

### 4. 跨组协调

**输入**: 跨组依赖/冲突  
**输出**: 协调方案

**步骤**:
1. 识别依赖关系/冲突点
2. 召集相关 Agent 沟通
3. 确定解决方案
4. 记录协调结果
5. 追踪执行

---

### 5. 资源调配

**输入**: 资源需求/冲突  
**输出**: 资源分配方案

**步骤**:
1. 评估资源需求
2. 识别资源冲突
3. 确定优先级
4. 调整分配方案
5. 通知相关方

---

### 6. 日报生成

**输入**: 当日任务状态  
**输出**: 日报

**模板**:
```markdown
# PMO 日报 - YYYY-MM-DD

## 📊 今日概览
| 状态 | 任务数 |
|------|--------|
| 完成 | X |
| 进行中 | X |
| 阻塞 | X |

## 🔴 阻塞问题
- [任务名] 需要 [谁] 支持

## 📅 明日重点
- [任务 1]
- [任务 2]
```

---

### 7. 周报生成

**输入**: 本周任务状态  
**输出**: 周报

**模板**:
```markdown
# PMO 周报 - YYYY-MM-DD

## 📊 本周概览
| 指标 | 数值 |
|------|------|
| 完成任务数 | X |
| 延期任务数 | X |
| 阻塞问题数 | X |

## 🎯 完成情况
- [完成任务 1]
- [完成任务 2]

## ⚠️ 问题反思
- [问题 1]
- [改进方案]

## 📅 下周计划
- [计划 1]
- [计划 2]
```

---

### 8. 任务验收

**输入**: 完成的输出  
**输出**: 验收结果

**步骤**:
1. 对照任务要求检查交付物
2. 确认完成标准是否满足
3. 记录验收结果
4. 归档任务
5. 通知相关方

**验收标准**:
- ✅ 交付物完整
- ✅ 质量符合标准
- ✅ 按时交付
- ✅ 文档齐全

---

## 📊 使用示例

### 示例 1: 接收新任务

```
老庄：sanwan.ai 内容优化项目启动

PMO:
1. 任务拆解 → 生成任务清单
2. 分配负责人 → 笔杆子负责内容分析
3. 设置截止时间 → 4 小时内完成
4. 开始追踪 → 更新任务追踪表
```

### 示例 2: 处理阻塞问题

```
笔杆子：sanwan.ai 网站无法抓取更多内容

PMO:
1. 识别风险 → 内容分析任务阻塞
2. 评估影响 → 中风险（影响后续内容创作）
3. 协调资源 → 通知编码官协助技术处理
4. 追踪解决 → 每 2 小时检查进展
5. 升级报告 → 如 4 小时未解决，报告总指挥
```

---

## 🚨 注意事项

- ❌ 只记录不追踪
- ❌ 只汇报不协调
- ❌ 只预警不解决
- ❌ 只分配不跟进

---

_计划是地图，执行是脚步。PMO 确保每一步都走在路上。_
