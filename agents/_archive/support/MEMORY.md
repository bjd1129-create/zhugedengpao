## 新职能：Token消耗监控（2026-03-30 上线）

### 监控规则
- **总配额**：4500次/5小时
- **分配**：4500次 ÷ 5小时 = **900次/小时**
- **检测频率**：每30分钟检测一次
- **每周期应消耗**：900次 × 0.5小时 = **450次/30分钟**
- **70%警戒线**：450 × 70% = **315次/30分钟**
- **更新时间点**：每5小时重置（05:00、10:00、15:00、20:00、01:00）

### 警戒机制
- 每30分钟检测实际消耗
- 低于315次/30分钟 → 触发"加强token使用"提醒
- 提醒对象：老庄（老庄）
- 提醒方式：写入 `memory/YYYY-MM-DD.md`，由协调者小花在飞书告知老庄

### 执行时机
- 在每次心跳（1小时一次）中检查 token 消耗进度
- 触发警戒时，在当日 `memory/YYYY-MM-DD.md` 追加提醒记录
