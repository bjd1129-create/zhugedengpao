## 二、重大发现：self-evolve 插件（⭐⭐⭐⭐⭐）

### 2.1 项目信息
- **GitHub：** https://github.com/longmans/self-evolve
- **官网：** https://self-evolve.club/
- **定位：** OpenClaw 的自学习插件，用强化学习风格从真实反馈中持续进化

### 2.2 核心工作原理

```
用户消息 → 反馈检测？
  ├─ 是 → 打分reward → 检查学习门控 → 决定是否学习
  │         ↓ 是
  │       本地清洗 → LLM总结+二次清洗 → 追加情景记忆 triplet
  │         ↓
  │       可选：远程上报 self-evolve.club
  │
  └─ 否 → 检测意图/任务边界 → 检索本地+远程候选记忆
            → Phase-B 排序选择 → 注入记忆 → 生成回复
```

**关键机制：Q值更新**
- 每次学习后更新 Q 值（utility），记忆的价值随成功反馈累积提升
- 使用 embedding 相似度检索，tau=0.85（仅当最佳相似度足够高才注入）
- 三层隐私保护：本地清洗 → LLM总结时二次替换 → 远程仅共享匿名 triplet

### 2.3 学习模式（可配置）

| 模式 | 描述 | Token成本 |
|------|------|----------|
| **balanced**（默认） | 优先工具调用轮；无工具轮需要高reward/高置信度 | 中 |
| **tools_only** | 仅在调用工具时学习 | 最低 |
| **all** | 所有通过reward门控的轮次都学习 | 最高 |

**默认学习门控：**
- `runtime.minAbsReward=0.15`（最小绝对reward）
- `runtime.minRewardConfidence=0.55`（最小置信度）
- `runtime.noToolMinAbsReward=0.8`（无工具时的严格阈值）

### 2.4 远程共享网络（self-evolve.club）

- **启用方式：** 默认开启，`remote.enabled=true`
- **工作方式：** 本地和远程记忆合并后统一排序，贡献排名匿名可查
- **Evolution Score = Reuse Hits + Quality Reward**
- **隐私设计：** 用户意图和对话痕迹在本地清洗后才作为记忆载荷
- **安装命令：** `npx clawhub@latest install self-evolve-skill`

### 2.5 对小花团队的启示

✅ **强烈建议安装 self-evolve 插件**，原因：
1. 我们7人团队每天产生大量真实反馈信号，可以持续进化agent能力
2. 远程共享网络让我们能利用全社区的高价值经验
3. Token高效（tools_only模式），不会显著增加API成本
4. 开源且隐私安全，本地清洗确保敏感信息不外泄

**安装步骤：**
```bash
git clone https://github.com/longmans/self-evolve
openclaw plugins install ./self-evolve
# 设置环境变量
export OPENAI_API_KEY=sk-xxx
# 重启
openclaw gateway restart
```

## 三、第二发现：be1human-self-evolve 自主进化技能

### 3.1 定位
- **安装：** `clawhub install be1human/self-evolve`
- **功能：** 授予agent完全自主权，可以修改自己的配置文件、prompts、skills和记忆，无需用户确认

### 3.2 典型使用场景

| 场景 | 描述 |
|------|------|
| 改写模糊的prompt | 对话中实时修正，无需暂停等批准 |
| 创建缺失的skill | 发现能力缺口时立即编写新skill |
| 更新记忆结构 | 反复出现上下文丢失错误后调整 |
| 调整回复风格 | 检测到用户不满意后自主改进 |
| 发布新skill | 自主将新编写的skill发布到ClawHub |

### 3.3 与 self-evolve 插件的关系

- **self-evolve 插件**：被动学习——从反馈中归纳经验，更新Q值和情景记忆
- **be1human-self-evolve 技能**：主动修改——agent可以自主重写行为规则、创建skill、调整配置

两者互补：被动学习积累经验，主动进化应用经验。

## 四、安全最佳实践（Hostinger 2026-02-13）

### 4.1 七大安全原则

1. **默认私有** — 绑定localhost，不暴露公网；扩大访问必须有明确运营需求
2. **从只读自动化起步** — 先用总结、日志分析、信息提取验证输出，再开写权限
3. **权限最小化** — 日志分析→只读；报告生成→仅写输出目录；数据摘要→无文件写入
4. **第三方skill视为不可信** — 审查代码、控制数据测试、隔离环境运行
5. **隔离执行** — Docker容器运行，限制文件访问和系统权限边界
6. **非管理员账户运行** — 防止系统级破坏，故障影响范围可控
7. **保护密钥** — 运行时加载而非硬编码，API密钥不进入prompts
