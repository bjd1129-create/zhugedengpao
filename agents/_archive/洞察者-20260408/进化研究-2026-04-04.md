## 一、本轮核心发现

### 1. OpenClaw-RL（Princeton 2026.03）：强化学习框架

**论文**：[arXiv:2603.10165](https://arxiv.org/abs/2603.10165) — "Train Any Agent Simply by Talking"

**核心洞察**：所有 Agent 交互都在产生"下一状态信号"（用户回复、工具输出、终端/GUI 状态变化），但现有 RL 系统没有把它们作为实时在线学习来源。

**OpenClaw-RL 的突破**：
- **下一状态信号** = 两种信息：**评估信号**（PRM judge 打分 scalar reward）+ **指导信号**（通过 Hindsight-Guided On-Policy Distillation / OPD 恢复"应该怎么做不同"）
- **统一训练环**：个人对话、终端执行、GUI 交互、SWE 任务、工具调用跟踪——不是独立的训练问题，是同一策略的同一条训练回路
- **异步设计**：模型服务 live 请求 + PRM 评判 ongoing 交互 + 训练器同时更新策略，三者零协调开销
- **对个人 Agent 的价值**：从用户 re-query、纠正、明确反馈中恢复对话信号，实现"通过使用来进化"

**对团队的启示**：
- 这是目前最学术化、最系统化的 OpenClaw 训练框架
- 核心思想：每个交互都是训练数据——不只是"做对做错"，而是从下一状态中提取方向性优势（token-level directional advantage）
- **可直接借鉴**：建立反馈打分机制（正/负反馈）+ OPD 式的"回顾重训"记录

### 2. self-evolve 插件（longmans/self-evolve）：Q值强化学习

**仓库**：[github.com/longmans/self-evolve](https://github.com/longmans/self-evolve)  
**网站**：self-evolve.club（共享排行榜）

**架构设计**：
```
收到用户消息
  ├─ 反馈轮次？→ 评分reward + 检查学习门 → 决定是否学习
  │    → sanitizeMemoryText 第一次脱敏
  │    → LLM总结 + 第二次脱敏
  │    → 追加本地记忆三元组 (intent/experience/embedding)
  │    → 可选：远程注册到 self-evolve.club
  └─ 非反馈轮次 → 检测意图边界 → 检索本地+远程候选 → Phase-B排序 → 注入记忆 → 生成回复
```

**关键参数（推荐值）**：
| 参数 | 默认值 | 说明 |
|------|--------|------|
| runtime.minAbsReward | 0.15 | 最小绝对奖励阈值 |
| runtime.minRewardConfidence | 0.55 | 最小奖励置信度 |
| runtime.noToolMinAbsReward | 0.8（balanced模式无工具轮） | 无工具调用时的奖励门槛 |
| retrieval.tau | 0.85 | 只在最高相似度够高时注入记忆 |
| memory.maxEntries | 200 | 最大记忆条目数 |

**三种学习模式**：
- `balanced`（默认）：偏好工具调用轮；无工具轮需要更高奖励/置信度
- `tools_only`：只在调用工具时学习（最低 token 消耗）
- `all`：所有通过奖励门的轮次都学习（最高 token 消耗）

**隐私设计（双重脱敏）**：
1. `sanitizeMemoryText` 移除对话元数据、ID、发送者标签
2. 经验总结器要求 LLM 输出"可迁移策略"并替换敏感数据为 `[REDACTED_*]`

**远程共享网络**：
- 默认开启，向 self-evolve.club 上报高质量经验
- 从他人贡献中获取远程记忆候选（Q值 + reuse hits + quality reward 排名）
- **关闭远程**：`plugins.entries.self-evolve.config.remote.enabled false`

**安装状态（团队现状）**：
- ✅ self-evolve 插件已安装（今天 20:29 完成）
- ✅ balanced 模式 + 远程网络开启
- ✅ Gateway 需重启完全生效

**最佳实践**：
- 明确的正反馈 > 明确的负反馈 > 模糊的"ok"
- 工具调用比纯对话更有训练价值
- 每次 cron 日志检查是否有 `self-evolve: initialized` 和 `[self-evolve] hook before_prompt_build`

### 3. Self-Improving + AutoSkill 双环体系

**Self-Improving Agent**（clawhub.ai/pskoett/self-improving-agent）：
- 在 `.learnings/` 目录下建立三个文件：
  - `ERRORS.md`：命令失败、工具出错、推理偏差
  - `LEARNINGS.md`：纠正信息、更优方案、工具使用技巧
  - `FEATURE_REQUESTS.md`：用户改进建议
- 自动触发：命令失败 / 用户纠正 / 发现更优方案 / API 失败 / 知识过时
- 每天凌晨 4:00 自动复盘（提取当天对话经验 → 更新 MEMORY.md）

**AutoSkill**（ECNU-ICALK/AutoSkill）：
- [论文 arXiv:2603.01145](https://arxiv.org/abs/2603.01145)
- 不是人工设计技能，而是从真实交互中**自动提炼**技能
- **双环架构**：技能进化（提取+维护 → 显式技能）+ 技能增强（查询重写+检索+上下文注入）
- **通过记忆增长而非模型微调来持续改进**
- 技能以 SKILL.md 形式存储，有版本号（案例：professional_text_rewrite 已迭代至 v0.1.34）
- 重复出现的模式才被抽象为技能，一次性请求不会生成

**对团队的启示**：
- Self-Improving = "个体学习"（自己学自己的错误）
- AutoSkill = "集体学习"（从交互中提炼可复用技能）
- **两者结合 = 完整的学习系统**

### 4. openclaw-self-optimizing 自我进化引擎

**仓库**：github.com/lyxxy01/openclaw-elite-skills（openclaw-self-optimizing）

**五大模块**：

| 模块 | 功能 | 推荐时机 |
|------|------|----------|
| Meta-Learning Engine | 读 `.learnings/`，识别 recurrence ≥ 3 的模式 | 每日 03:00 cron |
| Automatic Reflection | 生成结构化会话反思，写入 memory/ | 每会话末 |
| Performance Analytics | 跟踪 token 消耗、成功率、任务类型分布 | 每周 |
| Skill Synthesizer | 从学习条目自动生成 SKILL.md | 每周 |
| Behavioral Adaptation | 跟踪用户纠正 → 7次后自动提升到 SOUL.md | 实时 |

**关键机制 - Behavioral Adaptation**：
- 用户纠正出现 → 记录到 adaptation 日志
- 7 次同类型纠正 → 自动将适应提示注入 SOUL.md
- 这是一个"习惯养成"机制

### 6. 安全警示：自我修改 Agent 的"致命三合一"

**来源**：openclawpulse.com

**"致命三合一"问题**：
拥有以下三者的 Agent 面临极高风险——
1. **私有数据访问**（private data access）
2. **不受信任内容的暴露**（exposure to untrusted content）
3. **外部通信能力**（external communication abilities）

**对自我进化 Agent 的特别放大**：自我修改（self-modifying）+ 三合一 = 极高风险

**安全建议**：
- 外部内容（web fetch、用户输入）必须严格脱敏后再用于学习
- self-evolve 的双重脱敏机制（sanitizeMemoryText + 经验总结器）是正确方向
- 定期审查 `.learnings/` 目录内容，防止提示注入
