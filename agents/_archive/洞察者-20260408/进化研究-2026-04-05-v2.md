## 一、Self-Improving Agent v3.0.5 重大升级（268k+下载量）

这是 ClawHub 下载量最高的技能，2026-03-17 最新更新。

### 1.1 核心机制：Analyze → Log → Report 循环
```
每次会话结束 → 自动分析对话质量
                ↓
         记录到 improvement_log.md
                ↓
         定期生成改进报告
                ↓
         推送到项目记忆文件（SOUL.md/AGENTS.md 等）
```

### 1.2 v3.0.5 新增：完整晋升（Promotion）工作流
**核心创新**：不是把 learnings 堆在那里，而是有明确的晋升规则：

| 层级 | 触发条件 | 晋升目标 |
|------|---------|---------|
| 即时日志 | 任何错误/纠正/成功模式 | improvement_log.md |
| 每日复盘 | 每日会话结束 | memory/YYYY-MM-DD.md |
| 项目级晋升 | 3次同类型问题 或 跨session重复 | SOUL.md / AGENTS.md / TOOLS.md |
| 全局共享 | 团队通用价值 | self-evolve 远程网络 |

**晋升标准**：
- 同一错误出现 3+ 次 → 直接写入 SOUL.md（永久化）
- 跨 session 复现的成功模式 → 写入 AGENTS.md
- 工具使用最佳实践 → 写入 TOOLS.md
- 用户偏好模式 → 写入 USER.md

### 1.3 日志类型与格式规范
**错误日志模板**：
```markdown
## ERROR-YYYY-MM-DD-HHMM
**Task**: [任务描述]
**Failure**: [失败表现]
**Fix Applied**: [修复方法]
**Session Link**: [相关会话]
**Repeatable**: YES/NO  ← 3× YES → 晋升SOUL.md
```

**Feature Request 日志模板**：
```markdown
## FEATURE-YYYY-MM-DD-HHMM
**Capability**: [所需能力]
**Trigger Scenario**: [触发场景]
**Proposed Approach**: [建议方案]
**Priority**: P0/P1/P2
```

### 1.4 对我们团队的直接价值
- **立即可安装**：`npx clawhub@latest install self-improving-agent`（但我们已有 self-evolve，这两个不冲突）
- **补充价值**：self-evolve 管 Q 值学习，self-improving-agent 管结构化日志，两者互补
- **晋升规则**："3次同一错误 → SOUL.md" 可以直接借鉴

## 二、Thoth System：最完整的 OpenClaw 超集系统

GitHub: AlekseiUL/openclaw-superagent，MIT 协议

### 2.1 核心架构
把空白 OpenClaw 变成完整 AI agent 的 2 分钟安装包。核心组件：

```
thoth-system/
├── workspace/          # Agent 大脑
│   ├── SOUL.md         # 人格+价值观
│   ├── IDENTITY.md     # 名字/角色
│   ├── AGENTS.md       # 工作规则+安全+记忆管理
│   ├── TOOLS.md        # 可用工具配置
│   ├── MEMORY.md       # 长期事实
│   ├── USER.md         # 所有者信息
│   ├── HEARTBEAT.md    # 主动检查调度
│   ├── SECURITY-RULES.md
│   └── HONESTY-RULES.md
├── memory/
│   ├── handoff.md              # session 间上下文传递
│   ├── decisions/lessons-learned.md  # ⭐ 自我改进日志
│   └── patterns.md             # 重复模式检测+自动化提案
├── skills/
│   ├── agent-doctor/    # 自我诊断 skill
│   └── agent-forge/    # Skill 创建 skill
└── scripts/
    ├── smart-watchdog.sh     # WAL + gateway 自动修复
    ├── post-update-check.sh  # 更新后验证
    └── pre-restart-handoff.sh  # 重启前保存上下文
```

### 2.2 ⭐ Graduation（毕业）机制——最有价值的创新
**规则**：如果同一个错误发生 3+ 次 → 教训永久晋升到 SOUL.md

```
Lesson Learned × 1  → memory/decisions/lessons-learned.md
Lesson Learned × 3  → SOUL.md（永久生效）
                       ↓
               Agent 永远不会重蹈覆辙
```

**对我们团队的直接借鉴**：
- 现有问题：同样的坑反复踩（如 portfolio.json 被误删）
- 解决方案：在 memory/ 建立 lessons-learned.md，3次同一错误 → 写入 AGENTS.md 安全规则

### 2.3 Self-Healing（自愈）机制
**Smart Watchdog** 监控：
- SQLite WAL（Write-Ahead Log）状态
- Gateway 进程健康
- 自动检测并修复问题（无需人工介入）

**对我们**：可以借鉴这个脚本逻辑，建立小花团队的系统健康监控

### 2.4 三自动 Cron
| Cron | 时间 | 任务 |
|------|------|------|
| 每日备份 | 每日某时 | workspace 备份 |
| 晨间简报 | 每日早 | 今日任务预览 |
| 晚间日记 | 每日晚 | 每日总结写入 memory |

**对比我们**：我们有 heartbeat-state.json 的心跳检查，但没有系统的晨/晚自动日志 cron。建议参考。

### 2.5 Security + Honesty Rules
**安全规则**：
- Gateway 绑定 127.0.0.1（不从外部访问）
- API keys 只在 .env，不进 config/logs
- allowFrom 限制 Telegram 访问
- 数据不外发，除非明确授权
- 删除永远进 trash，不永久删除

**诚实规则（Honesty Rules）**：
- Agent 绝不捏造事实
- 立即承认错误
- 不懂就说不懂

**对我们**：我们的 AGENTS.md 已有安全规则，但缺少"诚实规则"。建议补充。

### 3.1 9 Agent 分工体系
| Agent | 职责 |
|-------|------|
| Email Agent | 读邮件/排日程 |
| Calendar Agent | 管理日历 |
| Research Agent | 深度研究 |
| Writing Agent | 内容创作 |
| Sales Agent | 销售跟进 |
| Social Agent | 社交媒体 |
| Reminder Agent | 提醒事项 |
| Coordination Agent | 协调各 Agent |
| Quality Agent | 结果质检 |

### 3.2 ⭐ 最重要安全忠告（我们完全忽略了这点！）
> **"Never install OpenClaw on an actively used work or personal computer."**
> **"OpenClaw can technically have access to all the files on the computer it runs on."**

**教训**：不要把 OpenClaw 装在正在使用的主力电脑或工作机上。
- 我们当前：MBB's iMac（主力机）❌
- 建议：Mac Mini 或 VPS（专用隔离机器）

### 3.3 具体用法示例
```
发一条消息 → 自动变成持续运行的 Agent

示例："Let's make sure our website always has the latest 
reasons why we're better than competitors."

→ 自动执行：Web搜索 + GitHub + 公开API
→ 自动提交 PR 更新页面
→ 每周自动刷新（基于新功能/市场动态）
```

**对我们内容团队的直接参考**：我们的"每日资讯→文案→配图"自动化链路，完全可以借鉴这个模式。
