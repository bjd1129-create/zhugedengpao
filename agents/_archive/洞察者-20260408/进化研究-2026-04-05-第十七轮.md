## 一、本轮新增发现

### 1. ContextEngine 智能剪裁机制（OpenClaw 2026.3核心新能力）

**来源**: eastondev.com OpenClaw 2026.3 Advanced Practice

**机制原理**:
旧版本context管理是"粗放型"——喂多少吞多少，模型能否处理取决于上下文长度是否超限。新版本ContextEngine增加"智能剪裁"机制，自动识别当前任务真正需要的信息，其余暂时搁置。

**实测效果**: 50轮复杂对话任务，context使用量降低约30%。

**对我们团队的意义**: 小花团队目前9个子agent，context管理压力会随对话复杂度增长。ContextEngine是内置优化，无需额外配置，应确认当前版本是否已包含。

### 2. 三层插件架构 + Bundle/Provider/Plugin 分离

**来源**: eastondev.com

**架构演进**:
- 旧版: 单一 Skills 结构，迁移成本高
- 新版: Bundle + Provider + Plugin 三层
  - Bundle: 技能打包单位
  - Provider: 核心逻辑（可跨agent复用）
  - Plugin: 执行层（平台相关配置）

**核心价值**: Codex到Claude的迁移，过去需要改一堆配置，现在只需换Provider，核心逻辑不动。

**我们现状**: 团队54个skill文件，应评估是否需要按新架构重组，降低维护成本。

---

### 3. 多模型路由最佳实践（Tiered Model Selection）

**来源**: mindstudio.ai 200+小时经验

**三层任务分级**:

| 任务类型 | 推荐模型 | 代表场景 |
|----------|----------|----------|
| 简单决策 | GPT-4o Mini / Claude Haiku / Gemini Flash | 分类、路由、是/否判断 |
| 中等推理 | Mid-tier models | 结构化提取、摘要、中等推理 |
| 复杂推理 | Frontier models (Claude Opus/GPT-4) | 代码生成、复杂写作、多步逻辑 |

**关键洞察**: 在10字路由决策上，Haiku和Opus的延迟差异可忽略，但成本差异巨大。习惯用最强模型做所有事是最大的预算杀手。

**我们现状**: 小花团队使用MiniMax M2.7为主，应建立明确的模型选择策略，避免小事用大模型。

### 4. Agent架构设计原则

**来源**: mindstudio.ai

**单职责原则**: 研究agent只研究，格式化agent只格式化，路由agent只路由。不要给单个agent超过3个条件分支。

**并行优先**: 三个独立研究任务并发执行，45秒串行 → 20秒并行，成本不变，时间减半。

**架构先行**: 在打开builder之前，先在纸上画agent graph，标注输入/输出/决策点/失败级联风险。

**我们现状**: 9个agent已有分工，应对照自检：是否有agent职责过于集中？是否有串行任务可以并行？

### 5. Self-Improving + AutoSkill 协同模式详解

**来源**: cloud.tencent.com 腾讯云开发者社区

**Self-Improving Agent 三文档机制**:
- `ERRORS.md`: 命令失败、工具出错、推理偏差
- `LEARNINGS.md`: 用户纠正、更优方案、工具更好用法
- `FEEDBACK_REQUESTS.md`: 用户改进建议

**触发机制**: 命令执行失败、用户纠正回答、发现更优方案 → 自动记录

**AutoSkill 技能生成流程**:
1. **摄取**: 收集对话、行为轨迹、用户反馈等原始信号
2. **提炼**: 判断是否存在可复用模式（一次性请求不生成技能）
3. **维护**: 新技能入库、已有技能更新合并、无价值模式过滤

**版本管理**: 技能像软件一样有版本号（文中案例: professional_text_rewrite 已迭代到 0.1.34）

**与我们现有self-evolve插件的关系**: self-evolve更偏RL式Q值更新，Self-Improving更偏文档式复盘，两者可互补。AutoSkill则解决"技能如何持续迭代"的问题。

### 6. OpenShell 沙箱后端（Docker替代）

**来源**: eastondev.com

**新选择**: OpenShell（支持mirror和remote模式）+ SSH沙箱，Mac用户无需装Docker，直接用本地shell，速度更快。

**适合场景**: Mac本地开发/测试OpenClaw，不想折腾Docker。

## 三、可应用到团队的策略

### 立即可做

1. **确认ContextEngine状态**
   ```
   openclaw doctor
   ```
