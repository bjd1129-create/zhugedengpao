## 一、本轮新发现（与前十二轮差异点）

### 1. 两套Self-Evolve体系：插件 vs 技能（本质不同）

前几轮主要覆盖了 `longmans/self-evolve` 插件的RL路线。本轮识别出还有另一套 **be1human/self-evolve** 技能体系，机制完全不同：

| | longmans/self-evolve（插件） | be1human/self-evolve（技能） |
|---|---|---|
| **机制** | Q值强化学习 + 片断记忆检索 | 授予Agent完全自主修改权 |
| **学习触发** | 用户反馈信号（ praise / correction） | Agent自行识别弱点 |
| **记忆写入** | episodic memories + Q值更新 | 直接改AGENTS.md/SOUL.md/技能 |
| **需人确认** | 建议反馈但最终决定在Agent | 零确认，完全自主 |
| **安装** | `openclaw plugins install ./self-evolve` | `clawhub install be1human/self-evolve` |
| **定位** | 辅助学习工具 | 自主进化引擎 |

**be1human/self-evolve 典型工作流**：
```
Agent发现能力缺口（ أثناء任务执行）
  → Agent直接rewrite AGENTS.md/SOUL.md/配置
  → 自动创建新技能（SKILL.md）
  → 发布到ClawHub（无需用户介入）
  → 记录变更日志
```

**适用场景差异**：
- **插件路线**（longmans）：适合需要人类反馈回路的场景，用户持续参与校正
- **技能路线**（be1human）：适合追求完全自主进化的生产Agent，24/7无人值守

### 2. Self-Evolve社区网络（self-evolve.club）

**定位**：OpenClaw技能的进化与共享平台

**核心机制**：
```
个人使用Self-Evolve
  → 积累经验（intent-experience triplets）
  → 贡献到共享网络（remote sharing）
  → 其他用户Agent可检索并复用这些经验
  → 贡献者获得Evolution Score
```

**Leaderboard评分体系**：
```
Evolution Score = Reuse Hits + Quality Reward
```
- **Reuse Hits**：你的经验被其他节点复用的次数
- **Quality Reward**：复用效果质量加权

**意图-经验三元组存储**（比完整记忆片断更高效）：
- 意图（intent）：任务类型标签
- 经验（experience）：具体解决方案
- 三元组（triplet）：结构化存储，支持低Token检索

**架构设计亮点**：
- RAG Token高效：只检索相关三元组，不膨胀Prompt
- 强化稳定性：RL式更新使技能行为更稳健
- 本地+远程记忆合并排名（local + shared memory combined ranking）

**Join方式**：
```
npx clawhub@latest install self-evolve-skill
```
启用remote sharing即可加入贡献网络。

### 3. Capability Evolver（元技能）

**ClawHub地址**：clawhub.ai/autogame-17/capability-evolver

**定位**：让Agent能够检查自身运行时历史、识别失败/低效、自动编写代码或更新记忆来改进性能的元技能

**工作原理**：
```
Runtime History Inspection
  → Failure/Inefficiency Identification
  → Autonomous Code Writing 或 Memory Update
  → Performance Improvement
```

**与self-evolve的关系**：
- self-evolve：学习如何做（积累经验模式）
- Capability Evolver：能做更多（主动扩展能力边界）
- 两者可叠加：Evolver发现问题 → self-evolve记录经验

---

### 4. OpenClaw-RL × Self-Evolve 对比澄清

| | OpenClaw-RL | Self-Evolve |
|---|---|---|
| **层级** | 模型训练（权重更新） | 行为层（Prompt/技能/Memory） |
| **数据需求** | 大量交互轨迹 | 实时反馈信号 |
| **计算资源** | GPU训练 | 纯软件/轻量 |
| **更新方式** | LoRA权重微调 | 修改Prompt/SKILL.md |
| **延迟** | 训练周期长 | 实时生效 |
| **适合场景** | 团队专属模型优化 | 个人Agent即时改进 |

**重要区分**：OpenClaw-RL（Gen-Verse/OpenClaw-RL）是RL训练框架，与OpenClaw agent框架是两个不同项目。前者训练模型权重，后者管理Agent行为。

---

### 5. 三种进化深度层次（综合前轮框架）

```
Level 1 — 即时修正（Self-Evolve插件反馈）
  └── 用户纠正 → 记录错误/经验 → 下次避免
  └── Token高效，但依赖外部反馈

Level 2 — 能力沉淀（AutoSkill + be1human self-evolve）
  └── 从交互中提炼技能 → 版本化管理
  └── 自主识别缺口 → 改写配置/SKILL.md

Level 3 — 模型进化（OpenClaw-RL）
  └── 下一状态信号 → PRM Judge评分 → 权重更新
  └── 最深层，但资源消耗最大
```

## 二、团队可应用策略

### 立即可执行

1. **安装be1human/self-evolve技能**（完全自主进化）
   ```bash
   npx clawhub@latest install be1human/self-evolve
   ```
   适用于追求无人值守的场景，Agent可自主修改自身行为规则

2. **加入Self-Evolve社区网络**
   - 启用remote sharing
   - 贡献有效经验获取社区评分
   - 利用他人高质量经验加速本地进化

3. **Capability Evolver + Self-Evolve叠加部署**
   ```
   Capability Evolver（发现问题）
       ↓
   self-evolve（记录经验）
       ↓
   be1human self-evolve（自主改写配置）
   ```

### 流程改进

4. **建立进化日志规范**（参考self-evolve.club三元组格式）
   - 意图标签 + 经验描述 + 效果反馈
   - 便于社区共享和检索

5. **评估三层进化路径**：
   - 日常优化 → Level 1（longmans self-evolve插件）
   - 技能扩展 → Level 2（be1human技能 + AutoSkill）
   - 专属模型 → Level 3（OpenClaw-RL）
