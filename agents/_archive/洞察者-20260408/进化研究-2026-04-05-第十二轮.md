## 一、本轮新发现

### 1. OpenClaw-RL 重大更新（2026-04-04）

**项目**：[Gen-Verse/OpenClaw-RL](https://github.com/Gen-Verse/OpenClaw-RL)  
**定位**：通过自然对话进行强化学习训练的个人化AI Agent框架

**最新突破（2026/4/4）**：
> 👨👦👦 We support optimizing a single model based on feedback from a group of people.

**三项学习范式**：

```
Binary RL (GRPO)
├── PRM Judge评分每个turn
├── 下一状态反馈作为自然信号
├── GRPO advantage estimation
└── PPO-style clipped surrogate loss

On-Policy Distillation (OPD)
├── 下一状态揭示hindsight
├── Judge模型提取文本提示
├── 构建增强教师上下文
└── Token级方向性监督（比标量奖励信息量更大）

Combination Method
├── Binary RL + OPD统一训练
├── 密集标量监督 + Token级方向信号
└── 更强更稳健的优化效果
```

**技术亮点**：
- 全异步4组件循环：模型服务、Rollout收集、PRM评判、策略训练
- 完全私有：所有数据在本地基础设施内
- 零人工标注：从对话自动组织训练轨迹
- 同时支持个性化OpenClaw优化 + 可扩展RL（terminal/GUI/SWE/tool-call）

**相关论文**：arXiv:2603.10165（2026-03-10），发布首日即登HuggingFace Daily Papers榜首

### 3. OpenClaw进阶调教7大实战技巧（ai23.cn）

**来源**：[ai23.cn](https://ai23.cn/tools/openclaw-advanced-tuning-guide/)

#### 技巧1：AGENTS.md — 给AI写一部行为宪法

**Session启动标准流程**（Every Session）：
```
1. Read SOUL.md — this is who you are
2. Read USER.md — this is who you're helping
3. Read memory/YYYY-MM-DD.md (today + yesterday) for recent context
4. If in MAIN SESSION: Also read MEMORY.md
```

**为什么按这个顺序**：
- SOUL.md/USER.md很小（通常<1KB），每次读成本可忽略
- 读昨天的日志确保凌晨1点时今天日志为空的情况下仍有内容
- MEMORY.md只限主session（私人不应在群聊泄露）

#### 技巧2：记忆写入规范（记结论不记过程）

**日志格式标准**：
```markdown
### [PROJECT:名称] 标题
- 结论: 一句话总结
- 文件变更: 涉及的文件
- 教训: 踩坑点（如有）
- 标签: #tag1 #tag2
```

**铁律**：
- 记结论不记过程（"部署成功，用了nginx反代" ✅ vs 三页操作日志 ❌）
- 标签便于memorySearch检索
- "Mental notes" don't survive session restarts. Files do.

#### 技巧3：安全边界（External vs Internal）

```markdown
## External vs Internal

**Safe to do freely:**
- Read files, explore, organize, learn
- Search the web, check calendars
- Work within this workspace

**Ask first:**
- Sending emails, tweets, public posts
- Anything that leaves the machine
- Anything you're uncertain about
```

**群聊原则**：
- AI能访问你的文件和记忆，但不等于可以在群聊里分享
- 在群聊里，你是参与者，不是用户的代言人

#### 技巧4：memoryFlush（上下文压缩保护）

**问题**：聊了一个小时后，AI突然"失忆"了  
**原因**：上下文窗口接近限制，自动压缩触发  
**解决**：memoryFlush在压缩前主动把重要信息写入memory/文件

### 4. 17大高手技巧（阿里云开发者社区）

**来源**：[developer.aliyun.com](https://developer.aliyun.com/article/1720262)

#### 三层记忆体系

```
L1 工作记忆（最快、最短）
├── 当前对话上下文
├── 用完即清理
└── 不适合长期知识

L2 会话记忆（跨对话、可持久）
├── 文件、状态、配置
├── 重启不丢失
└── memory/state.json、files/

L3 长期记忆（可检索、可归纳）
├── 结构化数据库、向量索引
├── 可搜索、可归纳、可复盘
└── 让AI越用越像"专属助理"
```

#### 遗忘机制（好的记忆必须有）

```
权重建议：
- 重要记忆：95%
- 普通记忆：60%
- 次要记忆：30%

定期整理（每周）：
- 删除重复、过时、错误记忆
- 合并相似内容
- 重新计算权重
```

#### 温度参数调优（按任务类型）

| 任务 | 温度 | 效果 |
|------|------|------|
| 代码生成 | 0.2–0.4 | 精确、稳定、少幻觉 |
| 逻辑推理 | 0.3–0.5 | 平衡、可靠、步骤清晰 |
| 创意写作 | 0.7–0.9 | 发散、丰富、表达灵活 |

#### systemd稳定守护

```ini
[Unit]
Description=OpenClaw Agent Service
After=network.target

[Service]
Type=simple
User=root
WorkingDirectory=/opt/openclaw
ExecStart=/usr/bin/docker start -a openclaw
Restart=always
RestartSec=10
CPUQuota=60%
MemoryMax=2G
```

#### 多Agent协作四大难题解决

| 问题 | 解决方案 |
|------|---------|
| 消息丢失 | 重试机制+队列确认 |
| 状态不同步 | 共享状态文件 |
| 任务分配不均 | 任务队列+负载均衡 |
| 执行冲突 | 锁机制、乐观并发 |

#### 分层人格管理

```
表层人格：语气、表情、聊天风格
核心人格：专业判断、逻辑、原则
→ 闲聊用表层，工作用核心
```
