## 📌 重大发现一：OpenClaw-RL — 强化学习框架重磅更新

**来源：** https://github.com/Gen-Verse/OpenClaw-RL

### 关键信息（2026年4月最新）

- **2026/4/4** — 支持基于多人反馈优化单一模型（Group Feedback → Single Model）
- **2026/3/25** — 获得 Tinker 算力支持
- **2026/3/20** — 支持使用自己的 OpenClaw（安装 rl-training-headers 扩展）
- **2026/3/13** — 同时支持本地 GPU 和云端（Tinker）部署，Hybrid RL / OPD / Binary RL 全部支持
- **2026/3/12** — 支持 **LoRA 训练**
- **2026/3/10** — 技术报告发布（arXiv:2603.10165），登顶 HuggingFace 每日 Paper 榜单 #1
- **2026/3/10** — 发布组合方法（Combination Method = Binary RL + OPD）

### 技术架构

OpenClaw-RL 是一个**全异步强化学习框架**，将日常对话转化为训练信号：

**四大组件异步循环：**
1. Agent Serving（模型服务）
2. Rollout Collection（经验收集）
3. PRM/Judge Evaluation（过程奖励模型评分）
4. Policy Training（策略训练）

**三种学习范式：**
- **Binary RL（GRPO）：** 每个 turn 基于 next-state 反馈打分，用 GRPO 优势估计 + PPO 裁剪损失
- **OPD（On-Policy Distillation）：** 用 judge 模型从下一状态提取文本 hint，丰富 token 级方向信号
- **Combination Method：** 结合 Binary RL 的密集标量监督 + OPD 的 token 级方向信号，效果最强

**Track 2 — 通用 Agent 场景：**
- Terminal RL（终端操作）
- GUI RL（图形界面）
- SWE RL（软件工程）
- Tool-call RL（工具调用）

### 🎯 对团队的启发

- **可直接应用：** 本地运行 RL 训练，私有数据不离机器
- **LoRA 支持：** 低成本微调，不需要全量训练
- **组合方法最优：** Binary RL + OPD 结合，效果比单一方法更强

## 📌 重大发现二：Self-Improving Agent 完整架构

**来源：** https://blog.exit.st/openclaw-self-improving-agent-complete-guide/

### 四阶段自我改进循环

```
检测(Detection) → 分析(Analysis) → 生成(Generation) → 集成(Integration)
```

**Phase 1 — 检测：**
- 任务失败信号
- 重复模式（同一类型请求反复出现）
- 用户反馈（显式纠正或不满表达）
- 效率指标（耗时过长、步数过多）

**Phase 2 — 分析：**
- 缺什么能力？
- 解决方案长什么样？
- ClawHub 上已有相关 skill 吗？
- 改现有 skill 还是新建？

**Phase 3 — 生成：**
- 写 SKILL.md（带指令）
- 创建支撑脚本
- 测试 skill
- 验证不破坏现有能力

**Phase 4 — 集成：**
- 放入 pending-skills/ 目录
- 通知人工审核
- 审批后移至活跃 skills 目录
- 高风险 skill（影响外部系统）必须人工签字

### 自我改进触发条件

> "如果你注意到过去7天内同一类型失败3次以上，分析模式，判断新 skill 是否有帮助。如果有，起草 skill 并记录待审。"

**好的触发条件：**
- 同类失败 ≥3 次
- 任务耗时超过预期 2 倍
- 用户明确请求新能力
- 发现可自动化的重复性手工流程

### 现实案例：客服 Agent

- Week 1：基线，解决70%工单
- Week 2：检测到15个关于 API rate limit 的工单，模式识别
- Week 3：生成"API Rate Limit Support" skill（包含限流层级、错误信息、处理步骤、升级标准）
- Week 4：集成，解决率升至82%

### 🎯 对团队的启发

- **可立即实施：** 性能日志（performance-log.json）+ heartbeat 审查
- **安全护栏：** pending-skills/ + 人工审核 gate，不过度自动化
- **复合 skill：** agent 不仅创建独立 skill，还能组合现有 skill 形成新能力

## 📌 发现三：Self-Evolving AI Team 架构

**来源：** https://dev.to/guanfu_lab/how-we-built-a-self-evolving-ai-team-with-openclaw-6gd

### 多 Agent 团队角色

- CEO Agent：任务协调、优先级管理
- CTO Agent：技术决策
- CFO Agent：财务和报告
- COO Agent：日常运营
- Specialists：内容、销售、分析

### 三大关键机制

1. **持久记忆（Persistent Memory）：** 每个 agent 维护自己的记忆文件（每日日志 + 长期知识 + 进化记录）
2. **Skill 系统：** 模块化、可共享的专业能力
3. **自我进化循环：** 每日反思 → 学到什么？应该改变什么？保留什么知识？

### 核心教训

- **记忆即一切（Memory is Everything）：** 无持久记忆 = 每次 reset；有 = 持续积累
- **结构防止混乱（Structure Prevents Chaos）：** 清晰的角色分工防止 agent 互相干扰
- **通信渠道至关重要：** 专用渠道处理不同类型通信（团队讨论 / 任务分配 / 个人更新）

## 📌 发现四：OpenClaw 2026 Best Practices

**来源：** https://openclaw-ai.com/en/blog/best-practices

### 性能优化

- **模型选择：** 简单任务用轻量模型，复杂推理用强模型
- **缓存：** 启用 TTL 缓存，减少 API 调用
- **上下文精简：** 系统 prompt 保持聚焦，避免冗余

### 成本管理

- 设置使用限制（messagesPerMinute / maxTokensPerDay / monthlyBudget）
- 预算警报（0.8 警告阈值，1.0 超出停止）
- 定期 `openclaw stats` 审查

### 安全实践

- 敏感信息绝不硬编码（用 .env 文件）
- 频道级访问控制（allowFrom 白名单）
- 定期 `openclaw update` 更新

### 可靠性

- 健康检查（interval 60s）
- 自动恢复（autoRestart + maxRetries）
- 定期配置导出备份（`openclaw config export`）
