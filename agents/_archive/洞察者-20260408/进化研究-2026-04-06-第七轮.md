## 一、核心发现

### 1. Self-Evolve Network（自我进化网络）—— 重磅更新

**官网**：`self-evolve.club`

这是目前最完整的 OpenClaw 自我进化框架，核心是一个"共享经验网络"：

**运作机制**：
- **经验 triplet（三角）存储**：intent（意图）+ experience（经验）+ embedding（向量）
- **Q 值学习**：每次反馈后更新 Q 值，类似强化学习的 reward 机制
- **本地 + 远程双层记忆**：本地记忆 + 社区共享记忆，检索时合并再排序
- **激励机制**：re-use hits + quality reward = Evolution Score，可上 leaderboard

**关键参数**（值得直接应用到小花）：
| 参数 | 默认值 | 说明 |
|------|--------|------|
| `runtime.minAbsReward` | 0.15 | 最低绝对 reward 才触发学习 |
| `runtime.minRewardConfidence` | 0.55 | 最低 reward 置信度 |
| `retrieval.tau` | 0.85 | 只在相似度 > 0.85 时注入记忆 |
| `memory.maxEntries` | 200 | 最多 200 条经验，超出按 Q 值淘汰 |

**学习模式**：
- `balanced`（默认）：优先 tool turns，无 tool turns 需要高 reward
- `tools_only`：只在调用工具时学习（最低 token 消耗）
- `all`：所有通过 reward 门的 turn 都学（最高 token 消耗）

**隐私设计**：本地双重 redact（sanitizeMemoryText + LLM summarizer），共享数据仅为 sanitized triplets。

### 2. OpenClaw-RL — 训练任意 Agent 仅通过对话

**论文**：`arXiv:2603.10165`（2026年3月10日发布）

核心洞察：**next-state signal（下一状态信号）是通用学习信号**

- 用户回复、工具输出、终端状态变化、GUI 交互——全部可以作为 RL 的 reward 信号
- **OPD（Hindsight-Guided On-Policy Distillation）**：从下一状态提取 directive signal（指导性信号），告诉模型"应该怎样不同"
- **PRM（Process Reward Model）**：对每个 action 给 scalar reward，支持异步在线学习

**对小花团队的意义**：OpenClaw-RL 的框架说明——对话本身就是训练数据。老庄的每一次纠正/补充都是 reward 信号。

### 3. Capability Evolver — GEP 标准（35K+ 下载）

**安装**：`claw install capability-evolver`

- 扫描 memory 文件、error logs、interaction history 找优化机会
- 运行 `/evolve` 命令或自动每 24 小时运行
- **GEP（Guided Evolution Protocol）**：标准化自我改进流程，防止 agent"失控"
- 目前 35K+ 下载，最受欢迎的自我进化类 skill

### 4. Self-Improving Agent Skill（3K+ 下载）

**架构四阶段**：
1. **Detection**：检测失败模式、重复模式、用户反馈、效率指标
2. **Analysis**：分析缺什么能力、现有 skill 是否可修改
3. **Generation**：用 Skill Creator 生成/修改 SKILL.md
4. **Integration**：测试 → 人类审核 gate → 上线

**关键实践**：新 skill 先放 `pending-skills/` 目录，人类批准后才移动到活跃目录。
