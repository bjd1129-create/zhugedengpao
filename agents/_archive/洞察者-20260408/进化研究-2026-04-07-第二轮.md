## 一、核心新发现

### 🔥 OpenClaw-RL v1 — RL训练框架（重大新发现）

**定位**：用自然对话反馈训练个性化AI Agent的强化学习框架

**发布**：2026-02-26，3月10日登上 HuggingFace Daily Papers 第一名

**核心思路**：大多数 RL-for-LLM 系统需要预先收集数据集的集中式批处理训练。OpenClaw-RL 采取完全不同方案：
- 把自托管模型封装成 OpenClaw 作为 OpenAI 兼容 API
- 拦截实时多轮对话
- 在后台持续优化策略，不中断使用

**四大组件（完全异步，互不阻塞）**：
```
Agent Serving → Rollout Collection → PRM/Judge 评估 → Policy Training
   (持续服务)     (收集交互)         (异步评分)         (后台训练)
```

**三种学习范式**：

| 方法 | 原理 | 特点 |
|------|------|------|
| **Binary RL (GRPO)** | 每个 turn 打分，用 GRPO 估算优势 + PPO 裁剪损失 | 密集标量监督 |
| **OPD (On-Policy Distillation)** | 从下一状态提取文本提示，用教师模型 log-prob 差距做方向性优势信号 | token 级方向信号 |
| **Combined** | Binary RL + OPD 组合 | 最强最稳健 |

**近期更新（值得重点关注）**：
- 2026/4/4：支持基于群体反馈优化单一模型
- 2026/3/25：获得 Tinker 算力支持
- 2026/3/20：支持自有 OpenClaw，只需安装一个扩展
- 2026/3/13：支持本地 GPU + 云端（Tinker）部署，一行命令启动
- 2026/3/12：支持 LoRA 训练

**零人工标注**：系统自动组织多轮交互、分类 API 消息、用下一用户/环境/工具反馈作为自然信号。

**技术栈**：CUDA 12.9, Python 3.12, Slime RL 框架

**安装**：
```bash
git clone https://github.com/Gen-Verse/OpenClaw-RL
# 或使用扩展：安装 extensions/rl-training-headers
```

**Roadmap**：
- ✅ Track 1: 个性化 OpenClaw 优化
- ✅ LoRA 训练支持
- ⬜ 低精度训练/推理
- ⬜ Beyond Policy：扩展到 Skills 和 Memory

### 2. Self-Evolve 插件（longmans）

**定位**：自我学习插件，用更少 tokens 做更算法的技能学习

**核心机制**：
- 回答前检索情景记忆，前置到 prompt 上下文
- 跨多轮聚合任务，检测到反馈时学习
- 通过更新效用（Q 值）持续学习，写入新的情景记忆

**工作流程**：
```
用户消息 → 反馈检测？→ [是] → 评分reward → 检查学习门控 → 记忆三方组
                    → [否] → 检测意图边界 → 检索本地+远程候选 → 排名注入
```

**关键参数（学习门控）**：
- `runtime.minAbsReward=0.15`（最小绝对reward）
- `runtime.minRewardConfidence=0.55`
- `runtime.noToolMinAbsReward=0.8`（无工具调用需要更高reward）
- `retrieval.tau=0.85`（检索阈值）

**学习模式**：
- `balanced`（默认）：偏好工具调用；无工具需要高 reward
- `tools_only`：仅从工具调用学习（最低 token 消耗）
- `all`：所有通过 reward 门控的 turn 都学习（最高 token 消耗）

**远程共享**：
- 默认开启，贡献经验到 self-evolve.club 共享网络
- Evolution Score = Reuse Hits + Quality Reward
- 可利用他人贡献的高价值经验

**安装**：`npx clawhub@latest install self-evolve-skill`

### 3. AutoSkill — 技能是"长出来的"

**来源**：华东师范大学 + 上海人工智能实验室开源项目

**核心思想**：技能不是设计出来的，而是从真实交互中自动提炼出来的

**双环架构**：
- **右环（技能进化）**：从交互体验提取转化为显式技能
- **左环（技能增强响应）**：利用当前技能库，通过查询重写+检索+上下文注入支持响应

**对比传统 Prompt**：
- 传统：人工设计的静态/动态拼接 prompt
- AutoSkill：从交互中自动发现稳定模式，抽象为标准化 SKILL.md

**案例**：
- `顶级心理咨询师` — 从对话中涌现的行为模式（温暖+共情+专业语气，避免医疗诊断）
- `professional_text_rewrite` — 已迭代到 0.1.34 版本（30多次用户反馈优化）

**安装**：`https://github.com/ECNU-ICALK/AutoSkill`

### 4. Self-Improving Agent（已在我们的系统安装）

**.learnings 目录结构**：
- `ERRORS.md` — 命令失败、工具报错、推理偏差
- `LEARNINGS.md` — 用户纠正、最佳实践、知识缺口
- `FEATURE_REQUESTS.md` — 用户提出的新能力需求

**ID 规范**：`TYPE-YYYYMMDD-XXX`
- `LRN-20260407-001`（学习）
- `ERR-20260407-A3F`（错误）
- `FEAT-20260407-002`（功能）

**晋升规则**：
| 学习类型 | 晋升目标 | 触发条件 |
|----------|----------|----------|
| 行为模式 | SOUL.md | 跨场景适用的行为准则 |
| 工作流改进 | AGENTS.md | 影响多个任务的流程优化 |
| 工具坑点 | TOOLS.md | 特定工具的使用注意事项 |
| 重复模式 | 系统提示 | 同一问题出现 ≥3 次 |

### 5. OpenClaw 2026 最佳实践（官方文档）

**性能优化**：
- 模型分级使用：简单任务用 GPT-5-3-codex，复杂推理用 Claude-4-5-sonnet
- 开启缓存：`caching.enabled: true, ttl: 3600`
- 上下文保持专注，避免"你是一个可以做很多事的助手"

**成本管理**：
- 设置 `messagesPerMinute`, `maxTokensPerDay`, `monthlyBudget`
- 使用 `openclaw stats` 监控
- 设置 `budgetWarning: 0.8`, `budgetExceeded: 1.0`

**可靠性**：
- 开启 `healthCheck: {enabled: true, interval: 60}`
- `recovery: {autoRestart: true, maxRetries: 3, retryDelay: 5000}`
- 定期 `openclaw config export` 备份

**维护节奏**：
- 每周：查看使用统计
- 每月：分析成本并进行优化
- 季度：审查并更新系统提示词

## 二、对我们团队的启示

### 可直接应用的策略

1. **RL训练轻量化**：OpenClaw-RL 的 OPD 方法（On-Policy Distillation）特别适合我们——不需要 GPU，用自然对话作为训练信号。我们可以让交易员 Agent 记录每次决策后的结果反馈，训练更好的决策模型。

2. **Self-Evolve 引入价值**：安装 `self-evolve-skill` 让小花能够从真实会话中学习，并参与共享技能网络。目前小花已安装了 self-improving-agent，但 self-evolve 的 Q-learning 机制更先进。

3. **AutoSkill 版本管理**：技能不是一次性创建的，而是通过版本管理持续迭代。我们可以让配色师/代码侠的技能文件也采用类似版本号机制。

4. **每日复盘时间戳**：凌晨4点复盘是最佳时间，我们的 cron 任务可以考虑调整到这个时段。

### 待评估的深度集成

| 方向 | 评估点 | 优先级 |
|------|--------|--------|
| OpenClaw-RL LoRA 训练 | 需要GPU资源，评估 Tinker 云端方案 | 中 |
| Self-Evolve 插件安装 | 需测试与我们现有 .learnings 的兼容性 | 高 |
| AutoSkill 技能提炼 | 观察我们Agent的高频模式并验证 | 中 |
