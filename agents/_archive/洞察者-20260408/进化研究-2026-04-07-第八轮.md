## 一、核心发现：OpenClaw Self-Evolution 技术详解

### 1.1 longmans/self-evolve 插件（最关键）

**地址**：https://github.com/longmans/self-evolve

这是目前最成熟的 OpenClaw 自我进化插件，核心机制：

**Q-Learning + Episodic Memory 架构**
- 在回答前检索情景记忆（episodic memories），注入到 prompt context
- 跨多轮聚合任务，检测到反馈时学习
- 通过更新 utility（Q values）实现渐进式学习，写入新的情景记忆

**反馈机制**
- 赞美 = 正向强化
- 指出错误 = 降权
- 明确反馈比"好的"这类模糊消息更有效

**学习门控（Learning Gates）**
```
runtime.observeTurns = 0
runtime.minAbsReward = 0.15
runtime.minRewardConfidence = 0.55
retrieval.tau = 0.85（仅在最佳相似度够高时才注入记忆）
```

**学习模式**
- `balanced`（默认）：偏好工具调用轮；无工具轮需要高奖励/高置信度
- `tools_only`：仅在调用工具时学习（最低 token 消耗）
- `all`：所有通过奖励门控的轮次都学习（最高 token 消耗）

**隐私设计**
- 第一层脱敏：sanitizeMemoryText 移除对话元数据、ID、敏感标签
- 第二层脱敏：经验总结器要求 LLM 输出可迁移策略，将敏感数据替换为 `[REDACTED_*]`
- 远程共享仅限脱敏后的三元组（intent / experience / embedding）

3. **通信渠道很重要**：用专门渠道处理不同类型通信
