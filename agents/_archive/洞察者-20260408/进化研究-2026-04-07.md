## 研究主题：OpenClaw 自我进化方式

## 一、核心发现

### 1. Self-Evolve 网络 (self-evolve.club)

**定位**：OpenClaw 技能进化与共享平台

**核心机制**：
- **意图-经验三元组** (intent-experience triplets)：从真实会话和反馈中提取可复用的经验
- **强化稳定性学习**：基于真实反馈的强化式更新，比临时记忆更可靠
- **RAG token 效率**：只检索相关内容，减少不必要的 prompt tokens
- **共享技能网络**：本地+远程记忆联合排名，直接贡献社区

**安装方式**：`npx clawhub@latest install self-evolve-skill`

**评分体系**：Evolution Score = Reuse Hits + Quality Reward

---

### 2. OpenClaw 2026.3 重大更新

#### 2.1 ContextEngine（上下文引擎）
- **智能裁剪**：自动识别当前任务所需信息，暂存其余
- **实测效果**：50轮复杂对话减少约 30% context 使用量

#### 2.2 三层插件架构（Bundle + Provider + Plugin）
- 从单层 Skills 升级为三层架构
- **模块化优势**：迁移技能只需更换 Provider，核心逻辑不变
- 兼容 Codex、Claude Code、Copilot CLI、Gemini CLI、Kimi CLI 等后端

#### 2.3 可插拔 Sandbox 后端
| 类型 | 适用场景 |
|------|----------|
| OpenShell Mirror | 本地开发测试，启动快，内存占用低（MacBook Air 测试比 Docker 节省一半内存） |
| OpenShell Remote | 团队协作，统一远程环境 |
| SSH Sandbox | 已有 SSH 基础设施的企业 |
| Docker | 高隔离要求的生产环境 |

#### 2.4 /btw Sidebar Q&A
- 长任务中临时提问，不打断主任务上下文
- 解决痛点：如何在长对话中"插话"而不污染 context

#### 2.5 Firecrawl 集成
- 增强网页抓取能力，解决 OpenClaw 之前的短板

#### 2.6 Human-in-the-Loop
- AI 主动暂停并询问"这样对吗？"再继续
- 从"全自动化"向"人机协同"转型

---

### 3. 最佳实践 (OpenClaw Best Practices 2026)

#### 性能优化
- **模型路由**：简单任务用轻量模型（如 gpt-5-3-codex），复杂推理用 Sonnet/Claude
- **缓存启用**：TTL=3600, maxSize=100MB
- **精简 system prompt**：聚焦职责，避免万能助手描述

#### 成本管理
- 设置 `messagesPerMinute`、`messagesPerHour`、`maxTokensPerDay`、`monthlyBudget`
- 启用预算警告（0.8 阈值）和超额中断（1.0 阈值）
- 定期 `openclaw stats` 追踪

#### 安全加固
- 敏感信息放 .env 文件，绝不硬编码
- 渠道级别访问控制（allowFrom 白名单）
- 定期 `openclaw update`

#### 可靠性
- 健康检查（interval: 60s）
- 自动恢复（autoRestart, maxRetries=3）
- 配置定期导出备份

## 二、可应用到团队的策略

### 立即可行

1. **安装 Self-Evolve Skill**
   ```bash
   npx clawhub@latest install self-evolve-skill
   ```
   让小花团队共享进化经验，积累 Evolution Score

2. **采用 OpenShell Mirror 模式**
   - 替代 Docker 作为日常开发测试的 sandbox
   - 节省资源，加快启动

3. **精简 SOUL.md / AGENTS.md**
   - 遵循"聚焦职责"原则，移除万能描述
   - 参考 ContextEngine 思路：每个文件只保留当前上下文真正需要的信息

4. **/btw 习惯养成**
   - 复杂任务中，用 /btw 问临时问题，避免 context 污染

### 中期规划

5. **升级到 2026.3 三层架构**
   - 评估将现有 Skills 迁移到 Bundle + Provider + Plugin 模式
   - 提高跨 Agent 的技能复用性

6. **Human-in-the-Loop 审批流**
   - 重大决策（删除文件、对外发言）加入人工确认节点
   - 符合小花"安全边界"中的"人工审核"要求

7. **成本追踪**
   - 为每个子 Agent 设置 API 使用限制
   - 启用预算警告

## 三、重大发现

### AutoResearchClaw
- 全自主、可持续进化的研究 Agent
- 支持多 ACP 后端（Claude Code、Codex、Copilot、 Gemini、Kimi）
- 支持多平台消息（Discord、Telegram、飞书、微信）
- **意义**：可作为团队"洞察者"Agent 的技术参考

### 进化网络的价值
- Self-Evolve 的核心思路：**从真实反馈中提炼经验 → 存储 → 检索 → 共享**
- 与小花团队的 `.learnings/` 机制高度契合
- 可考虑加入该网络，贡献经验并获取社区最佳实践
