## 一、核心发现（结论先行）

OpenClaw 的自我进化体系已形成 **三层架构**：

| 层级 | 机制 | 触发方式 | 适用场景 |
|------|------|----------|----------|
| L1 实时学习 | self-evolve 插件 | 自动检测反馈信号 + Q-Learning | 高频重复任务 |
| L2 事后进化 | skill-evolution + skill-improver | 任务结束后人工触发 | 技能链优化 |
| L3 长期记忆 | elite-longterm-memory + capability-evolver | 周期性分析日志 | 系统性能力提升 |

**关键洞察**：没有单一"银弹"，三层组合使用效果最佳。

## 二、各机制深度分析

### 1. self-evolve 插件（L1 实时学习）

**核心机制**：Q-Learning + RAG 向量检索

**工作流程**：
```
任务结束 → 检测用户反馈 → 脱敏处理 → 提取三元组 → 更新 Q 值 → 下次检索注入
```

**技术参数**：
- 检索门控 τ=0.85（相似度>85% 才注入，省 token）
- 学习门控 minAbsReward=0.15, minRewardConfidence=0.55
- 记忆上限 maxEntries=200（超出后淘汰低价值）

**三种模式**：
| 模式 | 学习范围 | Token 成本 | 推荐场景 |
|------|----------|-----------|----------|
| tools_only | 仅工具调用轮次 | 低 | 初期试跑 |
| balanced | 优先工具调用 | 中 | 日常使用（默认） |
| all | 所有通过奖励门的轮次 | 高 | 高价值任务密集期 |

**优势**：
- 自动检测正/负反馈（无需人工标记）
- 实时生效，下次相似任务立即受益
- 隐私保护：两轮脱敏（本地→LLM）后才上传共享

**局限**：
- 需要一定的任务量才能积累有效记忆
- Q 值体系对简单场景可能过度设计

---

### 2. skill-evolution + skill-improver（L2 事后进化）

**核心机制**：运行日志分析 → 生成补丁建议 → 人工审核应用

**工作流程**：
```
任务完成 → 读取 events.jsonl/state.json → 识别失败模式 → 生成改进建议 → 人工审核
```

**Hooks 配置**（~/.claude/settings.json）：
```json
{
  "hooks": {
    "PreToolUse": [{"matcher": "Bash|Write|Edit", "command": "pre-tool.sh"}],
    "PostToolUse": [{"matcher": "Bash", "command": "post-bash.sh"}]
  }
}
```

**输出产物**：
- `evolution-candidates.md`：待改进点列表
- `improvements.md`：具体改进建议（可直接转为变更任务）

**优势**：
- 基于真实运行数据，非主观猜测
- 保留人工审核，安全可控
- 可批量处理多个技能

**局限**：
- 需要手动触发（非自动）
- 依赖完整的日志记录

---

### 3. capability-evolver-pro（L3 系统健康）

**核心机制**：确定性日志分析（无 LLM）→ 健康评分 → 改进路线图

**核心功能**：
- `analyze`：错误模式识别 + 健康评分
- `evolve`：生成优先级改进建议
- `status`：系统健康状态快照

**优势**：
- 纯确定性逻辑，无幻觉风险
- 亚 100ms 处理速度，零 token 成本
- 支持批量跨 Agent 分析

**局限**：
- 需要导出日志到指定格式
- 更适合系统性诊断，非日常进化

---

### 4. elite-longterm-memory（长期记忆）

**核心机制**：向量嵌入 + 跨会话检索

**与 self-evolve 的区别**：
- self-evolve：任务级短期记忆（200 条上限）
- elite-longterm-memory：项目级长期记忆（无上限）

**适用场景**：
- 跨项目的通用模式
- 需要长期保留的决策依据
- 团队共享知识

## 三、对小花团队的适用性评估

### 立即可以采用的（本周内）

**1. 简化版反馈信号检测**

当前 AGENTS.md 的"3 次失败触发进化"效率低。建议：

```markdown
### 进化触发信号

**负信号**（自动记录到 .learnings/ERRORS.md）：
- 用户明确否定（"不对"、"错了"、"重做"）
- 工具连续失败≥2 次
- 任务超时/中断

**正信号**（自动记录到 .learnings/LEARNINGS.md）：
- 用户表扬（"好"、"不错"、"就这样"）
- 任务顺利完成
- 发现更优解法
```

**2. 三元组记忆格式试点**

在 `memory/triplets/` 下建立新格式：

```markdown
## [TRP-20260408-001]

**意图**：OpenClaw 自我进化研究
**经验**：web_search 不稳定时用 web_fetch 替代；last30days 脚本不存在时检查本地技能目录
**结论**：研究任务优先查本地技能文档，网络搜索作为补充
**标签**：#进化研究 #工具使用
```

**3. self-improving-agent 日志系统**

启用 `.learnings/` 目录：
- `LEARNINGS.md`：知识更新、最佳实践
- `ERRORS.md`：工具失败、命令错误
- `FEATURE_REQUESTS.md`：功能需求
