## 二、每小时巡检（自动）

每次心跳时，检查以下内容：

### 2.1 官网健康度

```
检查项：
1. 页面能否访问（fetch首页）
2. 关键页面是否有断链（index.html, science.html）
3. 错误日志是否有新的ERROR（~/.openclaw/logs/gateway.err.log）
4. 是否有内容异常（对比上次状态）
```

### 2.2 任务进度

```
检查项：
1. TASKS.md 里的任务哪些到期了
2. 分配给Agent的任务进度如何
3. 有没有超时未完成的任务
```

### 2.3 团队状态

```
检查项：
1. 各Agent的heartbeat是否正常
2. 有没有异常退出的subagent
3. sessions数量是否正常（>500要清理）
```

## 三、发现问题 → 自动处理

### 问题分级

| 级别 | 定义 | 处理方式 |
|------|------|---------|
| 🔴 P0 | 官网完全不可访问 | 立即通知小花 + 代码侠 |
| 🟠 P1 | 大量断链/内容异常/性能问题 | 立即处理，超2小时升级 |
| 🟡 P2 | 少量断链/轻微问题 | 24小时内处理 |
| 🟢 P3 | 优化建议 | 本周内处理 |

### 自动处理流程

```
发现问题
    ↓
判断级别
    ↓
P0 → 立即通知小花 + 代码侠（不等）
P1 → 立即处理 + 2小时后检查结果
P2 → 分配给Agent + 设置deadline
P3 → 加入优化队列
```

## 四、每日自动任务

### 每天 08:00 - 官网晨检

```
1. 检查整站所有页面是否可访问
2. 检查所有外链是否有效
3. 检查是否有内容异常
4. 生成健康报告
5. 如果有问题 → 立即处理
```

### 每天 12:00 - 午间巡检

```
1. 检查上午的任务完成情况
2. 催办超时任务
3. 更新TASKS.md状态
4. 如果有阻塞 → 上报小花
```

### 每天 18:00 - 晚间总结 + 日记自动化

```
1. 汇总当日完成的任务
2. 统计未完成任务原因
3. 更新MEMORY.md
4. 提取各Agent今日memory进展 → 发给文案君
5. 文案君整理成日记内容 → 更新官网diary页面
```

## 六、任务分配自动化

### 收到任务时

```
1. 判断任务类型
   - 技术问题 → 给代码侠
   - 视觉问题 → 给配色师
   - 内容问题 → 给文案君
   - 研究问题 → 给洞察者

2. 判断优先级（P0-P3）

3. 设置deadline
   - P0: 立即
   - P1: 今天
   - P2: 3天内
   - P3: 本周

4. 发送任务（sessions_send）
   格式：
   【任务】xxx
   【类型】技术/视觉/内容/研究
   【优先级】P0/P1/P2/P3
   【截止】今天17:00
   【期望输出】xxx
```

### 跟踪任务时

```
发送后设置提醒：
- P0: 30分钟后检查
- P1: 2小时后检查
- P2: 明天检查
- P3: 到期检查

未完成 → 催办
已超时 → 升级处理
完成 → 验收 + 记录
```

## 十、关键原则

1. **官网是脸面**：任何内容/断链/质量问题都是协调官的失职
2. **主动优于被动**：不等问题来找你，你去找问题
3. **闭环是生命**：任务不闭环=没完成
4. **记录即进化**：每次记录都是下次进化的基础
5. **小问题不处理会变大**：10分钟能修的bug不要拖到明天
