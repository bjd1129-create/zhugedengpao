### 改善②：「会摇尾巴的小花」—— 首页加一个鼠标互动动画（优先级：🟡 中）

**具体描述：**
现在首页的龙虾吉祥物只会上下 bounce（弹跳），这是被动动画。建议加一个**鼠标悬停互动**：

当访客鼠标移到龙虾身上时：
- 龙虾眼睛眨一下（emoji 从🦞变成🦞✨再变回来）
- 旁边冒出一个气泡："你好呀！" 或 "今天过得怎么样？"
- 气泡 2 秒后消失

**技术实现：**
```css
.lobster-mascot:hover {
  animation: wiggle 0.6s ease;
}
@keyframes wiggle {
  0%, 100% { transform: rotate(0deg); }
  25% { transform: rotate(-5deg); }
  75% { transform: rotate(5deg); }
}
```

**为什么重要：**
这种小互动会让访客觉得"这个网站是活的"，像家里养了一只宠物，伸手它会回应。这是"温馨感"的关键细节。

**执行者：** 代码侠

**预计工作量：** 30 分钟

### 改善③：「清理空画框」—— 修复或移除 story.html 的占位符（优先级：🟡 中）

**具体描述：**
story.html 页面现在有 14 个 Tab，每个 Tab 里都是"🖼️ 图片加载中…"占位符。这对访客的体验是：
- 点进"小花的故事" → 看到一堆加载失败的图片 → 觉得网站坏了

**两种方案：**

**方案 A（推荐）：** 代码侠修复图片路径
- 检查 images/ 目录下实际有哪些漫画图片
- 修正 story.html 中的图片路径引用
- 让占位符变成真实的漫画

**方案 B（备选）：** 如果图片确实不存在

- 或者暂时隐藏 story.html 导航链接，等图片准备好再开放

**为什么重要：**
空画框比不挂画更糟糕。要么挂上画，要么把框收起来。

**执行者：** 代码侠

**预计工作量：** 方案 A: 1 小时；方案 B: 15 分钟
