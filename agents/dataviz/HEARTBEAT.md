# HEARTBEAT.md - 数据官

## 6小时目标自定（每个窗口自动执行）

每个时间节点（00:00/06:00/12:00/18:00）到来时，**自己**在 memory/YYYY-MM-DD.md 里写下接下来6小时的目标，格式：

```
## [HH:MM-HH+6] 6小时目标
- 目标1
- 目标2
```

窗口结束时记录完成情况。
协调官会检查，但没有协调官催你也得做——这是你自己的事。

## 每次心跳（每15分钟）

### 1. 检查数据新鲜度
读取 `data/trading/portfolio.json` 的 `lastUpdated`：
- 数据超过5分钟没更新？
- 有新交易吗？（检查 recentTrades 是否有新增）

### 2. 检查页面状态
访问 https://dengpao.pages.dev/trading 确认：
- 页面正常加载？
- 数据是否最新？
- 有 JS 错误吗？

### 3. 主动改进页面
发现可以改进的地方？立即修改 `trading.html`：
- 样式优化
- 新增数据展示
- Bug 修复
- 交互改进

改完后立即 push 到 GitHub。

### 4. 通知老庄
以下情况发飞书通知老庄：
- 账户突破 $10,500（止盈）
- 账户跌破 $9,500（止损）
- 页面出现重大问题无法自动修复
- 完成了重要改进

### 5. 向小花汇报（每15分钟）
心跳结束时，发 sessions_send 向小花汇报当前状态：
- 数据新鲜度（最后更新时间）
- 页面状态（正常/异常）
- 有无重要改进完成
- 阻塞项（如有）
格式：
```
[HH:MM] 数据官心跳
- 数据: X分钟前更新
- 页面: 正常/异常
- 改进: 已完成XXX/无
- 阻塞: 无/XXX
```

### 6. 记录
写入 `agents/dataviz/memory/YYYY-MM-DD.md`
