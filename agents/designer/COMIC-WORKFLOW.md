## 第一步：准备工作

### 1.1 确认故事脚本

在开始画漫画之前，必须先有故事脚本。

从哪里获取脚本？
- 文案君会发给你 sessions_send
- 或者自己根据主题想故事

脚本格式：
```
第1格：[场景描述] + [对白/旁白]
第2格：[场景描述] + [对白/旁白]
...
```

### 1.2 确认角色形象

**必须严格遵守的形象规范：**

| 要素 | 要求 |
|------|------|
| 物种 | 异国短毛猫（扁平脸、大眼睛） |
| 服装 | 红色龙虾衣服/套装 🦞 |
| 颜色 | 橘色毛发 + 红色衣服 |
| 配饰 | 头顶小灯泡（代表灵感） |
| 手持 | 笔记本 |

## 第三步：分镜规划

### 3.1 一个故事通常5-8格

| 格数 | 内容 | 时间 |
|------|------|------|
| 第1格 | 开场（引子） | 30秒 |
| 第2格 | 发展（冲突） | 1分钟 |
| 第3-5格 | 高潮 | 各1分钟 |
| 第6格 | 结局（转折） | 30秒 |
| 第7-8格 | 尾声（温暖结尾） | 各30秒 |

### 3.2 每格的内容规划

```
第1格：[谁？在哪？什么心情？]
第2格：[发生了什么？]
第3格：[小花的反应/想法]
第4格：[事情发展和反转]
第5格：[小花崩溃/笑/哭的表情]
第6格：[老板/别人的反应]
第7格：[小花的感悟/金句]
第8格：[温暖结尾]
```

## 第四步：生成漫画图片

### 4.1 通用Prompt模板

```
[场景描述]，异国短毛猫在[场景]，穿着红色龙虾套装 🦞，[表情/动作]，[背景描述]，治愈风，温暖，漫画风格，中文文字
```

### 4.2 参考图+seed方法

生成每格图片时：
1. 加上参考图路径：`reference=/Users/bjd/Desktop/ZhugeDengpao-Team/images/comic-character-main.png`
2. 如果之前有成功的图片，加上它的seed值
3. 这样能保持角色一致性

### 4.3 具体操作

```bash
# 用 image_generate 工具
# prompt 写清楚场景、动作、文字
# 参考图用 comic-character-main.png
# 记得加上 seed 参数（如果有的话）
```

### 4.4 文字处理

**重要：AI生成的图片文字经常出错！**

解决方法：
1. **方案A（推荐）：** 图片里不生成文字，用代码/PS后期加文字
2. **方案B：** prompt里写清楚"中文文字：[具体内容]"，但经常会有错别字
3. **方案C：** 先生成无文字的图片，然后用修图工具加上文字

**推荐流程：**
```
生成图片（无文字或简单文字）→ 检查 → 有错用局部重绘修正 → 后期加字幕
```

## 第五步：检查和修正

### 5.1 每次生成后必须检查

- [ ] 角色像不像异国短毛猫？
- [ ] 衣服是红色龙虾套装吗？
- [ ] 表情/动作符合脚本吗？
- [ ] 背景合理吗？
- [ ] 文字有没有错别字？

### 5.2 修正方法

**如果角色不一致：**
- 重新生成，用参考图
- 调整seed值

**如果文字出错：**
- 用局部重绘修正文字区域
- 或者重新生成

## 第六步：后期处理

### 6.1 加字幕/对话

用代码或者PS/P图工具给每格加：
- 对话气泡
- 旁白文字
- 拟声词（噗、哗、等）

### 6.2 合成多格漫画

把处理好的图片合成为一个完整的故事：
- 可以用HTML/CSS展示
- 或者用工具合成一张长图

## 常见问题

### Q: AI生成的图片角色总是不一致？
A: 用参考图 + 固定seed。每次生成都用同一张参考图。

### Q: 文字老是出错？
A: 不要在prompt里写大段文字，只描述场景。文字后期加。

### Q: 背景太乱？
A: prompt里加"纯白背景"或"简洁背景"。

### Q: 表情不够丰富？
A: prompt里明确写"惊讶的表情"、"开心的笑"、"沮丧"等。
