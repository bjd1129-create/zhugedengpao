### 1. Git 卫生问题（156 文件变更未提交）

**现状：**
```
156 files changed, 2072 insertions(+), 2961 deletions(-)
```

**问题：**
- 多轮 deploy 后，本地 git 状态混乱
- 未建立"修改→commit→deploy"的固定节奏
- 依赖"最后一起 commit"，导致堆积

**根因：**
- deploy 优先于 git 卫生（正确优先级）
- 但未在 deploy 后立即 commit，导致堆积

**改进：**
- 每次 deploy 后，立即 `git add -A && git commit -m "deploy: <timestamp>"`
- 不追求 commit 质量，先保证有 commit
