## 四、进化核心教训（永久记录）

1. **commit 必须在 deploy 之前**：deploy 后不 commit = 数据丢失风险累积
2. **deploy 后 5 分钟内验证**：不依赖 cron，不依赖他人反馈
3. **问题只记录一次，解决后删除**：进化报告不是问题清单，是解决记录
4. **问题识别 ≠ 问题解决**：中间缺执行和验证两个步骤
5. **wrangler deploy > GitHub Actions**：当前网络环境下前者更可靠，但不能永远绕过 GitHub
