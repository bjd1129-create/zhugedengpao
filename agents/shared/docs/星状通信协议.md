## 二、通信方式

### 方式 1：共享目录（推荐）⭐

```
agents/shared/messages/
├── trader-urgent.json      # 给交易员的紧急消息
├── analyst-urgent.json     # 给分析师的紧急消息
├── engineer-urgent.json    # 给工程师的紧急消息
├── coordinator-urgent.json # 给协调官的紧急消息
├── xiaohua-urgent.json     # 给小花的紧急消息
└── game-engineer-urgent.json # 给游戏工程师的紧急消息
```

**优点**：
- ✅ 简单直接
- ✅ 有历史记录
- ✅ 不需要中间人
- ✅ 支持批量消息

**延迟**：
- 紧急消息：30 秒轮询
- 普通消息：5 分钟轮询

---

### 方式 2：sessions_send 直接通信

```python
# 交易员 → 分析师
sessions_send(
    sessionKey="agent:analyst:main",
    message="请提供 BTC 历史数据"
)

# 分析师 → 交易员
sessions_send(
    sessionKey="agent:trader:main",
    message="BTC 历史数据已发送"
)

# 工程师 → 游戏工程师
sessions_send(
    sessionKey="agent:game-engineer:main",
    message="游戏页面已部署"

```

**优点**：
- ✅ 即时响应（<2 秒）
- ✅ 有确认机制
- ✅ 不需要轮询

**缺点**：
- ⚠️ 需要 OpenClaw 支持
- ⚠️ 可能需要权限配置

## 六、使用场景

### 场景 1：交易员 → 分析师（数据请求）

```bash
# 交易员写入消息
cat > agents/shared/messages/analyst-request.json << 'MSG'
{
  "from": "交易员",
  "to": "数据分析师",
  "priority": "P1",
  "type": "request",
  "content": "请提供 BTC 和 ETH 的历史波动率"
}
MSG

# 分析师轮询发现
# 处理并回复
cat > agents/shared/messages/trader-response.json << 'MSG'
{
  "from": "数据分析师",
  "to": "交易员",
  "priority": "P1",
  "type": "response",
  "data": {"btc_vol": 0.65, "eth_vol": 0.78}
}
MSG
```

### 场景 2：工程师 → 游戏工程师（协作）

```bash
# 工程师写入消息
cat > agents/shared/messages/game-engineer-request.json << 'MSG'
{
  "from": "工程师",
  "to": "游戏工程师",
  "priority": "P2",
  "type": "request",
  "content": "游戏文件准备好了，何时可以部署？"
}
MSG
```

### 场景 3：协调官 → 全体（通知）

```bash
# 协调官写入多个文件
for agent in xiaohua trader engineer analyst game-engineer; do
  cat > agents/shared/messages/${agent}-notice.json << 'MSG'
  {
    "from": "协调官",
    "to": "全体",
    "priority": "P2",
    "type": "notice",
    "content": "每日进度汇报时间：22:00"
  }
  MSG
done
```

## 七、最佳实践

### 发送消息

1. **选择优先级**
   - P0：紧急（市场暴跌、止损）
   - P1：重要（需要回复）
   - P2：普通（通知、数据）

2. **写入共享目录**
   ```bash
   cat > agents/shared/messages/目标 agent.json << 'MSG'
   {"from":"我","to":"目标 agent","content":"..."}
   MSG
   ```

3. **等待回复**（如需要）

### 接收消息

1. **轮询检查**（自动）
   - 紧急消息：30 秒
   - 普通消息：5 分钟

2. **处理消息**

3. **回复**（如需要）

4. **删除已处理消息**
