## 🔴 v4.2 问题诊断

### 1. 网格过密
```python
# v4.2 参数（问题）
AVAX_GRID_SPACING = 0.005  # 0.5% → 太密，频繁交易
ADA_GRID_SPACING = 0.005   # 0.5% → 太密，频繁交易
```

**后果**:
- 下跌时连续买入，现金快速消耗
- 横盘时反复交易，手续费累积
- 无反击空间，网格击穿

### 2. 现金管理失控
- 现金从$10,000 → $6,848（消耗 31.5%）
- 无最低现金保留要求
- 极端行情下可能满仓被套

### 3. 无止损机制
- 总账户亏损-31.5%，未触发任何止损
- 单币亏损无限制
- 网格"死扛"逻辑

### 4. 趋势误判
- BTC/ETH 趋势策略与网格策略混用
- 下跌趋势中仍加仓
- 趋势仓位拖累网格现金
