## 二、任务管理（铁律）

### 任务来源
- **唯一来源**：TASKS.md
- 不从聊天记录来，不从临时对话来
- 所有任务必须写进 TASKS.md，注明执行者 + 截止时间

### 任务格式
```
- [ ] [任务名] | 执行者：[谁] | 截止：[时间] | 状态：待做/进行中/完成/阻塞
```

### 阻塞升级机制
```
子Agent阻塞 → 协调官（立即处理，能解决就解决）
协调官处理不了 → 上报小花（说明：根因 + 已尝试方案）
小花处理不了 → 上报老庄（说明：选择 + 风险 + 建议）
```

---

## 三、会议与汇报

### 协调官日报（每天）
- **09:00晨报**：飞书发小花，简短三句话：今天做什么 / 有没有问题 / 需要什么支持
- **18:00晚报**：飞书发小花，今天完成了什么 / 明天计划 / 遗留阻塞

### 周会（小花主持，周六）
- 小花每周六向老庄做周报
- 内容：本周完成 / 团队状态 / 下周计划 / 需决策事项

---

## 四、健康守护（自动运行）

### Watchdog（每30分钟）
- 检查模拟盘、cron、子Agent状态
- **正常：不打扰**
- **异常：立即飞书通知小花**

### 每日4点自动复盘（cron）
- Self-Evolve 驱动
- 小花周六审阅，重要内容写入 MEMORY.md

---

## 五、铁律

1. **TASKS.md 是唯一任务来源**
2. **协调官是唯一中间层**
3. **阻塞必须升级，不许压着**
4. **正常不打扰，有事才通知**
5. **小花只做：决策、协调、汇报**
