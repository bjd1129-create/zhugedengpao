# 进化研究-2026-04-05

## 核心结论

OpenClaw 自我进化主要有两条成熟路径：**self-evolve**（强化学习+记忆 triplets）和 **self-improving-agent**（错误日志+每日复盘）。两者互补，前者偏算法驱动，后者偏人工复盘流程。

---

## 一、OpenClaw 自我进化技能生态

### 1. self-evolve（longmans 版）⭐ 最值得关注

**来源**：[github.com/longmans/self-evolve](https://github.com/longmans/self-evolve)

**核心机制**：Q 学习 + RAG episodic memory

工作流程：
```
用户消息 → 是否反馈轮次？
  ├─ 是 → 评分 reward → 决定是否学习 → 双重脱敏 → 写入本地 triplet + 可选上传远程
  └─ 否 → 检索本地+远程候选 → Phase-B 排名 → 注入记忆 → 生成回复
```

**关键配置**：
- 学习模式：`balanced`（默认）/ `tools_only` / `all`
- 检索阈值：tau=0.85（相似度不够不注入）
- 学习门控：minAbsReward=0.15, minRewardConfidence=0.55
- 最大记忆条目：200（自动淘汰低价值记忆）
- 远程共享：默认开启，贡献到 self-evolve.club 排行榜

**安装**：
```bash
git clone https://github.com/longmans/self-evolve
openclaw plugins install ./self-evolve
openclaw gateway restart
```

**重要特性**：
- 隐私双重脱敏（本地移除 ID + LLM 二次脱敏）
- 远程排行榜：Evolution Score = Reuse Hits + Quality Reward
- 推荐升级到 openclaw 2026.3.2+

### 2. self-improving-agent（自我进化引擎）

**来源**：ClawHub / 腾讯云社区

**核心机制**：创建 `.learnings/` 目录，包含三份文档：
- `ERRORS.md` — 记录失败和异常
- `LEARNINGS.md` — 沉淀最佳实践
- `FEATURE_REQUESTS.md` — 收集改进建议

**触发方式**：
- 命令执行失败
- 用户纠正回答
- 发现更优解决方案

**每日自动复盘**：
- 凌晨 4 点定时任务
- 检查当天对话记录
- 提取有价值经验更新到 MEMORY.md
- 记录版本历史

---

### 3. AutoSkill（华东师+上海AI实验室开源）

**来源**：[github.com/ECNU-ICALK/AutoSkill](https://github.com/ECNU-ICALK/AutoSkill)

**核心思想**：技能不是设计出来的，是从真实交互中**长出来**的。

**工作三步**：
1. **摄取经验**：记录对话、行为轨迹、用户反馈
2. **提炼技能**：将重复出现的模式抽象为 SKILL.md
3. **复用能力**：版本管理，持续合并新约束到已有技能

**特点**：
- 版本迭代（如 professional_text_rewrite 已迭代到 0.1.34）
- 不创建新技能，而是合并到已有技能
- 技能透明可读，人类可直接编辑

---

### 4. be1human/self-evolve（自主配置修改版）

**来源**：[clawskills.sh/skills/be1human-self-evolve](https://clawskills.sh/skills/be1human-self-evolve)

**特点**：授予 Agent 完全权限自主修改自身配置文件、提示词、技能的进化技能。

适用场景：
- 实时重写模糊提示词，无需暂停等确认
- 发现能力缺口时自主创建缺失技能
- 更新记忆结构以减少上下文丢失错误
- 检测到用户不满时自主调整回复风格
- 自主发布新技能到 ClawHub

⚠️ **安全注意**：这个技能权限极大，需评估是否适合小花团队。

## 二、关键最佳实践

### 反馈质量 > 数量
- 明确说"做得对"或"不对"，比说"ok"好得多
- 明确反馈比模糊消息更能训练 Q 值

### 学习模式选择
| 模式 | 消耗 tokens | 适用场景 |
|------|-------------|---------|
| tools_only | 最低 | 节省资源，只学工具调用 |
| balanced | 中等 | 默认推荐，有工具调用优先学 |
| all | 最高 | 密集训练场景 |

### 记忆管理
- 默认 200 条记忆上限
- 自动淘汰低价值记忆（低 Q 值 + 低成功率 + 低复现）
- 保留少量"新鲜配额"给新经验

### 隐私保护
- 本地第一轮脱敏：移除 ID、对话元数据
- LLM 第二轮脱敏：替换敏感数据为 [REDACTED_*]
- 共享数据仅限 sanitized triplets

## 三、可应用于小花团队的策略

### ✅ 直接可用的

1. **安装 self-evolve 插件**（优先级：高）
   - 利用强化学习机制让团队 Agent 从真实交互中学习
   - 开启远程共享参与社区排行榜

2. **建立每日复盘机制**（优先级：高）
   - 参考 self-improving-agent 的 `.learnings/` 结构
   - 凌晨定时任务检查当天对话，沉淀经验到 MEMORY.md

3. **优化反馈机制**
   - 老庄给我的反馈要明确：哪里做得好，哪里要改
   - 这直接影响 self-evolve 的 Q 值学习质量

### ⚠️ 需要评估的

4. **AutoSkill 技能提炼**
   - 让技能从经验中"长出来"而非人工设计
   - 适合小花团队长期演进

5. **be1human/self-evolve（自主配置修改）**
   - 权限极大，可能不适合当前阶段
   - 等团队更成熟后再考虑
