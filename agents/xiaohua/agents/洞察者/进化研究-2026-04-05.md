## OpenClaw 进化方式研究

### 一、OpenClaw-RL：Princeton 的强化学习框架

**来源**：arXiv:2603.10165，GitHub: Gen-Verse/OpenClaw-RL

**核心思想**：把日常对话转化为训练信号，用强化学习让 AI Agent 持续自我改进。

**两大场景**：
1. **个人 Agent**：从 OpenClaw 对话中提取信号微调
2. **通用 Agent**：支持终端、GUI、SWE、工具调用等多环境并行训练

**技术亮点**：
- 全异步框架，不依赖离线批处理数据
- 从真实交互中实时学习
- 解决了"集中式批处理训练"无法覆盖动态环境的问题

**对小花团队的启示**：我们没有用到 RL 训练，但可以借鉴"从真实交互中提取反馈信号"这个思路——让小花记录每次执行结果，作为自我分析的数据源。

### 二、自我进化 Agent 架构（Skill-based）

**来源**：blog.exit.st、openclawpulse.com

**四阶段循环**：

```
检测(Detection) → 分析(Analysis) → 生成(Generation) → 整合(Integration)
```

**Phase 1 检测**：
- 任务失败记录
- 重复失败模式（同类失败 3+ 次触发）
- 用户反馈
- 效率指标（耗时超过预期 2x）

**Phase 2 分析**：
- 判断缺少什么能力
- 现有 ClawHub 技能是否能解决
- 是否修改现有技能 vs 从零创建

**Phase 3 生成**：
- 写 SKILL.md
- 创建支撑脚本
- 测试验证

**Phase 4 整合**：
- 放入 pending-skills/ 目录
- 通知人类审核
- 通过后移到活跃技能目录

**关键原则**：Agent 提议改进，人类批准。人类审核是安全阀，不能省。

### 三、每日自我进化 Cron Job 方案

**来源**：openclawpulse.com - 每日研究助手实践

**做法**：
- 每天 8:30 AM（工作日）自动触发
- 扫描：Anthropic 工程博客、Simon Willison 博客、Hacker News、GitHub Trending
- 对比自身文件：AGENTS.md、TOOLS.md、LESSONS.md
- 输出：`experiments/improvements.json`
- 可选：Telegram 早报

**成本**：每次 £0.02–0.08（人民币不到 1 毛钱）

**小花团队的参考价值**：我们可以借鉴这个模式，但改为每周运行一次，而不是每天。

### 四、Power User 最佳实践（200+ 小时经验）

**来源**：mindstudio.ai

**最高价值发现**：

1. **先画 Agent 图，再动手建**：建前画架构图，标输入/输出/决策点/失败级联风险
2. **单一职责原则**：一个 Agent 只做一件事，违反这条就该拆
3. **Sub-agent 用于并行，而非只是分解**：独立任务并行执行，速度提升显著
4. **模型路由**：小任务用小模型（Haiku/Flash），复杂任务用前沿模型。按任务类型分配，而不是全局用最强模型。
5. **专用 Router Agent**：入口统一路由到合适模型层，一处改动全局生效
6. **输出缓存**：对重复查询做缓存，高频工作流可省 30-50% 成本
7. **Telegram 分类线程**：错误/完成/审批/信息四类，不混淆

### 五、v2026.3.28 最新功能

**重大更新**：
1. **Plugin Approval Hooks（重大安全功能）**：Agent 执行工具前可触发人工审批，解决了"醒来发现 Agent 自主操作了一堆东西"的问题
2. **xAI x_search 原生支持**

7. **ANI（Agent-Native IM）双向消息**：支持人-AI 协作消息

## 对小花团队的启示

### 可直接应用的

1. **建立失败日志机制**：小花已有 `performance-log.json`，但没有定期分析。要加入每周分析流程。

2. **增加"每周进化研究"Cron**：参考 openclawpulse 的方案，每周运行一次，扫描 OpenClaw 新动态、新技能、新版本。

3. **在 AGENTS.md 中增加"自我进化触发规则"**：当同类失败 3 次，触发技能创建流程。

4. **模型路由**：目前我们用 MiniMax-M2.7 全任务处理，可以考虑把简单任务路由到更小/更便宜的模型，节省成本。

### 短期不适用但值得关注的

- **OpenClaw-RL**：需要大规模并行训练环境，暂不适用，但关注其进展
- **NERVE 仪表盘**：实时可视化 Agent 执行状态的开源工具，关注 claw.hot 的最新集成
