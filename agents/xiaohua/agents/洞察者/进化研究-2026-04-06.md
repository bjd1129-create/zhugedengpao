## 📡 OpenClaw 自我进化全景（2026年4月）

### 一、OpenClaw-RL — 核心进化框架

**发布状态**：v1 已于 2026-02-26 发布（Gen-Verse 团队）

**核心理念**：把每一次交互的"下一个状态信号"都当作训练数据

> "Personal conversations, terminal executions, GUI interactions, SWE tasks, and tool-call traces are not separate training problems. They are all interactions that can be used to train the same policy in the same loop."

**两种信号同时学习**：
- **Evaluative signals（评估信号）**：判断动作好坏的标量奖励，用 PRM judge 提取
- **Directive signals（指令信号）**：告诉agent应该怎么做的方向性指导，用 OPD（Hindsight-Guided On-Policy Distillation）恢复

**关键架构**：全异步设计
- 模型实时服务请求
- PRM judge 评判进行中的交互
- Trainer 同时更新策略
- 三者零协调开销

**适用场景**：
- 个人Agent：用对话中的 re-query、correction、explicit feedback 来训练
- 通用Agent：terminal、GUI、SWE、tool-call 全支持

**安装**：`https://github.com/Gen-Verse/OpenClaw-RL`

### 二、Self-Evolve 生态

**官网**：self-evolve.club

**核心理念**：
- 从真实会话和反馈中学习
- 把经验总结成 intent-experience triplets
- 本地+远程记忆联合 ranking
- 贡献者可以获得 Quality Reward

**安装命令**：
```bash
npx clawhub@latest install self-evolve-skill
```

**三大特点**：
1. **Reinforcement Stability**：强化学习式更新，比记忆快照更稳定
2. **RAG Token Efficiency**：只检索相关经验，减少 token 消耗
3. **Shared Skill Network**：本地+远程记忆联合排名，直接改善全社区结果

**Leaderboard**：实时的贡献者排行，Evolution Score = Reuse Hits + Quality Reward

### 三、Capability Evolver（本地诊断）

**定位**：元技能（meta-skill），分析运行时日志检测错误模式

**特点**：
- 纯确定性逻辑（deterministic），无 LLM 调用
- 子 100ms 处理速度，零 token 成本
- 完全本地运行，隐私安全

**四大 action**：
- `analyze`：分析日志找错误模式 + 计算健康分
- `evolve`：生成改进建议（支持 balanced/harden/innovate/repair-only 策略）
- `status`：系统健康检查
- 批量分析：跨多 Agent 找共同问题

**适用场景**：
- 生产事故根因分析
- 部署前回归检测
- 多 Agent 舰队管理
- 持续改进流水线

### 四、关键认知更新

**OpenClaw 进化三条路**：

| 路线 | 工具 | 适合场景 |
|------|------|----------|
| **RL训练** | OpenClaw-RL | 想真正更新模型权重，需要自托管模型 |
| **经验共享** | Self-Evolve | 积累经验 triplets，参与社区贡献 |
| **日志诊断** | Capability Evolver | 快速定位问题，生成改进方案 |

**重要洞察**：
- OpenClaw-RL 的 OPD（Hindsight-Guided On-Policy Distillation）不需要人工标注——从"下一个状态"自动推断正确做法
- Self-Evolve 的 triplets（intent-experience-outcome）比普通记忆更结构化，检索效率高
- Capability Evolver 的确定性分析避免了 LLM 的幻觉风险，结果可复现

## 🔥 本次新增发现（2026-04-06 补充）

### 重大更新：OpenClaw-RL 群体反馈训练（2026/4/4）

**突破**：现在支持**基于多人反馈优化单一模型**，这是从个人 RL 到群体 RL 的跨越。

**对小花团队的意义**：
- 交易团队有多人反馈源（老庄的修正、用户的交易结果、风控官的警报）
- 可以把这三类反馈汇入同一个模型，实现"群体智慧蒸馏"
- 不需要等老庄逐条纠正，系统可以自动从多人交互中学习

**技术前提**：
- 需要自托管模型
- 每人反馈 → embedding → 汇入 PRM judge → 训练更新
- 适合有明确反馈闭环的场景（交易执行效果）

### Self-Evolve 内部机制详解（补充）

**Q-Learning 架构**：
```
用户反馈 → 评分reward → Q值更新 → 写入episodic memory
                 ↓
         新请求到达 → 检索本地+远程triplets → 注入上下文 → 回复
```

**关键机制**：
- `before_prompt_build`：检索相关经验，注入到 prompt 前
- `agent_end`：捕获助手响应，标记为 waiting_feedback
- 检测到反馈时：评分 + Q更新 + 写入记忆 triplet
- 可选：同步到 self-evolve.club 远程网络

**Triplet 结构**（intent-experience-outcome）：
比普通记忆更结构化，检索时只看 intent 是否匹配，不扫描全文。

**最佳实践**（来自官方 README）：
- 正面反馈要**明确清晰**（"这个处理得好" > "ok"）
- 错误反馈要**指出具体问题**（帮助下一次 down-rank 坏策略）
- Explicit feedback > vague messages
