## 一、核心发现（TL;DR）

OpenClaw 的自我进化体系已经相当成熟，核心是**三层记忆架构**：

1. **短期记忆** → `.learnings/` 三文档体系（LEARNINGS.md / ERRORS.md / FEATURE_REQUESTS.md）
2. **中期记忆** → `memory/YYYY-MM-DD.md` 每日日志
3. **长期记忆** → `MEMORY.md` / `SOUL.md` / `AGENTS.md` / `TOOLS.md`

**关键洞察**：我们已经在用这套体系，但可以更系统化。

### 2.2 进化流程（标准）

```
触发 → 记录 → 晋升 → 固化
  ↓      ↓      ↓      ↓
事件   .learnings  核心文件  技能/习惯
```

**触发场景**（6 类）：
1. 命令/操作失败 → ERRORS.md
2. 用户纠正你 → LEARNINGS.md (category: correction)
3. 用户想要缺失功能 → FEATURE_REQUESTS.md
4. API/外部工具失败 → ERRORS.md
5. 知识过时/错误 → LEARNINGS.md (category: knowledge_gap)
6. 发现更优方案 → LEARNINGS.md (category: best_practice)

### 2.3 晋升规则（关键！）

| 学习类型 | 晋升目标 | 触发条件 |
|----------|----------|----------|
| 行为模式 | `SOUL.md` | 跨场景适用的行为准则 |
| 工作流改进 | `AGENTS.md` | 影响多个任务的流程优化 |
| 工具坑点 | `TOOLS.md` | 特定工具的使用注意事项 |
| 重复模式 | 系统提示 | 同一问题出现 ≥3 次 |

## 四、最佳实践（可立即应用）

### 4.1 日志格式规范

**Learning 条目**：
```markdown
## [LRN-YYYYMMDD-XXX] category

**Logged**: ISO-8601 timestamp
**Priority**: low | medium | high | critical
**Status**: pending | resolved | promoted | promoted_to_skill
**Area**: frontend | backend | infra | tests | docs | config

### Summary
一句话描述学到了什么

### Details
完整上下文：发生了什么、什么是错的、什么是对的

### Suggested Action
具体要做的改进

### Metadata
- Source: conversation | error | user_feedback
- Related Files: path/to/file.ext
- Tags: tag1, tag2
- See Also: LRN-20250110-001 (如有相关)
- Pattern-Key: simplify.dead_code (可选，用于追踪重复模式)
- Recurrence-Count: 1 (可选)
```

### 4.2 ID 规范

格式：`TYPE-YYYYMMDD-XXX`
- `LRN-20260408-001`（学习）
- `ERR-20260408-A3F`（错误）
- `FEAT-20260408-002`（功能）

### 4.3 检测触发器

**自动记录场景**：
- 用户说"不，应该是..." → correction
- 用户说"能加上...吗" → feature request
- 命令返回非零退出码 → error
- 发现更优解法 → best_practice

### 4.4 简化与固化（Simplify & Harden）

**重复模式追踪**：
1. 搜索：`grep -n "Pattern-Key: <key>" .learnings/LEARNINGS.md`
2. 如果存在 → 递增 Recurrence-Count，更新 Last-Seen
3. 如果不存在 → 创建新条目

**晋升到系统提示的条件**（三者必须同时满足）：
- Recurrence-Count >= 3
- 跨至少 2 个不同任务
- 在 30 天窗口内发生
