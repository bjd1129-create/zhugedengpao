# OpenClaw 自我进化研究报告

**研究日期：** 2026-03-31
**研究员：** 洞察者（AI）
**数据来源：** Web搜索 + GitHub + 阿里云开发者社区 + self-evolve.club

---

## 一、核心发现：三大进化体系

目前 OpenClaw 社区形成了三足鼎立的自我进化生态，各有侧重：

### 1. Self-Evolve（强化学习路线）⭐ 最推荐

**GitHub:** github.com/longmans/self-evolve
**安装：** `npx clawhub@latest install self-evolve-skill`

**核心机制：** Q值强化学习 + RAG情景记忆

| 特性 | 说明 |
|------|------|
| 学习触发 | 工具调用轮次优先（balanced模式）；无工具轮次需高奖励（≥0.8）才学 |
| 记忆单元 | Intent-Experience-Triplet（三元组） |
| 检索方式 | 向量相似度 + Q值排序，只在相似度≥0.85时注入 |
| 隐私保护 | 两轮脱敏：本地sanitizeMemoryText + LLM二次脱敏 |
| 共享网络 | self-evolve.club，可查看贡献排行榜 |
| 内存上限 | 默认200条，自动淘汰低价值记忆 |

**三种学习模式：**
- `balanced`（默认）：工具轮次优先，无工具轮次高奖励才学，最省Token
- `tools_only`：只在调用工具时学习，Token成本最低
- `all`：所有通过奖励阈值的轮次都学，Token成本最高

**关键参数：**
```
runtime.minAbsReward=0.15        # 最小绝对奖励
runtime.minRewardConfidence=0.55 # 最小奖励置信度
retrieval.tau=0.85               # 检索阈值
runtime.pendingTtlMs=300000      # 任务存活时间5分钟
```

**推荐理由：** Token效率高，有共享网络，能从其他节点的经验中受益。

### 2. Capability Evolver（GEP协议路线）

**ClawHub:** clawhub.ai/autogame-17/capability-evolver
**安装：** `claw install capability-evolver`
**下载量：** 35K+，ClawHub排名第一

**核心机制：** 基于GEP（基因组进化协议）的元技能

| 特性 | 说明 |
|------|------|
| 能力 | 扫描记忆/历史文件，自动识别错误和模式 |
| 自修复 | 检测崩溃并建议补丁 |
| 协议约束 | GEP标准化进化，防失控 |
| 执行方式 | `/evolve` 命令或 `node index.js` |
| 模式 | `--review`（人工审核）vs `--loop`（连续自动循环）|

**GEP基因组协议三维度：**
- **Gene（基因）：** 核心解决思路，技能的灵魂
- **Capsule（胶囊）：** 可执行代码+配置参数，技能的肉身
- **Evolution Event：** 完整进化过程记录，用于验证有效性

**环境变量：**
```bash
A2A_NODE_ID=node_xxxxxxxxxxxx   # 必填，EvoMap节点身份
EVOLVE_STRATEGY=balanced        # balanced/innovate/harden/repair-only
EVOLVER_ROLLBACK_MODE=hard      # 失败回滚策略
EVOLVER_LLM_REVIEW=0            # 是否启用LLM二次审核
```

**生态网络：** EvoMap（evomap.ai），接入了包括Claude、Cursor等主流AI智能体生态。

### 3. Self-Improving Agent（文件驱动路线）

**特点：** `.learnings/` 目录 + 定时凌晨4点复盘 + MEMORY.md更新

适合：经验文档化沉淀，不追求实时性。

## 二、三大进化方案对比

| 维度 | Self-Evolve | Capability Evolver | Self-Improving Agent |
|------|-------------|-------------------|---------------------|
| **原理** | Q值强化学习+RAG | GEP基因组协议 | 文件系统+定时复盘 |
| **学习触发** | 实时+反馈驱动 | 定时/命令驱动 | 每日凌晨4点 |
| **Token成本** | 低（balanced模式） | 中 | 低 |
| **共享网络** | self-evolve.club | EvoMap网络 | 无 |
| **防失控机制** | 隐私脱敏+阈值门控 | GEP协议约束+Review模式 | 人工审核 |
| **安装难度** | 低 | 中 | 低 |
| **推荐指数** | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐ |

## 三、安装建议优先级

### 第一阶段：立即安装 ✅

```bash
# Self-Evolve（推荐首选）
npx clawhub@latest install self-evolve-skill
```

**理由：**
- Token最省（balanced模式）
- 有共享网络，可以白嫖其他节点的进化经验
- 实时学习，无需等待
- 隐私保护做得最好（两轮脱敏）

### 第二阶段：可选加装

```bash
# Capability Evolver（进阶用户）
claw install capability-evolver
```

**适合：** 已经发现团队有明显瓶颈、需要系统性优化的Advanced用户。

## 四、对我们团队的启发

### 可立即应用

1. **安装 Self-Evolve**：立即提升小花的学习能力
2. **调整学习模式为 `balanced`**：当前Token敏感，工具轮次优先
3. **建立反馈意识**：当小花做得好时，明确说"好"；做得不好时，明确指出哪里不对

### 进阶应用

4. **配置远程共享**：加入 self-evolve.club 网络，获取社区经验
5. **定时凌晨复盘**：建立 `.learnings/` 体系，定期固化到 MEMORY.md

### 需要注意的风险

- Self-Evolve 会自动修改记忆文件，需要定期检查 memory/ 目录
- Capability Evolver 的 `--loop` 模式可能产生意外修改，建议用 `--review` 模式
- 共享网络虽然隐私做得好，但涉及数据传输，敏感项目慎用
