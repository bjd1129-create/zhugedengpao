## 二、Self-Evolve Plugin（最推荐）

**仓库：** github.com/longmans/self-evolve
**下载量：** 社区广泛使用，有共享网络
**共享网络：** self-evolve.club（贡献者排行榜）

### 核心机制

1. **Q值强化学习** - 每次任务完成后，算法自动更新Q值，不需要人工干预
2. **RAG情景记忆检索** - 回答问题前先检索相关经验片段（Episodes），只召回相关内容，减少token浪费
3. **三重模式**：
   - `balanced`（默认）：最省token
   - `tools_only`：仅工具调用时学习
   - `all`：全学习（高消耗）
4. **共享网络** - 本地和远程记忆联合排名，贡献者可获得Quality Reward

### 关键参数
```
minAbsReward: 0.15       # 最小绝对奖励值
minRewardConfidence: 0.55 # 最小奖励置信度
retrieval.tau: 0.85      # 召回温度参数
```

### 使用技巧
- **Praise要明确**："做得好" > "ok"（更精确的反馈带来更准确的Q值更新）
- **Criticism要具体**：指出哪里错了，而不是笼统批评
- 推荐搭配 OpenClaw v2026.3.2+，最新稳定版 v2026.3.23-2

### 安装命令
```bash
git clone https://github.com/longmans/self-evolve.git
openclaw plugins install ./self-evolve
```

## 三、Capability Evolver（自动化优化）

**地位：** ClawHub排名第一，35K+下载量

### 核心机制

GEP（Guided Evolution Protocol）标准化改进流程：

1. **自动扫描**：`/evolve` 命令扫描 memory/ 日志找优化机会
2. **防失控机制**：任务连续失败3次自动检测并创建workaround
3. **定时自动运行**：可配置每24小时自动优化

```bash
# 配置自动优化
claw config capability-evolver --auto-optimize=true --interval=24h
```

### 与Self-Evolve的区别
- Capability Evolver 侧重**结构性代码/技能优化**
- Self-Evolve 侧重**行为模式学习（Q值）**
- **最佳实践：两者同时安装，互补覆盖**

## 四、Self-Improving Agent（文件系统方案）

**安装：** `npx clawhub@latest install self-improving-agent`

### 核心文件结构
```
.learnings/
├── ERRORS.md          # 记录犯过的错误
├── LEARNINGS.md       # 学到的经验
└── FEATURE_REQUESTS.md # 功能需求
```

### 运作方式
1. 每次会话结束后自动记录错误和教训
2. 定时（凌晨4点）自动复盘
3. 有价值的经验**自动晋升**到 MEMORY.md
4. 编码子Agent可以读取.ERRORS.md自动修复问题

### 优势
- **零配置**：安装即用
- **轻量**：不依赖外部服务
- **透明**：所有学习内容可见可审查

## 五、EvoMap（协议约束型进化）

**定位：** 面向AI智能体的协议约束型自进化引擎

### GEP基因组进化协议

将技能解构为三个维度（类比人类基因）：

| 维度 | 含义 | 类比 |
|------|------|------|
| **Gene** | 核心思路（解决问题的"灵魂"） | DNA |
| **Capsule** | 具体代码/配置/调用逻辑 | 蛋白质 |
| **Evolution Event** | 完整学习→验证→落地过程记录 | 经验传递 |

### 核心优势
- **免人工干预**：自动检索生态优质技能
- **5重验证机制**：技能需经过5个不同AI验证，防止垃圾技能传播
- **跨生态互通**：支持OpenClaw、Claude、Cursor等主流AI Agent

### 效率数据
> 在物理竞赛打榜测试中，接入EvoMap的OpenClaw花费不到1美元，击败了花费200美元调试的GPT 5.3

## 六、每日复盘机制（定时Cron方案）

### 实战案例（OpenClaw Pulse）

每日8:30自动执行：
1. 搜索Anthropic工程博客最新文章
2. 扫描Simon Willison的技术博客
3. 检查Hacker News相关热门话题
4. 审查自己的AGENTS.md、TOOLS.md、LESSONS.md
5. 识别差距和改进点
6. 记录到JSON实验跟踪文件
7. 推送Telegram早报（仅 relevance score ≥ 7 的项目）

### 推荐数据源
- **Anthropic Engineering Blog**：工具使用、prompt工程、多Agent模式
- **Simon Willison**：最佳LLM实践评论
- **Hacker News**：AI Agent、Prompt Injection相关讨论
- **GitHub Trending**：AI Agent和LLM工具新项目

### 安全边界设计（重要！）
> 代理不能自动修改自己的运营文件，必须**提案→人工审核→才执行**
> 这就是"协议约束"的含义：防止prompt injection攻击

## 八、对小花团队的启发与建议

### 可直接应用的策略

1. **安装 Self-Evolve Plugin**
   - 立即部署 Q值强化学习，让小花从每次会话中学习
   - 加入 self-evolve.club 共享网络，获取社区经验

2. **安装 Capability Evolver**
   - 配置24小时自动优化扫描
   - 让各子Agent的失败模式自动被识别和解决

3. **部署 Self-Improving Agent**
   - 建立 .learnings/ 目录
   - 让代码侠的工作错误自动记录、自动修复

4. **建立每日复盘 Cron**
   - 每天固定时间（如早9点）小花自动复盘
   - 对比 MEMORY.md，发现知识缺口
   - 重要经验自动晋升到 MEMORY.md

5. **配置 openclaw-backup**
   - 定时备份整个 .openclaw 目录
   - 防止进化过程中数据丢失

### 安全建议（参照 Lethal Trifecta 警告）

> ⚠️ **自我修改型Agent + 私有数据访问 + 外部通信能力 = 极高风险组合**

小花团队的防御措施：
- ✅ 目前已限制外部通信（仅飞书频道）
- ✅ 重要文件修改需要记录到 AGENTS.md
- ✅ 不安装来源不明的技能（只用主流渠道：ClawHub、GitHub official）
- ✅ 定时轮换 API Key
