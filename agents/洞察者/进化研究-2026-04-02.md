## 🔬 一、OpenClaw 进化三件套（最新状态）

### 1. Self-Evolve Plugin（longmans/self-evolve）
**状态：最推荐部署**

核心机制：
- **Q值强化学习**：每轮对话后根据反馈更新Q值，算法驱动而非随意
- **RAG情景记忆检索**：回答前先检索episodic memories，低token开销
- **经验聚合**：跨多轮聚合任务，检测到反馈时学习
- **共享网络**（self-evolve.club）：节点间可共享经验，双重隐私脱敏

三种学习模式：
| 模式 | 触发条件 | Token消耗 |
|------|---------|----------|
| balanced（默认）| 所有交互 | 最省 |
| tools_only | 仅工具调用 | 中 |
| all | 全量学习 | 最高 |

关键参数：`minAbsReward=0.15`, `minRewardConfidence=0.55`, `retrieval.tau=0.85`

安装：`git clone ... && openclaw plugins install ./self-evolve`

### 2. Capability Evolver（ClawHub排名第一，35K+下载）
**状态：推荐安装**

- **GEP（Guided Evolution Protocol）**：标准化改进流程，防失控
- 运行 `/evolve` 扫描 memory/ 日志找优化机会
- 任务连续失败3次 → 自动检测并创建workaround
- 支持定时自动运行：`claw config capability-evolver --auto-optimize=true --interval=24h`

## ⚡ 三、Power User 最佳实践（14条精华）

### 架构设计
1. **画Agent图再动手**：不要直接建，先画输入/输出/决策点/失败级联
2. **单一职责原则**：Agent只能用一句话描述，超过就拆分
3. **用Sub-Agent做并行**：独立任务并行执行，45秒→20秒

### 模型路由（成本优化核心）
4. **按任务匹配模型**：
   - 分类/路由/简单判断 → 小模型（Haiku/Flash）
   - 提取/摘要/中等推理 → 中端模型
   - 复杂推理/代码/精细写作 → 前沿模型
5. **不要用习惯匹配模型**：同一条路由决策，Haiku和Opus延迟差不多，但成本差很多

### 性能与成本
6. **实施缓存**：`caching.enabled: true, ttl: 3600`
7. **设置用量限制**：`messagesPerMinute/maxTokensPerDay/monthlyBudget`
8. **设置预算警报**：`budgetWarning: 0.8, budgetExceeded: 1.0`

### 可靠性
9. **健康检查**：`healthCheck.enabled: true, interval: 60`
10. **错误恢复配置**：`autoRestart: true, maxRetries: 3, retryDelay: 5000`
11. **定期备份配置**：`openclaw config export`

### 安全（当前高优先级）
12. **环境变量管理**：API Key绝不硬编码，写入.env
13. **访问控制**：channel允许列表（如Telegram allowFrom）
14. **定期更新**：`openclaw update`

## 📌 六、研究结论

**OpenClaw进化生态已成熟**。三个工具形成互补：
- Self-Evolve → 学"经验"（实时）
