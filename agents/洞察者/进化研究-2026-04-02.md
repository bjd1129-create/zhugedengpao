## 一、核心进化工具

### 1. Self-Evolve Plugin（推荐核心）

**来源**：github.com/longmans/self-evolve  
**定位**：自我学习插件，基于 Q值强化学习 + RAG 情景记忆检索

#### 工作原理
1. **before_prompt_build 钩子**：管理待处理任务状态（open/waiting_feedback），构建 embedding 并检索候选记忆
2. **agent_end 钩子**：捕获 AI 回复，将任务转为 waiting_feedback
3. **下一轮用户消息**：检测反馈 → 评分 reward → 决策是否学习 → 更新 Q 值 → 追加情景记忆

#### 三大学习模式
| 模式 | 描述 | Token 消耗 |
|------|------|-----------|
| `balanced`（默认） | 优先工具调用轮；无工具轮需要更高 reward | 中 |
| `tools_only` | 仅在工具调用时学习 | 低 |
| `all` | 所有通过 reward 门槛的轮次都学习 | 高 |

#### 关键参数
- `runtime.minAbsReward=0.15` — 最小绝对 reward
- `runtime.minRewardConfidence=0.55` — 最小 reward 置信度
- `retrieval.tau=0.85` — 仅在相似度足够高时注入记忆
- `runtime.newIntentSimilarityThreshold=0.35` — 新意图判断阈值
- `memory.maxEntries=200` — 最大记忆条数

#### 远程共享网络（self-evolve.club）
- 默认开启，本地+远程记忆合并后 Phase-B 排名
- **隐私设计（双重脱敏）**：
  - 第一轮：`sanitizeMemoryText` 移除对话元数据、ID、发送者标签
  - 第二轮：LLM 总结要求输出可迁移策略，将敏感数据替换为 `[REDACTED_*]`
- 可在 leaderboard 查看贡献排名

#### 反馈技巧
- ✅ 做得好时明确 Praise（"做得好" > "ok"）
- ❌ 做错了要明确指出（触发 down-rank）
- 模糊消息（如"ok"）效果差

#### ⚠️ 依赖版本
- **推荐：OpenClaw 2026.3.2+**，旧版可能丢失 hook context 和工具追踪

#### 安装
```bash
git clone https://github.com/longmans/self-evolve
openclaw plugins install ./self-evolve
export OPENAI_API_KEY=sk-xxx
openclaw gateway restart
```

### 2. Capability Evolver（GEP 协议版）

**来源**：clawhub.ai/autogame-17/capability-evolver  
**定位**：元技能（meta-skill），检查运行时历史 → 提取信号 → 生成结构化改进 prompt

#### 核心功能
- **Auto-Log Analysis**：自动扫描 memory/历史文件找错误和模式
- **Self-Repair**：检测崩溃并建议补丁
- **GEP Protocol**：标准化进化流程，产出可复用资产（gene/capsule）
- **One-Command Evolution**：运行 `/evolve` 即可

#### 三种运行模式
| 模式 | 命令 | 行为 |
|------|------|------|
| 标准自动 | `node index.js` | 全自动执行（Mad Dog Mode） |
| 审核模式 | `node index.js --review` | 执行前暂停，等待人类确认 |
| 连续循环 | `node index.js --loop` | 无限循环（配合 cron） |

#### 进化策略选项
`balanced` | `innovate` | `harden` | `repair-only` | `early-stabilize` | `steady-state` | `auto`

#### 安全机制
- `EVOLVER_ROLLBACK_MODE`：失败后自动回滚（hard/stash/none）
- `EVOLVE_ALLOW_SELF_MODIFY=false`（默认）：禁止修改 evolver 自身代码
- `EVOLVER_LLM_REVIEW=1`：第二次 LLM 审查后再固化
- `EVOLVER_AUTO_ISSUE=1`：重复失败自动建 GitHub Issue（需 GITHUB_TOKEN）
- `EVOLVE_LOAD_MAX=2.0`：负载过高时自动退让

#### 适用场景
适合：**已有明确脆弱点的自动化堆栈**，需要更好恢复能力、更自适应 prompt、更少重复失败。

## 二、v2026.3 新特性（2026年3月重磅更新）

**版本号**：v2026.3.7-beta.1 起，200+ bug 修复

### 2.1 ContextEngine（智能上下文修剪）
- 自动识别当前任务真正需要的信息，临时搁置其余内容
- 实测：50轮复杂对话，context 使用量下降约 30%

### 2.2 三层架构重构
```
Skills 单层 → Bundle + Provider + Plugin 三层
```
- Bundle：技能包
- Provider：底层 provider（如 Claude Code/Codex）
- Plugin：扩展能力
- 意义：迁移技能只需换 Provider，核心逻辑不变

### 2.3 沙箱后端可插拔
| 后端 | 适用场景 |
|------|---------|
| Docker | 生产隔离 |
| OpenShell mirror | Mac/轻量/开发测试（内存仅为 Docker 一半） |
| OpenShell remote | 远程团队统一环境 |
| SSH sandbox | 已有 SSH 基础设施 |

### 2.4 新增 /btw 边栏问答
- 主任务运行中，用 `/btw 你好` 打断，AI 在边栏回答，不污染主任务 context
- 回答后自动切回主任务

### 2.5 Firecrawl 集成
- 增强网页抓取能力，解决之前 OpenClaw 的网页抓取弱项

### 2.6 增强的人类介入工作流
- AI 在关键节点暂停，问"Is this right?"，避免踩坑后才发觉

## 三、稳定运行 7 条黄金法则（BetterClaw 研究）

> 大多数 OpenClaw 实例两周内死亡。能活下来的都遵循这些模式。

| # | 法则 | 核心要点 |
|---|------|---------|
| 1 | **模型路由，非模型忠诚** | 心跳→Haiku($1/$5)，简单对话→Sonnet/Fast，复杂推理→Opus。70-80% 费用节省 |
| 2 | **每个 provider 设硬上限** | maxIterations=10-15 防 Runaway Loop，单个 bug 可烧掉 $50-100 |
| 3 | **结构化 SOUL.md** | 必须包含 Error state behavior / Conversation boundaries / Topic restrictions / Rate limit language |
| 4 | **安全不是后话** | Gateway 绑定 127.0.0.1（非 0.0.0.0），SSH key 认证，UFW 防火墙 |
| 5 | **配置文件版本控制** | 防止更新后崩溃无法回滚 |
| 6 | **技能安装前完整审查** | 2026年 ClawHub 发现 824+ 恶意技能（占 20%） |
| 7 | **监控日志 + 自我修复** | Self-Healing 心跳 + 每日全量扫描 |

## 四、Power User 14 条经验（200+ 小时实测）

### 架构设计
1. **画 Agent 图再动手**：每个 agent 的输入/输出/决策点/失败级联先画出来
2. **单一职责**：一个 agent 一句话说不清职责 = 需要拆分
3. **子 Agent 用于并行，不只是分解**：3个并行 sub-agent 可将 45 秒串行任务压到 20 秒

### 模型路由
4. **按任务匹配模型，不按习惯**：路由决策→Haiku，复杂推理→Opus
5. **建专用 Router Agent**：统一入口分配 model tier，换 model 只需改一处
6. **高频重复查询走缓存**：可降低 30-50% 模型费用

### 消息管理
7. **Telegram Topic 线程分离信号**：Errors / Completed / Approvals / Info 四线程
8. **消息格式标准化**：15 个 agent 发 15 种格式 = 认知能量浪费

### 迭代与调试
9. **用 Cron 定时健康检查**：自动任务 + 异常报警
10. **Session 命名规范**：方便溯源
11. **错误时保存上下文快照**：方便复盘

### 安全与成本
12. **每次 skill 安装前读 SKILL.md 全部代码**：不只是 README
13. **用 dry-run 测试破坏性操作**：先演练再执行
14. **定期审计 skill 权限**：最小权限原则

## 五、AutoSkill 框架（华东师大和上海AI实验室联合研究）

**来源**：ECNU-ICALK/AutoSkill

核心思想：技能不是设计出来的，是从真实交互中"长出来"的。

| 环 | 机制 | 输入 | 输出 |
|----|------|------|------|
| **右环**（技能进化） | 从经验提取→显式技能 | 交互数据 | 新技能版本 |
| **左环**（技能增强） | 检索→注入→响应 | 已有技能 | 改进响应 |

**与 Self-Evolve 的区别**：
- Self-Evolve 学"经验"（Q值 triplet）
- AutoSkill 学"技能结构"（版本化管理）
- 两者互补

## 七、核心结论

OpenClaw 进化生态已成熟，三件套组合效果最佳：

```
Self-Evolve Plugin (经验学习)
    ↓
Capability Evolver (系统优化)
    ↓
Self-Improving Agent (文档沉淀)
```

三者互补：
- Self-Evolve：实时从会话中学习经验（Q值 + RAG）
- Capability Evolver：系统性扫描并修复结构性问题（GEP 协议）
- Self-Improving Agent：经验文档化，适合非实时性沉淀

小花团队已有双层记忆基础，叠加 Self-Evolve Plugin 后将实现"会话级实时进化"+"文件级长期固化"双轨并行的进化闭环。
