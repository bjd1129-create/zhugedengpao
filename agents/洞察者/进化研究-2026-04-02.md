## 一、核心进化体系梳理

### 1. Self-Evolve Plugin（强化学习 + RAG）

**仓库**: github.com/longmans/self-evolve
**地位**: OpenClaw 官方推荐的自我进化插件，基于 Q 值强化学习

**核心机制**:
- 观察对话 + 反馈信号 → 更新 utility(Q values) → 写入情景记忆片段
- 检索时：本地 + 远程记忆合并 → Phase-B 排序 → 注入上下文
- 隐私设计：本地双重脱敏（sanitizeMemoryText → LLM summarizer → [REDACTED_*] 占位符）
- 共享网络：self-evolve.club，可与其他节点共享经验（默认开启）

**三种学习模式**:
| 模式 | 说明 | Token 消耗 |
|------|------|-----------|
| balanced（默认）| 优先工具调用轮次；无工具轮次需要更高奖励 | 中 |
| tools_only | 仅在学习工具调用时 | 低 |
| all | 所有通过奖励门槛的轮次 | 高 |

**关键参数**:
- `runtime.minAbsReward = 0.15` — 最小绝对奖励
- `runtime.minRewardConfidence = 0.55` — 最小奖励置信度
- `retrieval.tau = 0.85` — 仅在最佳相似度足够高时注入记忆
- `runtime.newIntentSimilarityThreshold = 0.35` — 新意图判定阈值
- `runtime.pendingTtlMs = 300000` — 5 分钟 TTL
- `memory.maxEntries = 200` — 最大记忆条目数

**反馈原则**:
- Praise 要明确（"做得好" > "ok"）
- Criticism 要具体（"你错了因为…" > "不对"）
- 明确反馈 > 模糊消息

**安装要求**: OpenClaw 2026.3.2+

**远程共享**: 默认开启，贡献排行可在 self-evolve.club/#leaderboard 查看

### 2. Self-Improving Agent（.learnings 文件系统）

**下载量**: 260,000+（ClawHub 最热门技能之一）

**文件结构**:
```
.learnings/
├── ERRORS.md          # 错误日志
├── LEARNINGS.md       # 学到的经验
└── FEATURE_REQUESTS.md # 功能缺口
```

**核心工作流**:
```
用户纠正 → 创建知识卡片 → 促进到 rules/ → 更新 AGENTS.md
```

**知识卡片格式**:
```markdown
---
topic: "Correction: Always Use Feature Branches"
category: lessons
tags: [correction, git, workflow]
created: 2026-03-17
---

## What Happened
[发生了什么]

## Correct Approach
[正确做法]

## Why It Matters
[为什么重要]
```

**提升规则**:
| 条件 | 提升到 |
|------|--------|
| 适用于所有任务 | AGENTS.md（核心规则）|
| 特定领域（git/写作/安全）| rules/<domain>.md |
| 同一模式出现 3+ 次 | 必须提升 |

**自动错误检测模式**:
- Exit codes != 0
- "error:", "Error:", "ERROR"
- "command not found", "permission denied"
- HTTP 4xx/5xx
- "rate limit", "quota exceeded", "429"
- Timeout, OOM kills
- Git merge conflicts

## 二、稳定运行的 7 条黄金法则（BetterClaw 调研）

**来源**: betterclaw.io/blog/openclaw-best-practices（2026年4月最新）

### 法则 1：模型路由（最关键！）

**问题**: 一个模型处理所有任务 → $80-150/月
**方案**: 不同任务路由到不同模型 → $14-25/月（省 70-80%）

| 任务类型 | 推荐模型 | 价格 |
|---------|---------|------|
| 心跳检测 | Haiku / DeepSeek | $1/$5 per million |
| 简单对话 | Sonnet / Gemini Flash | 中等 |
| 复杂推理 | Opus / GPT-4o | 贵 |

### 法则 2：所有提供商设置消费上限

- 设置月度上限（建议 2-3x 预期用量）
- 设置 `maxIterations=10-15`（防止单轮次无限循环调用工具）

### 法则 3：结构化 SOUL.md

必须包含：
- Error state behavior（工具失败时说什么）
- Conversation boundaries（何时停止并升级）
- Rate limit language（限速时的说明）
- Topic restrictions（明确拒绝的话题）

### 法则 4：安全基线

- Gateway 绑定 127.0.0.1，不是 0.0.0.0
- SSH Key 认证，禁用密码
- UFW 防火墙配置
- 安装技能前必须完整阅读 SKILL.md 代码

### 法则 5：配置文件版本控制

防止更新后崩溃无法回滚

### 法则 6：监控日志

- Self-Healing 心跳（每小时）
- 每日全量扫描

### 法则 7：定期进化复盘

- 凌晨 4 点自动复盘
- Self-Evolve 持续学习

## 三、每日自动化进化流程（OpenClaw Pulse 方案）

**运行时间**: 工作日 8:30 AM（3-8 分钟，£0.02-0.08/次）

**执行步骤**:
1. 扫描 Anthropic Engineering blog
2. 扫描 Simon Willison Weblog（"LLM 领域最好的实践评论"）
3. 扫描 Hacker News（AI agents, OpenClaw, prompt injection 等标签，min 50 points）
4. 扫描 GitHub Trending（TypeScript/Python/Go，AI agent 相关 topics）
5. 读取 AGENTS.md, TOOLS.md, LESSONS.md 对比现有状态
6. 识别差距和改进点
7. 写入 experiments/improvements.json
8. 可选：发送早间简报到 Telegram

**重要设计原则**: agent 不自动修改自己的文件 → 提出变更 → 等待人工审核。这防止了 prompt injection 攻击的持久化风险。

## 四、记忆系统防崩溃体系（VelvetShark Memory Masterclass）

### 四层记忆

| 层级 | 内容 | 持久性 |
|------|------|--------|
| Bootstrap 文件 | SOUL/AGENTS/USER/MEMORY.md | **永久** |
| Session Transcript | JSONL 对话历史 | 半永久 |
| LLM Context Window | 模型实际看到的内容 | **临时**（溢出即压缩）|
| Retrieval Index | memory_search 索引 | 永久 |

### 三种失败模式

| 模式 | 描述 |
|------|------|
| Failure A | 从未存储 — 指令只存在于对话，压缩后永久消失 |
| Failure B | 压缩改变了上下文 — 长会话触发摘要，有价值细节丢失 |
| Failure C | Session Pruning 裁剪工具结果（磁盘不变）|

### 防御策略

- **所有规则必须写入 MEMORY.md / AGENTS.md**（对话中说的压缩后可能消失）
- 关键词触发写入：当老庄说"记得"、"请记录"、"记住这一点"
- 验证命令：`openclaw memory status` / `openclaw memory search` / `openclaw memory index --force`

### 1. 模型分级路由：省 50-80% 成本的终极方案

**来源**: VelvetShark 实测 | betterclaw.io | zenvanriel.com

**核心问题**: 默认配置下，心跳、简单查询、子 Agent 都用 Opus → 每月 $80-150

**解决方案：模型分级**

| 任务类型 | 推荐模型 | 价格对比 |
|---------|---------|---------|
| 心跳检测（30分钟/次）| Gemini 2.5 Flash-Lite | $0.50/百万token（比 Opus 便宜60倍）|
| 简单查询 | DeepSeek V3.2 | $0.53/百万token |
| 日常编码/内容创作 | DeepSeek R1 / Kimi K2 | $2-3/百万token |
| 复杂推理/架构决策 | Opus / GPT-5.2 | $15-30/百万token |

**实测效果**: 正确路由后，$80-150/月 → $14-25/月（省 70-80%）

**配置示例**（心跳专用廉价模型）:
```json
{
  "agents": {
    "defaults": {
      "model": {
        "primary": "anthropic/claude-opus-4-5",
        "fallbacks": ["openai/gpt-5.2", "deepseek/deepseek-reasoner", "google/gemini-3-flash"]
      }
    },
    "heartbeat": {
      "every": "30m",
      "model": "google/gemini-2.5-flash-lite",
      "target": "last"
    }
  }
}
```

**OpenRouter 自动路由**: 设置 `openrouter/openrouter/auto` 作为模型，AI 自动根据任务复杂度路由到合适模型

1. 公共数据池（墨媒07:50扫描热点 → 各Agent 08:00-09:40读取）

**关键洞察**: "没有记忆的AI，只是一个聪明的工具。有记忆且能进化的AI，才是会成长的伙伴。"

**安装命令**: `clawhub install openclaw-self-healing`

**心跳检查 Cron 示例**:
```bash
openclaw cron add \
 --id "self-healing-heartbeat" \
 --schedule "0 * * * *" \   # 每小时
 --message "读取 HEARTBEAT.md，执行心跳检查。任何失败项立即告警，检测到配置错误时尝试自动回滚。" \
 --model "haiku"
```

**自我修复 Cron 示例**:
```bash
openclaw cron add \
 --id "self-healing-daily-scan" \
 --schedule "0 2 * * *" \   # 每天凌晨2点
 --message "扫描过去24小时错误日志，识别重复模式，生成修复建议。" \
 --model "haiku"
```

**HEARTBEAT.md 核心检查项**:
- Gateway 运行状态
- 磁盘空间 >10GB
- 无僵尸进程
- openclaw.json 是有效 JSON
- 无关键任务失败日志

**⚠️ 安全提示**: 可设置 `autoRollback: false`，改为仅发送告警，人工确认后再回滚

### 2. AutoSkill 框架（技能不是设计出来的，是长出来的）

**来源**: cloud.tencent.com | GitHub ECNU-ICALK/AutoSkill
**论文**: arxiv.org/abs/2603.01145

**核心思想**: 不是人为设计所有技能，而是让系统从真实交互中自动提炼技能

**双环机制**:

| 环 | 名称 | 作用 |
|----|------|------|
| 右环 | 技能进化 | 提取和维护，将交互体验转化为显式技能 |
| 左环 | 技能增强 | 查询重写 → 技能检索 → 上下文注入 → 响应生成 |

**三步工作流**:

```
① 摄取经验：系统摄取用户交互过程中的各种信号（对话内容、行为轨迹、用户反馈）
② 提炼技能：分析是否存在可复用模式（重复出现 = 提炼，单次请求 = 忽略）
③ 复用能力：技能加入技能库，已有毒能被更新或合并，无复用价值被过滤
```

**技能版本管理**: 新经验合并到已有技能，非新建，版本号自动递增

**典型案例**:
- `professional_text_rewrite`：文本重写技能，已迭代至 **v0.1.34**（通过30+次用户反馈优化）
- `顶级心理咨询师`：从对话中涌现的行为模式（温暖、共情、专业语气、避免医疗诊断）

**安装**: `npx clawhub@latest install autoskill`

### 3. Capability Evolver（GEP 引导进化协议）

**来源**: clawhub.ai/autogame-17/capability-evolver
**地位**: ClawHub 排名第一 | 35,000+ 下载 | 2026年2月1日上线，10分钟内登顶

**定位**: 元技能（meta-skill），让 Agent 检查自己的运行时历史，识别失败或低效，主动写新代码或更新记忆

**核心机制**:
- 运行 `/evolve` 扫描 memory/ 日志找优化机会
- 任务连续失败3次自动检测并创建 workaround
- GEP（Guided Evolution Protocol）标准化改进流程，**防失控**

**推荐配置**:
```bash
claw config capability-evolver --auto-optimize=true --interval=24h
```

**与 Self-Evolve 的区别**:

| | Self-Evolve | Capability Evolver |
|--|------------|---------------------|
| 机制 | Q值强化学习 + RAG | GEP 协议 + 历史分析 |
| 触发 | 奖励信号（praise/criticism）| 连续失败3次 / /evolve 命令 |
| 防失控 | 共享网络脱敏 | GEP 协议约束 |

### 5. 进化 + 技能的融合架构（孟健案例精华）

**核心洞察**: "让AI从工具进化为会成长的数字员工"

**三个必备能力**:
1. **记忆能力**: 记住用户习惯和偏好
2. **复盘能力**: 从错误和失败中不断学习
3. **技能沉淀能力**: 把经验转化为稳定的解决方案

**最终形态**: 个人数字替身 — 时间越久，Agent 越像你自己

**Cron 定时复盘（各Agent晚间固定时间，错开10分钟）**:
```
21:30 墨微（公众号）
21:40 墨知（知乎）
21:50 墨视（视频号）
...
23:30 墨媒（汇总全局日报）
```

**数据驱动进化闭环**:
```
采集数据 → 分析对比 → 得出结论 → 更新Playbook → 下次执行
```
