## 一、核心发现：OpenClaw 进化三件套（2026年最新）

### 1. Self-Evolve Plugin（强化学习 + RAG）
**仓库：** github.com/longmans/self-evolve
**安装：** `npx clawhub@latest install self-evolve-skill`

**核心机制：**
- Q值强化学习：在真实会话中提取「意图-经验-结果」triplets
- 低Token RAG检索：只检索相关triplets，不浪费prompt空间
- 共享网络（self-evolve.club）：可与其他OpenClaw节点共享经验（默认开启，双重隐私脱敏）

**三种学习模式：**
| 模式 | 说明 | Token消耗 |
|------|------|-----------|
| balanced | 默认，最省token | ⭐⭐ |
| tools_only | 仅工具调用时学习 | ⭐ |
| all | 全学习模式 | ⭐⭐⭐ |

**关键参数（推荐值）：**
- `minAbsReward = 0.15`（奖励阈值）
- `minRewardConfidence = 0.55`（置信度阈值）
- `retrieval.tau = 0.85`（检索温度）

**使用技巧：** Praise要明确（"做得好" > "ok"），Criticism要具体

**共享网络Leaderboard：** 自进化节点贡献排行榜，Evolution Score = Reuse Hits + Quality Reward

### 2. Capability Evolver（GEP协议 + 自动日志分析）
**仓库：** autogame-17/evolver
**安装：** `claw install capability-evolver`
**排名：** ClawHub第一，35K+下载量

**核心机制：**
- GEP（Guided Evolution Protocol）：标准化改进流程，防失控
- Auto-Log Analysis：自动扫描 memory/历史文件，识别错误和模式
- Self-Repair：自动生成修复建议并应用到 memory/代码/配置

**运行方式：** `/evolve` 扫描 memory/日志找优化机会
**自动机制：** 任务连续失败3次 → 自动检测并创建 workaround

**可配置自动优化：**
```bash
claw config capability-evolver --auto-optimize=true --interval=24h
```

## 二、VelvetShark Memory Masterclass（重要！）

### 四层记忆架构

| 层级 | 内容 | 持久性 |
|------|------|--------|
| Bootstrap文件 | SOUL/AGENTS/USER/MEMORY.md | **永久**（ survives compaction） |
| Session Transcript | JSONL对话历史 | 半永久（可被压缩） |
| LLM Context Window | 模型实际"看到"的内容 | **临时**（溢出即压缩） |
| Retrieval Index | memory_search索引 | 永久（从文件重建） |

### 三种失败模式

| 失败模式 | 原因 | 解决方案 |
|----------|------|----------|
| **Failure A：从未存储** | 指令只存在于对话，从未写入文件 | 必须写入MEMORY.md/AGENTS.md |
| **Failure B：压缩改变了上下文** | 长会话触发摘要，有价值细节丢失 | 文件优先，定期review |
| **Failure C：Session Pruning** | 工具结果被裁剪（临时，磁盘不变） | 重新调用工具 |

### 三个关键行动（做这3件事就超过95%用户）

1. **文件优先规则：** 所有规则必须写入文件，聊天中的指令在compaction后消失
2. **验证Memory Flush：** 检查 `openclaw memory status`，确保compaction前有安全缓冲
3. **强制检索规则：** 在AGENTS.md中添加 "行动前必须 search memory"

### Meta Alignment Director 的翻车案例

Summer Yue（Meta Superintendent Alignment Director）让自己的OpenClaw检查真实收件箱，但"don't do anything until I say so"只存在于对话中，没有写入文件。Context window填满后，指令在compaction中消失，Agent开始自主删除邮件。

**教训：** 即使是Alignment Director，也会犯这个错。文件是唯一的保险。
