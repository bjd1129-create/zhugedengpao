## 🔬 工具详解

### 1. Capability Evolver（GEP协议）

**安装：** `claw install capability-evolver`

**核心机制：**
- 自动扫描 memory/ 和历史文件中的错误和模式
- 连续失败3次自动检测并创建 workaround
- **GEP（Guided Evolution Protocol）** — 标准化进化流程，防失控
- 可配置每24小时自动运行：`claw config capability-evolver --auto-optimize=true --interval=24h`
- 运行命令：`/evolve`

**进化流程（GEP）：**
1. 采集数据（memory + 错误日志）
2. 分析问题根因
3. 生成结构化改进提案
4. 人类审核（或自动应用）
5. 更新规则/代码

**关键优势：** 让所有技能随使用时间持续优化，不只是单一能力。

---

### 2. Self-Evolve Plugin

**来源：** github.com/longmans/self-evolve
**安装：** `git clone` 后 `openclaw plugins install`

**核心机制：**
- **Q值强化学习** — 在真实会话中提取「意图-经验-结果」triplets
- **情景记忆检索（RAG）** — 回答前检索并 prepend 到 prompt context
- **多轮任务聚合** — 检测到反馈时学习
- 共享网络：**self-evolve.club** — 可与其他 OpenClaw 节点共享经验（默认开启，双重脱敏）

**三种学习模式：**
- `balanced`（默认）— 最省token
- `tools_only` — 仅工具调用时学
- `all` — 全学习，高消耗

**关键参数：**
- `minAbsReward=0.15` — 最小奖励阈值
- `minRewardConfidence=0.55` — 奖励置信度
- `retrieval.tau=0.85` — 检索温度

**使用技巧：** Praise要明确（"做得好" > "ok"），Criticism要具体

## 🧠 记忆系统关键发现

### 四层记忆架构（VelvetShark 2026-04 实测）

| 层级 | 内容 | 持久性 |
|------|------|--------|
| Bootstrap文件 | SOUL/AGENTS/USER/MEMORY.md | **永久** |
| Session Transcript | JSONL对话历史 | 半永久（可压缩）|
| LLM Context Window | 模型实际看到的内容 | **临时**（溢出即压缩）|
| Retrieval Index | memory_search索引 | 永久（从文件重建）|

### 三种失败模式

- **Failure A：** 从未存储 — 指令只存在于对话，压缩后永久消失（最常见！）
- **Failure B：** 压缩改变了上下文 — 长会话触发摘要，有价值细节丢失
- **Failure C：** Session Pruning裁剪工具结果（临时，磁盘不变）

### Compaction vs Pruning

| | Compaction | Pruning |
|--|-----------|---------|
| **作用** | 总结旧对话 | 裁剪旧工具结果 |
| **保存？** | 是（摘要存入transcript） | 否（仅内存） |
| **范围** | 整个对话 | 仅工具结果 |
| **风险** | 高（有损） | 低（无损） |

### Memory Flush 安全网

OpenClaw 内置自动在 compaction 前执行 memory flush，但**需验证是否启用**：
```bash
openclaw memory status
```
