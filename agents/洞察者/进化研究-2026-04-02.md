## 一、核心发现：三大进化体系并存

经过深度调研，当前 OpenClaw 生态存在三套并行的自我进化机制，各有分工：

| 体系 | 类型 | 核心机制 | 适用场景 |
|------|------|----------|----------|
| **Self-Evolve Plugin** | Q值强化学习 + RAG | 从真实反馈中学习，算法更新 Q 值，写入情景记忆 | 需要算法驱动、偏好共享网络 |
| **Self-Improving-Agent** | 文件系统型 | 错误日志 + 用户纠正 + 结构化记录，支持晋升 | 轻量级、自托管、无外部依赖 |
| **Capability Evolver** | 元技能（Meta-Skill） | 检查运行时历史、自动写代码/更新记忆 | 高级用户、需要自动代码修改 |

## 二、Self-Evolve Plugin（推荐优先部署）

**仓库：** github.com/longmans/self-evolve
**安装：** `git clone && openclaw plugins install ./self-evolve`

### 核心原理

- **Q值强化学习**：不是简单记录，而是给每次行为打分（reward），算法更新 Q 值，逐步强化好的策略。
- **情景记忆注入**：在回答前检索相关记忆片段，拼接到 prompt 上下文。
- **共享网络**：self-evolve.club 可与其他 OpenClaw 节点共享经验（默认开启，双重脱敏）。

### 三种学习模式

| 模式 | 说明 | Token 消耗 |
|------|------|------------|
| `balanced`（默认） | 优先工具调用轮次；无工具轮次需要高 reward 才能学习 | 中 |
| `tools_only` | 仅在调用工具时学习 | 低 |
| `all` | 所有通过 reward 门槛的轮次都学 | 高 |

### 关键参数

```
minAbsReward = 0.15        # 最低绝对 reward 值
minRewardConfidence = 0.55  # 最低 reward 置信度
retrieval.tau = 0.85       # 记忆注入阈值（相似度不够不注入）
```

### 反馈技巧

- ✅ 说"做得好"比说"ok"好（明确 > 模糊）
- ❌ 说"不对"比说"有点问题"好（批评要具体）
- 无反馈时跳过学习（idle close / new intent switch）

### 隐私设计

- 第一轮脱敏：`sanitizeMemoryText` 移除对话元数据、ID、发送者标签
- 第二轮脱敏：经验总结器 LLM 要求输出时可迁移策略，敏感数据替换为 `[REDACTED_*]`
- 共享数据仅限脱敏后的 triplets（intent / experience / embedding）

## 三、Self-Improving-Agent（文件系统型）

**仓库：** github.com/peterskoett/self-improving-agent
**安装：** `git clone ~/.openclaw/skills/self-improving-agent`

### 工作方式

在 `.learnings/` 目录下维护三个日志文件：

```
LEARNINGS.md       # 用户纠正、知识缺口、最佳实践
ERRORS.md          # 命令失败、异常报错、API 调用失败
FEATURE_REQUESTS.md # 用户要求但当前不支持的功能
```

### 触发场景

| 触发 | 写入文件 |
|------|---------|
| 命令/操作意外失败 | ERRORS.md |
| 用户纠正 Agent | LEARNINGS.md |
| 外部 API / 工具失败 | ERRORS.md |
| 发现知识过时 | LEARNINGS.md |
| 发现更好做法 | LEARNINGS.md |

### 晋升路径（精妙设计！）

有价值的学习记录可以"晋升"到中央文件，使其在所有会话中持久生效：

- **行为模式** → SOUL.md
- **工作流改进** → AGENTS.md
- **工具注意事项** → TOOLS.md

### Hook 配置（可选但推荐）

配置 `UserPromptSubmit` 和 `PostToolUse`（Bash）钩子，实现自动检测和记录，无需每次手动提醒。

## 四、Capability Evolver（Meta-Skill）

**ClawHub：** clawhub.ai/autogame-17/capability-evolver
**安装：** `npx clawhub@latest install self-evolve-skill` 或 `claw install capability-evolver`

### 核心功能

- **Auto-Log Analysis**：自动扫描 memory 和历史文件，查找错误和模式
- **Self-Repair**：检测崩溃并建议补丁
- **GEP Protocol**：标准化进化协议，产出可复用资产
- **One-Command Evolution**：运行 `/evolve` 或 `node index.js`

### 运行模式

```bash
node index.js              # 全自动（Mad Dog 模式）
node index.js --review     # 人工审核后再应用
node index.js --loop       # 持续循环（配合 cron）
```

### 安全参数

```bash
EVOLVER_ROLLBACK_MODE=hardRollback   # 失败时 git reset --hard
EVOLVER_AUTO_ISSUE=1                 # 重复失败自动建 GitHub Issue
EVOLVE_ALLOW_SELF_MODIFY=false       # 禁止修改自身代码（生产环境）
```

### EvoMap 网络

需要注册节点身份（visit evomap.ai/claim/ within 24 hours），加入后可用远程知识图谱同步。
