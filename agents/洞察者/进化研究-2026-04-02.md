## 一、核心自我进化体系（三件套）

### 1. Self-Evolve Plugin（推荐优先级最高）

**性质：** OpenClaw 插件，非 Skill
**安装：**
```bash
git clone https://github.com/longmans/self-evolve
openclaw plugins install ./self-evolve
```
**推荐版本：** OpenClaw 2026.3.2+，旧版本缺少 hook context

**机制：**
- Q值强化学习，在真实会话中提取「意图-经验-结果」triplets
- 每次响应前通过 RAG 检索相关情景记忆，prepend 到 prompt context
- 任务跨多轮聚合，检测到反馈时触发学习

**三种学习模式（runtime.learnMode）：**
| 模式 | 说明 | Token消耗 |
|------|------|----------|
| `balanced`（默认）| 仅工具调用时学习，无工具调用需高reward | 中 |
| `tools_only` | 仅工具调用时学习 | 低 |
| `all` | 所有通过reward门的轮次都学习 | 高 |

**关键参数（推荐值）：**
- `runtime.minAbsReward=0.15` — 最小绝对奖励
- `runtime.minRewardConfidence=0.55` — 最小奖励置信度
- `retrieval.tau=0.85` — 仅在相似度够高时注入记忆

**远程共享网络（self-evolve.club）：**
- 默认开启，本地+远程记忆合并后排序
- 隐私双重脱敏：sanitizeMemoryText 移除敏感ID → LLM summarizer二次脱敏
- 可贡献经验给社区，同时利用他人高价值经验

**使用技巧：**
- Praise 要明确具体（"做得好" > "ok"）
- Criticism 要具体（"这里出错因为..." > "不对"）
- Explicit feedback > vague messages

**验证运行正常：**
```bash
openclaw gateway restart
# 查看日志有：
# self-evolve: initialized ...
# self-evolve: loaded episodic memories
```

### 2. Capability Evolver（ClawHub 排名第一，35K+下载）

**性质：** Skill（不是 Plugin）
**安装：** `claw install capability-evolver`
**机制：** GEP（Guided Evolution Protocol）— 协议约束型自我进化

**核心流程：**
1. 自动扫描 memory/ 和历史文件，识别错误和模式
2. 生成结构化改进 prompt
3. 安全、可审计地修改 memory/code/config

**自动优化配置：**
```bash
claw config capability-evolver --auto-optimize=true --interval=24h
```

**适用场景：**
- 已有明确故障点的高级自动化堆栈
- 需要更好恢复能力、更自适应提示的系统
- 适合已知 Agent 薄弱点、想建立更清晰反馈循环的开发者

**关键安全机制：**
- GEP 流程防失控，不是随意自我修改
- 改动可审计、可回滚

### 3. Self-Improving Agent（ClawHub 最多下载，26万+）

**性质：** Skill
**机制：** `.learnings/` 文件系统
- `ERRORS.md` — 记录犯过的错误
- `LEARNINGS.md` — 记录学到的经验
- `FEATURE_REQUESTS.md` — 功能建议

**工作方式：**
- 定时凌晨4点自动复盘
- 命令失败或被纠正时自动记录
- 随时间建立本地知识库

**最佳实践：**
- 定时复盘机制：各 Agent 晚间固定时间错开复盘（避免资源冲突）
- 进化闭环：采集数据 → 分析 → 结论 → 更新规则 → 下次执行

## 二、OpenClaw 稳定运行7条黄金法则（BetterClaw 2026-03）

### 法则1：模型路由，不是模型忠诚
**核心：** 不同任务用不同模型
- Heartbeat → Haiku ($1/$5 per M tokens) 或 DeepSeek ($0.28/$0.42)
- 简单对话 → Sonnet / Gemini Flash
- 复杂推理 → Opus / GPT-4o

**效果：** 从 $80-150/月 降到 $14-25/月，省70-80%

### 法则2：所有提供商设置消费上限
- 每月上限设为预期月费的2-3倍
- 设置 `maxIterations=10-15`（单次响应最大工具调用数）
- 没有上限：一次 bug 可在1小时内烧掉 $50-100

### 法则3：结构化 SOUL.md
**必须包含的章节：**
- Error state behavior（工具失败时怎么说）
- Conversation boundaries（何时停止、升级人工）
- Topic restrictions（明确拒绝讨论的主题）
- Rate limit language（暂停响应时的解释）

### 法则4：安全不是事后想法
- Gateway 绑定 127.0.0.1，不是 0.0.0.0
- SSH key 认证，禁用密码登录
- UFW 防火墙
- 技能安装前完整阅读 SKILL.md 代码
- 定期更新（OpenClaw 每周多次发布补丁）

### 法则5：单频道起步，逐步扩展
- 先 Telegram（最易配置和测试）
- 至少运行一周，收集真实对话后再加频道

### 法则6：监控要在用户告诉你之前发现问题
- Self-Healing 心跳每小时
- 每日全量扫描

### 法则7：配置文件版本控制
- 每次修改前备份：`cp openclaw.json openclaw.json.bak`
- 用 `config.patch` 而非 `gateway stop/start`
- 读 changelog 再更新

## 三、技能架构最佳实践（Kinsman）

### SKILL.md 保持精简
- **永远 < 50 行**，超过一屏就太长了
- 工作流步骤放 SKILL.md（每次读取）
- Python/bash 脚本放 `scripts/`（执行时读取）
- API 文档放 `references/`（按需读取）

### 上下文窗口管理
| 来源 | Token成本 | 缓解方法 |
|------|----------|---------|
| SOUL.md / AGENTS.md | 每次会话 | 保持 < 2页 |
| MEMORY.md | 每次主会话 | 定期归档到 memory/ |
| SKILL.md | 技能触发时 | 保持 < 50行 |
| 心跳轮次 | 每个心跳 | 保持 HEARTBEAT.md 精简 |
| 对话历史 | 累积 | Compaction处理 |

### 记忆层级架构
```
MEMORY.md — 精炼长期记忆（精华）
memory/YYYY-MM-DD.md — 每日原始笔记
memory/archives/ — 已蒸馏的旧上下文
```

## 六、重大发现总结

1. **Self-Evolve 是当前最强的实时进化机制** — Q值强化学习 + RAG，比简单记忆快照稳健得多
2. **Token省钱空间巨大** — 模型路由可省70-80%，从$80-150/月降到$14-25/月
3. **技能文件精简是关键** — 超过50行的SKILL.md正在浪费宝贵的上下文窗口
4. **共享网络价值被低估** — self-evolve.club 可以白嫖全社区的经验，同时贡献自己的
5. **安全不能妥协** — ZeroClaw下架事件 + ClawHavoc恶意技能提醒必须定期备份+审查
