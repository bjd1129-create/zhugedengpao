## 一、三大进化插件深度分析

### 1. Self-Evolve（自进化插件）
- **来源**：GitHub longmans/self-evolve，共享网络 self-evolve.club
- **机制**：Q值强化学习 + RAG情景记忆检索
- **核心流程**：
  1. 回答前检索情景记忆，拼接到Prompt上下文
  2. 聚合多轮任务，检测反馈信号
  3. 通过更新 Q值（效用值）持续学习，写入新情景记忆
- **三种模式**：balanced（默认，最省token）/ tools_only（仅工具调用）/ all（全学习，高消耗）
- **关键参数**：minAbsReward=0.15, minRewardConfidence=0.55, retrieval.tau=0.85
- **共享网络**：可与其他 OpenClaw 节点共享经验（默认开启，隐私双重脱敏）
- **安装**：`npx clawhub@latest install self-evolve-skill`

### 2. Capability Evolver（能力进化器）
- **下载量**：35K+，ClawHub 排名第一
- **机制**：GEP（Guided Evolution Protocol）——协议约束的自我改进，防止失控
- **核心功能**：
  - 自动日志分析：扫描 memory 和历史文件中的错误和模式
  - 自我改进：任务连续失败3次自动检测并创建 workaround
  - 自动优化：可配置每24小时自动运行
- **命令**：`claw install capability-evolver` + `/evolve` 扫描优化
- **自动配置**：`claw config capability-evolver --auto-optimize=true --interval=24h`
- **价值**：让所有技能随使用时间持续优化，不只是单一能力

### 3. Self-Improving Agent（自我改进代理）
- **下载量**：260K+，最热门的自我改进技能
- **机制**：self-reflection + self-criticism + self-learning + self-organizing memory
- **工作流**：Agent 评估自身工作 → 捕捉错误 → 永久改进
- **安装**：`clawhub install self-improving`

## 二、新兴实操模式：早晨研究简报（Cron）

**来源**：openclawpulse.com 深度教程

### 完整架构（可直接复用到团队）

**Cron 触发时间**：工作日 8:30 AM（成本约 £0.02-0.08/次）

**研究来源**：
1. Anthropic Engineering Blog（AI工程博客）
2. Simon Willison's Weblog（最佳LLM实践评论）
3. Hacker News（AI agents/OpenClaw/prompt injection相关，min 50 points）
4. GitHub Trending（AI agent/LLM工具类，Top 15）

**工作流**：
```
搜索博客 → 扫描HN → 检查GitHub趋势 → 读取自身AGENTS.md/TOOLS.md/LESSONS.md
→ 识别差距和改进点 → 记录到结构化JSON实验追踪文件 → 可选：Telegram早间简报
```

**关键设计原则**：
- Agent **不自动修改文件**，只**提出变更建议**（防止 prompt injection）
- 评分标准：(a)实用性 (b)是否已知 (c)实现难度，最高5项输出
- 记忆先行：必须先读取现有文件，对比后再记录新发现

## 三、记忆系统深度解析（VelvetShark Masterclass）

### 四层记忆架构

| 层级 | 内容 | 持久性 |
|------|------|--------|
| Bootstrap文件 | SOUL.md/AGENTS.md/USER.md/MEMORY.md | **永久** |
| Session Transcript | JSONL对话历史 | 半永久（可压缩）|
| LLM Context Window | 模型实际看到的内容 | **临时**（溢出即压缩）|
| Retrieval Index | memory_search索引 | 永久（从文件重建）|

### 三种失败模式（重大警示）

| 模式 | 原因 | 解决方案 |
|------|------|---------|
| **Failure A** | 指令只存在于对话，从未写入文件 | 写入 MEMORY.md/AGENTS.md |
| **Failure B** | 长会话触发摘要，有价值细节丢失 | 定期固化到文件 |
| **Failure C** | Session Pruning裁剪工具结果 | 临时，磁盘不变 |

### 三大必须操作（防丢失铁律）

1. **持久化规则写入文件**：MEMORY.md 和 AGENTS.md 生存 compaction，聊天中的指令不生存
2. **检查 memory flush 是否启用**：OpenClaw 内置安全网，但多数人从未检查
3. **强制检索规则**：AGENTS.md 添加"行动前必须 search memory"规则
