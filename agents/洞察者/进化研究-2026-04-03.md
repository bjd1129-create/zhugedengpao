## 二、VelvetShark Memory Masterclass（深度补充，续2026-04-02）

昨天报告已覆盖四层架构，今天补充**最关键但被大多数人混淆的概念**：

### Compaction vs Pruning（必须分清！）

**这是两个完全不同的系统：**

| 特性 | Compaction（危险！） | Pruning（友好） |
|------|---------------------|----------------|
| **做什么** | 把整个对话历史压缩成摘要 | 只裁剪内存中的旧工具结果 |
| **影响范围** | 所有消息（用户/助手/工具调用） | 仅工具结果，不碰对话 |
| **触发时机** | Context window快满了才触发 | 实时，自动 |
| **是否丢失** | **是，有损压缩** | 否，无损 |
| **磁盘影响** | JSONL文件被改写 | 不触碰磁盘 |
| **可逆性** | 不可逆 | 临时失效，重新调用即可 |

**实操建议：** Pruning是朋友，别关它。Compaction才是危险的那一个。

### Pruning 配置（如果需要调优）

默认cache-ttl模式（5分钟TTL）已足够智能。如需调整：
```json
{
  "agents": {
    "defaults": {
      "contextPruning": {
        "mode": "cache-ttl",
        "ttl": "5m"
      }
    }
  }
}
```

### 诊断命令（最有用的命令）

```bash
/context list
```

运行后能看到：
- Bootstrap文件注入状态（AGENTS.md/SOUL.md是否正常加载）
- 当前Context使用量（吃了多少Token）
- 哪些文件被TRUNCATED了（TOOLS.md经常被截断！）

**已知问题：** TOOLS.md经常被截断（raw 54K chars → injected 20K chars）。如果TOOLS.md有重要配置，可能在加载时被截断。

### 三个必须做的行动（超越95%用户）

1. **Put durable rules in files, not chat** — 聊天中的指令不survive compaction
2. **Check that memory flush is enabled** — compaction触发前有安全缓冲
3. **Make retrieval mandatory** — AGENTS.md里加"search memory before acting"

## 三、三层记忆系统详解（实操模板）

**来源：** getopenclaw.ai Permanent Memory Guide（2026年2月发布）

### Layer 1: MEMORY.md — 永久偏好和模式

**模板：**
```markdown
# MEMORY.md

## Communication Style
- 3句以内，除非被要求展开
- 禁止："Great!" "Certainly!" 
- 用要点列表，不用段落

## Key Facts
- 主项目：[名称+描述]
- 时区：Asia/Shanghai (UTC+8)

## Rules
- 代我发邮件前必须确认
- 删除文件前必须问
```

**核心原则：** 质量 > 数量，保持3000 tokens以下。只放真正永久的信息。

### Layer 2: Daily Notes — memory/YYYY-MM-DD.md

**用途：** 时间敏感信息，不适合放MEMORY.md的

**模板：**
```markdown
# memory/2026-04-03.md

## Today's Events
- [事件描述]
- [决策]
- [未完成的任务]

## Decisions Made
- [具体决策]

## Follow-ups
- [需要跟进的事项]
```

### Layer 3: Entity Files — 知识图谱

**结构：**
```
life/areas/
  people/
    [name]/summary.md    ← 3-5句现状描述
             items.json   ← append-only事实日志
  projects/
    [project]/summary.md
             items.json
```

**适用场景：** 人际关系 + 长期项目（需要数周以上维护的）

### 自动记忆更新 Cron（推荐！）

设置心跳cron，每30分钟自动提取：
```
扫描近期对话。提取持久性事实（新提到的人、做的决策、表达的偏好、状态变化）。
写入 ~/life/areas/ 相关文件。
跳过闲聊、笑话、临时信息。使用便宜模型。
```

成本极低（便宜模型 + 小context），但记忆文件会自动保持最新。

### 晚间总结 Cron（推荐！）

晚10点自动执行：
```
回顾今日对话。写入 memory/YYYY-MM-DD.md，包含：
- 做的决策
- 提到要跟进的事
- 关于我工作/生活的新事实
保持500字以内。
```
