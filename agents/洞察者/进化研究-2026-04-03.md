## 一、核心发现：进化三件套

### 1. Self-Evolve Plugin（推荐优先级：⭐⭐⭐⭐⭐）

**来源：** github.com/longmans/self-evolve
**定位：** 最推荐的自我进化插件，基于 Q 值强化学习的实时进化引擎

**核心机制：**
- 检索式记忆：在回答前 prepend 相关情景记忆到上下文
- 多轮任务聚合：跨多个对话轮次聚合任务，检测到反馈时学习
- Q 值更新：通过更新 utility 值（Q values）实现算法化学习

**三种学习模式：**
| 模式 | 说明 | Token 消耗 |
|------|------|-----------|
| `balanced`（默认） | 偏好工具调用轮次；无工具轮次需要高 reward | 中 |
| `tools_only` | 仅在调用工具时学习 | 最低 |
| `all` | 所有通过 reward gate 的轮次都学 | 最高 |

**关键参数（学习门控）：**
- `minAbsReward=0.15` — reward 绝对值门槛
- `minRewardConfidence=0.55` — reward 置信度门槛
- `retrieval.tau=0.85` — 仅在相似度够高时注入记忆

**远程共享网络（self-evolve.club）：**
- 本地+远程记忆合并后 ranking 注入
- 双重隐私脱敏：本地 sanitize → LLM summarizer 再脱敏
- 可查看共享贡献排行榜

**使用技巧：**
- Praise 要明确（"做得好" > "ok"）
- Criticism 要具体明确
- 建议升级到 OpenClaw 2026.3.2+（旧版可能丢失 hook context）

**安装命令：**
```bash
git clone https://github.com/longmans/self-evolve
openclaw plugins install ./self-evolve
# 配置 env var
export OPENAI_API_KEY=sk-xxx
openclaw gateway restart
```

### 2. Self-Improving Agent（推荐优先级：⭐⭐⭐⭐）

**来源：** ClawHub（self-improving-agent by pskoett）
**定位：** .learnings 文件系统 + 凌晨自动复盘

**核心结构：**
```
.learnings/
  ERRORS.md          # 记录失败和异常（命令执行失败、工具调用出错、推理偏差）
  LEARNINGS.md       # 沉淀新经验和最佳实践
  FEATURE_REQUESTS.md # 收集用户改进建议
```

**自动触发机制：**
- 命令执行失败 → 提醒记录
- 用户纠正回答 → 记录正确信息
- 发现更优方案 → 记录到 LEARNINGS.md

**凌晨复盘 Cron（推荐 4:00 AM）：**
- 检查当天对话记录
- 提取有价值的经验
- 更新到 MEMORY.md
- 记录版本历史

**适用场景：** 需要结构化复盘、错误记录和方法论沉淀的团队

### 3. AutoSkill（推荐优先级：⭐⭐⭐⭐）

**来源：** ECNU-ICALK/AutoSkill（华东师范大学 + 上海人工智能实验室）
**论文：** arxiv.org/abs/2603.01145
**定位：** 从真实交互中"长出来"的技能，而非人为设计

**双环机制：**
- **右环（技能进化）：** 提取 → 维护 → 将交互体验转化为显式技能
- **左环（技能增强）：** 查询重写 → 技能检索 → 上下文注入 → 响应生成

**关键特点：**
- 技能版本管理：v0.1.34 表示通过 30+ 次用户反馈优化
- 技能结构透明（SKILL.md），人类可查看和编辑
- 通过记忆增长而非模型微调持续改进

**技能不是设计出来的，是"长出来的"：** 系统观察用户交互，识别稳定模式（写作要求、工作流程、表达风格），抽象为可复用技能。

## 三、OpenClaw 2026 Best Practices 要点

### 性能优化
- **模型路由：** 简单问答 → GPT-5-3-codex；复杂推理 → Claude-4-5-Sonnet（可省 70-80% 费用）
- **缓存：** 启用 TTL=3600s 缓存减少 API 调用
- **上下文精简：** system prompt 要聚焦，避免冗长的"helpful assistant"铺垫

### 成本管理
- **硬上限：** 设置 messagesPerMinute/maxTokensPerDay/monthlyBudget
- **预算警报：** budgetWarning=0.8, budgetExceeded=1.0
- **监控：** `openclaw stats` 定期检查

### 安全
- API Key 永远不放 hardcode，用 .env
- 定期 `openclaw update` 保持安全补丁
- 配置 allowFrom 限制访问来源

### 可靠性
- 启用 healthCheck（interval=60s）
- 配置 autoRestart + maxRetries
- 定期 `openclaw config export` 备份
