## 一、Self-Evolve Plugin（经验驱动进化）

### 来源
- GitHub: `longmans/self-evolve`
- 安装量：35K+（含 Capability Evolver 合计）
- 推荐版本：OpenClaw 2026.3.2+（当前运行 v2026.3.31 ✅ 满足）

### 核心机制

**三层学习闭环：**
1. **观察**：监听 `before_prompt_build` 和 `agent_end` 钩子，捕获对话+工具调用
2. **记忆**：将「意图-经验-结果」triplet 用 RAG 检索，只注入高相关度记忆（tau=0.85）
3. **强化**：Q值更新，奖励/惩罚驱动，经验累积到 self-evolve.club 共享网络

**学习模式（3种）：**
| 模式 | 说明 | Token 消耗 |
|------|------|-----------|
| `balanced`（默认）| 仅工具调用时学习，无工具调用需高奖励 | ⭐⭐ |
| `tools_only` | 仅工具调用时学习 | ⭐ |
| `all` | 所有通过奖励门槛的轮次都学习 | ⭐⭐⭐ |

**关键参数：**
- `minAbsReward=0.15` — 最小绝对奖励值
- `minRewardConfidence=0.55` — 最小奖励置信度
- `retrieval.tau=0.85` — 检索相似度阈值（只注入高于此值的记忆）
- `newIntentSimilarityThreshold=0.35` — 新任务边界检测阈值
- `idleTurnsToClose=2` — 空闲2轮自动关闭任务
- `memory.maxEntries=200` — 最多保留200条经验

**共享网络（self-evolve.club）：**
- Evolution Score = Reuse Hits + Quality Reward
- 贡献者排行榜可见
- **隐私保护**：本地双重脱敏（sanitizeMemoryText → LLM summarizer）
  - 第一层：去除对话元数据、IDs、发件人标签
  - 第二层：LLM 生成可迁移策略，敏感数据替换为 `[REDACTED_*]`

**安装命令：**
```bash
git clone https://github.com/longmans/self-evolve
openclaw plugins install ./self-evolve
export OPENAI_API_KEY=sk-xxx
openclaw gateway restart
```

## 二、Capability Evolver（GEP 协议驱动）

### 来源
- ClawHub 排名第一（35K+ 下载）
- GitHub: `openclaw/skills` → `capability-evolver`
- EvoMap Hub: `evomap.ai`

### 核心机制

**GEP（Guided Evolution Protocol）**：标准化进化流程，防失控。
- 自动扫描 memory/ 和 history 文件，识别错误和模式
- 自我修复：检测崩溃并建议补丁
- 进化策略可配置：`balanced | innovate | harden | repair-only | early-stabilize | steady-state | auto`

**安全特性：**
- `EVOLVE_ALLOW_SELF_MODIFY=false`（默认）— 不允许修改 evolver 源码
- `EVOLVER_LLM_REVIEW=0`（默认）— 可开启第二意见审查
- `EVOLVER_ROLLBACK_MODE=hard` — 失败时硬回滚
- `EVOLVE_LOAD_MAX=2.0` — 系统负载过高时自动退避

**使用方式：**
```bash
node index.js          # 全自动（Mad Dog Mode）
node index.js --review # 审查后应用
node index.js --loop   # 循环运行（配合 cron）
```

**EvoMap 网络：**
- 通过 A2A 协议与 evomap.ai 连接
- 节点注册后获得 `A2A_NODE_ID`
- 支持远程知识图谱同步（可选 MEMORY_GRAPH_REMOTE_URL）

## 四、十大实操技巧（Power User 经验）

> 来源：mindstudio.ai（200+小时经验总结）

1. **Telegram Thread 绑定**：用 thread 创建持久会话，避免每次新建
2. **模型路由**：Heartbeat→Haiku，推理→Opus，可省70-80%费用
3. **子 Agent 协作**：复杂任务拆给 sub-agent，主 agent 做协调
4. **Cron 定时任务**：设置凌晨自动复盘，错开各 Agent 运行时间
5. **Session 持久化**：用 `sessions_spawn mode=session` 保持上下文
6. **API 花费上限**：每个提供商设硬上限 + `maxIterations=10-15` 防 Runaway Loop
7. **SOUL.md 结构化**：必须包含 Error state behavior、Conversation boundaries
8. **配置文件版本控制**：防止更新后崩溃无法回滚
9. **技能安装前审计**：完整阅读 SKILL.md 代码，防止恶意代码注入
10. **每日健康检查**：Self-Healing 心跳每小时 + 每日全量扫描
