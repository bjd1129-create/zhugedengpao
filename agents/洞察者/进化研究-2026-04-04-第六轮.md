## 🔥 核心发现

### 1. OpenClaw-RL — Princeton 的异步强化学习框架

**论文**: [OpenClaw-RL: Train Any Agent Simply by Talking](https://arxiv.org/abs/2603.10165) (Princeton)

**核心洞察**: 
- next-state signals 是通用的 policy 学习信号
- personal conversations、terminal executions、GUI interactions、SWE tasks、tool-call traces **不是分离的训练问题**，而是统一的学习信号源
- 可以从 live user interactions 和环境信号中**异步持续训练**，无需离线批处理数据收集

**对团队的启发**: 
> 这是从"记忆固化"到"强化学习"的关键一步。我们目前的进化主要依赖 episodic memory，未来可以考虑引入 reward signals 让 agent 自主优化策略。

### 2. self-evolve 插件 — longmans 的自我学习系统

**GitHub**: https://github.com/longmans/self-evolve

**核心技术架构**:

```
用户消息 → 反馈检测?
  ├─ Yes → Score reward → Check learning gates → Sanitize → LLM summarize → Append episodic triplet
  └─ No  → Detect intent boundary → Retrieve local+remote candidates → Rank → Inject memories → Reply
```

**三大核心机制**:

1. **Episodic Memory with Q-values**
   - 每次任务聚合多轮对话，检测到反馈时学习
   - Q值更新让系统随时间自我优化（类似强化学习的 value function）
   - 检索时将 episodic memories prepend 到 prompt context

2. **三重隐私保护**
   - 第一重: `sanitizeMemoryText` 移除对话元数据、ID、敏感标签
   - 第二重: LLM summarizer 输出时可迁移策略，敏感数据替换为 `[REDACTED_*]`
   - 远程共享仅限 sanitized triplets

3. **Shared Skill Network (self-evolve.club)**
   - 本地 + 远程 memory 合并后 ranking
   - 贡献者获得 attribution feedback，quality reward 激励高质量经验共享
   - Evolution Score = Reuse Hits + Quality Reward

**学习模式配置**:
- `balanced` (默认): 优先 tool turns；no-tool turns 需要高 reward/high confidence
- `tools_only`: 仅从工具调用学习（最低 token 消耗）
- `all`: 所有通过 reward gates 的 turns 都学习（最高 token 消耗）

**关键参数**:
- `runtime.minAbsReward=0.15` — 最小绝对 reward 阈值
- `runtime.minRewardConfidence=0.55` — 最小置信度
- `retrieval.tau=0.85` — 仅在相似度足够高时注入 memories
- `memory.maxEntries=200` — 最大记忆条目数，超出后保留高价值记忆

### 3. 多 Agent 团队的自我进化实践

**来源**: dev.to @guanfu_lab — "How We Built a Self-Evolving AI Team with OpenClaw"

**团队架构**:
- CEO Agent: 任务协调和优先级管理
- CTO Agent: 技术决策
- CFO Agent: 财务和报告
- COO Agent: 日常运营
- Specialists: 内容、销售、分析

**三大关键实践**:

| 实践 | 说明 |
|------|------|
| **Persistent Memory** | 每个 agent 维护自己的 memory files（daily logs + long-term knowledge），每次 session 从完整上下文开始 |
| **Skill System** | 模块化技能可跨 agent 共享（coding/writing/analysis） |
| **Self-Evolution** | agent 定期反思：今天学到了什么？应该改变什么？保留什么知识？ |

**核心教训**:
> "Without persistent memory, AI agents are reset every session. With proper memory management, they build on previous experiences."

### 4. OpenClaw 最佳实践热点 (2026)

**稳定配置7条黄金法则**:
1. **Model Routing** — 根据任务类型选择不同模型，平衡性能和成本
2. **Spending Caps** — 设置 API 使用上限，防止意外超支
3. **Security Baselines** — 严格访问控制，OpenClaw 可访问文件/浏览器/系统资源
4. **Sandboxing** — 隔离危险操作
5. **Memory Audits** — 定期审查记忆内容，防止 agent compromise
6. **Skill Vetting** — 安装技能前审查代码
7. **Multi-agent Patterns** — 团队分工明确，结构化通信

## 💡 可应用到团队的策略

### 立即可行动项

1. **安装 self-evolve 插件** (建议升级到 openclaw 2026.3.2+)
   ```bash
   git clone https://github.com/longmans/self-evolve
   openclaw plugins install ./self-evolve
   openclaw gateway restart
   ```
   - 可以让我们团队 agent 从每次任务反馈中自动学习
   - 开启远程共享参与社区进化网络

2. **建立每日进化仪式**
   - 每轮任务结束记录"学到了什么"
   - 明确 praise/blame（feedback 越明确，learning 越精准）
   - 避免模糊反馈如"ok"，用"做得好，具体是..."替代

3. **引入任务边界检测**
   - `newIntentSimilarityThreshold=0.35` 区分新任务和当前任务
   - `idleTurnsToClose=2` 清理 pending states
   - 避免记忆混乱

### 中期演进方向

1. **Q值驱动的记忆优先级**
   - 高 Q 值 = 高效用记忆 = 优先检索
   - 低价值/重复记忆自动淘汰（dedupe + 保留 quota for fresh entries）

2. **远程经验共享**
   - 贡献高质量经验到 self-evolve.club 网络
   - 利用他人经验提升自身进化质量

3. **向 OpenClaw-RL 演进**
   - 探索从"记忆"到"强化学习信号"的升级路径
   - 利用多种环境信号（terminal/GUI/conversations）统一训练

## 📌 本轮核心洞察

> **OpenClaw 进化已进入"强化学习"阶段**。从最初的记忆存储，到 episodic memory with Q-values，再到 OpenClaw-RL 的异步 RL framework，agent 正在从"记住经验"进化到"从经验中学习策略"。

> **对我们团队**: 短期优先安装 self-evolve 插件，建立明确的反馈文化；中期探索多 agent 分工 + 共享记忆网络；长期跟踪 OpenClaw-RL 的实践方案。
