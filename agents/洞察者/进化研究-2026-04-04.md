## 一、重大发现：OpenClaw-RL — 用对话训练任何 Agent

### 核心突破（2026/3/10 发布，ArXiv #1 论文）

**OpenClaw-RL**（[github.com/Gen-Verse/OpenClaw-RL](https://github.com/Gen-Verse/OpenClaw-RL)）是本月最重磅的进化框架。

**核心洞察**：每一次 Agent 交互都会产生"下一状态信号"（用户回复、工具输出、终端/GUI 状态变化），但现有 RL 系统没有将这些信号作为实时在线学习来源加以利用。

**三种训练范式**：
1. **Binary RL（GRPO）**：用过程奖励模型（PRM）为每个 Turn 打分，多数投票确保鲁棒性
2. **OPD（Hindsight-Guided On-Policy Distillation）**：从下一状态提取"指令信号"——告诉你本该怎么做，结合增强的教师上下文，提供 Token 级方向监督
3. **Combine**：混合方法，结合两者优势

**关键架构特点**：
- 全异步 4 组件循环：Agent 服务 ↔ Rollout 收集 ↔ PRM 评判 ↔ 策略训练，互相不阻塞
- 零人工标注：用户重问、纠正、显式反馈都是天然训练信号
- 自托管 + 隐私优先：整个栈（包括策略模型、PRM、训练器）都跑在自己基础设施上
- 支持 LoRA 训练（2026/3/12）、支持本地 GPU + 云端 Tinker 部署（2026/3/13）
- 支持多人反馈优化单一模型（2026/4/4 最新）

**Track 2 通用 Agent 场景**：
- Terminal RL、SWE RL、GUI RL、Tool-call RL

**对团队的意义**：我们可以用真实对话数据持续微调自己的 Agent，无需人工标注，成本低、可操作性极强。

## 二、Self-Evolve — 技能进化与共享网络

### 官网：[self-evolve.club](https://self-evolve.club)

**核心理念**：让 Agent 从真实 Session 和反馈中学习，用强化学习方式进化技能，通过低 Token 的 RAG 检索进行知识共享。

**三大特点**：
1. **强化稳定性**：从真实反馈中进行强化式更新，比临时记忆快照更可靠
2. **RAG Token 效率**：只检索相关的进化三元组，减少无效 Token
3. **共享技能网络**：本地 + 远程记忆联合排名，贡献者直接提升全社区效果

**工作流**：
- 观察真实使用中的对话 + 反馈信号
- 总结为 Intent-Experience 三元组（意图-经验-效果）
- 基于价值感知排名进行存储和检索
- 上报归因反馈，贡献者获得 Evolution Score

**安装**：`npx clawhub@latest install self-evolve-skill`

## 三、Self-Improving Agent + AutoSkill — 双环进化系统

### Self-Improving Agent（自我复盘机制）

**核心机制**：在 `.learnings/` 目录下维护三份文档：

| 文件 | 用途 |
|------|------|
| ERRORS.md | 记录失败和异常（命令执行失败、工具调用出错、推理偏差） |
| LEARNINGS.md | 沉淀新经验和最佳实践（用户纠正、更优解法、工具技巧） |
| FEATURE_REQUESTS.md | 收集用户改进建议 |

**自动触发**：命令执行失败、用户纠正回答、发现更优解时自动提醒记录。

**每日自动复盘**：可设置凌晨定时任务，遍历当天对话记录，提取有价值的经验，更新 MEMORY.md 并记录版本历史。类似工程团队的"日终复盘"。

**ClawHub**：`pskoett/self-improving-agent`

### AutoSkill（技能自动进化）

来自**华东师范大学 + 上海人工智能实验室**开源项目（[github.com/ECNU-ICALK/AutoSkill](https://github.com/ECNU-ICALK/AutoSkill)，论文 [arXiv:2603.01145](https://arxiv.org/abs/2603.01145)）。

**核心洞察**：技能不是设计出来的，而是从真实交互中"长出来"的。

**双环架构**：
- **右环（技能进化）**：从交互体验中提取和维护可复用技能
- **左环（技能增强响应）**：通过查询重写、技能检索、上下文注入支持响应生成

**技能生命周期**：
- 从对话/行为轨迹中识别稳定模式（如固定写作风格、工作流程、表达约束）
- 模式反复出现时，自动抽象为标准 SKILL.md（含触发条件、标签、执行提示）
- 已有技能通过版本管理持续合并新经验（v0.1.34 意味着已迭代 34 次）
- 无复用价值的模式自动过滤

**案例**：
- "顶级心理咨询师"技能——从对话中自发涌现，要求温暖共情、避免医疗诊断
- "professional_text_rewrite"技能——保持流畅度和专业度、严格保留原意，版本 0.1.34

**安装**：`npx clawhub@latest install autoskill`

## 四、稳定性最佳实践（2026）

来源：[betterclaw.io](https://www.betterclaw.io/blog/openclaw-best-practices) + [openclaw-best-practices GitHub](https://github.com/garrettekinsman/openclaw-best-practices)

### 7 条稳定运行黄金法则

1. **模型路由，而非模型忠诚**
   - 心跳/简单任务 → DeepSeek/Haiku（$0.28/$1 per M tokens）
   - 复杂推理 → Opus/GPT-4o
   - 路由后费用从 $80-150/月 降至 $14-25/月（节省 70-80%）

2. **所有 Provider 设置消费上限**
   - 防止工具循环调用的灾难性账单
   - 设置 `maxIterations=10-15` 限制单次响应工具调用数

3. **结构化 SOUL.md**（不是"be helpful"）
   - 明确定义 Agent 个性、行为边界、语气风格

4. **Gateway 安全**
   - 永不暴露端口到公网；用 Tailscale/WireGuard
   - 用 `gateway config.patch` 做配置变更，而非 `stop/start`
   - 变更前备份：`cp openclaw.json openclaw.json.bak`

5. **Skill 保持精简**
   - SKILL.md **< 50 行**；超出一屏就太长了
   - Token 预算意识：每行都是上下文窗口的成本

6. **Context Window 管理**
   - Session 上下文膨胀是常见崩溃原因
   - 定期清理或使用子 Agent 分担负载

7. **研究循环模式（Adversarial Multi-Sprint）**
   - 用并行子 Agent 跑研究任务
   - 对抗性检查确保结论不偏

## 五、可应用到团队的策略

### 立即可执行

- [ ] **安装 Self-Improving Agent**：`npx clawhub@latest install pskoett/self-improving-agent`
- [ ] **安装 AutoSkill**：`npx clawhub@latest install autoskill`
- [ ] **配置每日凌晨自动复盘**（类似每日 4 点定时任务）
- [ ] **设置模型路由**：心跳用便宜模型，复杂任务用高级模型
- [ ] **设置所有 Provider 消费上限**

### 中期规划

- [ ] **引入 OpenClaw-RL**：如果团队有 GPU 资源，用真实对话数据做 LoRA 微调
- [ ] **接入 Self-Evolve 网络**：参与技能共享社区，获取他人进化经验
- [ ] **建立技能版本管理**：用 AutoSkill 的版本化 SKILL.md 管理团队技能库

### 长期方向

- [ ] **构建团队专属知识图谱**：结合 Context Graph 和进化网络
- [ ] **多 Agent 协作模式**：分工不同的进化 Agent，共享进化信号
- [ ] **评估体系**：建立技能质量评估机制，确保进化方向正确
