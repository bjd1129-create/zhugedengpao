### 3.2 分层记忆架构（config.yaml）
```
L1: working_memory    → 最近50轮对话（内存）
L2: summary           → 关闭（性能优先）
L3: vector_index      → sqlite-vec，每次检索10条
L4: archive           → 100轮后压缩归档
```

**关键配置**：
- `working_memory.max_turns: 50`（性能优先）
- `summary.enabled: false`（关闭摘要，保留完整历史）
- `vector_index.top_k: 10`（每次注入10条高相关记忆）
- `vector_index.full_document_load: true`（加载完整文档而非摘要）

## 五、自我进化体系架构梳理

### 5.1 当前进化环路现状
```
[用户反馈]
    ↓
[Self-Evolve Plugin] → episodic-memory.json (Q值triplet)
    ↓
[Self-Evolve Club 远程] → 贡献经验/复用他人经验
    ↓
[Skill-Evolution] → 优化SKILL.md
    ↓
[Capability-Evolver-Pro] → 高级能力进化
    ↓
[AGENTS.md/SOUL.md] → 永久性进化
```

### 5.2 关键断点分析

**断点 #1：Self-Evolve Plugin 未激活**
- `buffer.md` 和 `episodic-memory.json` 都是空的
- 说明：插件从未真正写入任何经验
- 可能原因：Gateway 需要以插件模式重启才能激活 self-evolve Q值学习

**断点 #2：经验晋升链路断裂**
- lessons-learned.md 格式已建立（Thoth格式）
- 但晋升阈值（3次→SOUL.md）从未触发
- 没有定期复盘 cron 把低层经验推向高层

**断点 #3：多进化工具未整合**
- self-evolve（Q值）+ skill-evolution（SKILL优化）+ capability-evolver-pro（高级）三个工具并存
- 没有明确分工和协同机制
- 可能造成经验重复或相互覆盖
