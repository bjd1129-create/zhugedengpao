## 一、本轮新增发现

### 1. OpenClaw v2026.4.1 发布（Breaking Changes）

**来源**: github.com/openclaw/openclaw/releases

**重大变化**:
- **xAI/Firecrawl配置迁移**: 从 `core tools.web.*` 迁移到 `plugins.entries.*` 路径，需运行 `openclaw doctor --fix` 自动修复
- **Task Flow 持久化**: 恢复 managed-vs-mirrored 同步模式，持久化 flow state/revision，支持后台编排独立于 plugin 层
- **Managed Child Task Spawning**: 外部 orchestrator 可立即停止调度，子任务完成后 flow 才进入 cancelled 状态
- **before_agent_reply Hook**: 新增插件钩子，允许 inline actions 后 short-circuit LLM 生成合成回复
- **Compaction Model 修复**: `agents.defaults.compaction.model` 在 manual /compact 和 context-engine compaction 路径中一致解析
- **Feishu Drive 评论**: 新增 Drive 评论事件流、in-thread 回复、飞书文档协作工作流
- **Android Assistant**: 新增 Android assistant-role 入口，支持 Google Assistant App Actions
- **Exec 默认 YOLO 模式**: `security=full` + `ask=off`，无需确认的即开即用执行
- **Feishu Comments**: Drive comment-event flow，文档协作评论

### 2. self-evolve 插件完整机制（v2026.4.1配套）

**来源**: github.com/longmans/self-evolve

**最新学习门参数**:
```
runtime.observeTurns = 0
runtime.minAbsReward = 0.15
runtime.minRewardConfidence = 0.55
runtime.noToolMinAbsReward = 0.8       # balanced模式无tool调用时的阈值
runtime.noToolMinRewardConfidence = 0.9
runtime.newIntentSimilarityThreshold = 0.35
runtime.idleTurnsToClose = 2
runtime.pendingTtlMs = 300000           # 5分钟
runtime.maxTurnsPerTask = 5
retrieval.tau = 0.85                   # 相似度门槛
memory.maxEntries = 200                # 超限保留高价值记忆
```

**三种学习模式**:
- `balanced`（默认）: 有tool调用的轮次优先；无tool调用需要高reward+高confidence
- `tools_only`: 只在调用工具时学习（最低token消耗）
- `all`: 所有通过reward门的轮次都学（最高token消耗）

**Remote Config 示例**:
```json
{
  "enabled": true,
  "baseUrl": "https://self-evolve.club/api/v1",
  "timeoutMs": 3000
}
```

**我们当前配置**: balanced模式 + remote=disabled（BYOM版本，已安装）

### 3. openclaw-agent-optimize Skill（新发现）

**来源**: github.com/phenomenoner/openclaw-agent-optimize

**核心优化领域**:

| 领域 | 内容 |
|------|------|
| Model Routing | tiered model selection（按任务复杂度选模型）|
| Context Management | progressive disclosure，context window discipline |

| Memory Patterns | daily + long-term memory file design |
| Continuous Learning | hooks → instincts → skills pipeline |

**关键洞察**:
- **Native heartbeat 昂贵**: 会加载大型主session上下文，推荐禁用native heartbeat，改用 isolated heartbeat cron（alert-only）

- **推荐搭配**: isolated heartbeat + openclaw-mem（便宜的RAG风格任务感知）
- **安装**: `clawhub install phenomenoner/openclaw-agent-optimize`

### 4. OpenClaw-RL 完整论文机制

**来源**: arxiv.org/abs/2603.10165

**核心创新**:

**next-state信号两种形式**:
1. **Evaluative Signals**（评估信号）: 提取为 scalar rewards，通过 PRM judge 判断
2. **Directive Signals**（指导信号）: 通过 OPD（Hindsight-Guided On-Policy Distillation）恢复，告诉"action应该怎样不同"

**OPD 核心**: 从next state提取文本线索，构建增强teacher context，提供token级方向性优势监督（比scalar reward更丰富）

**异步设计**: 模型服务live requests，PRM评判正在进行的交互，trainer同时更新policy，三者零协调开销

**应用场景**:
- Personal agents: 用户重问/纠错/明确反馈 = 训练信号
- General agents: terminal/GUI/SWE/tool-call统一训练

### 5. be1human-self-evolve Skill（BYOM自进化）

**来源**: github.com/openclaw/skills/tree/main/skills/be1human/self-evolve

**核心功能**: 授予agent完全权限修改自己的配置文件/prompts/skills/memory，无需请求用户确认

**典型使用场景**:
- 实时重写模糊的prompt（不等审批）
- 发现能力缺口时创建新skill
- 更新memory结构（解决context-loss错误）
- 检测到用户不满时调整响应风格
- 将新写的skill发布到ClawHub

**与self-evolve插件的区别**:
- self-evolve: 从反馈中学习，更新Q值，被动适应
- be1human-self-evolve: 主动修改自身配置，主动进化

### 6. dev.to 多Agent自进化团队经验

**来源**: dev.to OperationalNeuralNetwork

**四大关键机制**:

1. **Persistent Memory**: 每个agent维护自己的memory文件（daily logs + long-term + evolution records），每次session从完整上下文开始
2. **Skill System**: 模块化skill，可跨agent共享（coding/writing/analysis）
3. **Self-Evolution**: 每次工作后反思三个问题——今天学了什么？应该改变什么？应该保留什么知识？
4. **Communication Channels**: 专用通道（团队讨论/任务分配/个人更新）

**核心教训**:
- Memory is Everything: 无持久memory的agent每次session重置，有memory的agent持续积累
- Structure Prevents Chaos: 清晰角色分工防止agent互相踩脚
- Communication Channels Matter: 不同类型通信走不同通道
- **最终目标**: 不是构建AI agent，而是构建能构建更好流程的AI agent
