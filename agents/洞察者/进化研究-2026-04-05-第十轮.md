## 一、核心新发现

### 1. Capability Evolver（35K+下载）— OpenClaw 下载量第一技能 ⭐⭐⭐

**定位**：元技能（meta-skill）——让Agent自主分析自身运行历史、识别失败/低效点，并自动编写新代码或更新记忆来改进性能。

**核心功能**：
- **自动日志分析**：扫描记忆文件和历史记录，识别错误和模式
- **自主代码生成**：发现问题后自动写workaround或更新Prompt
- **GEP协议**（Guided Evolution Protocol）：标准化自我改进流程，防止Agent行为失控
- **双轨执行**：先尝试第一路径，失败后自动切换第二路径，然后基于经验改进未来Prompt

**使用方式**：
- 手动：`/evolve` 命令（在聊天中触发）
- 自动：`claw config capability-evolver --auto-optimize=true --interval=24h`（每24小时自动运行）

**为什么重要**：大多数技能只是增加单一能力；Capability Evolver让所有其他技能随时间运作得更好——通过持续从失败中学习。下载量35K+，超过self-evolve（10K级别），是ClawHub最受欢迎的技能。

**安装**：`npx clawhub@latest install capability-evolver`（或 `claw install capability-evolver`）

**解决的问题**：之前技能用松散JSON Schema，运行时才发现错误。

**新机制**（Prism v3.0）：
```json
{
  "prism_version": "3.0",
  "skill_id": "finance.trading.limit_order",
  "inputs": {
    "symbol": {"type": "string", "pattern": "^[A-Z]{1,5}$"},
    "amount": {"type": "number", "minimum": 0.01}
  },
  "outputs": {
    "order_id": {"type": "string"},
    "status": {"enum": ["pending", "filled", "failed"]}

  "side_effects": ["network.write", "filesystem.append"]
}
```

**价值**：
- 编译时验证 + 静态类型检查

### 5. OpenClaw-RL 深度解读（arXiv:2603.10165）

**核心论文**：Princeton团队，2026-03-10发布

**突破性洞察**：
> 每个Agent交互都会生成"下一状态信号"——即每个动作之后的用户回复/工具输出/终端/GUI状态变化。**下一状态信号 = 通用训练信号**，不需要独立离线数据集。

**三种训练范式**：
1. **Binary RL**：标量奖励（0/1）
2. **OPD**（Outcome Processing with Distillation）：不只是告诉"对/错"，而是恢复"应该怎么做不同"——token级方向性监督
3. **Combine**：两者结合

**异步设计**：模型服务 + PRM评判 + 训练器三者零协调开销

**意义**：对话本身就是训练数据——不需要传统RLHF的标注流水线。对话驱动的训练信号让迭代Agent改进门槛大幅降低。

### 9. OpenClaw Pulse — 自我改进研究代理工作流

**设计思路**：每天早8:30自动研究OpenClaw工程博客/Hacker News/GitHub Trending，对比自身AGENTS.md/TOOLS.md/LESSONS.md，识别差距和改进点。

**输出格式**：
```json
{
  "date": "2026-02-17",
  "source": "Simon Willison",
  "title": "How to think about tool calling in LLMs",
  "relevance_score": 9,
  "summary": "...",
  "proposed_change": "更新TOOLS.md添加failure_modes section",
  "status": "pending_review"
}
```

**关键设计**：Agent不自动修改自身文件——只提出change，人类审批后再执行。这是对"自我修改"风险的对冲。
