## 一、OpenClaw 2026.3 核心更新（重点！）

版本号 v2026.3.7-beta.1 单次更新修复 **200+ bug**，官方 CHANGELOG 近 15000 字。

### 1.1 ContextEngine（上下文引擎）升级 ⭐
- 旧版：喂多少吃多少，上下文爆炸
- 新版："智能修剪"机制，自动识别当前任务所需信息，暂时搁置其余
- 实测：50轮复杂对话，context 使用量减少约 **30%**
- 意义：长任务稳定性大幅提升

### 1.2 /btw Sidebar Q&A（边栏问答）⭐
- 场景：长任务中途想问无关问题（如写个正则），之前要开新 session
- 新功能：输入 `/btw 你的问题`，在"边栏"回答，不影响主任务上下文
- 回答完自动切回主任务
- **解决痛点**：如何"中断"长任务但不破坏上下文

### 1.3 Pluggable Sandbox Backends（可插拔沙箱后端）
三种沙箱后端可选：

| 类型 | 特点 | 适用场景 |
|------|------|---------|
| Docker | 强隔离 | 生产环境 |
| OpenShell Mirror | 本地 shell 直连，轻量，内存是 Docker 一半 | Mac/本地开发测试 |
| OpenShell Remote | 连接远程服务器 shell | 团队协作/统一环境 |
| SSH Sandbox | 原生 SSH，无 OpenClaw 专有协议 | 已有 SSH 基础设施 |

Mac 用户推荐 OpenShell Mirror 模式，不装 Docker 也能跑。

### 1.4 Firecrawl 集成（网页抓取）
- 旧版：curl + 解析 HTML，遇到反爬/SPA 直接失败
- 新版：集成 Firecrawl，支持 JS 渲染、自动分页、结构化 Markdown 输出
- 实测：React 文档类 SPA 页面，之前要多轮 prompt 才能拿到干净内容，现在一步到位

### 1.5 Secrets Workflow（密钥管理）
完整 CRUD，支持加密本地存储（~/.openclaw/secrets/），Git 操作自动忽略。

```bash
/secrets set OPENAI_API_KEY "sk-xxx"
/secrets list      # 只显示名称，不显示值
/secrets delete OPENAI_API_KEY
```

### 1.6 Plugin System 三层架构重构（最大变化）
- **旧版**：Skill 单层，不同 Skill 可能依赖冲突，迁移模型要改一堆配置
- **新版**：Bundle + Provider + Plugin 三层架构
  - Bundle = 技能包集合
  - Provider = 模型接入层
  - Plugin = 功能模块
- 迁移模型只需换 Provider，核心逻辑不变

### 1.7 人类在环工作流（Human-in-the-Loop）
- 从"全自动化"转向"AI 暂停问人"模式
- AI 在关键步骤暂停问"这样做对吗？"再继续
- 适用场景：敏感操作、外部 API 调用、生产环境

- **Leaderboard 上线**：实时贡献者排行榜（Evolution Score = Reuse Hits + Quality Reward）
- **安装方式**：`npx clawhub@latest install self-evolve-skill`（我们的版本应该已经装了）

### 2.3 Self-Evolution Pro v1.0.0 新特性（March 26, 2026）
windy-001-crypto 发布，重大升级：
- **自动技能提取**：从对话中自动抽取可复用技能
- **多维根因分析（RCA）**：不只是记录，还分析为什么
- **知识图谱集成**：将学习、错误、经验链接成网络
- **进化追踪**：记录每次进化轨迹

### 2.4 Self-Evolve 运作机制（核心原理）
1. **观察**：从真实对话和反馈信号中观察
2. **总结**：将可复用经验提炼为"意图-经验三元组"
3. **存储+检索**：基于 Q 值排序（本地+远程结合）
4. **归因反馈**：顶级贡献者获得 Quality Reward

**Q 值学习核心**：
- 每条经验分配一个 Q 值（效用评分）
- Q 值随使用次数和反馈质量更新
- 高 Q 值经验优先被召回
- 三重隐私保护：本地优先，敏感数据不外传

## 三、OpenClaw 安全形势（重要提醒）

### 3.1 CVE-2026-25253
- **影响规模**：135,000+ 暴露实例
- **漏洞类型**：RCE（远程代码执行）
- **根因**：默认配置优先便利性而非安全性
- **恶意 Skills**：ClawHub 市场中有数百个被污染的 skills

### 3.2 安全最佳实践（来自 Bastion.tech）
1. **路由 + Gateway Token**：基础访问控制
2. **allowFrom 严格限制**：只允许受信任的用户/会话
3. **插件来源审计**：10% 的 ClawHub skills 已被污染，安装前审计
4. **禁止生产环境暴露**：不要将 OpenClaw 实例直接暴露在公网
5. **Secrets 加密存储**：用内置 Secrets workflow 不用明文环境变量
6. **定期更新**：OpenClaw 2026.3+ 有重大安全修复
7. **Human-in-the-Loop**：敏感操作开启暂停确认

### 3.3 我们的安全状态
- ✅ allowFrom 只接受老庄 open_id
- ✅ Secrets 在 .env 文件（.gitignore 保护）
- ✅ self-evolve 远程共享开启（balanced 模式，有隐私保护）
- ✅ 分支保护策略已建立
- ⚠️ OpenClaw 版本需确认是否 >= 2026.3（建议升级）
