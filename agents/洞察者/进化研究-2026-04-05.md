## 一、Self-Evolve（强化学习引擎）

### 工作原理
```
用户消息 → 反馈检测？ 
  ├─ 是 → 评分reward → 检查学习门 → 记忆清洗 → LLM总结 → 写入情景记忆
  └─ 否 → 意图检测 → 检索本地+远程 → Phase-B排名 → 注入记忆生成回复
```

### 三大学习模式
| 模式 | 描述 | Token消耗 |
|------|------|-----------|
| **balanced**（默认） | 偏好工具调用；无工具需高reward | 中 |
| **tools_only** | 只学习调用工具的轮次 | 低 |
| **all** | 学习所有通过reward门的轮次 | 高 |

### 关键参数
- `runtime.minAbsReward=0.15` — 最小绝对reward
- `runtime.minRewardConfidence=0.55` — 最小置信度
- `retrieval.tau=0.85` — 只注入高相似度记忆
- `memory.maxEntries=200` — 最大记忆条目

### 远程共享网络
- 本地+远程记忆合并排序
- 隐私设计：本地清洗 → LLM二次清洗 → 脱敏共享
- 可视化贡献排行榜：self-evolve.club/#leaderboard

**安装**：
```bash
git clone https://github.com/longmans/self-evolve
openclaw plugins install ./self-evolve
openclaw gateway restart
```

## 二、Self-Improving Agent（复盘机制）

### 三大文档
- **ERRORS.md** — 记录失败：命令执行失败、工具出错、推理偏差
- **LEARNINGS.md** — 沉淀经验：用户纠正、更优方案、工具技巧
- **FEATURE_REQUESTS.md** — 收集改进建议

### 每日自动复盘
- 定时任务：每天凌晨4点执行
- 检查当天对话记录，提取经验
- 更新核心记忆库 + 版本历史
- 等同于工程团队的"日终复盘"

**ClawHub地址**：https://clawhub.ai/pskoett/self-improving-agent

## 三、AutoSkill（技能孵化）

### 核心理念
> "技能不是设计出来的，而是长出来的"

### 工作流程
```
摄取经验（对话+行为+反馈）
    ↓
提炼技能（识别可复用模式）
    ↓
复用能力（技能库维护+版本管理）
```

### 技能版本化
- 0.1.34 意味着该技能已迭代34次
- 新约束/反馈合并到原技能，而非创建新技能
- 技能格式：SKILL.md（触发条件+标签+执行提示）

### 案例
- **顶级心理咨询师**：从对话中总结，温暖+共情+专业语气，避免医疗诊断
- **professional_text_rewrite**：保持流畅度和专业度，严格保留原意

**开源项目**：https://github.com/ECNU-ICALK/AutoSkill

## 五、团队应用建议

### 可立即实践
1. **安装 Self-Evolve** — 最完整的强化学习记忆系统
2. **启用 Self-Improving Agent** — 建立错误记录和复盘机制
3. **开启远程共享** — 贡献经验，获取社区集体智慧

### 适合团队特性
- 小花已有 `memory/` 日志体系和 `MEMORY.md` 长期记忆
- Self-Evolve 的 Q值学习可增强现有记忆体系
- 建议优先测试 Self-Improving Agent 的每日复盘功能

### 潜在风险
- Token消耗：all模式成本高，建议从balanced或tools_only开始
- 隐私：远程共享需确认敏感信息已清洗
- 版本兼容：建议升级到 openclaw 2026.3.2+
