## 🔬 重大发现一：self-evolve 插件（longmans/self-evolve）

GitHub: `github.com/longmans/self-evolve`

这是目前最成熟的**基于反馈的自我学习插件**，核心机制：

### 架构设计
```
用户消息 → 反馈检测？→ [是] 评分+学习门检查 → 双重脱敏 → 写入情景记忆
                    → [否] 意图识别 → 记忆召回 → 注入上下文 → 回复
```

### 三大学习模式
| 模式 | 说明 | Token 成本 |
|------|------|-----------|
| `balanced`（默认） | 优先工具调用轮次；无工具轮次需高奖励/置信度 | 中 |
| `tools_only` | 仅在学习工具被调用时（最低成本） | 低 |
| `all` | 所有通过奖励门的轮次都学习（最高成本） | 高 |

### 关键参数（学习门）
- `minAbsReward = 0.15` — 最小绝对奖励值
- `minRewardConfidence = 0.55` — 最小奖励置信度
- `noToolMinAbsReward = 0.8`（balanced模式无工具调用时）
- `tau = 0.85` — 检索相似度阈值（只注入高相关性记忆）

### 远程共享记忆
- 默认开启 remote，连接 `self-evolve.club`
- 本地+远程记忆合并后统一 ranking
- 隐私设计：双重 LLM 脱敏（本地+摘要层）
- 用户可查看贡献排行榜

### 反馈最佳实践
- ✅ 明确 Praise（正向强化）
- ✅ 明确 Point out failure（负向强化）
- ❌ 避免模糊 "ok" 类反馈

### 建议本团队采纳
**将 self-evolve 插件纳入工作流**，特别是反馈激励机制。团队目前使用 capability-evolver-pro 做分析，self-evolve 做实时学习，两者可以互补。

## 🔬 重大发现二：Capability Evolver（元技能）

多个版本存在于社区：
- `autogame-17/capability-evolver`（GitHub 官方 skills）
- `claw0x/capability-evolver-pro`（本地已安装，确定性分析引擎）

### capability-evolver-pro 特点（本地版本）
- **纯确定性**：无 LLM，无外部依赖，<100ms 处理，零 token 消耗
- **分析维度**：PCEC 执行、memory 使用、子 Agent 活跃度、响应质量、技能树状态
- **告警阈值**：PCEC 超 2h、子 Agent 超 5 个、错误率 > 20%

### 官方 capability-evolver（GitHub 版）
- 支持 GEP/A2A 协议
- 运行时历史检测 + 自动修复建议
- 支持回滚模式和进化策略配置

### 建议本团队采纳
**已领先** — 本团队已安装 `capability-evolver-pro` 和 `self-health-monitor`，这是最完整的自监控组合。建议确保 self-health-monitor 每 30 分钟触发并正常输出报告。

## 🔬 重大发现三：自我改进 Cron Job 模式（openclawpulse.com）

每天 8:30 工作日定时任务，完整流程：

1. 搜索 Anthropic 工程博客、Simon Willison 博客
2. 扫描 HN（关键词：openclaw、ai agent、prompt injection 等，min 50 points）
3. 检查 GitHub Trending（TypeScript/Python/Go，主题：ai-agent、llm、mcp）
4. 读取自身 AGENTS.md、TOOLS.md、LESSONS.md
5. 识别差距和改进点
6. 记录到 `experiments/improvements.json`
7. 可选：推送 Telegram 早间简报

### 关键设计原则
- **不自改文件**：agent 提出，人类审核后再改（安全护栏）
- **研究而非总结**：只记录"可执行的改进"，不堆砌信息
- **SOUL.md 约束**：禁止给自己增加权限（防 prompt injection）

### 建议本团队采纳
**值得引入** — 建立每周固定的自进化研究任务（可与现有 cron 融合），输入源增加中文技术博客（如稀土掘金、OSCHINA）和国内 AI 动态。

## 🔬 发现四：OpenClaw 2026 最新架构（13,700+ skills，335k GitHub stars）

### 关键数字
- **GitHub stars**: 335,000+
- **社区 skills**: 13,700+
- **消息平台支持**: 20+
- **历史名称**: Clawdbot → Moltbot → OpenClaw（2025年11月启动）

### 最佳实践 2026 版（openclaw-ai.com）
1. **模型分级使用**：简单问答用 GPT-5-3-Codex，复杂推理用 Claude-4-5-Sonnet
2. **缓存启用**：`caching.enabled: true`，TTL 3600s
3. **成本管控**：设置 `messagesPerMinute/ maxTokensPerDay / monthlyBudget`
4. **安全**：使用 .env 文件存储密钥，不用硬编码
5. **健康检查**：`healthCheck.interval: 60s`，自动恢复 `autoRestart: true`
