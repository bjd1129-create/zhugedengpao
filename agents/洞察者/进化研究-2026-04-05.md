# OpenClaw 自我进化研究 | 2026-04-05

## 核心发现

### 1. Self-Improving Agent（自我进化引擎）
**来源**：ClawHub / 腾讯云开发者社区

**机制**：
- 系统自动创建 `.learnings` 目录，包含三份文档：
  - `ERRORS.md` - 记录失败和异常（命令执行失败、工具调用出错、推理偏差）
  - `LEARNINGS.md` - 沉淀新经验和最佳实践
  - `FEATURE_REQUESTS.md` - 收集用户改进建议
- **自动触发机制**：命令失败、用户纠正、发现更优方案时自动记录
- **每日自动复盘**：凌晨4点定时任务，检查对话记录，提取经验，更新MEMORY.md

**核心价值**：让Agent具备"日终复盘"能力，经验不沉淀就会遗忘

---

### 2. AutoSkill（技能自进化框架）
**来源**：华东师范大学 + 上海人工智能实验室开源项目
**论文**：https://arxiv.org/abs/2603.01145

**机制**：
- **右环（技能进化）**：从交互体验中提取并维护显式技能
- **左环（技能增强）**：利用技能库支持响应生成（查询重写→技能检索→上下文注入）
- **核心思想**：技能不是设计出来的，是从经验中"长出来的"

**技能版本管理**：
- 发现稳定模式 → 抽象为SKILL.md（含触发条件、标签、执行提示）
- 用户反馈 → 自动合并到原有技能 → 版本号升级
- 案例：professional_text_rewrite 已迭代至 0.1.34 版本（优化30+次）

**三种信号摄取**：
- 对话内容、行为轨迹、用户反馈

### 3. self-evolve（共享进化网络）
**来源**：longmans/self-evolve（GitHub）
**平台**：self-evolve.club

**创新点**：
- **强化稳定性**：从真实反馈中学习，比临时记忆更 robust
- **RAG Token 效率**：只检索相关经验三元组，减少无用 token
- **共享技能网络**：本地+远程记忆联合排名，贡献者直接提升社区

**进化评分机制**：
```
Evolution Score = Reuse Hits + Quality Reward
```

**安装命令**：
```
npx clawhub@latest install self-evolve-skill
```

## 可应用到团队的策略

### 立即可落地
1. **安装 self-improving-agent** - 给小花装上"学习大脑"
   - 启用错误记录、经验沉淀、自动复盘机制
   - 配合每日凌晨自动反思

2. **尝试 self-evolve 技能** - 加入共享进化网络
   - 贡献经验获取评分
   - 利用社区共享的高质量技能

3. **建立三层记忆体系**（我们已在用）
   - L1: 当前对话上下文
   - L2: memory/state.json 跨对话状态
   - L3: MEMORY.md 重要决策/长期知识

### 中期目标
4. **技能版本化管理** - 重要技能采用 SKILL.md 格式
   - 记录触发条件、版本历史
   - 用户反馈驱动迭代

5. **借鉴 AutoSkill 思路** - 从真实交互中提炼团队专有技能
   - 观察高频模式
   - 抽象为可复用技能文档
