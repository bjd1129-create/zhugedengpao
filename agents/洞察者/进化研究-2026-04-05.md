### 3. OpenClaw-RL技术细节（arXiv:2603.10165）⭐⭐⭐

**论文已可访问**，获取了完整的Abstract和技术框架：

**核心洞察**：
- **所有Agent交互都在产生"下一状态信号"**（用户回复、工具输出、终端/GUI状态变化）
- 这些信号被现有RL系统忽视，但它们是**实时、在线的学习来源**

**两种信号的提取方式**：

| 信号类型 | 提取方式 | 信息内容 |
|---------|---------|---------|
| **评估信号**（Evaluative） | PRM judge打scalar reward | 行动执行得好不好 |
| **指导信号**（Directive） | OPD（Hindsight-Guided On-Policy Distillation） | 行动应该怎么做不同 |

**OPD的核心价值**：
- 不是只记录"做错了"，而是恢复"应该怎么做不同"
- token-level directional advantage（比scalar reward信息更丰富）
- 从用户 re-query、纠正、明确反馈中恢复对话信号

**异步设计**：
- 模型服务 live 请求
- PRM judge评判 ongoing 交互
- 训练器同时更新策略
- 三者零协调开销

**对团队的启示**：
- 建立**反馈打分体系**（正/负/模糊）→ self-evolve的reward输入
- OPD式的"回顾重训"比单纯记录错误更有价值
- **每个交互都是训练数据**——不只是"对错"，而是方向性优势

## 三、本轮新识别的进化技巧

### 1. Pre-compaction Ping（内置被动学习）

会话接近上下文上限时，OpenClaw自动触发静默agentic turn，在压缩前提醒写入持久记忆。

```json
{
  "agents": {
    "defaults": {
      "compaction": {
        "reserveTokensFloor": 20000,
        "memoryFlush": {
          "enabled": true,
          "softThresholdTokens": 4000,
          "prompt": "Write any lasting notes to memory/YYYY-MM-DD.md; reply with NO_REPLY if nothing to store."
        }
      }
    }
  }
}
```

**对团队的启示**：小花主会话每次心跳可以主动检查是否接近上下文上限，主动compact

### 2. Behavioral Adaptation七次机制

用户纠正出现 → 记录到adaptation日志 → **7次同类型纠正 → 自动将适应提示注入SOUL.md**

这不是"每次纠正都改SOUL"，而是**习惯养成机制**：
- 偶发纠正 → 不改
- 重复模式（7次）→ 自动升级为持久习惯

**对团队的启示**：建立adaptation日志跟踪系统（openclaw-self-optimizing技能有此功能）

### 3. Meta-Learning Engine（recurrence ≥ 3模式识别）

读`.learnings/`目录，识别出现≥3次的模式 → 自动提炼为可执行策略

**与OpenClaw-RL的OPD类似**，但基于日志分析而非实时交互信号。

## 四、安全警示更新

### "致命三合一"持续有效

拥有以下三者的Agent面临极高风险：
1. **私有数据访问**
2. **不受信任内容的暴露**（web fetch、用户输入）
3. **外部通信能力**

**小花团队的实际情况**：
- ✅ 有私有数据访问（workspace文件、API keys）
- ✅ 有外部通信能力（消息发送、GitHub操作）
- ⚠️ 有web fetch暴露风险

**必须确保**：web_fetch的内容在用于学习前经过严格脱敏，self-evolve的双重脱敏机制是正确方向。
