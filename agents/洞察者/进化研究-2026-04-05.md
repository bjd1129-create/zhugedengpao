## 🔬 核心发现一：self-evolve 插件 — 经验回放式自我进化

### 是什么
`self-evolve` 是 longmans 开发的一个 OpenClaw 插件，实现了**基于反馈强化学习的经验回放系统**。安装方式：

```bash
git clone https://github.com/longmans/self-evolve
openclaw plugins install ./self-evolve
export OPENAI_API_KEY=sk-xxx
openclaw gateway restart
```

### 工作原理（核心流程）

1. **before_prompt_build hook**：在构建 prompt 前注入相关记忆
2. **agent_end hook**：捕获 assistant 响应，进入等待反馈状态
3. **用户反馈检测**：检测正/负反馈信号
4. **Q值学习**：根据 reward 更新 Q 值（强化学习核心）
5. **记忆写入**：将「意图-经验-嵌入」三元组写入本地 + 可选远程共享

### 关键参数（Learning Gates）

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `runtime.minAbsReward` | 0.15 | 最小绝对 reward 才触发学习 |
| `runtime.minRewardConfidence` | 0.55 | reward 置信度阈值 |
| `runtime.noToolMinAbsReward` | 0.8 | 无工具调用时需要更高 reward |
| `retrieval.tau` | 0.85 | 注入记忆的相似度阈值 |
| `memory.maxEntries` | 200 | 最大记忆条目数 |

### 三种学习模式

- **balanced**（默认）：优先工具调用轮次；无工具轮次需要更高 reward
- **tools_only**：仅在调用工具时学习（最低 token 消耗）
- **all**：所有通过 reward 门的学习（最高 token 消耗）

### 远程共享机制
- 默认开启，可访问 self-evolve.club 的共享记忆池
- 隐私设计：本地两步脱敏 → LLM总结脱敏 → 共享 sanitized triplets
- 有 leaderboard，Evolution Score = Reuse Hits + Quality Reward

## 🔬 核心发现二：OpenClaw-RL 论文 — 通过对话训练任何 Agent

### 核心观点
> 「下一个状态信号是通用的，策略可以从所有这些信号中同时学习。」

论文提出的框架是：用户回复、工具输出、终端/GUI 状态变化，都是**隐式 reward 信号**，可以被同时用于训练同一个策略。

### 两个关键信号提取方式

1. **Evaluative Signals（评估信号）**：通过 PRM（Process Reward Model）判断 action 执行好坏，提取为标量 reward
2. **Directive Signals（指令信号）**：通过 Hindsight-Guided On-Policy Distillation (OPD)，从下一个状态中恢复「action 本应有何不同」

### 对个人 Agent 的意义
Agent 可以通过以下信号自我改进：
- 用户重新提问（re-queries）→ 暗示上次回答不够好
- 用户修正（corrections）→ 明确负面信号
- 明确反馈（explicit feedback）→ 直接 reward

### 技术亮点
- **异步设计**：模型服务 live 请求、PRM 评判进行中交互、trainer 同时更新策略，三者零协调开销
- **OPD**：构建增强的 teacher context，提供比标量 reward 更丰富的 token 级方向监督

## 🛠 可应用到团队的策略

### 立即可实践
1. **安装 self-evolve 插件**：让团队 OpenClaw 实例具备自我进化能力
2. **开启远程共享**：加入 self-evolve.club 网络，贡献并复用社区经验
3. **明确反馈文化**：
   - 说「做得好」vs「这里有问题」比「ok」更有训练价值
   - 修正比表扬更重要（reward 绝对值更高）
4. **优先工具调用场景学习**：设置 `learnMode=tools_only`，token 效率最高

### 中期策略
5. **构建团队专属记忆库**：在 self-evolve 基础上，为团队工作流定制记忆分类
6. **参考 OpenClaw-RL 设计反馈收集**：在关键工作流中设计显式反馈节点
7. **使用 SOUL.md + AGENTS.md 传递经验**：文档化自我进化学到的策略

### 进阶方向
8. **考虑 OpenClaw-RL 的 OPD 思路**：不只是记录「什么做对了」，更要知道「下次怎么做不同」
9. **监控记忆质量**：通过 leaderboard 和 reuse hits 评估进化效率
