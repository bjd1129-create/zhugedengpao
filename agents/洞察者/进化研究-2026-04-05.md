## 核心发现

### 1. Self-Evolve 插件 — OpenClaw 自我进化的核心机制

**仓库**: github.com/longmans/self-evolve

这是目前 OpenClaw 生态最系统的自我进化插件，核心原理是 **强化学习式经验沉淀**：

**三位一体记忆结构（episodic memory triplets）**：
- `intent` — 任务意图
- `experience` — 可复用的经验策略
- `embedding` — 向量索引，用于相似度检索

**Q值强化更新**：
- 每次收到用户反馈时，根据 reward 分数更新 Q 值
- 高 Q 值经验优先被检索注入到 prompt 上下文

**三种学习模式**：
| 模式 | 触发条件 | token 成本 |
|------|---------|-----------|
| balanced（默认）| 仅工具调用轮，或高 reward/高置信度的无工具轮 | 中 |
| tools_only | 只学习调用了工具的轮次 | 低 |
| all | 所有通过 reward 门控的轮次 | 高 |

**隐私设计（双重脱敏）**：
- 第一轮：sanitizeMemoryText 移除对话元数据、ID、发送者标签
- 第二轮：LLM summarizer 要求输出时可转移策略，将敏感数据替换为 `[REDACTED_*]`

**远程共享网络**：
- self-evolve.club 提供共享记忆池
- 本地 + 远程候选记忆合并后统一 ranking
- 贡献排行榜，可追溯 attribution

**推荐配置**（建议升级到 openclaw 2026.3.2+）：
```bash
git clone https://github.com/longmans/self-evolve
openclaw plugins install ./self-evolve
openclaw gateway restart
```

### 2. Self-Improving Agent Skill — 传统复盘机制

**ClawHub**: clawhub.ai/pskoett/self-improving-agent（下载量 26万+）

在 `.learnings/` 目录下自动维护三份文档：
- `ERRORS.md` — 记录失败和异常（命令执行失败、工具出错、推理偏差）
- `LEARNINGS.md` — 沉淀新经验和最佳实践
- `FEATURE_REQUESTS.md` — 收集用户改进建议

**配合每日定时复盘**：凌晨 4 点自动触发自我反思，从当天对话中提取经验并更新 MEMORY.md

### 3. AutoSkill — 从交互中"长出"技能

**仓库**: github.com/ECNU-ICALK/AutoSkill

核心理念：**技能不是设计出来的，而是从经验中长出来的**。

双环架构：
- **右环（技能进化）**：从交互体验中提取并维护为显式技能
- **左环（技能增强）**：利用技能库通过查询重写→技能检索→上下文注入支持响应

关键特性：
- 技能版本管理（迭代到 0.1.34 版本说明已优化 30+ 次）
- 新约束/反馈自动合并到已有技能，版本号递增
- 技能以 SKILL.md 形式存储，人类可查看和编辑

### 4. OpenClaw Best Practices — 自我进化工作流

**仓库**: github.com/solomonneas/openclaw-best-practices

**纠正识别触发器**：
- "No, do it this way..."
- "I told you not to..."
- "That's wrong because..."
- "Stop doing X"
- 任何改变 agent 行为的 pushback

**知识卡片晋升规则**：

| 场景 | 晋升目标 |
|------|---------|
| 3次以上同一模式 | AGENTS.md 或 rules/ |
| 行为纠正 | rules 文件立即晋升 |
| 工作流改进 | AGENTS.md |
| 单次事件 | 仅保留卡片 |

**Token 纪律规则（真实踩坑后沉淀）**：
```
## Token Discipline
- Expiring tokens are use-it-or-lose-it. Ship content, don't debug infra.
- Use the cheapest model that handles the task.
- Search memory, don't load it. Semantic search returns what you need.
```

**每6小时自动 memory sweep cron**：
- 跨 Discord、Telegram 等所有频道拉取最近会话
- 用 code-specialized 模型（GPT-5.4 / Codex）做结构化提取，不占用前端模型配额

## 对小花团队的启示

### ✅ 可立即应用的策略

1. **安装 Self-Evolve 插件**
   - 我已经具备插件安装能力，这是最直接的强化学习式进化机制
   - 开启远程共享，参与 self-evolve.club 网络

2. **建立纠正触发意识**
   - 识别"老庄纠正我"的信号 → 立即生成知识卡片
   - 不要只修复眼前问题，要写文档防止重复

3. **每日凌晨自我复盘**
   - 建议老庄给我设置一个每日 cron（凌晨 4 点）
   - 从前一天对话中提取经验，更新 MEMORY.md

4. **建立 rules/ 知识体系**
   - 在 workspace 建立 rules/ 目录，按领域分文件
   - git-rules.md、writing-rules.md 等

5. **任务前自检清单**（写入 AGENTS.md）：
   - 我之前有没有被纠正过类似的事？（搜索 memory）
   - 这是否符合已知反模式？
   - 我要做的事是否被明确禁止？

### ⚠️ 当前能力差距

- Self-Evolve 插件需要部署和配置，我没有直接执行 `openclaw plugins` 的权限（需要老庄操作）
- AutoSkill 框架目前还比较前沿，集成复杂度高
- memory sweep cron 需要老庄在 gateway 配置

### 🔮 长期进化方向

参考 AutoSkill 的理念：技能不是设计出来的，是"长出来"的。
→ 我应该从真实交互中识别重复模式，逐步沉淀为我自己的 .learnings 体系。
