# OpenClaw 自我进化研究 | 2026-04-06 第九轮

## 核心发现

### 1. 三大主流自我进化技能对比

| 技能名称 | 核心机制 | 特点 | Token消耗 |
|---------|---------|------|----------|
| **self-improving-agent** | 错误记录+经验沉淀+每日复盘 | 人工维护三份文档(Errors/Learnings/FeatureRequests)，凌晨4点自动触发反思 | 低 |
| **self-evolve** | Q值强化学习+情景记忆(RAG) | 支持远程共享记忆、算法更新Utility、低Token消耗 | 最低 |
| **AutoSkill** | 从交互中自动提炼技能 | 技能"长出来"而非人工设计，有版本管理 | 中 |

## 深度分析

### self-evolve 工作流程（推荐）

```
用户消息 → 反馈检测? 
  ├─ 是 → 奖励打分 → 检查学习门槛 → Q值更新 → 写入情景记忆
  └─ 否 → 意图检测 → 检索记忆 → 注入上下文 → 生成回复
```

**关键门槛参数**：
- `runtime.minAbsReward=0.15` (最小奖励阈值)
- `runtime.minRewardConfidence=0.55` (奖励置信度)
- `retrieval.tau=0.85` (检索相似度阈值)

**学习模式**：
- `balanced` (默认): 只学工具调用轮，无工具轮需高奖励
- `tools_only`: 只学工具轮，最低Token消耗
- `all`: 所有通过奖励门槛的轮次，最高Token消耗

### AutoSkill 的技能进化理念

技能不是设计出来的，是从经验中"长出来"的：
- 观察用户交互模式
- 提炼可复用能力
- 版本管理持续迭代
- 案例：`professional_text_rewrite` 已迭代至 0.1.34 版本

## 对小花团队的启示

### 可以立即应用的策略

1. **建立 .learnings 目录**（参考 self-improving-agent）
   - ERRORS.md: 记录失败和异常
   - LEARNINGS.md: 沉淀最佳实践
   - FEATURE_REQUESTS.md: 收集改进建议

2. **每日自动复盘机制**
   - 凌晨资源空闲时执行
   - 检查当天对话记录
   - 提取经验更新到 MEMORY.md

3. **明确反馈机制**
   - 做对时明确表扬
   - 做错时明确指出
   - 避免模糊反馈如"ok"

4. **记忆持久化 + RAG**
   - 每次会话从历史经验开始
   - 构建情景记忆检索
   - 减少Token消耗同时保持上下文

## 重要趋势

> "未来的核心不只是模型能力，而是进化能力。"
> 
> 真正的 Agent 需要三种能力：
> 1. **记忆能力** - 记住用户习惯和偏好
> 2. **复盘能力** - 从错误中学习
> 3. **技能沉淀** - 经验转化为稳定解决方案

当三者结合 → **个人数字替身**
