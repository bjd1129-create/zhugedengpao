## 一、Capability Evolver Pro（新增重点！）

### 1.1 定位差异
| 对比 | Self-Evolve插件 | Capability Evolver |
|------|----------------|-------------------|
| 原理 | 强化学习Q值更新 | 确定性规则引擎 |
| 依赖 | LLM评分 | 无LLM，纯代码 |
| 速度 | 秒级 | <100ms |
| 幻觉风险 | 有（LLM幻觉） | 无 |
| 隐私 | 差（远程上传） | 优（纯本地） |
| 适用 | 行为适应 | 日志分析 |

**结论**：两者互补！Self-Evolve管行为学习，Capability Evolver管系统健康。

### 1.2 四大分析策略
| 策略 | 适用场景 |
|------|----------|
| `auto` | 不知道用什么，让引擎决定 |
| `balanced` | 稳定系统，兼顾可靠性与新功能 |
| `innovate` | 健康系统，准备扩张 |
| `harden` | 故障频发，优先可靠性 |
| `repair-only` | 危机模式，只修critical |

### 1.3 输入格式（关键！）
```json
{
  "action": "analyze",
  "logs": [
    {
      "timestamp": "2025-01-15T10:00:00Z",
      "level": "error",
      "message": "ETIMEDOUT",
      "context": "payment-api.ts"
    }
  ]
}
```
必须包含：`timestamp`、`level`（error/warn/info/debug）、`message`
可选：`context`（文件/模块名）

### 1.4 应用场景
1. **生产事故**：导出最近1000条日志 → analyze → 找到级联故障根因
2. **发布前检查**：staging日志 vs baseline health score → 低于阈值则block部署
3. **舰队管理**：50+ agent聚合分析 → 找全局共同问题 → 修复一次全量生效
4. **每日cron**：`evolve` action → 每日生成改进建议 → 存入待办

### 1.5 可应用到小花团队
**立即可执行**：给小花的exec日志建一个轻量分析：
- 每次cron运行的exit code记录下来
- 每周用Capability Evolver跑一次
- 发现高频错误模式 → 更新HEARTBEAT.md检查项

## 二、self-evolve-remote-stats（新发现的skill！）

### 2.1 功能
通过curl查询self-evolve.club公开API，无需key：
- `GET /stats/overview` → 全网总体情况
- `GET /stats/leaderboard?limit=10` → 排行榜

需要key的（读取本地plugin配置）：
- `GET /stats/me` → 个人排名信息
- `POST /users/username` → 设置用户名

### 2.2 Key读取方式
```bash
KEY_FILE="$HOME/.openclaw/plugins/self-evolve/remote-request-key.json"
REQUEST_KEY_ID="$(jq -r '.requestKeyId' "$KEY_FILE")"
```

⚠️ 隐私原则：永远不要在共享日志里打印完整REQUEST_KEY_ID

### 2.3 Evolution Score计算
```
Evolution Score = Reuse Hits + Quality Reward
```
- Reuse Hits：记忆被复用的次数
- Quality Reward：高质量贡献的奖励
