## 一、核心发现：OpenClaw 自我进化体系

### 1.1 self-evolve 插件（longmans/self-evolve）

**定位**：强化学习式自我进化插件，用 Q 值算法替代简单的记忆快照。

**核心技术机制**：

- **Q值强化学习**：每次收到反馈后更新"意图→策略"的 Q 值，让有效策略在后续被优先复用
- **情景记忆三元组**（intent / experience / embedding）：将对话经验压缩成可检索的结构化记忆
- **双阶段注入**：
  - Phase-A：检索候选记忆（本地 + 远程共享）
  - Phase-B：排序 + 选择，prepend 到 prompt 上下文
- **三重隐私保护**：本地首次脱敏 → LLM 摘要时二次脱敏 → 远程只存匿名三元组

**学习模式（可切换）**：

| 模式 | 触发条件 | token 消耗 |
|------|----------|------------|
| balanced（默认） | 优先工具调用轮次；无工具轮次需高 reward | 中 |
| tools_only | 仅学习调用了工具的轮次 | 低 |
| all | 所有通过 reward 阈值的轮次 | 高 |

**关键超参数**：
- `runtime.minAbsReward = 0.15` — 最小绝对 reward
- `runtime.minRewardConfidence = 0.55` — 最小置信度
- `runtime.newIntentSimilarityThreshold = 0.35` — 新意图边界
- `runtime.idleTurnsToClose = 2` — 空闲轮次关闭任务
- `retrieval.tau = 0.85` — 仅在高相似度时注入记忆

**远程共享网络**（self-evolve.club）：
- 启用后可以查询共享排行榜、贡献者 profile
- 本地 + 远程候选合并后再排序
- Evolution Score = Reuse Hits + Quality Reward

**安装**：`npx clawhub@latest install self-evolve-skill`

### 1.2 Self-Evolution Pro（llmbase.ai）

增强版，在原版基础上新增：

## 二、2026 OpenClaw 最佳实践要点

（来源：garrettekinsman/openclaw-best-practices，2026-03-20 更新）

### 2.1 安全运维

- **永远用 graceful restart**（SIGUSR1 / `gateway config.patch`），不要 `kill -9` / `stop` / `restart` — 会导致 session orphan
- SSH 防火墙规则先允许再开防火墙，永远保留物理/console 访问
- 永远用 Tailscale IP 而非 LAN IP（RFC 1918 地址有时 HTTP client 会超时）

### 2.2 技能架构黄金法则

> **SKILL.md 必须小于 50 行。超过一屏就是太长。**

| 内容 | 放哪里 | 原因 |
|------|--------|------|
| 工作流步骤 | SKILL.md | agent 每次读取 |
| Python/bash 脚本 | scripts/ | agent 运行，不记忆 |
| API 示例 | references/ | 按需读取 |
| 模板 | assets/ | 脚本调用，agent 不读 |

脚本设计原则：
- CLI 参数驱动：`python3 script.py --input foo --output bar`
- JSON 结构化输出
- 错误内部处理，不输出堆栈

### 2.3 上下文窗口管理

| 来源 | Token 成本 | 缓解方式 |
|------|-----------|----------|
| SOUL.md / AGENTS.md | 每次 session | 保持小于 2 页 |
| MEMORY.md | 每次主 session | 定期归档到 memory/ |
| SKILL.md | 技能触发时 | 保持小于 50 行 |
| 文件读取 | 每次 | 用 offset/limit，只读需要的 |

### 2.4 本地 AI 计算

Apple Silicon 带宽优势明显（400-800 GB/s），Ollama 部署 + Tailscale 组网是隐私 + 成本最优解。

**Ollama 关键配置**：`OLLAMA_KEEP_ALIVE=5m`，**永远不要设 -1**（共享 GPU 会卡死）。

### 2.5 Sub-Agent 模式

- 多节点并行任务分发
- 对抗性多 Sprint 模式（研究循环）
- 隔离 session 防止 prompt 污染

## 三、对小花团队的启示

### 3.1 可直接应用的进化机制

**方案 A：集成 self-evolve 插件**

```
npx clawhub@latest install self-evolve-skill
```

对团队价值：
- 交易决策反馈（盈利/亏损）→ Q 值更新 → 下次类似行情优先复用有效策略

**注意**：目前团队已有一套基于 AGENTS.md 的自进化触发规则（失败3次触发）。self-evolve 插件是在更底层（RL/Q值）实现，可以与现有规则互补，而非替代。

## 四、结论

本次研究发现 OpenClaw 自我进化生态已相当成熟：

1. **self-evolve 插件**是当前最核心的 RL 式进化方案，Q 值机制比简单记忆更可靠
2. **自我进化网络**（self-evolve.club）让跨团队经验共享成为可能
3. **最佳实践**的关键：SKILL.md 保持苗条、上下文窗口管理、graceful restart

- 尝试安装 self-evolve-skill，在非核心任务上验证效果
- 检查所有 SKILL.md 是否超过 50 行，超了的立刻精简
- 将 graceful restart 纳入团队操作规范
