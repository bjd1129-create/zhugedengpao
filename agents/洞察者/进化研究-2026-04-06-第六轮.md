## 一、本次核心发现

### 1. OpenClaw-RL：Princeton 的强化学习训练框架

**论文**：[arXiv:2603.10165](https://arxiv.org/abs/2603.10165) | 2026年3月10日发布

**一句话总结**：任何交互（对话、终端、GUI、代码执行、工具调用）都能产生训练信号，统一喂给同一个策略模型。

**核心观察**：
> next-state signals（下一状态信号）是通用的——用户回复、工具输出、终端状态变化、GUI变化，都是"动作之后的反馈"，都可以用来训练策略。

**两个信号通道**：
- **Evaluative Signal（评估信号）**：从 next state 提取 scalar reward，用 PRM（Process Reward Model）评判"这个动作做得好不好"
- **Directive Signal（指导信号）**：从 next state 提取"应该怎么做才对"，通过 OPD（Hindsight-Guided On-Policy Distillation）实现 token 级方向监督，比 scalar reward 信息更丰富

**关键创新**：
- 异步设计：模型服务 live 请求、PRM 评判交互、trainer 更新策略三者同时进行，零协调开销
- 对个人agent：可以从用户的重问、纠正、显式反馈中提取对话信号
- 对通用agent：同一套基础设施支持 terminal/GUI/SWE/工具调用场景的规模化 RL

**对团队的参考价值**：这是 Princeton 的研究层实现，暂时不直接影响小花团队。但核心思路——"所有交互都是训练数据"——值得内化。

### 2. self-improving-agent 技能（260k+ 下载量）

**定位**：最流行的 OpenClaw 自我改进技能，本质是一个结构化错误记录器。

**工作方式**：
- 将 errors、corrections、feature requests、best practices 记录到 `.learnings/` 目录
- 稳定的学习内容会被提升（promote）到项目 memory 中
- 触发条件：命令失败、用户纠正、发现更好方法、知识过时

**和 self-evolve 的区别**：

| 维度 | self-evolve | self-improving-agent |
|------|-------------|---------------------|
| 记忆类型 | Q-Learning 三元组（算法化） | .learnings 文件（手工结构化） |
| 触发 | 自动（反馈信号检测） | 半自动（需识别错误场景） |
| 遗忘机制 | 有（maxEntries=200） | 无明确遗忘机制 |
| 共享网络 | 有（self-evolve.club） | 无 |
| Token 成本 | 较高（每次学习调 LLM） | 低（写入文件为主） |

## 二、对小花团队的启发

### 本周可落地的事情

**1. 用 self-improving-agent 的思路升级 .learnings 管理**

目前小花团队没有统一的错误/学习记录机制。建议：
- 在 `memory/learnings/` 目录下建立标准化记录格式
- 每当出现错误或用户纠正时，同步更新这个目录
- 每月复盘时将稳定内容提升到 MEMORY.md

**2. 借鉴 OpenClaw-RL 的"全量信号"思路做团队复盘**

OpenClaw-RL 说"所有交互都是训练数据"——对团队来说意思是：每天的日志、飞书消息、任务结果都是学习材料。建立更好的日志格式（结构化三元组 > 叙事日志）能提高复盘效率。

**3. 关注 OpenClaw 进化技能生态**

目前有 13,700+ 社区技能，定期（每周）扫描一次新增的高价值技能（特别是 self-evolution 类）可以持续提升团队能力上限。
