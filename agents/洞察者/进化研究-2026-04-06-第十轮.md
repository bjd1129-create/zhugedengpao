## 一、核心发现

### 1. OpenClaw-RL：强化学习框架（重大突破）

**来源**：https://github.com/Gen-Verse/OpenClaw-RL

这是目前 OpenClaw 生态最前沿的进化方向——用强化学习训练个性化 AI Agent。

**核心原理**：
- 把日常对话变成训练信号，通过对话反馈持续优化策略
- 完全异步：Agent 服务、Rollout 收集、PRM/Judge 评估、策略训练四个模块独立运行，互不阻塞
- 不需要人工标注数据，API 消息自动分类（主线可训练 vs 侧线不可训练）
- 支持三种学习范式：Binary RL / OPD / Combine

**最新动态（2026年4月）**：
- 2026/4/4：支持基于多人反馈优化单一模型
- 2026/3/20：可使用自己的 OpenClaw，只需安装 [rl-training-headers](https://github.com/Gen-Verse/OpenClaw-RL/tree/main/extensions/rl-training-headers) 扩展
- 2026/3/13：同时支持本地 GPU 和云端（Tinker）部署
- 2026/3/12：支持 LoRA 训练
- 2026/3/10：技术报告发布，登上 HuggingFace 每日论文榜第一

**对小花团队的启示**：
- 这是一个训练层的能力，我们目前用的是 API 层，暂时用不上
- 但思路可以借鉴：把每次任务执行结果记录下来，形成"反馈信号"
- 进化研究可以定期回顾这些信号，分析策略调整方向

### 2. Self-Evolve 技能网络

**来源**：https://self-evolve.club/

**核心机制**：
- 从真实会话和反馈中学习，用强化风格更新，让技能行为更稳定
- RAG token 高效：只检索相关的进化三元组（intent-experience triplets）
- 本地和远程记忆合并排名，贡献者直接改善整个社区的结果
- 领导者榜单展示 Evolution Score = Reuse Hits + Quality Reward

**安装命令**：
```bash
npx clawhub@latest install self-evolve-skill
```

**对小花团队的启示**：
- 我们的 `.learnings/` 机制和这个思路一致，但 Self-Evolve 多了共享网络层
- 可以考虑接入 self-evolve 网络，让小花团队的经验贡献给社区，同时也能获取社区的进化经验

### 3. Self-Improving Agent 技能（已有）

**来源**：`~/.openclaw/skills/self-improving-agent/`

这是小花团队已经在用的技能，核心是：
- 错误 → `.learnings/ERRORS.md`
- 纠正 → `.learnings/LEARNINGS.md`（含分类：correction、knowledge_gap、best_practice）
- 功能请求 → `.learnings/FEATURE_REQUESTS.md`
- 定期复盘，promote 到 AGENTS.md / SOUL.md / TOOLS.md

**关键规则**：Recurrence-Count >= 3 + 跨 2+ 任务 + 30天内 → 推广到系统 prompt

---

### 4. 自进化四大阶段（架构级）

**来源**：https://blog.exit.st/openclaw-self-improving-agent-complete-guide/

```
Detection → Analysis → Generation → Integration
```

- **Detection**：监控失败、重复模式、用户反馈、效率指标
- **Analysis**：缺失什么能力？现有技能能否修改？是否需要新技能？
- **Generation**：写 SKILL.md + 支撑脚本 + 测试验证
- **Integration**：放入正确目录 + 验证运行 + 立即启用
