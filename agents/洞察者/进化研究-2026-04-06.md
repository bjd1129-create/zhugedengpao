## 一、OpenClaw自我进化核心架构

### 1.1 三层记忆体系（已验证有效）

| 层级 | 文件 | 功能 | 更新频率 |
|------|------|------|----------|
| L1 | 脑内上下文 | 当前对话状态 | 实时 |
| L2 | memory/state.json | 跨对话配置/状态 | 按需 |
| L3 | MEMORY.md | 精选长期记忆 | 定期整理 |

**我们的差距**: 小花已有MEMORY.md，但L2层的state.json缺失，应补充。

### 1.2 双环进化引擎（Self-Improving + AutoSkill）

**Self-Improving Agent** (纠错型进化):
- 创建 `.learnings/` 目录
- ERRORS.md → 记录失败和异常
- LEARNINGS.md → 沉淀新经验和最佳实践
- FEATURE_REQUESTS.md → 收集改进建议
- 凌晨4点自动触发复盘cron

**AutoSkill** (技能型进化):
- 从真实交互中自动提炼技能（SKILL.md）
- 技能有版本管理（v0.1.34这样的迭代）
- 触发条件：用户反复强调的稳定约束
- 技能不是设计出来的，是"长出来的"

### 1.3 行为适应机制（Behavioral Adaptation）
- 追踪用户纠正（corrections）
- 自动注入适应提示（adaptation hints）
- 7次重复后自动升级到SOUL.md

## 补充研究（01:07）- self-evolve插件深度分析

### 4.1 self-evolve插件（longmans/self-evolve）

**比Self-Improving Agent更进阶的强化学习方案**:

核心差异：不是日志记录，而是**Q值强化学习**。

```
用户消息 → 反馈检测？
  ├─ Yes: 评分reward → 检查学习门 → Q值更新 → 写入情景记忆
  └─ No: 意图检测 → 检索本地+远程记忆 → 排序注入 → 回复
```

**三种学习模式**:
| 模式 | 说明 | Token成本 |
|------|------|----------|
| balanced | 只学工具调用；无工具需要高reward | 中 |
| tools_only | 只学调用工具的轮次 | 低 |
| all | 学所有通过reward门的轮次 | 高 |

**学习门默认值**:
- `runtime.minAbsReward=0.15`
- `runtime.minRewardConfidence=0.55`
- `retrieval.tau=0.85`（只注入高相似度记忆）

**隐私设计（双重卸载）**:
1. 本地sanitizeMemoryText移除ID和发送者标签
2. LLM总结时第二次脱敏，敏感数据替换为`[REDACTED_*]`

**远程共享网络self-evolve.club**:
- 本地+远程记忆合并后排序
- Evolution Score = Reuse Hits + Quality Reward
- 用户intent对话痕迹完全脱敏后才作为记忆payload
- 可选关闭：`remote.enabled=false`

**安装**:
```bash
git clone https://github.com/longmans/self-evolve
git clone https://github.com/longmans/self-evolve
openclaw plugins install ./self-evolve
export OPENAI_API_KEY=sk-xxx
openclaw gateway restart
```

### 4.2 self-improving技能（完整架构）

**目录结构**:
```
~/self-improving/
├── memory.md          # HOT: ≤100行,始终加载
├── index.md           # 主题索引
├── heartbeat-state.md # 心跳状态
├── projects/          # 按项目学习
├── domains/          # 按领域（代码/写作）
├── archive/          # COLD: 衰减模式
└── corrections.md    # 最近50条纠正
```

**热化规则**:
- 同一模式7天内出现3次 → 晋升HOT
- 30天未用 → 降级WARM
- 90天未用 → 归档COLD
- 从不删除（除非用户确认）

**检测触发词**:
- 纠正: "No, that's not right" / "Actually, it should be"
- 偏好: "I like when you" / "Always do X"
- 忽略: 一次性指令、上下文特定假设

### 4.3 skill-evolution（钩子驱动的技能进化）

**基于Claude的skill-evolution机制**:

钩子触发:
- `PreToolUse` → Bash/Write/Edit前
- `PostToolUse` → Bash/Write/Edit后
- `Stop` → 会话结束时

**结束任务三问**（60秒复盘）:
1. 是否基于本次运行优化技能？
2. 最大障碍是什么？
3. 优先优化哪个方向？（I/O契约/索引摘要/脚本模板/确认点）

如选"优化": 调用`skill-improver`生成最小补丁 → 人工审核后应用。

### 4.4 内置self-evolution技能（安全进化引擎）

**L1-L4权限分层**:
| 层级 | 权限 | 说明 |
|------|------|------|
| L1 | 自动 | 参数优化、提示词调整 |
| L2 | 自动 | 新增技能、知识补充 |
| L3 | 需确认 | 流程重构、架构调整 |
| L4 | 禁止 | 核心协议、安全规则 |

**进化流程**: 数据采集 → 短板分析 → 安全检查 → 执行进化 → 验证反馈
