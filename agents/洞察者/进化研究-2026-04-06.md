## 一、OpenClaw自我进化核心架构

### 1.1 三层记忆体系（已验证有效）

| 层级 | 文件 | 功能 | 更新频率 |
|------|------|------|----------|
| L1 | 脑内上下文 | 当前对话状态 | 实时 |
| L2 | memory/state.json | 跨对话配置/状态 | 按需 |
| L3 | MEMORY.md | 精选长期记忆 | 定期整理 |

**我们的差距**: 小花已有MEMORY.md，但L2层的state.json缺失，应补充。

### 1.2 双环进化引擎（Self-Improving + AutoSkill）

**Self-Improving Agent** (纠错型进化):
- 创建 `.learnings/` 目录
- ERRORS.md → 记录失败和异常
- LEARNINGS.md → 沉淀新经验和最佳实践
- FEATURE_REQUESTS.md → 收集改进建议
- 凌晨4点自动触发复盘cron

**AutoSkill** (技能型进化):
- 从真实交互中自动提炼技能（SKILL.md）
- 技能有版本管理（v0.1.34这样的迭代）
- 触发条件：用户反复强调的稳定约束
- 技能不是设计出来的，是"长出来的"

### 1.3 行为适应机制（Behavioral Adaptation）
- 追踪用户纠正（corrections）
- 自动注入适应提示（adaptation hints）
- 7次重复后自动升级到SOUL.md

- **旧问题**: 50轮对话上下文膨胀30%

Bundle (功能包) → Provider (模型提供者) → Plugin (具体插件)

## 三、最佳实践（来自200+小时用户）

### 3.1 内存管理
- **SKILL.md < 50行**：逻辑放到scripts/，SKILL.md只放步骤
- **MEMORY.md定期归档**：移到memory/YYYY-MM-DD.md
- **HEARTBEAT.md保持轻量**：每30-60分钟执行一次

### 3.2 安全红线
- 网关端口不暴露公网（用Tailscale/WireGuard/SSH tunnel）
- 设置强gateway auth token
- 用`gateway config.patch`而非stop/start重启
- 永远不要OLLAMA_KEEP_ALIVE=-1（共享GPU会锁死）
- git push前必须扫描API keys

### 3.3 子Agent模式
- **云端+本地混合**: Opus(云)编排，本地模型执行
- **并行化**: 多个独立任务同时spawn
- **永不轮询**: 完成是推送式的，不等轮询

### 3.4 研究循环（对抗性多冲刺）
```
Sprint1 研究员 → Sprint2 对手 → Sprint3 研究员 → ... → 最终报告
```
- 对抗性挑战消灭确认偏见
- 时间预算防止提前停止
