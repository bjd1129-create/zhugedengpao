## 核心发现

### 1. self-evolve 插件 — 最成熟的自我进化方案

**GitHub:** github.com/longmans/self-evolve

这是目前最完整、最可操作的自我进化插件，核心机制：

**学习循环（四步）：**
1. **观察（Observe）**：在每次对话中检测反馈信号（正/负）
2. **评分（Score）**：根据 reward gates 计算奖励值
3. **记忆（Learn）**：将有效的经验总结为「意图-经验」三元组存入 episodic memory
4. **检索（Retrieve）**：下次遇到类似场景时， prepend 相关记忆到 prompt context

**三种学习模式：**
- `balanced`（默认）：优先学习工具调用，少 token 消耗
- `tools_only`：仅在学习工具调用时记录，最低 token 成本
- `all`：所有通过 reward gate 的轮次都学习，最高 token 成本

**关键参数（可配置）：**
```
runtime.minAbsReward=0.15        # 最小绝对奖励值
runtime.minRewardConfidence=0.55  # 最小奖励置信度
retrieval.tau=0.85               # 检索阈值（只注入高相似度记忆）
memory.maxEntries=200            # 最大记忆条目数
```

**远程共享网络：**
- 启用后可在 self-evolve.club 共享经验三元组
- 本地+远程记忆合并后排序注入
- 隐私设计：本地两次脱敏（sanitize + LLM 重写）
- 有贡献者排行榜（Evolution Score = Reuse Hits + Quality Reward）

### 2. OpenClaw-RL 论文 — 统一训练框架

**arXiv:** arxiv.org/abs/2603.10165

核心洞察：**next-state 信号是通用的**，Policy 可以同时从所有信号学习：

- 私人对话
- 终端执行
- GUI 交互
- SWE 任务
- 工具调用轨迹

这些不是分离的训练问题，而是统一的信号源。

**对团队的启发：** 我们可以把不同类型的「成功/失败」信号统一记录，用强化学习的方式让团队进化。

### 3. 自我改进 Agent 架构（四阶段循环）

来源：blog.exit.st 完整指南

```
检测(Detection) → 分析(Analysis) → 生成(Generation) → 集成(Integration)
```

**Phase 1: 检测**
- 任务失败记录
- 重复模式识别（同类请求 3+ 次）
- 用户反馈
- 效率指标（耗时 > 预期 2x）

**Phase 2: 分析**
- 缺失什么能力？
- 解决方案什么样？
- ClawHub 是否有现成 skill？
- 修改现有还是新建？

**Phase 3: 生成**
- 编写 SKILL.md
- 创建支撑脚本
- 用原失败场景测试
- 验证不破坏现有能力

**Phase 4: 集成**
- 放入 pending-skills/ 目录待 review
- 通知人类审核
- 通过后移到 active skills
- 立即启用

**Human Review Gate 是必须的**：自我改进是增强人类监督，不是替代。

### 5. 最佳实践要点

**14条 Power User 经验（mindstudio.ai）：**
1. Telegram thread 模式用于长对话
2. 模型路由（不同任务用不同模型）
3. 子 Agent 用于并行处理
4. Cron 定时任务自动化
5. 安全配置（OPENCLAW_SKIP_ELEVATED=true）
6. 记忆系统分层（短期/长期/经验）
7. Skill 模块化复用
8. 日志定期审计

**自我进化最佳实践（self-evolve.club）：**
- 正向反馈要说清楚"什么成功了"
- 负向反馈要说清楚"什么失败了"
- 避免模糊反馈如"ok"
- Explicit feedback > vague messages

## 风险与注意事项

- **隐私**：使用远程共享前，确认脱敏充分
- **Token 成本**：balanced 或 tools_only 模式优先
- **Human in the loop**：自我进化不能完全自治，需要老庄把关
- **质量控制**：新增 skill 必须测试验证，不能直接上生产
