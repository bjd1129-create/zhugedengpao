## 一、核心进化机制深度解析

### 1.1 记忆系统 → 进化基础

OpenClaw的进化建立在**显式记忆写入**上——模型只能"记住"写入磁盘的内容，没有隐藏状态。

| 记忆类型 | 位置 | 加载时机 | 进化作用 |
|---------|------|---------|---------|
| **MEMORY.md** | workspace根目录 | 每个DM会话开始 | 长期记忆：偏好、决策、事实 |
| **memory/YYYY-MM-DD.md** | memory/目录 | 今天+昨天的文件自动加载 | 每日运行上下文 |

**关键洞察：** 进化 = 持续写入 + 定期提炼。每日文件 → 长期记忆的提炼过程就是"学习"。

---

### 1.2 上下文压缩（Compaction）→ 自动提炼机制

当对话接近context window限制时，OpenClaw自动：
1. 将早期对话总结为紧凑摘要
2. 保存到session transcript
3. 保留近期消息完整

**进化关联：** Compaction前会自动提醒agent保存重要笔记到memory，防止context丢失。这本质上是一个**被动提炼**机制。

**可用配置：**
```json5
{
  agents: {
    defaults: {
      compaction: {
        model: "openrouter/anthropic/claude-sonnet-4-6", // 用更强模型做摘要
        notifyUser: true
      }
    }
  }
}
```

### 1.3 系统提示词架构 → 自我感知基础

OpenClaw为每个agent构建自定义system prompt，核心结构：

```
Tooling → Safety → Skills → OpenClaw Self-Update → Workspace → Documentation
       → Bootstrap Files → Sandbox → Date/Time → Runtime → Reasoning
```

**进化关键——Self-Update section：**
```
OpenClaw Self-Update: how to run `config.apply` and `update.run`
```
这让agent知道如何更新自己！

---

### 1.4 Hook系统 → 事件驱动进化

Hooks是OpenClaw的**事件总线**，可监听并响应生命周期事件：

| 事件 | 触发时机 | 进化应用 |
|------|---------|---------|
| `session:compact:before` | 压缩前 | 强制保存重要记忆 |
| `session:compact:after` | 压缩后 | 验证记忆完整性 |
| `agent:bootstrap` | workspace初始化前 | 注入自定义人格/策略 |
| `gateway:startup` | 网关启动 | 执行自检和修复 |
| `message:received` | 收到消息 | 情绪检测→调整策略 |

**内置Hook：**
- `session-memory`：自动保存session上下文到memory/
- `bootstrap-extra-files`：注入额外引导文件
- `command-logger`：记录所有命令到日志

---

### 1.5 Bootstrap文件注入 → 个性化基础

OpenClaw在每个turn自动注入这些文件到context：

| 文件 | 内容 | 可进化性 |
|------|------|---------|
| SOUL.md | 核心人格定义 | ⭐ 直接改写=人格进化 |
| AGENTS.md | 工作空间规则 | ⭐ 规则迭代=能力扩展 |
| MEMORY.md | 长期记忆 | ⭐ 持续写入=知识积累 |
| USER.md | 用户画像 | ⭐ 学习用户=关系深化 |
| HEARTBEAT.md | 心跳检查清单 | ⭐ 任务调度=效率优化 |
| TOOLS.md | 本地工具配置 | ⭐ 工具偏好=效率提升 |

**重要：** `memory/*.md` 不自动注入，只在需要时通过memory_search/memory_get访问。

## 二、OpenClaw官方进化能力清单

根据文档，OpenClaw支持以下自我进化路径：

| 进化方式 | 实现路径 | 复杂度 |
|---------|---------|--------|
| **配置热更新** | `gateway config.patch` 或 `config.apply` | ⭐ 简单 |
| **技能安装/卸载** | `openclaw skills install <skill>` | ⭐ 简单 |
| **记忆写入** | agent自动写入MEMORY.md | ⭐ 简单 |
| **Hook编写** | 编写HOOK.md + handler.ts | ⭐⭐ 中等 |
| **自更新触发** | agent调用`update.run` | ⭐ 简单 |
| **系统提示词改写** | 修改SOUL.md/AGENTS.md | ⭐ 简单 |
| **Skill自创建** | agent创建新的SKILL.md | ⭐⭐ 中等 |
| **RL微调** | OpenClaw-RL框架 | ⭐⭐⭐ 复杂 |

## 三、进化触发点分析

### 何时会发生进化？

```
用户反馈 → 消息分析 → 情绪检测 → Hook触发
    ↓
记忆写入 → Daily Log → MEMORY.md提炼
    ↓
Compaction → 上下文总结 → 关键洞察保留
    ↓
下次bootstrap → 更新的SOUL.md/AGENTS.md生效
```

### 进化信号检测（可自动化）

| 信号 | 来源 | 预期动作 |
|------|------|---------|
| 重复错误 | 对话历史 | 更新AGENTS.md避免规则 |
| 用户调整指令 | 消息 | 更新USER.md偏好 |
| 效率下降 | compaction频率 | 优化TOOLS.md配置 |
| 新技能需求 | 任务失败 | 安装/创建Skill |
| 人格偏差 | 用户反馈 | 调整SOUL.md |
