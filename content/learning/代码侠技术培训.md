## 一、建站篇：从零到一个能上线的网站

### 1.1 选对工具，少走三年弯路

| 网站类型 | 推荐工具 | 原因 |
|----------|----------|------|
| 展示型官网 | Next.js + Tailwind | 速度快SEO友好 |
| 电商站 | Shopify / WooCommerce | 生态完善 |
| 博客/内容站 | Notion + Next.js / Hexo | 写作体验好 |
| 快速原型 | Webflow | 可视化拖拽 |

**不建议：** 纯用WordPress除非你有特殊原因。维护成本高，安全风险大。

### 1.2 Next.js 项目初始化（标准流程）

```bash
# 创建项目
npx create-next-app@latest my-site --typescript --tailwind --eslint

# 进入目录
cd my-site

# 启动开发服务器
npm run dev
```

**目录结构规范：**
```
my-site/
├── app/              # Next.js App Router（最新写法）
│   ├── page.tsx      # 首页
│   ├── about/       # 关于页
│   └── blog/        # 博客列表
├── components/       # 复用组件
├── lib/             # 工具函数
└── public/          # 静态资源
```

### 1.3 页面开发规范

**每个页面必须包含的元素：**
1. `<title>` 和 `<meta description>`
2. 语义化HTML标签（`<main>`, `<article>`, `<section>`）
3. 图片alt属性
4. 移动端友好的viewport设置

**示例：基础SEO配置**
```tsx
export const metadata = {
  title: '页面标题 | 网站名',
  description: '这里写不超过160字的页面描述，包含核心关键词。',
  openGraph: {
    title: '分享到社交平台时显示的标题',
    description: '分享到社交平台时显示的描述',
    images: ['/og-image.jpg'],
  },
}
```

## 二、SEO篇：让搜索引擎读懂你的网站

### 2.1 SEO的本质

SEO不是"欺骗搜索引擎"，而是**把网站变成搜索引擎能读懂的格式**。核心逻辑：

> 搜索引擎 = 图书馆管理员
> SEO = 用管理员能理解的方式写书

**三个核心要素：**
1. **抓取（Crawl）** —— 搜索引擎能找到你的页面
2. **索引（Index）** —— 搜索引擎把你的页面存进数据库
3. **排名（Ranking）** —— 用户搜某个词时你的页面出现

### 2.2 技术SEO清单（上线前必查）

#### 基础配置
- [ ] 所有页面有唯一且描述性的 `<title>`
- [ ] 所有页面有 `<meta name="description">`
- [ ] 使用语义化HTML（H1/H2/H3层级正确）
- [ ] 所有图片有 `alt` 属性
- [ ] `robots.txt` 允许搜索引擎抓取
- [ ] 有 `sitemap.xml` 供搜索引擎发现所有页面

#### 进阶配置
- [ ] 使用结构化数据（Schema Markup）
- [ ] 页面加载速度 < 3秒
- [ ] 移动端完全适配
- [ ] HTTPS 全站加密
- [ ] 无重复内容（规范化URL）

### 2.3 关键词研究：找到正确的战场

**关键词分类：**
- **头部词**（竞争激烈，流量大）：如"营销"——新站别碰
- **腰部词**（竞争中等，转化高）：如"B2B营销获客方案"——重点攻
- **长尾词**（竞争低，意图明确）：如"2024制造业B2B营销策略"——精准占位

**关键词布局原则：**
- Title前60字符内包含主关键词
- H1标签必须是页面核心主题
- 正文中自然出现关键词（密度2-3%，不要堆砌）
- URL路径包含关键词（如 `/blog/seo-guide`）

### 2.4 内容SEO：让内容自己带流量

**好内容的标准：**
1. 解决用户的具体问题
2. 比搜索结果第一页的内容更全面
3. 有数据、有案例、有截图
4. 结构清晰，有小标题方便扫描

**内部链接策略：**
- 每篇新文章至少链出3篇相关旧文
- 重要页面（首页、服务页）从多处获得链接
- 使用描述性锚文本（不要"点击这里"，要"查看SEO教程"）

### 2.5 外链建设：让别人给你投票

**高质量外链来源：**
1. 行业媒体报道/投稿
2. 合作伙伴网站的互相推荐
3. 优质内容被自然引用
4. 目录提交（谨慎选择，别用垃圾目录）

**绝对不要做的事：**
- 购买链接（Google会惩罚）
- 垃圾目录群发
- 隐藏链接或文字

## 三、性能优化篇：速度是用户体验的生死线

### 3.1 为什么性能重要

- Google明确表示：页面速度是排名因素
- 加载超过3秒，53%的用户会离开
- 电商网站每慢1秒，转化率下降7%

### 3.2 性能诊断工具

| 工具 | 用途 | 地址 |
|------|------|------|
| PageSpeed Insights | Google官方评分 | pagespeed.web.dev |
| Lighthouse | 开发者工具内置 | Chrome DevTools |
| GTmetrix | 详细分析报告 | gtmetrix.com |
| WebPageTest | 深度请求分析 | webpagetest.org |

**目标分数：**
- PageSpeed Desktop: 90+ 
- PageSpeed Mobile: 80+
- LCP ( Largest Contentful Paint ): < 2.5秒
- FID ( First Input Delay ): < 100ms
- CLS ( Cumulative Layout Shift ): < 0.1

### 3.3 图片优化（影响最大的单项）

**问题所在：**
未经优化的图片通常是页面最大、最慢的资源。

**解决方案：**
```bash
# Next.js 自动图片优化（开箱即用）
import Image from 'next/image'

// 自动转WebP/AVIF，自动懒加载
<Image 
  src="/hero.jpg" 
  alt="描述"
  width={1200}
  height={600}
  priority={true}  // 首页首图加这个
/>
```

**手动优化技巧：**
1. 上传前用 TinyPNG 压缩（可减少70%体积）
2. 照片用 JPEG/WebP，图标用 SVG
3. 截图和图表导出为 PNG 时，压缩到 80% 质量即可
4. 背景大图控制在 200KB 以内

### 3.4 代码分割与懒加载

**Next.js 自动做好的：**
- 每个页面只加载该页面的代码
- 图片懒加载（视口外的图不加载）

**你需要做的：**
```tsx
// 第三方库按需引入
import dynamic from 'next/dynamic'

// 这个组件只在需要时才加载
const HeavyChart = dynamic(() => import('./HeavyChart'), {
  loading: () => <p>加载中...</p>,
  ssr: false  // 不需要SEO的交互组件关掉SSR
})
```

### 3.5 字体优化

**问题：** Google Fonts 默认加载会阻塞渲染

**正确做法（Next.js）：**
```tsx
# npm install next/font
import { Inter } from 'next/font/google'

const inter = Inter({ 
  subsets: ['latin'],
  display: 'swap',  // 字体加载时用系统字体替代
})

// app/layout.tsx 中使用
<body className={inter.className}>
```

### 3.6 缓存策略

```javascript
// next.config.js 配置永久缓存
module.exports = {
  images: {
    formats: ['image/avif', 'image/webp'],
  },
  // 静态资源永久缓存（文件名带hash，改动才变）
}
```

**缓存配置最佳实践：**
- HTML：不缓存或短期缓存（确保更新及时生效）
- CSS/JS：永久缓存（文件名hash确保版本控制）
- 图片：永久缓存（同一URL永远返回同一文件）

### 3.7 Core Web Vitals 优化指南

**LCP（最大内容绘制）优化：**
- 预加载关键图片：`<link rel="preload" as="image" href="hero.webp">`
- 消除渲染阻塞资源
- 使用CDN加速

**CLS（布局偏移）优化：**
- 所有图片指定 width/height（Next.js自动处理）
- 广告位预留固定空间
- 字体加载用 `display: swap`

**FID/INP（交互响应）优化：**
- 去掉不必要的三方脚本
- 减少主线程 JavaScript 体积
- 交互逻辑代码分割
