# 洞察者研究方法论
## 市场研究的标准流程与实战指南

> "不是没有数据，是不知道该收集什么数据。不是没做研究，是用错了方法。"

---

## 一、研究的起点：先搞清楚你要回答什么问题

研究最大的敌人不是"不知道"，是"不知道要问什么"。

**研究开始前，必须明确：**

1. **决策问题（Decision Question）**
   - "我们要不要进入东南亚市场？"
   - "新产品的定价应该定在哪个区间？"

2. **核心假设（Core Hypothesis）**
   - "我们假设30-40岁的一线城市女性愿意为品质支付30%溢价"
   - "我们假设现有用户在推荐朋友时最大的障碍是'不知道怎么说'"

**先写下来，再开始研究。** 口头上的"先看看"往往导致研究完了还是不知道该做什么决定。

---

## 二、研究的两条路：定量与定性

### 2.1 定量研究：回答"有多少"

**适用场景：**
- 市场规模估算
- 用户画像描述（比例分布）
- A/B测试结果判断
- 趋势跟踪（年度对比）

**常用方法：**

| 方法 | 样本量 | 成本 | 适合场景 |
|------|--------|------|----------|
| 问卷调查 | 100-1000+ | 低 | 大样本描述 |
| 数据分析 | 全部用户 | 低 | 行为数据挖掘 |
| 第三方报告 | — | 中高 | 市场规模参考 |

**问卷设计要点：**
- 选项穷尽且互斥
- 避免引导性问题
- 开放式问题尽量少（分析成本高）
- 预测试：发给10个人先跑一遍

### 2.2 定性研究：回答"为什么"

**适用场景：**
- 探索未知领域（还没假设）
- 理解用户深层动机
- 发现竞品没注意到的痛点
- 优化产品细节

**常用方法：**

| 方法 | 人数 | 时长 | 适合场景 |
|------|------|------|----------|
| 一对一深访 | 5-15人 | 45-90分钟 | 高价值用户理解 |
| 用户座谈会 | 6-8人/组 | 2小时 | 初步探索 |
| 可用性测试 | 5-8人 | 45分钟 | 产品交互优化 |
| 实地观察 | 1-3人 | 半天起 | 真实使用场景 |

**深度访谈的精髓：**
> 5个WHY连续追问法

```
用户：我喜欢这个功能
访客：为什么喜欢？
用户：因为方便
访客：为什么方便？
用户：因为不用打字
访客：为什么不想打字？
用户：因为在开车
访客：为什么开车时还用？
→ 发现核心场景：行车中的语音需求
```

## 三、市场研究标准流程（六步法）

### 第一步：定义研究问题（1天）

**输出物：**
- 研究问题清单（不超过3个核心问题）
- 背景说明（为什么要研究这个问题）
- 决策建议（研究结果会如何影响决策）

**常见错误：**
- 问题太宽泛（"了解一下市场"）
- 问题太多（超过5个核心问题等于没重点）

---

### 第二步：建立分析框架（1-2天）

**框架类型：**

**① PEST分析（宏观环境）**
```
P - Political（政策）
E - Economic（经济）
S - Social（社会）
T - Technological（技术）
```

**② 波特五力（行业竞争）**
```
1. 现有竞争者
2. 潜在进入者
3. 替代品
4. 供应商议价能力
5. 买家议价能力
```

**③ 用户旅程地图（用户体验）**
```
认知 → 考虑 → 购买 → 使用 → 推荐
每个阶段的：行为 / 痛点 / 情绪 / 机会
```

---

### 第三步：收集二手信息（3-5天）

**信息来源：**

**免费来源：**
- 艾瑞咨询、36Kr研究院、QuestMobile（免费报告摘要）
- 行业协会报告
- 上市公司年报（财务数据、战略描述）
- 政府统计数据（国家统计局、行业主管部门）

**付费来源：**
- Gartner / Forrester（企业服务市场）
- eMarketer（数字营销）
- 各行业定制研究报告

**二手信息评估三问：**
1. 这个数据的来源是哪里？
2. 样本量和口径是什么？
3. 是什么时候做的调研？（时效性）

---

### 第四步：一手数据采集（5-10天）

**访谈筹备：**
```
招募条件 → 访谈大纲 → 预测试 → 正式执行
```

**访谈大纲结构（以产品需求调研为例）：**
```
开场（5分钟）
- 自我介绍，说明目的，保密承诺

背景了解（10分钟）
- 使用习惯、工作场景

核心问题探索（30分钟）
- 当前最大挑战是什么？
- 现有解决方案是什么？
- 为什么会这样/那样选择？

产品场景模拟（20分钟）
- 如果有一个功能可以解决X问题，会用吗？
- 为什么？
- 付费意愿？

收尾（5分钟）
- 还有什么想补充的？
- 有没有推荐的其他访谈对象？
```

---

### 第五步：数据分析与综合（3-5天）

**定量数据分析：**
1. 数据清洗（剔除无效问卷）
2. 交叉分析（按年龄/行业/使用频率分组）
3. 关键指标提炼（2-3个核心发现）

**定性数据分析：**
1. 录音转文字（oto）→ 逐字稿
2. 编码归类（把相似回答归为一类）
3. 提炼主题（每个主题要有支撑证据）

**综合分析：**
> 定量告诉"是什么"，定性告诉"为什么"。
> 两者结合才完整。

---

### 第六步：研究输出与呈现（2-3天）

**研究报告结构：**
```
1. 执行摘要（1页，重点结论）
2. 研究背景与问题
3. 研究方法说明
4. 核心发现（不超过5个，每个有数据支撑）
5. 含意与建议（每个发现对应的行动建议）
6. 局限性与后续研究建议
```

**好报告的标准：**
- 决策者10分钟内能看完核心结论
- 每个结论有明确的证据支撑
- 建议是具体的、可执行的
- 方法透明，可供review

## 四、竞品分析：专项研究方法

### 4.1 竞品选择

**竞品分级：**
- **直接竞品**：解决同样问题，目标用户重叠
- **间接竞品**：解决相关问题，可借鉴或存在替代关系
- **潜在竞品**：今天不是对手，但明天可能是

### 4.2 竞品分析维度

| 维度 | 关注点 | 信息来源 |
|------|--------|----------|
| 产品功能 | 核心功能差异，迭代方向 | 官网、版本记录 |
| 定价策略 | 价格区间，收费模式 | 官网、用户访谈 |
| 用户评价 | App Store / G2 / Capterra | 公开评价平台 |
| 流量来源 | SEO关键词，广告投放 | SEMrush / SimilarWeb |
| 融资情况 | 融资金额，投资方 | 公开报道 |
| 团队规模 | 公司规模（估算） | LinkedIn / 官网 |

### 4.3 竞品体验测试

自己用竞品走一遍核心流程：
1. 注册/激活流程是否顺畅？
2. 核心功能体验如何？
3. 哪里卡住了？哪里惊喜了？
4. 客服响应速度和质量？

**输出：竞品体验地图 + 功能对比表**

## 五、实操工具箱

### 数据收集工具
- **问卷**：腾讯问卷（免费）、Typeform（体验好）、Google Form
- **访谈**：腾讯会议 + otter.ai 自动转录
- **数据分析**：Excel 够用，复杂用 Python / R

### 信息整理工具
- **笔记整理**：Notion（结构化好）
- **用户旅程图**：Figma / FigJam
- **竞品对比表**：飞书多维表格

### 信息来源
- [艾瑞咨询](https://report.iresearch.cn/) — 免费报告摘要
- [36Kr研究院](https://36kr.com/) — 科技互联网
- [QuestMobile](https://www.questmobile.com.cn/) — 移动互联网
- [SimilarWeb](https://similarweb.com/) — 竞品流量分析
- [Gartner](https://www.gartner.com/) — 企业级市场

## 六、研究伦理与质量底线

**必须遵守：**
- 用户隐私保护（访谈内容未经允许不外传）
- 数据来源标注（不把推测当数据）
- 结论谨慎（样本量不够大时不夸大）
- 利益冲突披露（如果有的话）

**常见研究陷阱：**
1. **确认偏误** —— 只找支持自己假设的证据
2. **幸存者偏差** —— 只问现有用户，忽略流失用户
3. **样本选择偏差** —— 用小样本得出大结论
4. **数据挖掘过度** —— 在随机波动中找规律

---

## 七、研究能力成长路径

**Level 1（执行）：** 能独立执行访谈、回收问卷、输出记录
**Level 2（分析）：** 能提炼洞察，识别关键发现，给出建议
**Level 3（设计）：** 能设计研究方案，选择正确方法，评估研究质量
**Level 4（战略）：** 能将研究与商业决策挂钩，影响产品/战略方向

**成为优秀洞察者的习惯：**
1. 每天关注3条行业新闻并记录观点
2. 定期使用竞品，记录体验变化
3. 建立个人数据库（行业数据、政策、数据源清单）
4. 读研究报告时先看摘要和结论，再读方法论

---

> **洞察者信条：**
> 好的研究不是证明自己正确，而是发现真相——哪怕真相和预期不一样。
