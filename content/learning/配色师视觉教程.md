# 配色师视觉教程
## 从零开始系统学设计

> "设计不是'我觉得好看'，是'用户觉得合适'。配色是设计中最能快速提升的技能，也是最容易暴露功力的地方。"

---

## 一、设计基础：先理解再动手

### 1.1 设计的核心目标

**设计的本质是沟通。** 
每一个视觉元素的选择，都是在向用户传递信息。

配色三大原则（记住，这是前提）：
1. **可读性** —— 信息能不能被看清楚？
2. **层级** —— 什么重要，什么次要？
3. **情感** —— 用户感受到的情绪对不对？

## 二、色彩基础：设计师的语言

### 2.1 色彩三要素

**色相（Hue）：** 颜色本身叫什么（红、绿、蓝）
**明度（Value）：** 颜色有多亮或多暗
**饱和度（Saturation）：** 颜色有多鲜艳或多灰

### 2.2 色轮：配色的起点

```
红 → 橙 → 黄 → 绿 → 青 → 蓝 → 紫 → 红（360度闭环）
```

**原色：** 红、黄、蓝（色轮上等距的三个点）

**间色：** 橙（红+黄）、绿（黄+蓝）、紫（蓝+红）

**冷暖之分：**
- 暖色：红、橙、黄 —— 能量、热情、紧迫
- 冷色：蓝、绿、紫 —— 平静、专业、信任

### 2.3 专业色彩工具

| 工具 | 用途 | 推荐场景 |
|------|------|----------|
| Adobe Color | 色轮配色、提取图片色 | 所有场景 |
| Coolors | 快速生成配色方案 | 灵感探索 |
| Color Hunt | 收集现成配色方案 | 效率优先 |
| Material Design | Google色彩系统 | 数字产品 |
| Tailwind CSS | 内置调色板 | 开发对接 |

## 三、经典配色方法（核心技能）

### 3.1 单色配色（Monochromatic）

**方法：** 同一色相 + 不同明度/饱和度

```
主色：#3B82F6（蓝色）
浅色：#60A5FA
深色：#1D4ED8
背景：#EFF6FF
```

**优点：** 统一感强，视觉和谐，适合专业/科技类
**缺点：** 变化少，容易单调
**适用：** 金融、法律、医疗、科技SaaS

---

### 3.2 互补色（Complementary）

**方法：** 色轮上相对的两种颜色

```
蓝：#3B82F6
橙：#F97316
```

**使用技巧：** 不要五五开，用主色70% + 互补色30%
- 主色做背景，互补色做强调
- 或互补色只用于图标和按钮

**优点：** 对比强烈，视觉冲击大
**缺点：** 容易刺眼，搭配不当很灾难
**适用：** 需要高对比的场景（CTA按钮、促销Banner）

---

### 3.3 类比色（Analogous）

**方法：** 色轮上相邻的3-5个颜色

```
蓝：#3B82F6
青：#06B6D4
紫：#8B5CF6
```

**优点：** 和谐自然，视觉流畅
**缺点：** 对比弱，层级不清晰
**适用：** 背景、渐变、品牌色

---

### 3.4 三等分配色（Triadic）

**方法：** 色轮上等距的三个点

```
红：#EF4444
黄：#EAB308
蓝：#3B82F6
```

**优点：** 色彩丰富，视觉活跃
**缺点：** 容易杂乱，需要主次分明
**适用：** 儿童、创意、娱乐类内容

---

### 3.5 分裂互补（Split-Complementary）

**方法：** 一个主色 + 它的互补色两侧的两个色

```
主色：蓝 #3B82F6
辅助：#F97316 的两侧 → #FB923C 和 #FBBF24
```

**优点：** 保留了强对比，但更柔和
**缺点：** 搭配复杂度提升
**适用：** 想有活力但不想太刺眼的场景

## 四、色彩心理学：颜色传递什么信息

### 常用色彩含义

| 颜色 | 正面含义 | 负面含义 | 常用于 |
|------|----------|----------|--------|
| 红 | 热情、紧急、爱 | 危险、侵略 | 促销、CTA按钮 |
| 橙 | 活力、创意、温暖 | 廉价、轻浮 | 电商、折扣 |
| 黄 | 快乐、乐观、注意 | 焦虑、警告 | 提醒、注意 |
| 绿 | 成长、健康、自然、money | — | 金融、成功状态 |
| 蓝 | 信任、专业、平静 | 冷淡、疏离 | 企业服务、科技 |
| 紫 | 奢华、创意、神秘 | — | 高端、美妆 |
| 黑 | 精致、力量、权威 | 压抑、死亡 | 高端、奢侈品 |
| 白 | 纯净、简洁、空间感 | 冷漠、空洞 | 科技、极简设计 |

### 行业配色参考

**金融/银行：** 深蓝 + 金色 + 白 —— 信任 + 价值
**医疗健康：** 蓝绿 + 白 + 浅灰 —— 专业 + 清洁
**电商促销：** 红/橙 + 白 —— 紧迫 + 清爽
**高端奢侈品：** 黑 + 金 + 白 —— 质感 + 稀缺
**餐饮美食：** 暖橙 + 红 + 自然色 —— 食欲 + 温暖
**科技/SaaS：** 蓝 + 深灰 + 白 —— 专业 + 清晰

## 五、色彩系统：如何建立品牌色板

### 5.1 建立基础色板（Design Token）

```css
:root {
  /* 主色 */
  --primary-50: #EFF6FF;   /* 极浅（背景） */
  --primary-100: #DBEAFE;
  --primary-200: #BFDBFE;
  --primary-500: #3B82F6;   /* 标准 */
  --primary-600: #2563EB;
  --primary-700: #1D4ED8;
  --primary-800: #1E40AF;   /* 深色文字 */
  --primary-900: #1E3A8A;   /* 极深 */
}
```

**至少需要：**
- 主色（Primary）1个
- 辅助色（Secondary）1个
- 强调色（Accent）1个
- 中性色（Neutral）2-3个
- 语义色（Success/Error/Warning）各1个

### 5.2 语义色定义

```css
:root {
  --success: #10B981;   /* 绿：成功状态 */
  --warning: #F59E0B;   /* 黄：警告 */
  --error: #EF4444;     /* 红：错误 */
  --info: #3B82F6;      /* 蓝：信息提示 */
}
```

## 六、排版与色彩的关系

**色彩不是孤立存在的，是和文字、留白、图形一起工作的。**

### 文字与背景对比度

**WCAG 标准（无障碍 accessibility）：**
- 正文内容：对比度 ≥ 4.5:1
- 大字/图标：对比度 ≥ 3:1
- 背景色过浅会导致文字无法阅读

**检查工具：**
- WebAIM Contrast Checker（webaim.org/resources/contrastchecker）
- Figma 插件 Contrast

### 色彩面积法则

```
背景色：70%面积
主色：25%面积
强调色：5%面积
```

强调色不是越多越好。越少越值钱。

## 七、实战：做一个完整的配色方案

### 场景：SaaS产品后台

**Step 1：确定主色**
选择蓝色系（科技、信任、专业）
主色：#3B82F6

**Step 2：建立色阶**
```
浅背景：#EFF6FF
默认背景：#FFFFFF
深色背景：#1E40AF
主色：#3B82F6
辅助色：#8B5CF6（偏紫蓝，丰富层次）
```

**Step 3：定义中性色**
```
文字主色：#1F2937（深灰，不是纯黑）
文字次色：#6B7280（中灰）
边框色：#E5E7EB（浅灰）
背景色：#F9FAFB（极浅灰）
```

**Step 4：定义功能色**
```
成功：#10B981
警告：#F59E0B
错误：#EF4444
信息：#3B82F6
```

**Step 5：验证对比度**
- 主文字 #1F2937 在 #FFFFFF 上：15.8:1 ✅
- 次文字 #6B7280 在 #FFFFFF 上：5.7:1 ✅
- 按钮文字 #FFFFFF 在 #3B82F6 上：4.5:1 ✅

## 八、配色学习路径

### 入门阶段
1. 用 Adobe Color 色轮生成一套配色
2. 用 Figma 把同一套色应用在不同组件上
3. 对比哪种配色"感觉对"

### 进阶阶段
1. 临摹优秀App的配色系统
2. 学会用颜色表达层级，不依赖字号
3. 建立自己的配色灵感库（收集200+配色方案）

### 高手阶段
1. 能根据品牌调性反向推导配色策略
2. 动态配色（明/暗主题切换）
3. 色彩无障碍设计

## 十、配色检查清单

每次做完设计，逐项检查：

- [ ] 整体配色不超过5种颜色
- [ ] 有明确的主色，视觉焦点清晰
- [ ] 文字与背景对比度符合WCAG标准
- [ ] 不同状态有对应颜色区分（hover/active/disabled）
- [ ] 强调色只用于最重要的元素
- [ ] 深色模式和浅色模式的颜色逻辑一致
- [ ] 色盲测试通过（不要只用红绿区分）
