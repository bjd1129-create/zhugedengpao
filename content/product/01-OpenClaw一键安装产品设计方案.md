## 1. 产品概述

### 1.1 背景

当前 OpenClaw 安装依赖命令行操作，对非技术用户存在较高门槛：
- 需要打开终端/命令行
- 输入安装命令
- 配置环境变量
- 运行 onboarding

调研显示，**78%的潜在用户**在看到需要命令行安装后选择放弃（来自竞品AutoClaw数据）。

### 1.2 目标

打造**零门槛、一键式**的安装体验，让任何人都能在**60秒内**完成OpenClaw的安装和首次使用。

### 1.3 一键安装定义

用户只需：
1. 下载安装包
2. 双击运行
3. 等待自动完成

全程**无需输入任何命令**，安装完成即默认启动Gateway并进入引导界面。

## 2. 当前安装方式分析

### 2.1 现有安装方式

| 安装方式 | 命令 | 目标用户 | 门槛等级 |
|---------|------|---------|---------|
| 安装脚本 | `curl -fsSL https://openclaw.ai/install.sh \| bash` | 开发者 | ⭐⭐ |
| npm/pnpm | `npm install -g openclaw@latest` | 开发者 | ⭐⭐ |
| Docker | `docker run...` | 高级用户 | ⭐⭐⭐⭐ |
| 源码编译 | `git clone && pnpm build` | 贡献者 | ⭐⭐⭐⭐⭐ |
| 第三方AutoClaw | 桌面安装包 | 非技术用户 | ⭐ |

### 2.2 AutoClaw竞品分析

AutoClaw是目前市场上唯一的一键安装方案：
- **优点**：真正的一键安装，支持Windows/macOS桌面应用
- **缺点**：非官方维护，可能存在版本滞后和安全风险
- **数据**：宣称"Setup in under 1 minute"

## 3. 一键安装产品设计

### 3.1 方案A：官方桌面安装包（推荐）

#### 3.1.1 技术方案

使用 **Electron** 或 **Tauri** 构建跨平台桌面安装程序：

```
OpenClaw-Setup-{version}-{os}.{ext}
├── OpenClaw-Setup-1.2.0-win.exe
├── OpenClaw-Setup-1.2.0-mac.dmg
└── OpenClaw-Setup-1.2.0-linux.AppImage
```

#### 3.1.2 安装流程

```
用户双击安装包
    ↓
启动安装向导（Electron Installer）
    ↓
显示许可协议 → 用户同意
    ↓
选择安装目录（默认推荐）
    ↓
创建桌面快捷方式/开始菜单
    ↓
后台静默安装：
    ① 检测Node环境（无则自动安装）
    ② 安装OpenClaw CLI
    ③ 启动Gateway服务
    ④ 配置开机自启
    ↓
显示"安装完成" → 勾选"立即启动OpenClaw"
    ↓
打开Web UI引导界面
```

#### 3.1.3 关键功能点

| 功能 | 描述 | 优先级 |
|-----|------|--------|
| 静默安装 | 无需用户交互的自动安装 | P0 |
| 自动依赖检测 | 检测并安装Node运行时 | P0 |
| Gateway自启动 | 安装后自动启动本地Gateway | P0 |
| Web UI入口 | 打开浏览器进入引导界面 | P0 |
| 卸载程序 | 一键卸载，清理干净 | P1 |
| 更新检测 | 安装后检查更新 | P2 |

## 5. 技术实现

### 5.1 技术选型

| 组件 | 推荐方案 | 备选方案 |
|-----|---------|---------|
| 安装框架 | Electron Builder | Tauri |
| 打包工具 | electron-builder | @tauri-apps/builder |
| Node管理 | volta | fnm / nvm |
| 发行格式 | NSIS(Win) / DMG(Mac) / AppImage(Linux) | |

### 5.2 安装包结构

```
openclaw-setup/
├── installer/           # 安装逻辑
│   ├── preinstall.js   # 安装前检查
│   ├── install.js      # 安装流程
│   └── postinstall.js  # 安装后配置
├── resources/
│   ├── node/          # 嵌入式Node运行时
│   ├── icons/         # 应用图标
│   └── templates/     # 配置文件模板
└── uninstall/        # 卸载程序
```

### 5.3 自动更新机制

```
┌─────────────┐
│ 检查更新    │ ← 启动时自动检查
└──────┬──────┘
       ↓
┌──────────────┐
│ 发现新版本？  │ ──否──→ 启动主程序
└──────┬───────┘
       ↓ 是
┌──────────────┐
│ 提示用户更新  │ ──用户取消──→ 启动主程序
└──────┬───────┘
       ↓用户确认
┌──────────────┐
│ 下载更新包   │
└──────┬──────┘
       ↓
┌──────────────┐
│ 安装并重启   │
└──────────────┘
```

## 8. 风险与对策

| 风险 | 概率 | 影响 | 对策 |
|-----|-----|-----|-----|
| Electron包体积过大 | 高 | 中 | 使用Tauri替代（Rust更轻量） |
