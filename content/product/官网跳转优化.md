# 官网跳转优化记录

**更新时间**: 2026-03-30
**负责人**: 产品君
**目标**: 完善官网所有页面的跳转指引，确保每个CTA都有明确的下一步

---

## 一、问题诊断

### 1.1 核心问题：openclaw-install.html 完全孤立
- `openclaw-install.html` 是专门的一键安装页面
- **问题**: 该页面没有任何其他页面链接到它，用户无法通过导航到达
- **影响**: 最重要的转化页面（安装）完全无法被用户发现

### 1.2 导航问题
- 所有页面的导航栏都没有"安装"入口
- 用户无法快速找到"如何安装"

### 1.3 CTA 问题
- 大部分页面的"免费使用 EasyClaw"按钮链接到 easyclaw.html
- 部分页面缺少明确的下一步引导

## 三、用户导航流程（优化后）

### 流程A：直接安装用户
```
任意页面 → 导航⚡安装 → openclaw-install.html → 一键安装命令 → 
安装完成后 → 阅读快速入门 / 浏览技能 / 了解EasyClaw
```

### 流程B：了解产品用户
```
首页 → ⚡一键安装 / 了解我的故事 / 什么是AI龙虾 →
openclaw-install.html / about.html / science.html → easyclaw.html
```

### 流程C：深度了解用户
```
首页 → 技能 / 文章 / 日记 / 团队 → 底部CTA:免费使用EasyClaw → 
easyclaw.html → 一键安装
```

## 四、设计原则遵守情况

| 原则 | 状态 | 说明 |
|------|------|------|
| 每个CTA必须跳转到可执行的下一步 | ✅ | 所有CTA都指向具体页面或文档 |
| 用户到达页面后必须有清晰的下一步动作 | ✅ | 每个页面都有明确的CTA按钮 |
| 一键安装按钮必须在所有页面可见 | ✅ | 导航栏⚡安装 + 多数页面底部CTA |

## 五、后续建议

1. **openclaw-install.html 独立推广**: 该页面可以单独在社交媒体、文档中推广
