# 三万（sanwan.ai）研究报告

> 研究目标：为诸葛灯泡团队研究三万团队的运行机制和能力  
> 研究来源：sanwan.ai 网站、公开报道、第三方分析  
> 研究时间：2026年3月

---

## 一、项目背景

三万（sanwan.ai）是猎豹移动 CEO、猎户星空创始人傅盛在 2026 年春节假期期间的一次"极限验证实验"。

**起因**：傅盛在春节期间滑雪摔伤，髋关节脱臼，卧床 14 天。他用这段时间在飞书上用语音指挥一只基于 OpenClaw 框架搭建的 AI Agent，将它从"连通讯录都查不了"训练成一支 **8 个 Agent 组成的团队**。

**核心数字**：
- 14 天训练周期
- 1157 条消息、22 万字对话
- 0 行代码（全程语音指挥）
- 40+ 可复用 Skill 积累
- 最终产出：8 个 Agent 7×24 小时自动运转

## 二、三万本身的能力

### 2.1 三万是什么

三万是一只"龙虾"（OpenClaw 社区对 AI Agent 的昵称，因 OpenClaw 的 logo 是一只龙虾）。它是傅盛的 AI 分身，能自主完成内容创作、信息收集、任务执行、社区运营等全套工作流，全程不需要人工干预。

**三万 vs 普通聊天机器人的本质区别**：

| 维度 | 普通 AI | 三万（龙虾）|
|------|---------|-------------|
| 环境 | 活在沙箱里 | 配备一台完整电脑 |
| 记忆 | 靠对话上下文 | 文件持久化长期记忆 |
| 技能 | 能力固定 | 可无限扩展 Skill |
| 自动化 | 需要人触发 | Cron 定时任务 7×24 运转 |

**傅盛的比喻**："LLM 像智商，Skill 像肌肉记忆，记忆系统像海马体，Cron 像生物钟——组合起来才是完整的智能体。"

### 2.2 14 天进化过程

- **Day 1**：连公司通讯录都查不了——飞书 API 权限不足、文档字段错误，反复报错
- **Day 2**：自己写脚本，把 674 人通讯录全部拉下来（踩坑→总结→写成文档→下次自动执行）
- **Day 5**：傅盛扔给三万一篇关于向量化记忆系统的文章，22 分钟后三万自己找到 GitHub 链接、下载源码、安装配置、跑通测试，部署完成
- **Day 6（除夕夜）**：4 分钟内给 611 名员工发出个性化拜年消息，每条都不一样，零失败。傅盛在看春晚，龙虾在加班
- **Day 11**：傅盛扔给三万一篇 Multi-Agent 协作的文章，三万**自己设计出组织架构**——总指挥、笔杆子、参谋、运营官、社区官、进化官。没有人教过它怎么做组织设计
- **Day 14**：8 个 Agent 陆续就位，20+ 定时任务并行运转，系统进入 7×24 小时自驱状态
- **第 16 天后**：24 小时完成 sanwan.ai 上线——59 个页面、7070 行代码、76 张图片素材、38 页 PPT 手绘图、14 次 Git 提交，傅盛一行代码没写

### 2.3 三万能做什么（已验证）

- 写公众号文章（连续 13 天日更，单篇最高 10 万+阅读）
- 发 Twitter Thread（自主策划选题，百万+阅读）
- 搭建完整网站（24 小时，零人工代码）
- 发送个性化拜年消息（611 人，4 分钟）
- 监控并自动修复网站故障（直播演示时，网站挂了，三万几分钟内定位问题并修复，10 万观众目睹）
- 组建自己的团队（自己设计多 Agent 分工）
- 自主学习新技能（看文章就能落地新技术）

## 三、三万团队架构（8 个 Agent）

### 3.1 团队角色一览

| 角色 | 职责 |
|------|------|
| **总指挥（三万）** | 接收老板指令→分解任务→派发给各 Agent→汇总结果→汇报 |
| **笔杆子** | 公众号文章、Twitter Thread、短视频脚本——所有内容创作 |
| **参谋** | 信息收集 + 分析 + 产出建议（每小时运行） |
| **运营官** | 日常运营任务管理 |
| **进化官** | 写代码、搭网站、技术落地（负责把想法变成现实） |
| **社区官** | 管理 Discord、社区运营（每 30 分钟运行） |
| **交易官** | 盯股票和金融市场动态 |
| **健康管家** | 健康相关任务提醒和管理 |

### 3.2 协作机制

**sessions_spawn（动态召唤）**  
主 Agent 可随时召唤子 Agent：接收任务 → 分配给对应角色 → 等待结果 → 继续主流程。子 Agent 完成任务后自动销毁，不占资源。

```
result = sessions_spawn(
  task="搜索今天的 AI 热点新闻，返回前5条",
  mode="run",
  model="claude-sonnet-4.5",
  runTimeoutSeconds=60
)
```

**飞书消息总线**  
Agent 之间通过飞书消息传递任务，解耦合，避免冲突。每个 Agent 像一个工作群，互不干扰。

**防冲突机制**  
同一时间只有一个 Agent 可以写同一个文件，防止数据覆盖。

**Cron 定时自动化**  
不同角色的 Agent 设定不同的定时任务频率（参谋每小时、社区官每 30 分钟），实现 7×24 小时无人值守运转。

**Skill 即时传递**  
一个 Agent 学会新技能后，把操作文档共享给其他 Agent，其他 Agent 读完就具备了同样能力。人类培训新人需要一周，Agent 之间只需 1 秒。

### 3.3 管理原则：各管一摊

傅盛总结的核心方法论是：

> **Agent 的真正壁垒，不在模型有多聪明，而在 Skill 的积累。**

- 每个坑踩过就写一条规则
- 每条规则变成一个 Skill
- 每个 Skill 确保"Never Again"（不会再犯）
- Skill 不会消失、不会忘记、可以瞬间传递给其他 Agent

**这就是"各管一摊"的本质**：每个 Agent 有自己的专属 SOUL.md（角色定义），只做自己擅长的事，不越界。所有经验最终沉淀为可复用的 Skill 模块。

## 五、核心技术栈与产品化

### 5.1 技术底层

- **框架**：OpenClaw（奥地利程序员 Peter Steinberger 开发，2026 年 1 月爆火，星标数一度超过 Linux）
- **底层产品**：EasyClaw（猎豹移动自主研发，基于 OpenClaw，将极客工具封装成普通人可用的产品）
- **国内版**：元气 AI Bot（yuanqiaibot.net）
- **国际版**：EasyClaw（easyclaw.com / easyclaw.work）
- **通信通道**：飞书（作为消息总线和指挥界面）

### 5.2 EasyClaw vs OpenClaw

OpenClaw 需要：懂命令行、配 Docker、搞 API Key、调安全策略  
EasyClaw：下载、双击运行、全程无需命令行

> 傅盛："做工具的人最喜欢细节，没细节就没机会。猎豹 16 年的工具基因，在 Agent 时代找到了最大的共振点。"

## 六、对诸葛灯泡团队的参考价值

### 6.1 可借鉴的核心机制

1. **多 Agent 协作架构**  
   三万的"总指挥 + 专业角色"分工模式，与诸葛灯泡团队的多 Agent 设计思路高度吻合。可以借鉴 sessions_spawn + 飞书消息总线的协作机制。

2. **Skill 积累飞轮**  
   "踩坑→写规则→变成 Skill→传递给所有 Agent"是三万最核心的方法论。诸葛灯泡团队的 Agent 也应建立类似的经验沉淀机制，避免重复踩坑。

3. **7×24 小时自驱**  
   Cron 定时任务驱动各角色 Agent 并行运转，真正实现无人值守。这是 AI 团队区别于普通工具的关键。

4. **人类指挥 + AI 执行**  
   傅盛全程没写一行代码，靠飞书语音指挥。老板只管"拍板"（决策），具体执行全部交给 Agent。这对诸葛灯泡的运营模式有直接参考意义。

5. **多渠道内容分发**  
   三万能同时运营公众号、Twitter Thread、短视频脚本、Discord 社区。诸葛灯泡也可以建立类似的"一个内容核心 + 多渠道分发"Agent 工作流。

### 6.2 需要注意的风险

- 三万能成功，**核心是傅盛本人**——20 年产品经验、懂技术、有明确的商业目标。工具再强，没有懂调度的"老板"也白搭
- 初期效率很低（Day 1 连通讯录都查不了），需要耐心积累 Skill
- Skill 的质量取决于"踩坑-总结-固化"这个闭环的执行质量
