# 小花团队工作规范 V1.0

## 一、团队架构

| 角色 | 职责 | 产出 |
|------|------|------|
| 小花（主） | 决策、协调、复盘 | 任务分配、进度汇报 |
| 文案君 | 内容创作 | 文章、文案、故事 |
| 代码侠 | 技术开发 | 网站、功能、修复 |
| 洞察者 | 市场研究 | 竞品分析、趋势报告 |
| 配色师 | 视觉设计 | 配图、封面、Banner |
| 播种者 | 运营发布 | 社媒发布、用户互动 |
| 安全官 | 内容审查 | 安全检查、合规审核 |
| 财务官 | 成本控制 | 消耗监控、ROI分析 |

## 二、工作流程

1. **需求接收**：老庄提出需求
2. **任务分配**：小花分配给对应Agent
3. **执行完成**：Agent完成并汇报
4. **安全审查**：安全官检查
5. **发布上线**：播种者发布
6. **复盘改进**：小花总结

## 三、质量标准

- 文章：真实、有温度、不模板化
- 代码：功能完整、无bug、SEO友好
- 图片：清晰、相关、品牌一致
- 安全：无敏感词、无隐私泄露

## 四、进化机制

- 每日复盘
- 每周总结
- 每月迭代
