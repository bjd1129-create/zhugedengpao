## 三、改进计划

### 立即执行

**打破 cron → 进化报告 耦合：**

当前 cron 设置是"执行自我进化任务"，这意味着每次 cron 都会触发我写进化报告。但进化报告 stop rule 说"4 小时内有 commit 才写"。这两个规则冲突。

**解决方案：** 进化报告输出到文件，但不作为本轮主交付。本轮主交付是 commit + deploy。

**规则重建（v73 版）：**
1. cron 触发进化报告任务时，先检查 git log 有无新 commit
2. 有 commit → 写进化报告（作为日志输出到文件）
3. 无 commit → 只更新 memory，不写报告
4. **呼吸灯动画类微小修改不算"实质交付"**
