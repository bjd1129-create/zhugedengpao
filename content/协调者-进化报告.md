## 二、自身不足识别

### 2.1 角色边界执行仍然不够彻底

**问题**：上次进化承诺"不碰代码文件和文案内容文件"，但实际情况是：

- 我仍然在主动审阅和判断子Agent的产出（配色师报告、文案君报告、战略官报告）
- 我花了大量时间在理解团队产出，而不是直接推进
- 有些报告我做了大量批注，但实际上批注不是协调者该做的事

**本质**：协调者是裁判，不是教练。我不需要理解每个细节，我只需要知道方向对不对、资源够不够、风险大不大。

### 2.2 风险预判机制仍然缺失

**上次进化说要建立批量任务预判机制，但这周没有建立。**

**这次新教训**：
- 配色师120图任务，事前没有确认月度配额，只确认了RPM限速
- bailian API调用没有确认是否支持，在任务跑了两次失败后才知道不对
- 对比优化cron没有评估计算量，导致超时

**本质**：我还是在用"试试看"的思维做事，而不是"先算清楚再做"。

### 2.3 任务委派标准模糊

**问题**：哪些任务应该委派给哪个子Agent，没有明确标准。

**现状**：
- 有些任务重复委派给多个Agent（设计师品牌内容规划被多人做）
- 有些任务分配后，子Agent方向走偏，需要返工
- 任务描述不够具体，缺少"做什么+做到什么程度+什么时候完成"三要素

**新认知**：委派任务时，标准格式应该是：
```
任务：[具体描述]
标准：[做到什么程度算完成]
截止：[什么时候]
```

### 2.4 跨Agent协作流程不清晰

**问题**：多个Agent需要协作时，没有标准流程。

**教训**：设计师品牌平台分析，文案君/洞察者/配色师都在做，但没有明确谁主导、谁配合、产出如何合并。结果是各自做了很多重复工作。

**需要建立的标准**：
- 多Agent协作，必须指定一个"主导Agent"
- 其他Agent做"配合"，只提供素材，不单独产出完整报告
- 主导Agent负责合并和最终输出

### 2.5 对子Agent能力边界了解不够

**问题**：我分配任务时，有时低估或高估了子Agent的能力。

**案例**：
- 配色师做MiniMax文生图时，我只给了场景和风格变体列表，但没给具体prompt模板，导致生成的图良莠不齐
- 对比优化cron任务，我以为代码侠能做，但实际上这个任务需要AI实时对比两个网站，120秒不够

**本质**：我不了解每个子Agent能做什么、不能做什么、擅长什么、不擅长什么。

## 三、改进计划

### 3.1 立即执行：建立Agent能力档案（本周内）

为每个子Agent建立一张卡片，包含：
```
Agent：[名字]
职责范围：[能做的]
能力边界：[不能做的]
擅长：[具体强项]
短板：[具体弱项]
常用工具：[skill名称]
```

**本周完成**，并存入 `agents/AGENT-CAPABILITIES.md`。

### 3.2 立即执行：任务委派标准格式（立即生效）

所有委派任务，必须包含：
```
任务：[具体要做什么]
标准：[做到什么程度]
截止：[时间]
是否需要协作：[是/否]
主导Agent：[名字]（如需要协作）
```

不再发模糊的任务描述。

### 3.3 本周内：批量任务预判清单（本周内建立）

**任何批量任务启动前，必须完成这个checklist：**

| 检查项 | 问题 | 如果不符合怎么办 |
|--------|------|----------------|
| API配额 | 任务总量 vs API限制 | 先报告老庄，确认资源够再做 |
| 预计耗时 | 单次 × 总量 vs 截止时间 | 调整任务规模或延长截止 |
| 超时风险 | 单次执行是否可能超时 | 优化方案或拆分任务 |
| 成本估算 | Token/费用消耗预估 | 先报预算 |

这个清单写入 TOOLS.md，供每次批量任务前查阅。

### 3.4 本周内：失败任务复盘机制（本周内建立）

**任何cron任务连续失败2次，立即：**
1. 暂停任务
2. 分析失败原因
3. 制定修复方案
4. 修复后重新启动，并设置观察期（连续3次成功才算恢复）

**当前失败任务清单：**
- bailian API（glm-5）：400错误 → 暂停，检查API配置
- 每3小时对比优化cron：超时 → 暂停，重新评估任务规模

### 3.5 持续改进：协调者"只退不退"原则

**新承诺：**
- 我只做：决策、协调、分配、记录、风险预警
- 我不做：直接审阅子Agent报告内容（除非老庄要求）
- 子Agent产出有质量问题，交给对应的子Agent自己改进，我不代替他们思考

## 五、核心认知更新

> 这次进化最重要的认知：**协调者的核心能力不是"做判断"，而是"让判断发生"。**

之前我花大量时间在理解子Agent的产出对不对、好不好。这其实是越界——那是子Agent自己的责任。

我应该花时间在：
- 任务是否清晰、是否能完成
- 资源是否足够、风险是否可控
- 多Agent协作是否有冲突

不是判断产出好不好，而是确保做事的条件是否具备。
