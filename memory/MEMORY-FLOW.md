# 记忆系统规范

## 核心理念

**"Mental notes don't survive session restarts."**
**必须写文件！**

---

## 三层记忆架构

### 第一层：每日日记（memory/YYYY-MM-DD.md）
- 每天结束前必须写
- 记录：做了什么事、学到什么、灵感、决定
- 格式：日期标题 + bullet points
- 由主Agent主动写入

### 第二层：长期记忆（MEMORY.md）
- 精选重要的内容
- 老庄的背景、品牌定位、踩过的坑
- 每周整理一次
- 不会被清理

### 第三层：上下文（当前对话）
- 只在当前会话有效
- 对话结束自动压缩

---

## 触发机制

### 1. 重要决定 → 立即存档
当你做一个重要决定时（如：品牌定位、团队配置），立即写入MEMORY.md

### 2. 每天结束前 → 自动写入记忆
每天对话结束前，检查是否写了memory/今天.md，如果没有就写

### 3. 定期整理 → 每周日晚
把本周的重要内容精选到MEMORY.md

---

## 记忆写入标准

### ✅ 必须写入
- 老庄说的重要背景（职业、家庭、经历）
- 品牌定位、方向的决定
- 用户反馈（喜欢什么、不喜欢什么）
- 踩过的坑、教训
- 灵感、想法

### ❌ 不写入
- 临时性的问题
- 一次性任务
- 纯技术调试过程

---

## 格式模板

### 每日日记
```markdown
# YYYY-MM-DD 日记

## 今日完成
- ...

## 老庄说的重要的事
- ...

## 灵感/想法
- ...

## 踩过的坑
- ...

## 明天要做什么
- ...
```

### 长期记忆
```markdown
# 老庄背景
- ...

# 品牌定位
- ...

# 踩过的坑
- ...

# 重要决定
- YYYY-MM-DD: ...
```

---

## 检查清单

每次对话结束前自检：
- [ ] 今天的事写入memory/今天.md了吗？
- [ ] 有重要决定写入MEMORY.md了吗？
- [ ] 有灵感记录下来了吗？
