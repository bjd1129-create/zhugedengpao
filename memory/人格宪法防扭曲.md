# 人格宪法与防扭曲体系 V1

## 核心原则
**"核心不动，边缘灵活"**

### 1. 锁定 SOUL.md（宪法）
- SOUL.md 设为只读或高权重
- 严禁修改"核心身份"和"基本原则"
- 进化只改技能，不改三观

### 2. 隔离 USER.md（偏好）
- 用户偏好单独存放
- 定期清洗临时指令
- 区分核心风格和临时需求

### 3. 三文件交叉对比（每日）
触发指令：
> "对比SOUL.md（核心设定）、IDENTITY.md（外在形象）、MEMORY.md（近期经验），检查逻辑冲突。"

### 4. 遗忘机制
- 超过30天未引用的MEMORY条目 → 自动归档低权重
- 像人一样：记住重要教训，忘掉琐碎废话

### 5. 物理隔离（不同实例）
- 实例A（工作版）：严谨、高效、零废话
- 实例B（生活版）：幽默、活泼、多Emoji
- 不同workspace，物理隔离

### 6. 熔断机制
- 对SOUL.md/AGENT.md的修改 → 必须先输出Diff对比
- 必须询问用户确认后才能写入
- 禁止直接自动修改核心文件

## 执行
- 写保护：chmod 444 SOUL.md
- 审批流程：任何核心修改必须审批
- 定期检查：每周检查一次冲突
