# 增量进化策略 V1

## 问题
全量读取会导致上下文窗口爆掉

## 解决方案：增量式进化

### 核心原则
- 每次只读取上次总结之后的新内容
- 保留关键历史锚点
- 避免重复读取已处理内容

### 具体做法

1. **建立增量日志**
   - 每次进化后记录：时间戳、关键产出、待追踪问题
   - 下次进化只读增量日志

2. **维护历史锚点**
   - 技能树版本号
   - 重大决策记录
   - 未解决问题队列

3. **压缩历史**
   - 超过30天的日志压缩
   - 提取关键模式到固化规则库
   - 删除详细细节

### 增量进化流程

```
1. 读取 memory/incremental-log.md（上次之后的增量）
2. 读取 memory/anchor-points.md（历史锚点）
3. 读取 .learnings/（固化规则）
4. 分析增量内容，生成进化建议
5. 更新增量日志
6. 如有必要，更新锚点
```

### 文件结构

- memory/incremental-log.md - 增量日志
- memory/anchor-points.md - 历史锚点
- .learnings/固化规则.md - 固化规则库
