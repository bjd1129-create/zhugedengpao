# Case 010: 工作区切换后配置文件路径丢失反斜杠

## 元信息
- **case_id**: 010
- **发生时间**: 2026-03-31
- **症状分类**: Config
- **关键词**: 路径错误, C:openclaw-pc, 反斜杠丢失, 工作区切换

---

## 症状表现
| 检查项 | 状态 |
|--------|------|
| 配置文件路径 | `C:openclaw-pc`（缺少 `\`）|
| 正确路径 | `C:\openclaw-pc` |
| 影响范围 | 默认配置 + 11 个 Agent（共 12 处）|
| Gateway | 需重启才生效 |

---

## 根因

**工作区切换导致配置文件路径反斜杠丢失**：

1. Windows 路径格式应为 `C:\openclaw-pc`
2. 配置写入时反斜杠被转义或丢失，变成 `C:openclaw-pc`
3. 影响默认配置 + 所有 11 个 Agent 的工作区路径（共 12 处）
4. 导致工作区切换失败或指向错误路径

---

## 诊断步骤
```
[1/3] 检查配置文件
    命令：cat ~/.openclaw/config.yaml
    结果：发现 12 处路径错误

[2/3] 查找具体错误位置
    命令：grep "C:openclaw-pc" ~/.openclaw/config.yaml
    结果：列出所有包含错误路径的位置

[3/3] 验证影响范围
    结果：默认配置 2 处 + 11 个 Agent 各 1 处
```

---

## 修复步骤
```
[1/2] 批量修复路径（逐一替换）
    命令：sed 's/C:openclaw-pc/C:\\openclaw-pc/g' ~/.openclaw/config.yaml
    结果：12 处路径全部修复

[2/2] 重启 Gateway 使配置生效
    命令：openclaw gateway restart
    结果：Gateway 重启，配置生效
```

## 经验教训
1. **Windows 路径必须用双反斜杠** `\\` 或正斜杠 `/`，避免转义丢失
2. **工作区切换时配置会重新写入**，要确保路径格式正确
3. **配置写入后要验证**，检查路径是否被正确解析
4. **影响所有 Agent** 的配置要一起修复，不能遗漏
