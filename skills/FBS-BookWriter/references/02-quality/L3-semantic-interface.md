# L3 语义评分接口规范

> **版本**：v1.0
> **日期**：2026-03-21
> **用途**：企业B2B语义相关性评分接口设计（供平台/SDK实现参考）

---

## 1. 接口定位

L3语义评分属于**质量评分体系第三层**，位于：
- L1（语言层）：字词级错误检测
- L2（结构层）：B层篇级节奏/标点/结构检测
- **L3（语义层）：全文级语义相关性检测** ← 本文定义

**调用时机**：
- S5终审时触发（全文 assembled 后）
- 企业B2B订单强制执行
- 分数输出给 `references/metrics.md` 评分记录

---

## 2. 评分维度

| 维度 | 说明 | 权重 |
|------|------|------|
| 主题一致性 | 全文是否围绕书籍主题展开 | 25% |
| 论点连贯性 | 章节间逻辑是否连贯、递进 | 30% |
| 案例相关性 | 引用案例/数据是否紧扣论点 | 20% |
| 结论支撑度 | 结论是否有足够论据支撑 | 15% |
| 反腐检测 | 是否存在自言自语/AI填充废话 | 10% |

---

## 3. 输入输出规范

### 3.1 请求格式

```json
{
  "method": "L3_SEMANTIC_SCORE",
  "params": {
    "book_id": "string",
    "full_text": "string (全书MD内容)",
    "theme": "string (书籍主题)",
    "target_reader": "string (目标读者)",
    "chapter_outline": ["array of chapter titles"],
    "expected_length": "number (字数)"
  },
  "id": "string (请求唯一ID)"
}
```

### 3.2 响应格式

```json
{
  "id": "string (对应请求ID)",
  "result": {
    "l3_score": 8.5,
    "dimensions": {
      "theme_consistency": { "score": 8.8, "issues": [] },
      "argument_coherence": { "score": 8.2, "issues": ["第三章论点与第二章有轻微跳跃"] },
      "case_relevance": { "score": 9.0, "issues": [] },
      "conclusion_support": { "score": 8.0, "issues": ["第五章结论支撑不足"] },
      "ai_filler_detection": { "score": 9.5, "issues": [] }
    },
    "overall_pass": true,
    "warnings": ["第八章案例与主题相关性偏低"],
    "processing_time_ms": 2340
  }
}
```

## 4. 评分标准

| 分数段 | 评级 | 说明 |
|--------|------|------|
| ≥9.0 | 优秀 | 语义高度相关，可直接发布 |
| 8.0-8.9 | 良好 | 有少量优化空间 |
| 7.0-7.9 | 合格 | 需修改后发布 |
| <7.0 | 不合格 | 触发打回，需要大幅修改 |

**企业B2B门槛**：L3 ≥ 8.0（硬性要求）

## 5. 实现建议

### 5.1 LLM驱动方案（推荐）

```
输入 → LLM(角色=严格评审) → 多维度打分 → 结构化输出JSON
```

提示词模板：
```
你是一位严格的内容质量评审专家。请对以下书籍全文进行L3语义评分。

主题：{theme}
目标读者：{target_reader}
章节结构：{chapter_outline}

评分维度（每项1-10分）：
1. 主题一致性（25%）：全文是否围绕主题展开
2. 论点连贯性（30%）：章节间逻辑是否连贯
3. 案例相关性（20%）：案例是否紧扣论点
4. 结论支撑度（15%）：结论是否有足够论据
5. AI废话检测（10%）：是否存在无效填充内容

输出格式（严格JSON）：
{
  "l3_score": 总分,
  "dimensions": {各维度分数和具体问题},
  "warnings": ["优化建议"],
  "overall_pass": boolean
}
```

### 5.2 规则降级方案

当LLM不可用时，使用规则匹配降级：
- 主题关键词覆盖率 < 60% → 主题一致性警告
- 连续3段无数据引用 → 论据不足警告
- AI废话模式匹配 > 10% → 降分处理

## 6. 与现有B3的关系

| 维度 | B3（全局节奏） | L3（语义层） |
|------|---------------|-------------|
| 关注点 | 节奏/风格/结构 | 语义/逻辑/相关性 |
| 粒度 | 篇级（章节） | 全局（全书） |
| 评估方式 | Critic-L3人工+规则 | LLM语义分析 |
| 用途 | 所有书籍 | 企业B2B强制 |

**关系**：B3是L3的前置检测；B3通过后L3才执行
