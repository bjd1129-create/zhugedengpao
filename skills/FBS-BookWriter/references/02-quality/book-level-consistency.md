# 全书级一致性（C0 · 汇编门禁）

> **定位**：弥补「单章合规、全书漂移」的结构性缺口。与 S/P/C/B/V1 的 **20 分原始分**并列执行，**不计入**该 20 分；**未通过 C0 则不得宣告终稿通过或对外发布**（与 `quality-check.md` C0 表一致）。  
> **触发场景**：多路并行写编、分章通过后合稿、S5 终审前与修订后。  
> **版本**：1.1.2（C0-4 主题一致性 + **CX** 全书跨章挂载）

---

## 1. 问题根因（为何单章体系会漏）

| 现象 | 根因 |
|------|------|
| 「——」规则写得很严，全书仍超标 | S6 多按**章**计量；每章略低于阈值，**全书累加**仍可能远超；规则若只被当作「审校打分」而**未绑定 Writer 改写—复扫闭环**，执行易流于形式。 |
| 同一缩写多义（示例条目见 `abbreviation-audit-lexicon.json`） | S/P/C/B 侧重**句段章**的形式与节奏，**缺少**「核心术语跨章唯一含义」检查项；并行写作时各路只见己方章节，矛盾在**汇编鸟瞰**时才可发现。 |

**结论**：除逐章 S6 外，必须增加 **全书尺度** 的总账与 **术语契约**，并在流程上强制 **违规→改写→再扫** 直到达标或用户书面接受风险。

---

## 2. Writer ↔ Critic（S6）执行闭环

**原则**：S6（及同类标点/去 AI 味项）对 **Writer 与 Critic 同时生效**——不是「只给 Critic 看」的参考条。

1. **初稿**：Writer 产出章节（或修订段）后，触发 Critic-S（或等价自动化）按 `quality-S.md` **S6** 计数。  
2. **若 S6 未达标**（含破折号、分号、感叹号等任一项越阈）：  
   - **禁止**仅以「替换标点符号」敷衍（如仅把「——」改为逗号导致语义残缺）。  
   - Writer 须做**句式级重写**，必要时拆句、调换信息顺序。  
3. **改写验收**：修订后必须对**本次改动覆盖范围**重新执行 S6（整章重扫或变更段增量扫描，由宿主实现；**不得**默认免检）。  
4. **主编/主会话**在合并多路稿件时，若发现某路长期「警告即放行」，须在质量账本记录**技术债**，并在 S5 前集中清零或取得用户确认。

> **与 `quality-check.md` §1.0 的关系**：§1.0 要求改稿后重扫 S2/S4/S5/S6；本条强调 **S6 未达标时的改写义务与复扫闭环**，二者叠加执行。

## 3. C0-1：全书破折号（——）总密度

**编排层可对齐实现**：技能包内 `integration/lib/BookLevelConsistency.js` 提供与下表一致的 **ρ_em** 计算；`BookWorkflowOrchestrator` 的 `s5_release_gate` 在传入 `fullManuscriptText` 时自动写入门禁结果（见 `efficiency-implementation.md`）。

**计量**：对**计划纳入终稿的正文**（可含各编章 MD，排除代码块/ frontmatter 由工具约定）统计：  
`ρ_em = 全书中文等价字数内「——」出现次数 ÷ 千字`（与 S6 口径一致，尺度改为**全书**）。

**阈值**（与 `quality-S.md` S6 对齐，避免双标准）：

| 档位 | 条件 | 处理 |
|------|------|------|
| ✅ 通过 | ρ_em ≤ **1** | 记录数值即可 |
| ⚠️ 警告 | **1** < ρ_em ≤ **3** | 须列出超标章段清单，Writer 按 §2 闭环改写后重算 |
| ❌ 阻断 | ρ_em > **3** | **禁止终稿发布**；全书级降密通过后重新跑 C0-1 |

**执行时机**（至少满足其一，建议都做）：  
- **S5 合稿后、用户确认终稿前**必跑一次；  
- **多路并行**各编完成后、**汇编合并后**再跑一次（防止「分章都过、全书爆量」）。

## 4. C0-2：核心术语表与缩写唯一性

**建表**（建议在 **S1 定位 / S2 目录** 阶段由主编落地为可检索工件，如 `GLOSSARY.md` 或 `[S1]/[S2]` 固定小节）：  

- 列出：**缩写 / 英文全称（若有）/ 中文首选译名 / 禁止混用的异义**。  
- 对易撞车缩写须在表内**显式选定本书唯一含义**，其余含义**改用全称或其他缩写**，不得在同一本书正文内混用；**专项审计用词示例清单**见同目录 [`abbreviation-audit-lexicon.json`](./abbreviation-audit-lexicon.json)（可增删条目，**勿**在业务脚本中写死具体缩写）。

**汇编检查**（S5 前由主编或指定 Proofer 执行）：  

1. 对表内每个缩写做全文检索，核对每次出现语境是否与表定义一致。  
2. 对**未入表但反复出现的自造缩写**，补录或改为全称。  
3. **并行写作**：合并分支时，**比较各编「术语表」**；冲突须在合稿中消解，不得留到终稿。

> **与 G 层区别**：G4 偏数据与来源；C0-2 专指**概念定义与缩写语义**，不可互相替代。

---

## 5. C0-3：分号 / 感叹号（全书总账，可选强化）

若项目启用「全书总账」模式，可对 S6 中**分号（；）密度、感叹号（！）总数**做与 §3 同构的全书核算，阈值仍以 `quality-S.md` S6 为准。  
**默认强制**：至少执行 **C0-1 + C0-2**；C0-3 由用户在 S0/S1 与主编约定是否开启。

## 6. C0-4：主题一致性（P0 强制，新增）

> **目的**：防止上下文跳变（如「龙虾」→「家谱」），确保全书主题一致。  
> **关联**：[`topic-consistency-gate.md`](../05-ops/topic-consistency-gate.md)

### 检查项

| 检查点 | 检查内容 | 通过标准 |
|--------|----------|----------|
| C0-4a | S0 简报首行必须包含主题声明 | `**主题**：{topic}` 存在且非空 |
| C0-4b | 各阶段产出物主题一致性 | 所有 `[S1]`–`[S6]` 文件首行主题与 S0 一致 |
| C0-4c | 工作记忆读取校验 | 读取的记忆文件主题与当前主题一致 |
| C0-4d | 多智能体并行主题广播 | 所有并行 Agent 收到相同锁定主题 |

### 执行时机

- **S0 完成时**：锁定主题并写入简报首行  
- **S1–S6 各阶段开始时**：校验主题一致性  
- **S5 终审时**：作为发布门禁之一  

### 失败处理

| 场景 | 处理方式 |
|------|----------|
| 主题不一致 | 阻断发布，询问用户是否切换主题或修正文件 |
| 无主题声明 | 阻断进入下一阶段，要求补充主题声明 |
| 多路主题冲突 | 要求主编统一主题后再合并 |

## 7. 与阶段工作流的挂载点

| 阶段 | 动作 |
|------|------|
| S0 | **锁定主题**并写入 S0 简报首行；建立主题校验基准。 |
| S1–S2 | 创建并维护 **核心术语表**；锁定易混缩写本书含义；**主题一致性校验**。 |
| S3–S4 | 每章仍执行 **逐章 S6**；Writer 对未达标项按 §2 闭环；**阶段切换时主题校验**。 |
| 汇编（多路合并后） | 执行 **CX** 跨章一致性（见 §9 与 [`cross-chapter-consistency.md`](./cross-chapter-consistency.md)）；再执行 **C0-1**（及可选 C0-3）；**C0-2** 交叉检索；**C0-4** 主题一致性。 |
| S5 | `quality-check.md` **C0**（含 C0-4 主题一致性）与既有 S5-G* 一并满足后，方可进入发布确认；**多路并行**时 **CX** 须在 C0 前或与之并行完成并留痕。 |

## 8. 报告字段建议

在终稿质量报告或 `[S5]` 附件中建议单列：

- `bookEmDashPerThousand`（数值 + ✅/⚠️/❌）  
- `glossaryPath` + `glossaryConflictResolved`（布尔或说明）  
- `s6RemediationRounds`（全书或终盘阶段改写—复扫轮次摘要）  
- `lockedTopic` + `topicDriftEvents`（主题锁与漂移处置摘要，可与 `.fbs/topic-lock.json` 对齐）  
- `crossChapterAuditPath` + `cxOpenIssues`（CX 报告路径与未闭环项，多路并行时建议必填）

## 9. CX：全书跨章一致性（手册级）

**条文全文**：[`cross-chapter-consistency.md`](./cross-chapter-consistency.md)  

**摘要**：在 **≥2 Writer 并行** 或 **对称结构章节** 场景下，主编须组织 **CX-1～CX-6**（论述一致、数据交叉验证、对称性、重复论述、框架呼应、口径一致），输出《全书一致性审查报告》。**CX** **不替代** C0（术语/标点/主题）；二者叠加。*勿与 `quality-PLC.md` §C1（章内表态）混淆。*
