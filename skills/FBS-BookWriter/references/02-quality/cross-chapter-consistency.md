# 跨章一致性（全书层 **CX** · 手册）

> **定位**：弥补 S/P/C/B 以**单章**为主、**不覆盖章间逻辑**的缺口。**不计入** S/P/C/B 二十分；与 **C0** 并列，在 **多路并行合稿后、S5 终审前** 执行。  
> **术语**：本书 **CX** = 全书跨章层；**勿与** `quality-PLC.md` §**C1**（章内「表态」条款）混淆（测试报告 05）。  
> **版本**：1.18.9

---

## 1. 何时必须做 CX

| 场景 | 是否执行 CX |
|------|----------------|
| 单会话单 Agent 逐章串行，且后章写作时可见前章定稿 | 可选（仍建议 S5 前抽检） |
| **≥2 名 Writer 并行写不同章节** | **必须** |
| 对称结构章节（如产品 A vs 产品 B 深度剖析） | **必须**（含对称性检视） |

---

## 2. 检查项（CX-1 — CX-6）

主编或 **全书审校角色（Book Auditor）** 按下列维度出一份《全书一致性审查报告》（Markdown 即可）。

| 编号 | 内容 | 通过标准（建议） |
|------|------|-------------------|
| **CX-1** | 同一主题在不同章的**论断**是否矛盾（含推测 vs 已证实混用） | 列出矛盾点 = 0 或已标注「估算/待核实」 |
| **CX-2** | **同一数据点**多章出现时数值、单位、口径是否一致 | 不一致 = 0 或已在附录说明口径 |
| **CX-3** | **对称章节**小节结构、深度是否大体对等（避免一详一略造成偏见） | 显著不对称须说明或改稿 |
| **CX-4** | **重复论述**：核心段落是否在多章完整复述 | 后续章改为引用前文或压缩复述 |
| **CX-5** | 前文提出的**分析框架**（如对比表、瓶颈清单）后文是否呼应 | 无头无尾 = 0 |
| **CX-6** | 同一来源在不同章引用时**口径**一致 | 冲突须统一 |

> **自动化**：`integration/quality-auditor.mjs` 仍以启发式为主；CX 以 **人工 / 独立 Auditor 智能体** 为主；独立提示词模板见 [`../03-product/book-auditor-prompt.md`](../03-product/book-auditor-prompt.md)。

## 3. 与 C0、S5 的顺序

```
多章初稿齐备 → 汇编合并
  → CX（本书跨章逻辑与数据）
  → C0（本书）术语 / 破折号 / 主题锁
  → S5 终稿流程（quality-check、Proofer、G 层等）
```
