# 禁用词表维护流程（技能包内可执行）

> **定位**：审计提出的「动态进化」在工程未接入 API 前，用**固定节奏的人工维护**落地；与 [`quality-S.md`](./quality-S.md) frontmatter `version` 联动。

## 1. 触发节奏

| 触发 | 动作 |
|------|------|
| **每季度首周** | 维护者联网浏览「AI 生成文本检测」「中文公文/营销高频套话」等公开讨论，记录**新增模式**提案 |
| **本技能大版本发布前** | 必审 A/B/C 三类词表与 `quality-S.md` 版本号 |
| **用户集中反馈误报/漏报** | 开 issue 或内部表登记例句，纳入下轮词表评审 |

## 2. 变更流程

1. 在 `quality-S.md` **禁用词总表**增删词（B 类慎用「正常语义」词，须加注释说明误判边界）。  
2. 递增 `quality-S.md` frontmatter **`version`** 与 **`last_updated`**。  
3. 在 `quality-S.md` changelog 或本文件末尾**简要记录**变更原因（一行即可）。  
4. 运行 `node scripts/audit-skill-consistency.mjs`；更新 [`spec-sync-checklist.md`](../05-ops/spec-sync-checklist.md) 若涉及跨文档定义。  

## 3. 禁止事项

- 不在 stub 文件（根级镜像入口）改词表。  
- 不把「第三方检测报告」写进本包为强制依赖；若宿主接入 GPTZero 等，在宿主文档中单独约定。
