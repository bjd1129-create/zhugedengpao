# FBS-BookWriter 指标体系与评分系统

> 更新日期：2026-03-21 | 状态：P1落地
>
> 本文档定义 FBS-BookWriter 全生命周期评分体系，包含 20 分折算 10 分模型、指标追踪系统、自动化验收检查点。

---

## 1. 评分体系总览

### 1.1 20 分折算 10 分模型

| 层级 | 分项 | 分值 | 权重（折算后） | 说明 |
|------|------|------|----------------|------|
| **S层** | S1-S6（6条，与 `quality-S.md` / `quality-check.md` 一致） | 6分 | 3.0分 | 句级逐句扫描 |
| **P层** | P1-P4（4条） | 4分 | 2.0分 | 段级逐段检测 |
| **C层** | C1-C4（4条） | 4分 | 2.0分 | 章级逐章审查 |
| **B层** | B1 + B2-A + B2-B + B2-C + B3（5维） | 5分 | 2.5分 | 篇级全文体检 |
| **V1** | 视觉密度（1项） | 1分 | 0.5分 | 见 `quality-check.md` |
| **合计** | **S6+P4+C4+B5+V1** | **20分** | **10分** | 折算分 = 原始分÷2 |

**C0（全书级汇编门禁）**：**不计入**上表 20 分；终稿前须满足 `quality-check.md` **C0** 与 [`book-level-consistency.md`](./book-level-consistency.md)（全书破折号总账、术语唯一性等），与 **S5-G5** 门禁一致。

**折算公式**：`折算分 = 原始分 / 2`

---

### 1.2 S层评分细则（6条 / 6分）

> Critic-S 逐句扫描，详见 `quality-S.md`；**不设 S7**，与 `quality-check.md` §1 检查表一致。

| 编号 | 名称 | 满分 | 检测标准 |
|------|------|------|----------|
| S1 | 首句具体化 | 1分 | 全文首句违规数 = 0 |
| S2 | 消灭冗余修饰 | 1分 | A类程度副词出现次数 = 0 |
| S3 | 短句优先 | 1分 | 超40字句占比 ≤ 10% |
| S4 | 连接词降频 | 1分 | C类连接词密度 ≤ 2次/千字 |
| S5 | Buzzword黑名单 | 1分 | B类禁用词出现次数 = 0 |
| S6 | 标点符号纪律 | 1分 | 破折号≤1/千字，分号≤2/千字，感叹号≤2个全文 |

**S层通过阈值**：≥ 5 分（满分6分，约 83%）

---

### 1.3 P层评分细则（4条 / 4分）

> Critic-L1 逐段检测，详见 `quality-PLC.md §P`

| 编号 | 名称 | 满分 | 检测标准 |
|------|------|------|----------|
| P1 | 问题驱动 | 1分 | 每节开头必须是问题或场景，不得是定义 |
| P2 | 对话代替转述 | 1分 | 每2000字 ≥ 1处直接引语 |
| P3 | 禁止对称排比 | 1分 | 全文对称排比 = 0处 |
| P4 | 拒绝注水 | 1分 | 注水段 = 0（删掉后上下文通顺的段落） |

**P层通过阈值**：≥ 3 分（满分4分）

---

### 1.4 C层评分细则（4条 / 4分）

> Critic-L2 逐章审查，详见 `quality-PLC.md §C`

| 编号 | 名称 | 满分 | 检测标准 |
|------|------|------|----------|
| C1 | 承认局限+明确表态 | 1分 | 每章 ≥ 1处"好消息/坏消息"+明确表态 |
| C2 | 结尾指向行动 | 1分 | 章尾末句含动词+具体对象 |
| C3 | 打破结构均匀 | 1分 | 各节字数标准差 ≥ 均值的30% |
| C4 | 数据具体化 | 1分 | 整数百分比 ≤ 全文数据点的20% |

**C层通过阈值**：≥ 3 分（满分4分）

---

### 1.5 B层评分细则（5条 / 5分）

> Critic-L3 全文体检，详见 `quality-PLC.md §B`

| 编号 | 名称 | 满分 | 检测标准 |
|------|------|------|----------|
| B1 | 标题去公式化 | 1分 | 公式化标题 = 0 |
| B2-A | 段落节奏 | 1分 | 变异系数 ≥ 0.3，短段落15-25%，长段落10-20% |
| B2-B | 标点多样性 | 1分 | 每千字使用 ≥ 5种标点类型 |
| B2-C | 结构雷同检测 | 1分 | 连续3章相同开头=警告，连续2段相同结尾=警告 |
| B3 | 全局节奏综合 | 1分 | 章节长度变异系数>0.4=警告，语气偏离度>0.3=警告 |

**B层通过条件**：B1=1分 + B2-A/B2-B/B2-C各≥0.8分 + B3≥0.8分
**B层通过阈值**：≥ 4 分（满分5分）

---

### 1.6 达标阈值体系

| 等级 | 折算分 | 原始分 | 说明 |
|------|--------|--------|------|
| **精品级** | ≥ 9.0 | ≥ 18 | 允许发布，推荐标注 |
| **合格** | ≥ 7.5 | ≥ 15 | 通过，允许发布 |
| **待修改** | ≥ 7.0 | ≥ 14 | 轻微问题，修改后重审 |
| **打回** | < 7.0 | < 14 | 严重问题，打回修改 |

### 1.7 AI味一票否决制

以下情况直接打回，不进入评分：

| 违规类型 | 说明 |
|----------|------|
| **彩虹渐变** | 文字颜色使用彩虹渐变效果 |
| **过度3D效果** | 标题/装饰使用过度立体效果 |
| **AI味超标** | S层+P层+C层综合AI味指数 > 阈值 |
| **抄袭/侵权** | 内容存在抄袭风险或侵权问题 |

**AI味指数计算**：
```
AI味指数 = (S层扣分/S层满分6)×40% + (P层扣分/P层满分4)×30% + (C层扣分/C层满分4)×30%
```
AI味指数 > 30% 触发一票否决。

## 3. 自动化验收检查点

### 3.1 S1完成验收

**触发时机**：S1（定位确认）完成后

**验收条件**：
| 维度 | 阈值 | 说明 |
|------|------|------|
| S层 | ≥ 5 分 | S层6条至少得5分（满分6） |
| P层 | ≥ 3 分 | P层4条至少得3分 |
| 学术警告 | 已确认 | 学术场景已识别并确认 |
| 定位确认稿 | 已交付 | S1-C已用户确认 |

**验收结果**：
```
✅ S1验收通过：S层5.8分，P层3.5分，可进入S2。

❌ S1验收未通过：S层5.2分，P层2.5分。
   - [ ] S2需要修复：S3短句优先不达标（超40字句占15%）
   - [ ] S4Buzzword发现2处（"赋能"出现在第3段）
   - [ ] 建议：修改后再进入S2
```

### 3.2 S3完成验收

**触发时机**：S3（大纲审批+初稿）完成后

**验收条件**：
| 维度 | 阈值 | 说明 |
|------|------|------|
| S+C层合计 | ≥ 10 分 | S层+C层至少得10分（S 满 6 + C 满 4） |
| S层 | ≥ 5 分 | S层最低门槛（满分6，与 §3.1 一致） |
| C层 | ≥ 3 分 | C层最低门槛 |
| B层 | ≥ 4 分 | B层5条至少得4分（预览） |
| AI味指数 | < 30% | 无一票否决风险 |

**验收结果**：
```
✅ S3验收通过：S+C层10.0分（S6.0+C4.0），B层4.5分，AI味指数18%。

❌ S3验收未通过：S+C层9.0分（S5.0+C4.0），B层3.5分。
   - [ ] S层S2/S5需修复
   - [ ] C层C3结构均匀问题突出
   - [ ] B层B1标题公式化发现3处
   - [ ] AI味指数32%，超阈值
   - [ ] 建议：S3修改后重新审核再进入S4
```

### 3.3 S5完成验收（发布前终审）

**触发时机**：S5（终稿审批）完成后，发布前

**验收条件**：
| 维度 | 阈值 | 说明 |
|------|------|------|
| B层 | ≥ 4 分 | B层5条至少得4分 |
| 总分 | ≥ 15 分 | 全书平均分≥15（折算≥7.5） |
| AI味指数 | < 20% | 发布级严格阈值 |
| 学术合规 | 通过 | G层合规检查通过 |

**B层分项要求**：
| 分项 | 通过阈值 | 说明 |
|------|----------|------|
| B1 | = 1分 | 标题去公式化必须满分 |
| B2-A | ≥ 0.8分 | 段落节奏必须达标 |
| B2-B | ≥ 0.8分 | 标点多样性必须达标 |
| B2-C | ≥ 0.8分 | 结构雷同检测必须达标 |
| B3 | ≥ 0.8分 | 全局节奏必须达标 |

**验收结果**：
```
✅ S5终审通过：B层5.0分（S层6.0+P层4.0+C层4.0+B层5.0+V1 1.0），总分20.0分（折算10.0），AI味指数8%。

精品级评分，可发布。

❌ S5终审未通过：B层3.5分，总分14.2分。
   - [ ] B1标题公式化：第3章/第7章/第11章
   - [ ] B2-A段落节奏：变异系数0.22（需≥0.3）
   - [ ] B2-B标点多样性：仅3种/千字（需≥5种）
   - [ ] B3全局节奏：章节长度方差过大
   - [ ] 建议：修改后重新提交S5审核
```

## 4. 评分流程

### 4.1 审核时序

```
S1完成 → Critic-S/S/L审查 → 评分记录 → 验收检查点 → S2

S3完成 → Critic-S初筛 → Critic-L1/L2/L3复审 → 评分记录 → 验收检查点 → S4

S5完成 → 全维度终审 → 评分记录 → B层专项检查 → 学术合规检查 → 发布
```

### 4.2 审核角色分工

| 角色 | 职责 | 加载模块 |
|------|------|----------|
| Critic-S | S层逐句扫描 | quality-S.md |
| Critic-L1 | P层逐段检测 | quality-PLC.md §P |
| Critic-L2 | C层逐章审查 | quality-PLC.md §C |
| Critic-L3 | B层全文体检 | quality-PLC.md §B |

### 4.3 评分输出格式

每次审核完成后，输出：

```
## 审核报告：CH-03

**审核时间**：2026-03-21 19:35
**审核角色**：Critic-L3（篇级审校）

### 评分汇总
| 层级 | 得分 | 满分 | 得分率 |
|------|------|------|--------|
| S层 | 5.5 | 6 | 92% |
| P层 | 3.5 | 4 | 88% |
| C层 | 4.0 | 4 | 100% |
| B层 | 4.5 | 5 | 90% |
| V1 | 1.0 | 1 | 100% |
| **总分** | **18.5** | **20** | **93%** |
| **折算分** | **9.25** | **10** | - |

### AI味指数
当前值：12%（阈值30%），通过。

### 发现问题
1. [S3] 第3段句长42字，略超40字阈值
2. [P2] 第800字缺少直接引语
3. [B2-A] 段落长度差异偏小

### 验收结论
✅ 通过（精品级），可进入下一阶段。
```

## 5. 特殊规则

### 5.1 加分项（最高+1分，折算前）
| 加分项 | 分值 | 说明 |
|--------|------|------|
| 师生案例集 | +0.3 | 包含真实师生案例 |
| 数据可视化 | +0.2 | Mermaid/图表质量优秀 |
| 创新结构 | +0.2 | 有独特叙事结构 |
| 深度洞察 | +0.3 | 有独到行业洞察 |

### 5.2 彩虹渐变/过度3D效果检测规则

```
彩虹渐变 = 文字渐变色数量 ≥ 3种
过度3D = text-shadow或box-shadow层数 ≥ 3层，或blur值总和 ≥ 20px
满足任一条件 → 直接打回，不评分
```

### 5.3 学术内容检测

```
学术关键词命中 ≥ 1次 → 触发学术警告 → 用户强制确认 → 记录到评分报告
学术场景：论文/期刊/学术研究/署名/数据造假相关
```

| ≥ 9.0 | 可按「精品档」对外展示；适合案例库「优秀」候选（须真实授权） |
| 7.5–8.9 | 达标可交付；建议记录改进项后结项 |
| 7.0–7.4 | 建议修订后重审，不宜作对外标杆案例 |
| < 7.0 | 打回重写；不计入成功案例统计 |
