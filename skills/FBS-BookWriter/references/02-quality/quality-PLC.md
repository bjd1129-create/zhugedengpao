# 去AI味·P/C/B层规则

> 按节加载：Critic-L1 加载 §P | Critic-L2 加载 §C | Critic-L3 加载 §B
> **权威规范**：本文档为完整 P/C/B 层规则，无需外部引用

---

## §P P层·段级（4条）— Critic-L1 逐段检测

**正向目标：每段回答一个具体问题，读者读完知道"所以呢"。**

---

### P1 问题驱动

**禁令**：禁止"概念→特性→应用"的AI默认叙事结构。

**正向**：正确顺序是"问题→困境→工具"。

**阈值**：每节开头必须是问题或场景，不得是定义。

| 概念驱动（AI味） | 问题驱动 |
|------------------|---------|
| "场景包是一种将AI能力标准化封装的方法论..." | "客户问：机器人装好了，然后呢？这就是场景包要回答的。" |
| "知识集是企微提供的知识管理能力..." | "你有200页文档，客户问了个问题，你希望AI 30秒找到答案。知识集就干这个。" |

---

### P2 对话代替转述

**禁令**：禁止全篇转述无直接引语。

**正向**：用真实（或合理虚构的）对话拉近距离。

**阈值**：每2000字 ≥ 1处直接引语。

| 转述（AI味） | 对话 |
|-------------|------|
| "某位顾问表示AI对其业务产生了显著影响" | "老陈说，他当时心里咯噔了一下。" |
| "用户反馈该功能有效提升了工作效率" | "'以前写方案要两天，现在半天出初稿。'第一个试用客户的原话。" |

---

### P3 禁止对称排比

**禁令**：同段禁止"首先/其次/最后"或"不仅/而且/更"式平行结构。

**正向**：列举用bullet list或表格；叙述用节奏变化替代工整对仗。

**阈值**：全文对称排比 = 0处。

| AI味 | 去AI味 |
|------|--------|
| "首先提高效率；其次降低成本；最后改善体验。" | "效率提了3倍。成本砍半。续费率？87%。" |

---

### P4 拒绝注水

**禁令**：删掉一段后上下文通顺 = 该段是注水，删除。

**正向**：每段不可替代，删了就断裂。

**阈值**：注水段 = 0。

常见注水模式：
- 换说法重述（"换言之...""也就是说..."）
- 过渡句填充（"接下来我们来看...""下面将详细介绍..."）
- 万能拖尾（"等等""以及其他方面"）

---

## §C C层·章级（5条）— Critic-L2 逐章审查

**正向目标：每章有观点、有立场、有节奏起伏。**

---

### C1 承认局限 + 明确表态

**禁令**：禁止万能叙事（"AI将彻底改变一切"）和无立场对冲（"一方面...另一方面..."然后不下结论）。

**正向**：坦诚说能/不能，然后给出明确建议。

**阈值**：每章 ≥ 1处"好消息/坏消息"+明确表态。

模板：
```
**好消息**：[能做的事]。
**坏消息**：[不能做的事]。
**所以**：[明确建议/立场]。
```

**AI味**：`一方面AI提升了效率，另一方面也带来了新的挑战，企业需要综合考虑。`
**去AI味**：`AI能帮你把200页文档变成可检索的知识库。但它判断不了哪些信息对这个客户最重要，这需要你的行业经验。先上知识库，别急着上智能推荐。`

---

### C2 结尾指向行动

**禁令**：章节结尾禁止抽象总结。

**正向**：最后一句必须是具体的下一步行动或指向下一章的悬念。

**阈值**：章尾末句必须含动词+具体对象。

| AI味 | 去AI味 |
|------|--------|
| "综上所述，AI将为企业带来巨大变革。" | "现在打开后台，建一个测试机器人。10分钟后你就知道答案。" |
| "本章介绍了三种方法，可根据实际情况选择。" | "三种方法里，先试第二种。原因？门槛最低，当天见效。" |

---

### C3 打破结构均匀

**禁令**：禁止所有小节结构对称（每节都是"定义→例子→总结"三段论，每节字数相近）。

**正向**：节奏有起伏。有的节200字点到即止，有的节800字深挖，有的节只是一张表。

**阈值**：
- 同章内各节字数标准差 ≥ 均值的30%（即不能全部差不多长）
- 至少1节的结构与其他节明显不同（如其他节是叙述，这节只有一张对比表）

**AI特征**：
```
## 3.1 工具A    [300字: 定义→例子→总结]
## 3.2 工具B    [310字: 定义→例子→总结]
## 3.3 工具C    [295字: 定义→例子→总结]
```

**去AI味**：
```
## 3.1 工具A    [150字: 一句话+一张表搞定]
## 3.2 工具B    [600字: 完整案例深挖]
## 3.3 工具C    [80字: "这个别用，原因见3.2的教训。"]
```

---

### C4 数据具体化

**禁令**：禁止使用整数估计值和模糊量词冒充精确数据。

**正向**：真实数据带小数/非整数；无数据时坦诚说"没有公开数据"而不是编一个。

**阈值**：整数百分比（50%/80%/30%等）出现次数 ≤ 全文数据点的20%。

| AI味（虚化数据） | 去AI味（具体数据） |
|-----------------|-------------------|
| "效率提升了50%" | "效率提升了47%（基于XX 2025年报告）" |
| "大量企业已经采用" | "截至2026Q1，已有1,247家企业部署（来源：XX）" |
| "节省约80%的时间" | "实测：从4.5小时降到52分钟" |
| 编不出数据 | "目前无公开基准数据，以下为单客户实测" |

---

### C5 章节衔接（多路并行时 P0）

**目的**：降低「各写各的」导致的断档、重复开篇与无钩子。

**检查项**：

1. **末段钩子**：非终章时，建议末段或末句含指向下文的线索（如「下一章」「第 N 章将」「后文」等）；终章豁免。  
2. **首段连贯**：开篇避免与上章已收束的语境完全无关的突兀新概念；若必须引入新概念，须有一句承接。  
3. **跨章重复**：与 `.fbs/book-context-brief.md`（或主编提供的全书摘要）对照，避免大段复述他章已展开内容（建议相同论述复制 **≤200 字** 须改为参见前文）。

**阈值**：主编在并行模式下将 C5 与 **CX**（[`cross-chapter-consistency.md`](./cross-chapter-consistency.md)）合稿审查联动；单 Agent 串行时 C5 作建议项即可。

## §B B层·篇级（5条）— Critic-L3 全文体检

**正向目标：全文读完像一个有经验的人在说话，不像机器在组装。**

---

### B1 标题去公式化

**禁令**：禁止公式化标题模式。

**正向**：标题类型只能是以下三种之一：
- **提问式**：引发读者思考，如"为什么你的用户流失了？"
- **结论式**：直接给答案，如"用户流失的五大原因"
- **悬念式**：制造好奇，如"第三章的策略为何在第四章被推翻"

**阈值**：公式化标题 = 0。

| 公式化（AI味） | 去公式化 |
|---------------|---------|
| "XX的重要性" | "为什么XX决定了你能不能活过明年" |
| "浅谈XX" | 删除"浅谈"，直接说内容 |
| "XX的现状与未来" | "XX：现在能用，但别指望三年后还是这样" |
| "如何利用XX实现XX" | "用XX做XX：三步，不用写代码" |
| "基于XX的XX研究" | 改为人话标题 |
| "一、背景介绍"、"二、方法论" | 禁止分类标签式标题 |

---

### B2-A 段落节奏

**禁令**：禁止全文段落等长，节奏平坦。

**正向**：段落长度有差异，形成波浪式节奏。

**阈值**：
- 全书段落长度变异系数 ≥ 0.3（避免等长段落）
- 短段落（≤50字）占比建议 15-25%
- 长段落（≥300字）占比建议 10-20%

**检测方法**：
```
变异系数 = 标准差 / 平均值
计算全书所有段落字数的变异系数
```

---

### B2-B 标点多样性

**禁令**：禁止全文只用句号和逗号。

**正向**：标点类型丰富，阅读有节奏感。

**阈值**：每千字使用 ≥ 5种标点类型

| 标点类型 | 说明 |
|---------|------|
| 句号（。） | 句子结束 |
| 逗号（，） | 停顿 |
| 顿号（、） | 列举 |
| 分号（；） | 分句 |
| 冒号（：） | 引导 |
| 引号（「」或""） | 引用/强调 |
| 感叹号（！） | 情感 |
| 问号（？） | 提问 |
| 省略号（……） | 留白 |
| 破折号（——） | 转折 |

**检测方法**：统计全文标点种类数 / 总字数(千字)

---

### B2-C 结构雷同检测

**禁令**：禁止连续章节/段落使用相同结构。

**正向**：每章有独特开头，每段有不同收尾。

**阈值**：
- 连续3个以上章节使用相同开头结构 → 触发警告
- 连续2个以上段落使用相同结尾句式 → 触发警告

**常见违规模式**：
```
章节开头：
  ❌ "首先...其次...最后..."（三段论式开头连续出现）
  ❌ "让我们来看看..."（万能过渡句连续出现）

段落结尾：
  ❌ "...是非常重要的"（万能判断句连续出现）
  ❌ "...见下图"（引导句连续出现）
```

---

### B3 全局节奏综合

**禁令**：禁止全书节奏单一、语气恒温、风格一致到假。

**正向**：全书有起伏、有变化、有呼吸感。

**阈值**：
- 检测全书章节长度分布曲线，变异系数 > 0.4 时触发警告
- 检测全文语气/风格一致性，偏离度 > 0.3 时触发警告
- 全文情绪曲线应有峰有谷，不能全程高能或全程平淡

**检测维度**：
| 维度 | 指标 | 警告阈值 |
|------|------|---------|
| 章节长度 | 各章字数变异系数 | > 0.4 |
| 句式长度 | 平均句长变异系数 | > 0.5 |
| 语气温度 | 情感词密度分布 | 方差过大 |
| 风格一致性 | 用词/句式偏离度 | > 0.3 |

**综合评分**：
- B层通过条件：B1=0 + B2-A/B2-B/B2-C达标 + B3无警告
- B层满分：10分，B1占3分，B2-A/B2-B/B2-C各占2分，B3占1分
- 通过阈值：≥ 7.5分
