# 去AI味·S层规则（句级自扫描）

> Critic-S Task 专用 | 自包含，零外部依赖  
> **权威规范**：本文档为完整 S 层规则，无需外部引用

---

## 元原则

**去AI味 = 品牌信任基建。**

读者读完全文的体感应该是：**像一个有经验的人在说话，不像机器在组装。**

---

## S层·句级（6条）— 逐句扫描

**正向目标：每句有信息增量，读者读完获得一个事实或画面。**

---

### S1 首句具体化

**禁令**：禁用套话开场。任何章节/段落的第一句不得是抽象的趋势、概念、定义。

**正向**：首句必须含人物/场景/数据/问题之一。

**阈值**：全文首句违规数 = 0。

| 禁止 | 替代 |
|------|------|
| "在当今数字化转型浪潮中..." | 直接切入场景或问题 |
| "随着AI技术的飞速发展..." | 用一个具体人/事开场 |
| "众所周知..." | 删除，直接说内容 |

**正例**：`你花了10分钟按教程装好了。机器人上线了。然后呢？`
**反例**：`随着AI Agent技术的快速发展，越来越多企业开始关注如何将AI落地。`

---

### S2 消灭冗余修饰

**禁令**：用数据替代程度副词。删掉修饰词后句意不变 = 该词多余。

**正向**：每个修饰词都携带不可替代的信息。

**阈值**：全文程度副词出现次数 = 0（见下方 A类禁用词表）。

| 原文 | 修正 |
|------|------|
| "非常重要的关键步骤" | "关键步骤" |
| "在很大程度上显著提升了效率" | "效率提升了3倍" |
| "从根本上彻底改变了" | "改变了" |

---

### S3 短句优先

**禁令**：单句超40字（中文）必须拆分。

**正向**：句子节奏明快，信息密度高。

**阈值**：超40字句占比 ≤ 10%。

| 原文 | 修正 |
|------|------|
| "通过将AI与企微工作流相结合，可实现从咨询到方案生成的全流程自动化。" | "AI+企微工作流=全流程自动化。从咨询到方案生成，三步完成。" |

---

### S4 连接词降频

**禁令**：禁止高频使用过渡连接词做段间桥梁。

**正向**：段与段之间靠逻辑衔接，不靠连接词胶水。

**阈值**：同一连接词全文不超过3次；C类连接词总密度 ≤ 2次/千字。

| 禁止（高频连接词） | 替代策略 |
|-------------------|---------|
| "此外/另外/同时" | 直接说下一个事实，不加过渡 |
| "值得注意的是/需要指出的是" | 删除，直接说内容 |
| "不仅如此/更重要的是" | 用短句另起，靠内容本身推进 |

**正例**：`效率提了3倍。成本？砍半。`（无连接词，靠节奏推进）
**反例**：`此外，成本也得到了显著降低。同时，用户体验也有所改善。`

---

### S5 buzzword黑名单

**禁令**：禁用中文科技圈AI高频术语。

**正向**：用读者能直觉理解的日常用语。

**阈值**：B类禁用词出现次数 = 0（见下方 B类禁用词表）。

| buzzword | 替代 |
|----------|------|
| "赋能" | "帮/让...能做到" |
| "抓手" | "切入点/工具" |
| "底座/底层能力" | "基础功能" |
| "链路/全链路" | "流程/从...到..." |
| "沉淀" | "积累/留下来" |
| "闭环" | "从头到尾跑通" |
| "颗粒度" | "细致程度/精度" |
| "打通/拉通" | "连起来/接上" |
| "对齐/拉齐" | "统一/一致" |
| "心智" | "认知/印象" |

---

### S6 标点符号纪律

**禁令**：禁止标点模式单调化。AI特征是破折号滥用、分号堆砌、感叹号过度、标点类型单一。

**正向**：标点服务于节奏，类型多样，疏密有致。

**频率上限**：

| 标点 | 上限 | 说明 |
|------|------|------|
| 破折号（——） | ≤ 1次/千字 | AI最显著指纹，严控 |
| 分号（；） | ≤ 2次/千字 | 用句号断开替代 |
| 感叹号（！） | 全文 ≤ 2个 | 克制使用 |
| "具体来说："式冒号 | = 0 | 删除引导语，直接起列表 |

> **全书尺度**：上表阈值对 **章内扫描** 与 **全书合稿总账** 同时适用。逐章均略低于阈值时，全书累加仍可能严重超标，须在 **S5 前**按 [`book-level-consistency.md`](./book-level-consistency.md) **C0-1** 做总密度核算。  
> **执行闭环**：Critic 判 S6 未达标后，Writer **须重写句式并复扫 S6**，禁止只做「——→逗号」类无效替换（见 `quality-check.md` §1.0.1）。

**鼓励使用**：

| 标点 | 用途 | 示例 |
|------|------|------|
| 问号（？） | 反问/设问散布段中 | "续费率？87%。" |
| 省略号（……） | 留白、停顿、欲言又止 | "他犹豫了一下……然后把合同签了。" |
| 短句群（，。交替） | 制造节奏感 | "效率提了。成本砍了。人没换。" |

**AI特征**：`AI正在改变一切——从生产到销售——每个环节都在被重新定义。此外，效率也得到了显著提升；成本大幅降低；用户体验明显改善！`
**去AI味**：`AI改了什么？生产线。销售端。客服。但改得最狠的，是你的预期。`

## 禁用词总表

### A类·程度副词（S2适用）

非常、显著、大幅、极大地、深刻地、全面地、前所未有、根本性、彻底、高度、充分、深入、尤其、特别、相当、极其、格外、切实

### B类·buzzword（S5适用）

赋能、抓手、底座、底层能力、链路、全链路、沉淀、闭环、颗粒度、打通、拉通、对齐、拉齐、心智、势能、卡位、卡点、组合拳、矩阵（非数学语境）、生态（非生物语境）、中台（非技术架构语境）、协同、联动、融合、整合、维度、视角、层面

> **说明**：「维度/视角/层面」在 **空洞套话**（无具体对象、连续堆砌）时按 B 类处理；正常技术语义（如「三维坐标维度」）不判违规。

### C类·过渡连接词（S4适用）

此外、另外、同时、值得注意的是、需要指出的是、不仅如此、更重要的是、与此同时、除此之外、总的来说、综上所述、由此可见、不难看出、显而易见、再者、再者说、从另一个角度看、换个角度说
