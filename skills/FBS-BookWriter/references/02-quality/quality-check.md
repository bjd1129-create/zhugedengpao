## §1 四层检查单 + 评分公式

> **计分体系（与 [`metrics.md`](./metrics.md) §1 一致，已消除旧版「满分10却分项加总>10」矛盾）**  
> - **原始分**满分 **20** = S(6) + P(4) + C(4) + B(5) + V1(1)。  
> - **折算分** = 原始分 ÷ 2，满分 **10**（可同时汇报「原始/20」与「折算/10」）。  
> - **通过**：折算分 **≥ 7.5** ⇔ 原始分 **≥ 15**。  
> - **边界带**：折算分 **≥ 7.0 且 < 7.5**（原始分约 **14–14.5**）→ **建议修订**后重审；**< 7.0** → 打回（与 SKILL §0 置信度阈值一致）。  
> - **Bonus**：可选加分，**不计入**上述 20 分满额；须在报告中单列。  
> - **G 层**：合规与阻断，**不计入** 20 分；见 §G。  
>
> **各层原始分**：S 层 6 条（S1–S6）各 1 分；P/C 各 4 条各 1 分；**B 层 5 个维度**（B1、B2-A、B2-B、B2-C、B3）**各 1 分**——下表中 B2-A、B2-C、B3 下的多行阈值为**同一维度**的细则，**不重复计分**。V1 达标 1 分。

### §1.0 改稿后防反弹（S 层再扫）

> **审计缺口补齐**：大段改写、去 AI 味或用户粘贴替换后，须**重新跑**与 S 层相关的自动项（至少 S2 程度副词、S4 连接词、S5 buzzword、S6 标点），避免「改前通过、改后未检」。  
> **执行**：宿主在 Writer 提交新版本后触发 Critic-S 或等价任务；若仅改局部，可对变更段做增量扫描，但**不得**默认整章免检。

### §1.0.1 Writer 与 S6 强制闭环（去标点味）

- **Critic 报 S6 未达标**（破折号/分号/感叹号等任一项越阈）：**Writer 须做句式级重写**并**重新跑 S6** 直至达标，或取得**用户书面接受风险**；**禁止**仅以替换标点敷衍导致语义受损。  
- **单章 S6 通过**不能替代**全书**破折号总账：汇编后须执行 **C0-1**，见下表与 [`book-level-consistency.md`](./book-level-consistency.md)。

> **为何需要**：逐章 S/P/C/B 合规时，仍可能出现「章内略低、全书累加超标」或「并行各路对同一缩写赋予不同含义」。C0 在**合稿/终稿**尺度补漏。细则、Writer↔S6 闭环与报告字段见 [`book-level-consistency.md`](./book-level-consistency.md)。

| # | 检查项 | 阈值 / 规则 | 执行方 | 失败处理 |
|---|--------|-------------|--------|----------|
| C0-1 | 全书破折号（——）密度 ρ_em（千字计数） | 同 `quality-S.md` S6：**≤1** ✅；**(1,3]** ⚠️ 须列出超标段落并改写；**>3** ❌ **阻断终稿** | Proofer / 自动化 + 主编确认 | 全书降密 → 重跑 C0-1 |
| C0-2 | 核心术语 / 缩写全书唯一 | S1–S2 起维护术语表；汇编时全文检索缩写与定义一致，**禁止**同书多义；易多义缩写示例见 [`abbreviation-audit-lexicon.json`](./abbreviation-audit-lexicon.json) | 主编 / Proofer | 统一释义或改全称后重检 |

### 国标与行业编校（Proofer 专项，不计入二十分）

> **五路检索结论摘要**：现行图书**编校差错判定与计错**以 **CY/T 266—2023** 为主；**合格线**见《图书质量管理规定》（**差错率不超过万分之一**等，以现行文本为准）。**标点/数字**对照 **GB/T 15834—2011、GB/T 15835—2011**；**参考文献**对照 **GB/T 7714**。  
> **与上表关系**：S6/C0-1 的破折号密度等为**去 AI 味与本书风格**，**不可替代** CY/T 的全套计错；若对外承诺「编校合格」，须按 [`national-standards-editorial-checklist.md`](../05-ops/national-standards-editorial-checklist.md) 安排独立质检或第三方校对。

### G层·合规检查（可开关，默认关闭）

> **注意**：G3敏感内容审查为始终启用，不可关闭。其他G层检查项需用户主动启用。
>
> **启用方式**：用户输入"启用合规检查" / "学术模式" / "商用模式"
> **关闭方式**：用户输入"禁用合规检查"

#### 多路并行写作时的 G4 默认策略（P0，审计对齐）

当 **≥2 名 Writer 并行撰写不同章节**（WorkBuddy 多成员各写一章）时：

- **G4（数据来源标注）视为默认启用**：主编 / team-lead 须在团队任务中写明遵守 [`citation-format.md`](./citation-format.md)；各章须有 **本章数据来源索引**（或等价结构），并与检索门禁一致。
- **G1/G5** 仍按用户口令「学术模式」「商用模式」开关，不因并行自动改写。
- **单会话单 Agent 逐章串行**时，G 层默认策略不变（仍以用户启用合规检查为准）。

## §G G层合规检查实现规范

> **补充**：G1-G4检测逻辑 + 人工复核通道

### G1 违规词汇检测实现

**学术场景检测**（学术模式启用）：
```
检测位置：用户输入的书籍主题/核心主张
检测方式：关键词库匹配（keywords.md）
触发等级：
  - 强触发（弹窗+强制确认）：论文/毕业/期刊/署名/实验数据/诚信类
  - 中触发（提示条）：教育作业场景
  - 弱触发（标签提示）：需上下文的学术词
```

**敏感话题检测**（始终启用）：
```
检测位置：全书正文
检测方式：基础词库匹配
词库分类：
  - 政治敏感词（发现即G3人工复核）
  - 暴力/恐怖词
  - 色情低俗词
  - 违法犯罪词
```

**过度绝对化检测**：
```
检测位置：全书正文
检测方式：绝对词表匹配
绝对词表：必须、一定、所有、全部、完美、绝对、彻底、无可否认、无与伦比
阈值：每千字≤1次
超标处理：提示修改，不阻断
```

### G2 引用格式检测实现

**无来源表述检测**：
```
检测位置：全书正文
检测方式：模式匹配
需检测的表述模式：
  - "据显示" → 必须改为"[XXX](来源)显示"或删除
  - "有研究表明" → 必须添加来源标注
  - "数据显示" → 必须添加来源
  - "证明指出" → 必须添加来源
  - "专家称" → 必须添加具体专家信息和来源
处理：缺失来源=提示补充，不阻断但记录
```

**数据引用规范**：
```
要求：所有数据引用必须包含
  1. 具体数值
  2. 来源（媒体/机构/报告名）
  3. 检索日期
格式示例："根据[艾瑞咨询《2026年中国SaaS报告》](https://example.com)，2025年..."
```

### G3 阻断规则实现

**触发条件**：发现 G3 表所列敏感/违法/严重谣言等问题时。  
**处理方式**：与上文 G3 检查表「出现=阻断」一致——**须停止继续生成或发布该段内容**，提示用户修改后重试；同时可记入人工复核队列。**禁止**在已确认 G3 阻断的内容上静默继续出稿。

**G3级问题分类**：
| 问题类型 | 关键词/模式 | 处理 |
|----------|-------------|------|
| 学术造假嫌疑 | 实验数据伪造、调查数据篡改 | 标记+人工复核 |
| 版权侵权风险 | 长引用未标注、合理使用超限 | 标记+人工复核 |
| 政治敏感内容 | 领导人相关、分裂国家相关 | 标记+阻断+人工复核 |
| 严重虚假信息 | 谣言、已被证伪的数据 | 标记+阻断+人工复核 |

**人工复核通道**：
```
触发后输出：
🔒 G3级问题发现
   问题类型：[具体分类]
   涉及位置：[章节/段落]
   建议处理：[修改建议]
   状态：待人工复核
用户可选择：
  [1] 修改内容后继续
  [2] 提交人工复核
  [3] 确认为误报，继续创作
```

### G4 学术警告机制实现（S1-A关键词扫描）

**执行时机**：用户启动"写书"后，S1-A接收用户输入时

**扫描流程**：
```
1. 接收用户输入（核心主张/目标读者/书籍主题）
2. 加载keywords.md关键词库
3. 执行匹配检测
4. 根据触发等级执行对应行为
```

**强制弹窗确认（强触发）**：
```
检测到关键词时，输出：
⚠️ 学术合规风险提示

检测到您的项目可能涉及学术写作场景。

[风险详情]
• 触发关键词：[检测到的词]
• 风险等级：强触发

[重要提醒]
本书-generated内容可能无法直接用于学术场景。
建议根据学校/期刊要求披露AI使用情况。

[确认选项]
□ 我理解AI生成内容不能直接作为学术成果提交
□ 我确认核心研究由本人完成

[ 继续创作 ]  [ 重新描述需求 ]
```

**关键词库位置**：`references/keywords.md`（≥50个学术风险词）

## §2 章节质量报告模板

## §4 典型失败模式速查

| 模式 | 示例 | 违反规则 | 修正方向 |
|------|------|---------|---------|
| 套话开场 | "在数字化浪潮中..." | S1 | 人物/场景/数据开场 |
| 概念先行 | "XX是一种..." | P1 | 问题/困境开场 |
| 排比堆砌 | "首先...其次...最后..." | P3 | bullet list或表格 |
| 万能叙事 | "AI将彻底改变..." | C1 | 好消息/坏消息+表态 |
| 注水重述 | "换言之...也就是说..." | P4 | 删除重述段 |
| 抽象结尾 | "综上所述..." | C2 | 具体下一步行动 |
| 连接词胶水 | "此外...另外...同时..." | S4 | 靠逻辑衔接 |
| buzzword | "赋能底座打通闭环" | S5 | 用日常用语替代 |
| 结构克隆 | 每节300字三段论 | C3 | 长短交错，结构变化 |
| 公式标题 | "XX的重要性" | B1 | 提问/结论/悬念式标题 |
| 整数数据 | "提升50%" | C4 | 47.3%或标注来源 |
| 破折号滥用 | 每段一个"——" | S6 | ≤1次/千字 |
| 分号串联 | "提效率；降成本；改体验" | S6 | 句号断开 |
| 无立场对冲 | "一方面...另一方面..." | C1 | 加"所以我建议..." |
| 节奏平坦 | 连续5个200字段落 | B2-A | 长短穿插 |
| 感叹号轰炸 | "太重要了！必须做！" | S6 | 全文≤2个 |

## §5 文体适配表

| 规则层 | 书/白皮书 | 短文/报告 | 营销文案 |
|--------|----------|----------|---------|

| 三校 | 完整三校 | 一校+二校 | 一校 |
