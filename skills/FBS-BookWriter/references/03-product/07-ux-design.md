# UX 设计规范

> **说明**：交互与等待体验约定（示例文档）
> **核心目标**：等待体验优化 + 透明度设计 + 增量预览机制

---

## 一、等待体验设计（P1-U1）

### 1.1 骨架屏规范

**定义**：章节生成过程中显示的占位结构，让用户感知"正在生成"而非"卡住"。

**骨架屏结构**：
```
┌─────────────────────────────────────┐
│ 第三章 [章节标题]                    │
│ ████████████████████░░░░░ 67%      │
├─────────────────────────────────────┤
│                                     │
│  [▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓]           │  ← 标题占位（渐变动画）
│  [▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓]       │  ← 段落占位（脉冲动画）
│  [▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓]               │
│  [▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓]       │
│                                     │
│  ⏳ 正在生成内容...                  │
└─────────────────────────────────────┘
```

**动画规范**：
- 标题占位：左侧滑入渐变（300ms ease-out）
- 段落占位：脉冲动画（1.5s infinite，opacity 0.5→1.0）
- 不使用旋转加载（2026趋势：骨架屏 > 旋转加载）

### 1.2 实时进度指示

**进度条设计**：
```
章节进度：[████████████░░░░░░░] 12/20 章 (60%)
全局进度：[████████░░░░░░░░░░░░] 45%
预计剩余：约 8 分钟
```

**进度粒度**：
| 操作类型 | 进度单元 | 更新频率 |
|----------|----------|----------|
| 单章生成 | 章节级 | 每完成一段 |
| 多章并行 | 全局级 | 每章完成 |
| 审核阶段 | 规则级 | 每通过一条规则 |
| 排版导出 | 步骤级 | 每完成一步 |

### 1.3 预估时间算法

**基于移动平均的预估**：
```
首章实测时间 T₁
次章预估 = T₁ × 1.1（并行开销）
N章预估 = Σ(Tᵢ) × 1.15（汇总结算）
```

**状态播报格式**：
```
⏳ 正在生成第 3 章「团队协作」...
   [████████████████░░░] 75%
   预计还需：约 5 分钟
```

## 二、透明度设计（P1-U1续）

### 2.1 AI参与程度标识

**三级标识系统**：

| 标识 | 含义 | 显示位置 | 触发条件 |
|------|------|----------|----------|
| 🤖 **AI初稿** | AI生成，待人工审核 | 章节标题旁 | Writer输出后 |
| ⚠️ **需复核** | 存在数据引用或敏感内容 | 段落/数据点旁 | 引用外部数据 |
| ✅ **已审核** | 通过质量门禁（≥7.5分） | 章节末尾 | Critic-L3终审通过 |

**视觉规范**：
- AI初稿：灰色标签 `#9CA3AF`
- 需复核：橙色标签 `#F59E0B`，点击展开来源
- 已审核：绿色标签 `#10B981`

## 三、增量预览机制（P1-U2）

### 3.1 四层预览架构

| 层级 | 内容 | 时机 | 用户动作 |
|------|------|------|----------|
| **结构预览** | 大纲框架 + 章节标题 | S2大纲设计完成后 | 审批/调整 |
| **增量内容预览** | 内容逐步生长（逐段展开） | S3写作中 | 静观/不打断 |
| **质量标记预览** | AI初稿/需复核/已审核标识 | 每章完成后 | 查看状态 |
| **完整预览** | 全量展示 + 可交互编辑 | S5终稿完成后 | 编辑/发布 |

### 3.2 增量内容预览（核心创新）

**"正在生长"的内容展示**：
```

├─────────────────────────────────────────────────────────┤
│ ████████████████████████████████████████████████████   │ ← 进度条
│                                                         │
│ 2.1 找到对的合伙人...                                   │
│     ▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓  │ ← 已生成
```

**设计原则**：
- 不遮挡已生成内容（向上滚动可见）
- 不预加载未生成内容（节省资源）
- 用户可随时停止/调整（通过"暂停"按钮）

### 3.3 交互式编辑（完整预览层）

**编辑能力分级**：
| 操作 | 可编辑 | 说明 |
|------|--------|------|

| 删除章节 | ⚠️ | 需确认 |

## 四、隐式确认最低确认清单（P1-U3）

### 4.1 三层确认机制

**A层：自动确认（极低风险）**
> 无需用户确认，AI自动执行

| 场景 | 示例 | 处理方式 |
|------|------|----------|
| 错别字修正 | 的/地/得 | 自动修正 |
| 标点规范化 | 引号/省略号 | 自动规范化 |
| 格式统一 | 标题层级 | 自动对齐模板 |
| 数据保鲜标注 | 来源URL | 自动补全 |

**B层：隐性确认（必做，用户无感知）**
> 在关键节点自动插入，需用户知道但不需要每次点确认

| 节点 | 确认项 | 展示方式 |
|------|--------|----------|
| S1-C | 主题 + 读者 + 风格 + 篇幅 + 大纲 | 定位确认卡（一屏展示） |
| S2.5 | 脉络确认（内容方向） | 脉络摘要卡 |
| S3 | 大纲审批（章节结构） | 结构预览图 |
| S5 | 终稿预览（全文质量） | 评分卡 + 预览 |

**C层：显性确认（必做，需用户操作）**
> 必须用户点击确认，否则阻断执行

| 场景 | 阻断条件 | 确认方式 |
|------|----------|----------|
| 删除章节 | 任何删除操作 | 弹窗确认 |
| 方向变更 | S1→S2后改主题 | 显性确认 + 影响说明 |
| 定稿发布 | 进入发布流程 | 双重确认（内容 + 渠道） |
| 学术警告 | 检测到学术关键词 | 强制阅读 + 勾选确认 |

### 4.2 学术警告强制确认流程

**触发条件**：检测到学术关键词（论文/毕业/期刊/SCI/第一作者等）

**强制确认流程**：
```
┌────────────────────────────────────────────────────────┐
│ ⚠️ 学术用途警告                                        │
├────────────────────────────────────────────────────────┤
│                                                        │
│ 我们检测到您的内容可能涉及学术用途：                   │
│ • 论文写作                                             │
│ • 学位申请                                             │
│ • 期刊发表                                             │
│                                                        │
│ [内容合规提醒]                                         │
│ 1. AI生成内容应作为参考素材，不宜直接作为学术成果提交  │
│ 2. 引用数据需自行核实来源和准确性                      │
│ 3. 学术诚信责任由使用者自行承担                        │
│                                                        │
│ □ 我已知晓上述提醒，内容为 [下拉：学习参考/商业使用/  │
│   内部培训/其他非学术用途]                             │
│                                                        │
│                        [继续生成]  [取消]             │
└────────────────────────────────────────────────────────┘
```

## 五、状态设计规范

### 5.1 统一状态标识

| 状态 | 图标 | 颜色 | 含义 |
|------|------|------|------|
| 进行中 | ⏳ | `#3B82F6` 蓝 | 正在执行 |
| 已完成 | ✅ | `#10B981` 绿 | 成功结束 |
| 待确认 | ⚠️ | `#F59E0B` 橙 | 需用户确认 |
| 已撤销 | ❌ | `#6B7280` 灰 | 用户取消 |
| 异常 | 🚨 | `#EF4444` 红 | 需干预 |

### 5.2 进度展示规范

**2026 UX趋势**：
- 骨架屏替代旋转加载（感知更快）
- 预估时间替代不确定等待
- 分层透明度替代一刀切
- 增量展示替代全量阻塞
