# S0 前置调研模块设计方案

> 版本：v1.0 | 更新日期：2026-03-21 | 状态：已确认

---

## 一、定位与价值

### 1.1 为什么需要S0

**现状痛点**：
- 70%时间投入在简报准备阶段（用户往往不知道自己要写什么）
- 用户直接启动"写书"后，AI在信息不足情况下开始工作
- 导致后续大量返工，浪费算力

**S0核心价值**：
- 在正式写作前，通过系统性调研帮助用户厘清：
  - 主题市场定位（是否值得写）
  - 目标读者是谁（写给谁看）
  - 差异化空间（能否脱颖而出）
  - **本书/本稿**变现潜力（能否通过出版、付费内容、ToB 等路径变现——**非**泛化「赛道有多赚钱」）

### 1.2 S0位置

```
用户启动"写书"
    ↓
【S0 前置调研】 ← 新增
    ↓
S1 定位确认
    ↓
S2 目录设计
    ↓
...
```

**默认执行，可跳过**：
- 如果用户明确说"我已经有详细大纲"，可直接跳过S0
- 如果用户说"先写第3章"，可跳过S0直接进入S3

## 二、S0执行流程

### 2.1 总体流程

```
S0 = S0-A竞品扫描 → S0-B读者痛点 → S0-C变现潜力 → 用户确认
```

### 2.2 S0-A：竞品扫描（并行搜索）

**目标**：了解**与本稿可替代**的同类内容（书/白皮书/深度文章/付费内容等）——**默认不是**主题产业链上的商业产品矩阵；后者仅适用于工具评测、采购指南等已确认体裁，且须与「内容竞品」**分表**输出。

**执行方式**：4个Researcher并行搜索
- Researcher-1：搜索同类书籍/白皮书/长文
- Researcher-2：搜索市场规模/趋势（**简报中归「市场背景」**，不顶替「本书变现」结论）
- Researcher-3：搜索读者评价/痛点
- Researcher-4：搜索**本类读物的**价格/渠道/分销（指向内容产品，非泛化「赛道融资」）

**搜索关键词模板**：
```
[主题]书籍
[主题]手册
[主题]白皮书
[主题]在线课程
[主题]市场规模 2026
[主题]读者评价
同类产品 评测 对比
```

**产出**：
- 竞品列表（含书名/价格/评分/销量估算）
- 市场空缺点识别
- 差异化机会点

### 2.3 S0-B：读者痛点（联网+分析）

**目标**：确认目标读者的核心痛点

**执行方式**：
1. 搜索读者痛点相关讨论（知乎/豆瓣/小红书/公众号）
2. 分析高频需求和抱怨
3. 验证用户初始假设是否准确

**搜索关键词模板**：
```
[主题]痛点
[主题]难点
[主题]常见问题
[主题]教程
[读者群体]学习[主题]的困难
```

**产出**：
- 读者痛点清单（5-8个核心痛点）
- 需求优先级排序
- 用户假设修正意见（如有）

### 2.4 S0-C：变现潜力评估

**目标**：评估本书的变现潜力，给出建议路径

**执行方式**：
1. 分析竞品变现方式
2. 搜索同类内容变现案例
3. 给出1-3条推荐变现路径

**变现ROI参考**（来自变现/定价策略文档）：
| 变现路径 | ROI | 适合类型 |
|---------|-----|---------|
| 电子书 | 2:1 | 入门级/指南类 |
| 在线课程 | 5:1 | 方法论/实战类 |
| 训练营 | 8:1 | 高端/深度类 |
| 企业培训 | 10:1 | B2B/行业类 |
| 咨询项目 | 15:1 | 专业/顾问类 |
| 版权授权 | 20:1 | 爆款/IP类 |

**产出**：
- 1-3条推荐变现路径
- 各路径预估收益范围
- 建议优先级

## 四、S0触发与跳过

### 4.1 强制触发条件

以下情况**必须执行S0**：
1. 用户首次使用"写书"功能
2. 用户说"我想写一本关于[模糊主题]的书"
3. 用户没有提供详细大纲

### 4.2 跳过条件

以下情况**可直接跳过S0**：
1. 用户明确说"我有详细大纲"并提供
2. 用户选择"继续写上一本书"并选择已有项目
3. 用户直接说"先写第X章"且上下文明确

### 4.3 跳过确认

跳过S0时，系统应询问：
```
是否跳过S0调研直接进入S1？
- 我已准备好详细大纲 ✓
- 我想快速开始（AI将根据有限信息开始）
- 我需要S0帮我厘清方向
```

S0结束后，将调研简报保存为项目状态：

## 六、与其他模块的接口

### 6.1 与S1的接口

- S0输出：调研简报
- S1输入：调研简报 + 用户补充信息
- S1需在简报基础上与用户确认：
  - 主题是否调整？
  - 读者是否准确？
  - 风格是否合适？

### 6.2 与变现模块的接口

- S0-C输出变现方向建议
- S6后期执行时获取详细执行方案
- 变现路径记录在项目状态中

### 6.3 与竞品监控的接口

- S0-A 竞品数据可推送至竞品分析文档/竞品库维护流程以更新竞品库
- 建议定期（每季度）更新竞品分析报告
