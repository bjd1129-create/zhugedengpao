# 心跳与阶段播报协议

> **配套**：`SKILL.md` §1 心跳与阶段播报协议  
> **目的**：防止长时间无输出导致用户误判为卡死

---

## 阶段过渡播报（每个阶段结束时必须输出）

```
───────────────────────────
✅ 阶段N [阶段名] 完成
   [1-2句本阶段成果摘要]
⏭️ 即将进入：阶段N+1 [阶段名]
   预计动作：[将要做什么的一句话说明]
───────────────────────────
```

---

## 静默心跳（长耗时操作期间）

当单次操作预计超过 **30 秒** 时，每 **15 秒** 输出一行心跳（**用户可见节奏**；与 `search-policy.json` 中 **`searchAccessPolicy.singlePageTimeoutMs`（默认 15s）** 是不同概念：后者为**单次页面/请求**级超时，不替代本节的对话心跳）：

```
⏳ [操作说明] 进行中...（已用时 Xs）
```

---

## 研究员 / 检索心跳

联网检索或调研类任务：

```
🔍 检索中：[关键词或子任务简述] …
```

---

## 关闭与同步

多成员场景下，阶段结束前应确认各子任务状态已汇总；具体关闭与 `shutdown` 语义以宿主平台为准。

---

## 禁止行为

- 长时间无任何输出（含心跳）
- 仅输出内部调试信息替代用户可读进度
