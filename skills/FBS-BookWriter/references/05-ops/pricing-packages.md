# 定价与产品包（Starter / Pro / Enterprise）

## 1. 定价原则

1. 价值锚点优先：按“结果确定性 + 质量保障 + 时效”定价，不按字数线性定价。
2. 路线分层：Route-G 走标准化订阅，Route-E 走项目制与里程碑。
3. 可升级：Starter -> Pro -> Enterprise 路径清晰，避免能力断层。

## 2. 三档产品包

| 套餐 | 目标用户 | 适配路线 | 核心能力 | 建议计费方式 |
|---|---|---|---|---|
| Starter | 个人作者/小团队试用 | Route-G | 基础工作流、标准模板、基础质量自检 | 低门槛月付 |
| Pro | 稳定产出团队 | Route-G + Route-E 轻量 | 进阶模板、更多审校层、运营复盘支持 | 标准月付/季付 |
| Enterprise | 企业项目制客户 | Route-E | 里程碑交付、审计报告、定制流程与 SLA | 项目制/年框 |

## 3. 各档建议能力边界

### Starter

- 提供：S0-S6 基本流程、通用模板、基础自检
- 不提供：深度定制、企业级审计报告、专属实施支持

### Pro

- 提供：行业模板包、增强门禁策略、协作与复盘支持
- 不提供：企业 SLA、深度定制交付治理

### Enterprise

- 提供：项目实施、交付看板、审计链路、专项培训与复盘
- 可选：私有化流程约定、管理层汇报模板

## 4. 销售话术锚点（简版）

- Starter：快速启动，低成本验证内容生产闭环。
- Pro：标准化规模生产，提升团队交付稳定性。
- Enterprise：高风险内容项目可审计、可追踪、可复盘。

## 5. 升级触发条件（建议）

- Starter -> Pro：
  - 月产出需求 > 既有配额，或需要跨角色协作
- Pro -> Enterprise：
  - 需要项目制管理、审计证明、SLA 或定制流程

## 6. 运营建议

- 将升级路径放在交付节点而不是结算节点提示。
- 用“通过率、时效、复购”三指标证明升级价值。
