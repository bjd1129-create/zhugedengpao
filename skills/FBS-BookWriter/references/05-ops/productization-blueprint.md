# FBS-BookWriter 双路线产品化总蓝图

## 1. 目标与边界

本蓝图将 FBS-BookWriter 在 WorkBuddy 生态中拆分为两条并行路线：

- 企业高附加值路线（高客单、高交付确定性）
- 增长规模路线（高转化、高复购、标准化服务）

边界约束：

- 本文仅描述可公开产品能力，不包含内部经营敏感细节。
- 对外承诺仅基于当前仓库可验证能力与宿主可用能力。

## 2. 双路线定位

| 路线 | 核心人群 | 核心价值 | 主要交付形态 |
|---|---|---|---|
| 企业高附加值路线 | ToB 管理层、咨询/培训团队、行业机构 | 降低高风险内容交付的不确定性，提供可审计流程 | 白皮书/手册项目制交付、审计报告、发布包 |
| 增长规模路线 | 中小团队、内容运营者、个人作者 | 快速产出可发布长文档与多平台内容 | 标准化 Skill 包、模板化服务、分层订阅 |

## 3. 统一产品底座（两路线共用）

两条路线共用同一底座能力，避免双轨重复建设：

1. 工作流底座：S0-S6 分阶段推进、审批点与硬门禁。
2. 质量底座：S/P/C/B/V1 分层校验与阻断。
3. 证据底座：联网检索、引用、质量账本与审计脚本。
4. 交付底座：`SKILL.md + references/` 公开包与边界防泄露机制。

## 4. 企业高附加值路线（Route-E）

### 4.1 核心场景

- 行业白皮书（政策、数据、竞品强依赖）
- 企业培训/操作手册（流程一致性与合规要求高）
- 高管报告体内容（口径稳定、可追溯）

### 4.2 产品包策略

- 以“项目制+里程碑”售卖，不以单次字数售卖。
- 每个里程碑绑定可审核产出：目录、章节、终审报告、发布包。
- 将“门禁阻断率下降、一次通过率提升”作为核心价值指标。

### 4.3 成功标志

- 关键里程碑按时通过率 >= 90%
- 终稿一次通过率 >= 75%
- 高价值客户复购率 >= 35%

## 5. 增长规模路线（Route-G）

### 5.1 核心场景

- 个人作者快速成书
- 小团队周更内容体系化生产
- 主题模板化复制（行业指南、手册、训练营文档）

### 5.2 产品包策略

- 以“标准化套餐 + 可选增强”售卖。
- 核心卖点：快启动、可复用、低学习成本。
- 通过模板、预设、短指令降低首单摩擦。

### 5.3 成功标志

- 新用户 7 日激活率 >= 45%
- 首次成稿时长中位数 <= 90 分钟
- 月留存率 >= 40%

## 6. 组织与执行机制

### 6.1 角色分工

- 产品负责人：维护路线优先级与版本节奏。
- 交付负责人：确保质量门禁与上线节奏一致。
- 运营负责人：追踪转化漏斗与复购指标。

### 6.2 版本节奏（建议）

- 每两周一次小迭代（模板、文档、门禁参数）。
- 每月一次运营复盘（转化、留存、阻断原因）。
- 每季度一次路线重评（Route-E 与 Route-G 资源配比）。

## 7. 关键风险与对策

| 风险 | 触发信号 | 对策 |
|---|---|---|
| 过度承诺 | 文档与可执行能力不一致 | 执行 `audit-fbs-efficiency` 与 `doc-code-consistency` 对照 |
| 泄露风险 | 内部文件混入公开包 | 执行 `audit-visibility-boundary` + 白名单打包 |
| 交付波动 | 章节质量反复打回 | 强化 S2->S3 与章节质量门禁，优化模板与示例 |
