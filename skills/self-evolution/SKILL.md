## ⚠️ 核心原则

**安全进化，协议约束**

| 原则 | 说明 |
|------|------|
| **不越权** | 只在授权范围内进化 |
| **不破坏** | 不修改核心系统文件 |
| **可回滚** | 进化记录可追溯回滚 |
| **需确认** | 重大进化需人工确认 |

## 技能描述

分析Agent运行历史，自动识别能力短板，在协议约束下安全进化。实现持续自我改进和能力扩展。

## 核心功能

### 1. 运行历史分析

| 功能 | 说明 |
|------|------|
| **任务记录** | 分析历史任务执行记录 |
| **错误统计** | 识别高频错误类型 |
| **效率分析** | 评估任务完成效率 |
| **能力图谱** | 生成能力分布图 |

### 2. 能力短板识别

| 维度 | 分析内容 |
|------|----------|
| **技能缺失** | 识别未掌握的技能 |
| **知识盲区** | 识别知识空白点 |
| **效率瓶颈** | 识别效率低的环节 |
| **错误模式** | 识别重复错误 |

### 3. 安全进化机制

| 层级 | 权限 | 说明 |
|------|------|------|
| **L1 微调** | 自动执行 | 参数优化、提示词调整 |
| **L2 扩展** | 自动执行 | 新增技能、知识补充 |
| **L3 重构** | 需确认 | 流程重构、架构调整 |
| **L4 核心** | 禁止自动 | 核心协议、安全规则 |

## 进化流程

```
┌─────────────────────────────────────────────────────────────┐
│                    自我进化引擎流程                          │
└─────────────────────────────────────────────────────────────┘

1. 数据采集
   ├── 收集运行日志
   ├── 分析任务记录
   └── 提取错误模式

2. 短板分析
   ├── 识别能力缺口
   ├── 评估优先级
   └── 生成进化建议

3. 安全检查
   ├── 协议约束检查
   ├── 权限级别判定
   └── 风险评估

4. 执行进化
   ├── L1/L2：自动执行
   ├── L3：请求确认
   └── L4：禁止执行

5. 验证反馈
   ├── 测试进化效果
   ├── 记录进化日志
   └── 更新能力图谱
```
