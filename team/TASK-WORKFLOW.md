# 任务工作流 v1.0

## 核心理念
**闭环 + 循环进化** — 每一个任务都有起点和终点，每一次终点都是下一次的起点。

---

## 阶段一：任务发起

### 1.1 发起来源
| 来源 | 触发方式 |
|------|---------|
| 老庄 | 飞书/微信直接下达 |
| 小花 | 判断需要行动时主动发起 |
| 协调官 | 例行巡检发现问题上报 |
| Cron | 自动触发（巡检/进化） |

### 1.2 任务录入
- 写入 `TASKS.md`（主目录）
- 格式：

```markdown
## [时间] 任务名称
- **发起人**：xxx
- **负责人**：xxx
- **截止**：xxx
- **内容**：具体要求
- **状态**：待确认
```

---

## 阶段二：任务派发（传达链）

### 2.1 传达链（双轨并行）

**主轨（正常情况）**：
```
小花 → 协调官 → 子Agent
```

**备轨（协调官失联时自动触发）**：
```
小花 → 子Agent（直接打通）
```

**失联判定标准**（满足任一即触发备轨）：
- 协调官超过30分钟无响应
- TASKS.md状态超过1小时未更新
- cron推送连续失败3次
- 收到老庄明确指令

**备轨触发动作**：
1. 小花直接在子Agent工作空间写入TODO.md
2. 小花直接用飞书/消息通知子Agent
3. 事后告知协调官（不追究，不补手续）

### 2.2 派发标准动作
协调官必须完成：
1. 在 `TASKS.md` 更新负责人和状态
2. 飞书通知负责人
3. 负责人回复"收到"
4. 负责人必须在memory/YYYY-MM-DD.md记录任务

---

## 阶段三：任务执行

### 3.1 执行标准动作
负责人必须：
1. **立即记录**：在memory/YYYY-MM-DD.md写下任务
2. **执行任务**
3. **更新状态**：完成后更新TASKS.md状态
4. **复盘**：在memory/YYYY-MM-DD.md写下教训/收获

### 3.2 节点检查
| 时间节点 | 检查内容 |
|---------|---------|
| 派发后5min | 负责人是否回复"收到" |
| 执行中 | 进度如何，有无阻塞 |
| 截止前 | 是否能按时完成 |
| 完成后 | 是否已复盘 |

---

## 阶段四：闭环确认

### 4.1 闭环标准
- 负责人回复"已完成"
- 在TASKS.md标记"已完成"
- 在memory/YYYY-MM-DD.md记录复盘
- 小花或协调官确认闭环

### 4.2 未闭环处理
- 48小时未完成 → 升级到小花处理
- 负责人失联 → 协调官指定替代人
- 任务取消 → 标记"已取消"并说明原因

---

## 阶段五：循环进化

### 5.1 每日进化循环
```
协调官每日复盘 → 发现问题 → 优化流程 → 更新AGENTS.md → 第二天执行
```

**复盘问题**：
- 今天有哪些任务没闭环？为什么？
- 哪个流程环节效率最低？
- 有没有重复犯错？

### 5.2 每周进化循环
```
周检视 → 流程优化 → 团队调整 → 新一周执行
```

### 5.3 进化触发条件
| 触发 | 条件 | 响应 |
|-----|------|-----|
| 任务失败 | 同一类任务失败2次 | 立即复盘，更新流程 |
| 流程阻塞 | 某环节超过30min无进展 | 绕道或升级 |
| 新机会 | 发现效率提升机会 | 记录并评估 |

## 关键规则（铁律）

1. **任务必须写memory** — 没写的等于没做
2. **闭环才能结束** — 完成≠闭环，确认闭环才算结束
3. **复盘才能进化** — 每次任务结束必须复盘
4. **禁止跳级** — 正常情况不走捷径
5. **紧急情况备案** — 绕过流程后24小时内补齐记录
